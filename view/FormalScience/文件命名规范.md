# 文件命名规范

> **创建日期**: 2025-01-27
> **目的**: 确保所有文件命名直观、一致、易于理解

## 📋 命名原则

### 1. 直观性原则

- 文件名应清晰表达文件内容
- 使用中文描述性名称
- 避免缩写和模糊术语

### 2. 一致性原则

- 统一使用 `XX.Y_主题名称.md` 格式
- 主题文件夹使用 `XX_主题名称` 格式
- 保持编号连续性

### 3. 层次性原则

- 主主题：`01_形式科学基础理论/`
- 子主题：`01.1_形式科学的定义与特征.md`
- 子子主题：`01.1.1_抽象性特征.md`（如需要）

## 📁 文件结构规范

### 主题文件夹命名

```
XX_主题名称/
├── README.md                    # 主题概览
├── XX.1_子主题1.md             # 子主题文件
├── XX.2_子主题2.md
└── ...
```

### 文件命名格式

#### 格式1：主题.子主题_描述性名称.md

- **示例**: `01.1_形式科学的定义与特征.md`
- **说明**: 主主题编号.子主题编号_描述性名称

#### 格式2：主题.子主题.子子主题_描述性名称.md

- **示例**: `01.1.1_抽象性特征.md`
- **说明**: 多层级结构（如需要）

## ✅ 命名检查清单

创建新文件时，请检查：

- [ ] 文件名是否清晰表达内容？
- [ ] 是否遵循 `XX.Y_描述性名称.md` 格式？
- [ ] 编号是否连续且不重复？
