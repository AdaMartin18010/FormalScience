## 1. 并发控制协议

### 1.1 事务的基本概念

**定义 17.2.1** (事务)：

事务是一个操作序列 $\mathcal{T} = \{op_1, op_2, ..., op_n\}$，满足ACID特性：

- **原子性 (Atomicity)**：$\forall op_i \in \mathcal{T}: \text{commit}(\mathcal{T}) \lor \text{abort}(\mathcal{T})$
- **一致性 (Consistency)**：$\mathcal{T}: S_{valid} \rightarrow S'_{valid}$
- **隔离性 (Isolation)**：并发执行等价于某个串行执行
- **持久性 (Durability)**：$\text{commit}(\mathcal{T}) \Rightarrow \text{permanent}(\text{effects}(\mathcal{T}))$

**定义 17.2.2** (事务操作)：

事务操作集合 $\mathcal{O} = \{r_i[x], w_i[x], c_i, a_i\}$，其中：

- $r_i[x]$：事务 $T_i$ 读数据项 $x$
- $w_i[x]$：事务 $T_i$ 写数据项 $x$
- $c_i$：事务 $T_i$ 提交
- $a_i$：事务 $T_i$ 回滚

### 1.2 两阶段锁协议 (2PL)

**定义 17.2.3** (两阶段锁协议)：

事务 $T_i$ 遵循2PL，如果其锁操作序列可分为两个阶段：

1. **扩展阶段 (Growing Phase)**：只能获取锁，不能释放锁
2. **收缩阶段 (Shrinking Phase)**：只能释放锁，不能获取锁

形式化表示：
$$\forall l_i[x], u_i[x] \in T_i: \text{time}(l_i[x]) < \text{time}(u_i[x])$$
且存在唯一时间点 $t_{2pc}$ 使得：

- $\forall t < t_{2pc}$: 所有操作是获取锁
- $\forall t > t_{2pc}$: 所有操作是释放锁

**定理 17.2.1** (2PL 可串行化保证)：

所有遵循2PL协议的事务调度都是冲突可串行化的。

**证明**：

假设存在遵循2PL的调度 $S$ 不是冲突可串行化的。

根据冲突可串行化理论，非冲突可串行化调度必然存在循环：
$$T_1 \rightarrow T_2 \rightarrow ... \rightarrow T_n \rightarrow T_1$$

其中 $T_i \rightarrow T_j$ 表示存在冲突操作对 $op_i[k]$ 在 $op_j[k]$ 之前。

对于任意边 $T_i \rightarrow T_j$：

- 若为写-写冲突：$w_i[x] < w_j[x]$，则 $T_i$ 必须在 $T_j$ 之前释放 $x$ 的锁
- 若为读-写冲突：$r_i[x] < w_j[x]$，则 $T_i$ 必须在 $T_j$ 写之前释放锁
- 若为写-读冲突：$w_i[x] < r_j[x]$，则 $T_i$ 必须在 $T_j$ 读之前释放锁

因此，对于循环 $T_1 \rightarrow T_2 \rightarrow ... \rightarrow T_n \rightarrow T_1$：
$$t_{2pc}(T_1) < t_{2pc}(T_2) < ... < t_{2pc}(T_n) < t_{2pc}(T_1)$$

这产生矛盾 $t_{2pc}(T_1) < t_{2pc}(T_1)$。

故假设不成立，2PL保证冲突可串行化。$\square$

```python
class TwoPhaseLock:
    """
    两阶段锁协议实现
    """

    def __init__(self):
        self.lock_table = {}  # {data_item: {lock_type, holder_set}}
        self.transaction_locks = {}  # {tx_id: {data_items}}
        self.tx_phase = {}  # {tx_id: 'growing'|'shrinking'}
        self.wait_for_graph = {}  # {tx_id: [waiting_for_tx_ids]}

    def acquire_lock(self, tx_id: str, data_item: str, lock_type: str) -> bool:
        """
        获取锁

        Args:
            tx_id: 事务ID
            data_item: 数据项
            lock_type: 'S'(共享) 或 'X'(排他)

        Returns:
            True if 成功获取
        """
        # 检查阶段
        if self.tx_phase.get(tx_id) == 'shrinking':
            raise Exception(f"Transaction {tx_id} in shrinking phase, cannot acquire lock")

        self.tx_phase[tx_id] = 'growing'

        # 检查锁兼容性
        if data_item not in self.lock_table:
            self.lock_table[data_item] = {'type': lock_type, 'holders': {tx_id}}
            self._add_lock_to_tx(tx_id, data_item)
            return True

        current = self.lock_table[data_item]

        # 锁兼容性矩阵
        if current['type'] == 'S':
            if lock_type == 'S':
                # 共享锁兼容
                current['holders'].add(tx_id)
                self._add_lock_to_tx(tx_id, data_item)
                return True
            else:  # X锁请求
                if len(current['holders']) == 1 and tx_id in current['holders']:
                    # 升级锁
                    current['type'] = 'X'
                    return True
                else:
                    # 需要等待
                    self._add_to_wait_graph(tx_id, current['holders'])
                    return False
        else:  # X锁已存在
            if tx_id in current['holders']:
                return True
            else:
                self._add_to_wait_graph(tx_id, current['holders'])
                return False

    def release_lock(self, tx_id: str, data_item: str):
        """释放锁，进入收缩阶段"""
        self.tx_phase[tx_id] = 'shrinking'

        if data_item in self.lock_table:
            self.lock_table[data_item]['holders'].discard(tx_id)
            if not self.lock_table[data_item]['holders']:
                del self.lock_table[data_item]

        if tx_id in self.transaction_locks:
            self.transaction_locks[tx_id].discard(data_item)

    def release_all_locks(self, tx_id: str):
        """事务结束时释放所有锁"""
        if tx_id in self.transaction_locks:
            for data_item in list(self.transaction_locks[tx_id]):
                self.release_lock(tx_id, data_item)
            del self.transaction_locks[tx_id]
        del self.tx_phase[tx_id]

    def _add_lock_to_tx(self, tx_id: str, data_item: str):
        """记录事务持有的锁"""
        if tx_id not in self.transaction_locks:
            self.transaction_locks[tx_id] = set()
        self.transaction_locks[tx_id].add(data_item)

    def _add_to_wait_graph(self, tx_id: str, holders: set):
        """添加到等待图"""
        if tx_id not in self.wait_for_graph:
            self.wait_for_graph[tx_id] = set()
        self.wait_for_graph[tx_id].update(holders)
```

### 1.3 严格两阶段锁 (Strict 2PL)

**定义 17.2.4** (严格两阶段锁)：

事务 $T_i$ 遵循严格2PL，如果：

1. 遵循基本2PL
2. 所有排他锁 ($X$) 直到事务提交/回滚后才释放

**定理 17.2.2** (严格2PL避免级联回滚)：

严格2PL调度不会产生级联回滚。

**证明**：

级联回滚发生当且仅当：

- 事务 $T_j$ 读取了未提交事务 $T_i$ 写入的数据
- $T_i$ 回滚导致 $T_j$ 必须回滚

在严格2PL中：

- 写操作需要排他锁 $X$
- $X$ 锁直到提交/回滚后才释放
- 因此 $T_j$ 只能在 $T_i$ 提交后读取其写入的数据

故不可能发生级联回滚。$\square$

### 1.4 多版本并发控制 (MVCC)

**定义 17.2.5** (多版本并发控制)：

MVCC 为每个数据项维护多个版本：
$$\text{Versions}(x) = \{(x_1, ts_1), (x_2, ts_2), ..., (x_n, ts_n)\}$$

其中 $ts_i$ 是版本创建时间戳。

**定义 17.2.6** (MVCC 读规则)：

事务 $T_i$ 在时间戳 $ts(T_i)$ 读取 $x$，返回满足以下条件的版本：
$$x_v = \arg\max_{(x_k, ts_k) \in \text{Versions}(x)} \{ts_k \leq ts(T_i) \land \text{committed}(x_k)\}$$

```python
from dataclasses import dataclass
from typing import Dict, List, Optional, Set
from datetime import datetime
import threading

@dataclass
class Version:
    """数据版本"""
    value: any
    created_ts: int
    expired_ts: Optional[int] = None  # None 表示当前有效版本
    created_by: str = ""

    def is_visible_to(self, read_ts: int) -> bool:
        """检查此版本对读取时间戳是否可见"""
        return (self.created_ts <= read_ts and
                (self.expired_ts is None or self.expired_ts > read_ts))


class MVCCStorage:
    """
    MVCC存储引擎实现
    """

    def __init__(self):
        self.versions: Dict[str, List[Version]] = {}  # {key: [versions]}
        self.active_transactions: Dict[str, int] = {}  # {tx_id: start_ts}
        self.global_timestamp = 0
        self.lock = threading.RLock()

    def begin_transaction(self, tx_id: str) -> int:
        """开始事务，返回读取时间戳"""
        with self.lock:
            ts = self.global_timestamp
            self.active_transactions[tx_id] = ts
            return ts

    def read(self, tx_id: str, key: str) -> Optional[any]:
        """
        MVCC读取操作

        读取规则：找到小于read_ts的最大已提交版本
        """
        with self.lock:
            read_ts = self.active_transactions.get(tx_id)
            if read_ts is None:
                raise Exception(f"Transaction {tx_id} not started")

            versions = self.versions.get(key, [])

            # 找到可见的最新版本
            for version in reversed(versions):
                if version.is_visible_to(read_ts):
                    return version.value

            return None

    def write(self, tx_id: str, key: str, value: any) -> bool:
        """
        MVCC写入操作

        创建新版本，旧版本标记为过期
        """
        with self.lock:
            write_ts = self.active_transactions[tx_id]

            if key not in self.versions:
                self.versions[key] = []

            # 检查写冲突（Write-Write Conflict）
            # 如果存在未提交版本，且不是我们创建的，则冲突
            for version in self.versions[key]:
                if version.expired_ts is None and version.created_by != tx_id:
                    if version.created_ts > write_ts:
                        # 检测到冲突，需要回滚或等待
                        return False

            # 标记旧版本过期
            for version in self.versions[key]:
                if version.expired_ts is None:
                    version.expired_ts = write_ts

            # 创建新版本
            new_version = Version(
                value=value,
                created_ts=write_ts,
                created_by=tx_id
            )
            self.versions[key].append(new_version)

            return True

    def commit(self, tx_id: str) -> int:
        """提交事务，返回提交时间戳"""
        with self.lock:
            commit_ts = self._advance_timestamp()
            del self.active_transactions[tx_id]
            return commit_ts

    def rollback(self, tx_id: str):
        """回滚事务"""
        with self.lock:
            # 删除此事务创建的所有未提交版本
            for key, versions in self.versions.items():
                self.versions[key] = [
                    v for v in versions
                    if not (v.created_by == tx_id and v.expired_ts is None)
                ]
                # 恢复被标记为过期的版本
                if self.versions[key]:
                    last_version = self.versions[key][-1]
                    if last_version.created_by == tx_id:
                        last_version.expired_ts = None

            del self.active_transactions[tx_id]

    def _advance_timestamp(self) -> int:
        """推进全局时间戳"""
        self.global_timestamp += 1
        return self.global_timestamp

    def garbage_collect(self, oldest_active_ts: int):
        """清理过期版本"""
        with self.lock:
            for key in list(self.versions.keys()):
                # 保留：1) 当前有效版本 2) oldest_active_ts之后的版本
                self.versions[key] = [
                    v for v in self.versions[key]
                    if v.expired_ts is None or v.expired_ts >= oldest_active_ts
                ]
```

### 1.5 乐观并发控制 (OCC)

**定义 17.2.7** (乐观并发控制)：

OCC 将事务分为三个阶段：

1. **读取阶段**：事务读取数据到私有工作区，不做验证
2. **验证阶段**：提交前检查冲突
3. **写入阶段**：验证通过后写入数据库

**定义 17.2.8** (OCC 验证规则)：

对于事务 $T_i$，定义：

- $RS(T_i)$：读取集
- $WS(T_i)$：写入集
- $ts_{start}(T_i)$：开始时间戳
- $ts_{finish}(T_i)$：完成时间戳

$T_i$ 通过验证当且仅当：
$$\forall T_j \text{ 已提交}: ts_{start}(T_i) < ts_{finish}(T_j) < ts_{finish}(T_i) \Rightarrow WS(T_j) \cap RS(T_i) = \emptyset$$

```python
class OptimisticConcurrencyControl:
    """
    乐观并发控制实现
    """

    def __init__(self):
        self.committed_versions = {}  # {key: [(value, commit_ts), ...]}
        self.global_timestamp = 0
        self.lock = threading.Lock()

    def begin(self) -> 'OCCTransaction':
        """开始新事务"""
        return OCCTransaction(self)

    def get_global_timestamp(self) -> int:
        with self.lock:
            self.global_timestamp += 1
            return self.global_timestamp


class OCCTransaction:
    """OCC事务"""

    def __init__(self, occ: OptimisticConcurrencyControl):
        self.occ = occ
        self.start_ts = occ.get_global_timestamp()
        self.read_set: Dict[str, any] = {}  # {key: value}
        self.write_set: Dict[str, any] = {}  # {key: value}
        self.read_timestamps: Dict[str, int] = {}  # {key: version_ts}
        self.state = 'active'  # active/validating/committed/aborted

    def read(self, key: str) -> Optional[any]:
        """读取数据"""
        if self.state != 'active':
            raise Exception("Transaction not active")

        # 检查本地写入
        if key in self.write_set:
            return self.write_set[key]

        # 从已提交版本读取
        versions = self.occ.committed_versions.get(key, [])
        for value, commit_ts in reversed(versions):
            if commit_ts <= self.start_ts:
                self.read_set[key] = value
                self.read_timestamps[key] = commit_ts
                return value

        return None

    def write(self, key: str, value: any):
        """写入数据到本地工作区"""
        if self.state != 'active':
            raise Exception("Transaction not active")

        self.write_set[key] = value

    def commit(self) -> bool:
        """
        尝试提交事务

        验证阶段：检查是否有其他事务在我们读取后修改了读集
        """
        self.state = 'validating'

        try:
            with self.occ.lock:
                finish_ts = self.occ.get_global_timestamp()

                # 验证：检查是否有事务在[start_ts, finish_ts]期间
                # 提交并修改了我们读取的数据
                for key in self.read_set:
                    versions = self.occ.committed_versions.get(key, [])
                    # 找到我们读取的版本之后的所有提交
                    for value, commit_ts in versions:
                        if (self.read_timestamps[key] < commit_ts <= finish_ts):
                            # 验证失败
                            self.state = 'aborted'
                            return False

                # 验证通过，写入阶段
                for key, value in self.write_set.items():
                    if key not in self.occ.committed_versions:
                        self.occ.committed_versions[key] = []
                    self.occ.committed_versions[key].append((value, finish_ts))

                self.state = 'committed'
                return True

        except Exception:
            self.state = 'aborted'
            return False

    def abort(self):
        """中止事务"""
        self.state = 'aborted'
```

## 2. 死锁检测与预防

### 2.1 死锁的形式化定义

**定义 17.2.9** (等待图)：

等待图 $WG = (V, E)$ 是有向图，其中：

- $V = \{T_1, T_2, ..., T_n\}$ 是活跃事务集合
- $E = \{(T_i, T_j) | T_i \text{ 等待 } T_j \text{ 释放锁}\}$

**定理 17.2.3** (死锁定理)：

事务系统存在死锁当且仅当等待图存在环。

**证明**：

**充分性**：若等待图存在环 $T_1 \rightarrow T_2 \rightarrow ... \rightarrow T_n \rightarrow T_1$：

- 每个 $T_i$ 持有 $T_{i+1}$ 需要的资源
- 每个 $T_i$ 等待 $T_{i+1}$ 释放资源
- 所有事务无限等待，形成死锁

**必要性**：若系统死锁：

- 存在事务集合 $\{T_1, ..., T_n\}$ 互相等待
- $T_i$ 等待 $T_{i+1}$ 释放资源
- 因此 $(T_i, T_{i+1}) \in E$
- 形成环

$\square$

### 2.2 死锁检测算法

```python
class DeadlockDetector:
    """
    死锁检测器
    """

    def __init__(self):
        self.wait_for_graph: Dict[str, Set[str]] = {}
        self.lock = threading.Lock()

    def add_edge(self, from_tx: str, to_tx: str):
        """添加等待边"""
        with self.lock:
            if from_tx not in self.wait_for_graph:
                self.wait_for_graph[from_tx] = set()
            self.wait_for_graph[from_tx].add(to_tx)

    def remove_edge(self, from_tx: str, to_tx: str = None):
        """移除等待边"""
        with self.lock:
            if from_tx in self.wait_for_graph:
                if to_tx:
                    self.wait_for_graph[from_tx].discard(to_tx)
                else:
                    del self.wait_for_graph[from_tx]

    def detect_cycle(self) -> Optional[List[str]]:
        """
        使用DFS检测环

        Returns:
            如果存在环，返回环中的事务列表
        """
        with self.lock:
            WHITE, GRAY, BLACK = 0, 1, 2
            color = {tx: WHITE for tx in self.wait_for_graph}
            parent = {}

            def dfs(tx: str, path: List[str]) -> Optional[List[str]]:
                color[tx] = GRAY

                for neighbor in self.wait_for_graph.get(tx, []):
                    if color.get(neighbor, WHITE) == GRAY:
                        # 发现回边，找到环
                        cycle_start = path.index(neighbor)
                        return path[cycle_start:] + [neighbor]

                    if color.get(neighbor, WHITE) == WHITE:
                        result = dfs(neighbor, path + [neighbor])
                        if result:
                            return result

                color[tx] = BLACK
                return None

            for tx in self.wait_for_graph:
                if color[tx] == WHITE:
                    result = dfs(tx, [tx])
                    if result:
                        return result

            return None

    def find_victim(self, cycle: List[str]) -> str:
        """
        选择牺牲者（ victim ）来打破死锁

        策略：选择代价最小的事务
        """
        # 简化策略：选择最近开始的事务
        return cycle[-1]

    def get_wait_chain(self, tx_id: str, max_depth: int = 10) -> List[str]:
        """获取事务的等待链"""
        chain = []
        current = tx_id
        visited = set()

        while current and len(chain) < max_depth:
            if current in visited:
                chain.append(f"{current} (cycle)")
                break

            visited.add(current)
            chain.append(current)

            # 找到它等待的事务
            next_tx = None
            for neighbor in self.wait_for_graph.get(current, []):
                next_tx = neighbor
                break

            current = next_tx

        return chain
```

### 2.3 死锁预防

**定义 17.2.10** ( wound-wait 策略)：

事务 $T_i$ 请求被 $T_j$ 持有的锁时：

- 若 $ts(T_i) < ts(T_j)$：$T_i$ **伤害 (wound)** $T_j$（$T_j$ 回滚）
- 若 $ts(T_i) > ts(T_j)$：$T_i$ **等待 (wait)**

**定义 17.2.11** ( wait-die 策略)：

事务 $T_i$ 请求被 $T_j$ 持有的锁时：

- 若 $ts(T_i) < ts(T_j)$：$T_i$ **等待 (wait)**
- 若 $ts(T_i) > ts(T_j)$：$T_i$ **死亡 (die)**（$T_i$ 回滚）

```python
class DeadlockPrevention:
    """
    死锁预防实现
    """

    def __init__(self, strategy: str = 'wait-die'):
        """
        Args:
            strategy: 'wait-die' 或 'wound-wait'
        """
        self.strategy = strategy
        self.transaction_timestamps: Dict[str, int] = {}
        self.global_counter = 0
        self.lock_table: Dict[str, Dict] = {}

    def begin_transaction(self, tx_id: str) -> int:
        """开始事务，分配时间戳"""
        self.global_counter += 1
        self.transaction_timestamps[tx_id] = self.global_counter
        return self.global_counter

    def request_lock(self, tx_id: str, data_item: str,
                     lock_type: str) -> str:
        """
        请求锁

        Returns:
            'granted' | 'wait' | 'abort'
        """
        if data_item not in self.lock_table:
            self.lock_table[data_item] = {
                'holder': tx_id,
                'type': lock_type
            }
            return 'granted'

        holder = self.lock_table[data_item]['holder']

        if holder == tx_id:
            return 'granted'

        ts_requester = self.transaction_timestamps[tx_id]
        ts_holder = self.transaction_timestamps[holder]

        if self.strategy == 'wait-die':
            # 老的等待，年轻的死亡
            if ts_requester < ts_holder:
                return 'wait'
            else:
                return 'abort'  # 年轻者死亡

        elif self.strategy == 'wound-wait':
            # 老的伤害年轻的，年轻的等待
            if ts_requester < ts_holder:
                return 'wound'  # 伤害持有者
            else:
                return 'wait'

        return 'wait'
```

## 3. 事务隔离级别实现

### 3.1 SQL标准隔离级别

**定义 17.2.12** (隔离级别异常)：

| 异常类型 | 描述 | 形式化定义 |
|---------|------|-----------|
| 脏读 (Dirty Read) | 读取未提交数据 | $\exists T_i, T_j: r_i[x_j] \text{ where } w_j[x] \text{ uncommitted}$ |
| 不可重复读 (Non-repeatable Read) | 两次读取结果不同 | $\exists T_i: r_i[x], r_i[x'] \text{ where } x \neq x'$ |
| 幻读 (Phantom Read) | 两次查询行数不同 | $\exists T_i: \text{pred}(R) \neq \text{pred}(R')$ |
| 写偏斜 (Write Skew) | 违反约束的并发写 | $\exists T_i, T_j: r_i[A], r_j[B], w_i[B], w_j[A]$ |

**定义 17.2.13** (SQL 隔离级别)：

| 隔离级别 | 脏读 | 不可重复读 | 幻读 | 实现机制 |
|---------|------|-----------|------|---------|
| READ UNCOMMITTED | ✓ | ✓ | ✓ | 无锁读取 |
| READ COMMITTED | ✗ | ✓ | ✓ | 读已提交版本 |
| REPEATABLE READ | ✗ | ✗ | ✓ | 快照读 |
| SERIALIZABLE | ✗ | ✗ | ✗ | 严格2PL/SI+检查 |

```python
class IsolationLevelManager:
    """
    隔离级别管理器
    """

    READ_UNCOMMITTED = 0
    READ_COMMITTED = 1
    REPEATABLE_READ = 2
    SERIALIZABLE = 3

    def __init__(self, mvcc_storage: 'MVCCStorage'):
        self.storage = mvcc_storage
        self.default_level = self.READ_COMMITTED

    def read(self, tx_id: str, key: str,
             isolation_level: int = None) -> Optional[any]:
        """根据隔离级别执行读取"""
        level = isolation_level or self.default_level

        if level == self.READ_UNCOMMITTED:
            return self._read_uncommitted(tx_id, key)
        elif level == self.READ_COMMITTED:
            return self._read_committed(tx_id, key)
        elif level == self.REPEATABLE_READ:
            return self._read_repeatable(tx_id, key)
        elif level == self.SERIALIZABLE:
            return self._read_serializable(tx_id, key)

    def _read_uncommitted(self, tx_id: str, key: str) -> Optional[any]:
        """读未提交：直接读取最新版本（无论是否提交）"""
        versions = self.storage.versions.get(key, [])
        if versions:
            return versions[-1].value
        return None

    def _read_committed(self, tx_id: str, key: str) -> Optional[any]:
        """读已提交：每次读取最新的已提交版本"""
        current_ts = self.storage.global_timestamp
        versions = self.storage.versions.get(key, [])

        for version in reversed(versions):
            # 检查版本是否已提交
            if version.expired_ts is None and version.created_ts <= current_ts:
                return version.value

        return None

    def _read_repeatable(self, tx_id: str, key: str) -> Optional[any]:
        """可重复读：使用事务开始时的快照"""
        return self.storage.read(tx_id, key)

    def _read_serializable(self, tx_id: str, key: str) -> Optional[any]:
        """串行化：快照读 + 谓词锁/索引间隙锁"""
        # 获取谓词锁
        # 这里简化实现，实际需要谓词锁管理
        return self._read_repeatable(tx_id, key)
```

### 3.2 快照隔离 (Snapshot Isolation)

**定义 17.2.14** (快照隔离)：

快照隔离保证：

1. **事务读**：从事务开始的快照读取数据
2. **事务写**：基于快照验证（first-committer-wins）

**定义 17.2.15** (写偏斜异常)：

即使在快照隔离下，仍可能发生写偏斜：

| 时间 | $T_1$ | $T_2$ |
|-----|-------|-------|
| 1 | $r_1[x=50]$ | |
| 2 | | $r_2[y=50]$ |
| 3 | $w_1[y=50-x]$ | |
| 4 | | $w_2[x=50-y]$ |
| 5 | commit | commit |

结果：$x + y = 50$（违反 $x + y \geq 0$ 约束）

```python
class SnapshotIsolation:
    """
    快照隔离实现
    """

    def __init__(self):
        self.storage = MVCCStorage()
        self.writesets: Dict[str, Set[str]] = {}  # {tx_id: {keys}}

    def begin(self, tx_id: str):
        """开始事务"""
        self.storage.begin_transaction(tx_id)
        self.writesets[tx_id] = set()

    def read(self, tx_id: str, key: str) -> Optional[any]:
        """从快照读取"""
        return self.storage.read(tx_id, key)

    def write(self, tx_id: str, key: str, value: any) -> bool:
        """
        写入数据

        使用 first-committer-wins 规则
        """
        self.writesets[tx_id].add(key)
        return self.storage.write(tx_id, key, value)

    def commit(self, tx_id: str) -> bool:
        """
        提交事务

        验证：检查写集是否与其他已提交事务冲突
        """
        with self.storage.lock:
            # 检查写冲突
            for key in self.writesets.get(tx_id, set()):
                versions = self.storage.versions.get(key, [])
                for version in versions:
                    # 如果在我们开始后，有其他事务提交了这个key
                    if (version.created_by != tx_id and
                        version.expired_ts is None and
                        version.created_ts > self.storage.active_transactions[tx_id]):
                        # 检测到写冲突，回滚
                        self.abort(tx_id)
                        return False

            # 提交成功
            commit_ts = self.storage.commit(tx_id)
            del self.writesets[tx_id]
            return True

    def abort(self, tx_id: str):
        """中止事务"""
        self.storage.rollback(tx_id)
        if tx_id in self.writesets:
            del self.writesets[tx_id]
```

### 3.3 可串行化快照隔离 (SSI)

**定义 17.2.16** (可串行化快照隔离)：

SSI 在快照隔离基础上检测写偏斜：

**定理 17.2.4** (SSI 可串行化)：

SSI 通过检测和阻止可能导致写偏斜的读写依赖循环，保证可串行化。

**检测规则**：

- **rw-依赖**：$T_i$ 读取了 $T_j$ 写入的版本，则存在 $T_i \xrightarrow{rw} T_j$
- 若存在 $T_i \xrightarrow{rw} T_j \xrightarrow{rw} T_i$，则检测到写偏斜风险

```python
class SerializableSnapshotIsolation:
    """
    可串行化快照隔离实现

    基于 PostgreSQL 的 SSI 实现
    """

    def __init__(self):
        self.storage = MVCCStorage()
        self.read_sets: Dict[str, Dict[str, int]] = {}  # {tx_id: {key: version_ts}}
        self.out_rw_conflicts: Dict[str, Set[str]] = {}  # T_i -> {T_j} (T_i 读取后 T_j 写入)
        self.in_rw_conflicts: Dict[str, Set[str]] = {}   # T_i -> {T_j} (T_j 读取后 T_i 写入)

    def begin(self, tx_id: str):
        """开始事务"""
        self.storage.begin_transaction(tx_id)
        self.read_sets[tx_id] = {}
        self.out_rw_conflicts[tx_id] = set()
        self.in_rw_conflicts[tx_id] = set()

    def read(self, tx_id: str, key: str) -> Optional[any]:
        """读取并记录读集"""
        result = self.storage.read(tx_id, key)

        # 记录读取的版本
        versions = self.storage.versions.get(key, [])
        for version in versions:
            if version.is_visible_to(self.storage.active_transactions[tx_id]):
                self.read_sets[tx_id][key] = version.created_ts
                break

        return result

    def write(self, tx_id: str, key: str, value: any) -> bool:
        """
        写入数据

        检查 rw-conflict
        """
        # 检查是否有其他事务读取了此key
        for other_tx, read_set in self.read_sets.items():
            if other_tx != tx_id and key in read_set:
                # 发现 rw-conflict: other_tx 读取后我们写入
                self.out_rw_conflicts[tx_id].add(other_tx)
                self.in_rw_conflicts[other_tx].add(tx_id)

                # 检查是否形成危险结构
                if self._is_dangerous(tx_id, other_tx):
                    # 需要回滚其中一个事务
                    self.abort(tx_id)
                    return False

        return self.storage.write(tx_id, key, value)

    def _is_dangerous(self, tx1: str, tx2: str) -> bool:
        """
        检查是否形成危险结构（写偏斜风险）

        危险结构：T1 -rw-> T2 -rw-> T1
        """
        # 检查 tx1 -> tx2 -> tx1
        if tx2 in self.out_rw_conflicts.get(tx1, set()):
            if tx1 in self.out_rw_conflicts.get(tx2, set()):
                return True

        return False

    def commit(self, tx_id: str) -> bool:
        """提交事务"""
        # 清理 rw-conflict 记录
        for other_tx in list(self.out_rw_conflicts.get(tx_id, set())):
            self.in_rw_conflicts[other_tx].discard(tx_id)

        for other_tx in list(self.in_rw_conflicts.get(tx_id, set())):
            self.out_rw_conflicts[other_tx].discard(tx_id)

        del self.read_sets[tx_id]
        del self.out_rw_conflicts[tx_id]
        del self.in_rw_conflicts[tx_id]

        return self.storage.commit(tx_id)

    def abort(self, tx_id: str):
        """中止事务"""
        # 清理冲突记录
        for other_tx in list(self.out_rw_conflicts.get(tx_id, set())):
            self.in_rw_conflicts[other_tx].discard(tx_id)

        for other_tx in list(self.in_rw_conflicts.get(tx_id, set())):
            self.out_rw_conflicts[other_tx].discard(tx_id)

        if tx_id in self.read_sets:
            del self.read_sets[tx_id]
        if tx_id in self.out_rw_conflicts:
            del self.out_rw_conflicts[tx_id]
        if tx_id in self.in_rw_conflicts:
            del self.in_rw_conflicts[tx_id]

        self.storage.rollback(tx_id)
```

## 4. 分布式事务

### 4.1 两阶段提交 (2PC)

**定义 17.2.17** (两阶段提交协议)：

2PC 由协调者 (Coordinator) 和参与者 (Participants) 组成：

**阶段一 (投票)**：

1. 协调者发送 $\text{PREPARE}$ 给所有参与者
2. 参与者执行本地事务，若可提交则写 Prepare 日志并回复 $\text{YES}$
3. 若不能提交则回复 $\text{NO}$ 并回滚

**阶段二 (提交/回滚)**：

1. 若所有回复 $\text{YES}$，协调者写 Commit 日志，发送 $\text{COMMIT}$
2. 若有 $\text{NO}$ 或超时，协调者写 Abort 日志，发送 $\text{ROLLBACK}$
3. 参与者收到后执行并回复确认

```python
import asyncio
from enum import Enum, auto
from typing import List, Dict, Callable
import logging

class TwoPhaseCommit:
    """
    两阶段提交协议实现
    """

    class State(Enum):
        INIT = auto()
        PREPARING = auto()
        PREPARED = auto()
        COMMITTING = auto()
        COMMITTED = auto()
        ABORTING = auto()
        ABORTED = auto()

    def __init__(self, coordinator_id: str, participant_ids: List[str]):
        self.coordinator_id = coordinator_id
        self.participant_ids = participant_ids
        self.state = self.State.INIT
        self.votes: Dict[str, bool] = {}
        self.prepare_timeout = 30  # 秒
        self.commit_timeout = 30
        self.logger = logging.getLogger(f"2PC-{coordinator_id}")

    async def commit(self, transaction: Dict) -> bool:
        """
        执行两阶段提交

        Args:
            transaction: 事务内容

        Returns:
            True if 提交成功
        """
        # 阶段一：准备
        self.state = self.State.PREPARING
        self.logger.info("Phase 1: Preparing transaction")

        # 发送PREPARE给所有参与者
        prepare_tasks = [
            self._send_prepare(pid, transaction)
            for pid in self.participant_ids
        ]

        try:
            results = await asyncio.wait_for(
                asyncio.gather(*prepare_tasks, return_exceptions=True),
                timeout=self.prepare_timeout
            )
        except asyncio.TimeoutError:
            self.logger.error("Prepare phase timeout")
            await self._abort()
            return False

        # 收集投票
        for pid, result in zip(self.participant_ids, results):
            if isinstance(result, Exception):
                self.votes[pid] = False
            else:
                self.votes[pid] = result

        # 决策
        all_prepared = all(self.votes.values())

        if all_prepared:
            # 阶段二：提交
            self.state = self.State.PREPARED
            self.logger.info("All participants voted YES, committing")
            return await self._do_commit()
        else:
            # 回滚
            self.logger.info(f"Some participants voted NO: {self.votes}")
            return await self._abort()

    async def _send_prepare(self, participant_id: str,
                           transaction: Dict) -> bool:
        """发送准备请求"""
        # 模拟网络调用
        participant = Participant(participant_id)
        return await participant.prepare(transaction)

    async def _do_commit(self) -> bool:
        """执行提交"""
        self.state = self.State.COMMITTING
        self.logger.info("Phase 2: Committing")

        # 写提交日志（持久化）
        self._write_log("COMMIT")

        # 发送COMMIT给所有参与者
        commit_tasks = [
            self._send_commit(pid)
            for pid in self.participant_ids
        ]

        try:
            await asyncio.wait_for(
                asyncio.gather(*commit_tasks),
                timeout=self.commit_timeout
            )
        except asyncio.TimeoutError:
            # 需要恢复处理
            self.logger.error("Commit phase timeout, need recovery")

        self.state = self.State.COMMITTED
        return True

    async def _abort(self) -> bool:
        """执行回滚"""
        self.state = self.State.ABORTING
        self.logger.info("Phase 2: Aborting")

        # 写中止日志
        self._write_log("ABORT")

        # 发送ROLLBACK给所有参与者
        abort_tasks = [
            self._send_rollback(pid)
            for pid in self.participant_ids
        ]

        await asyncio.gather(*abort_tasks, return_exceptions=True)

        self.state = self.State.ABORTED
        return False

    async def _send_commit(self, participant_id: str):
        """发送提交请求"""
        participant = Participant(participant_id)
        await participant.commit()

    async def _send_rollback(self, participant_id: str):
        """发送回滚请求"""
        participant = Participant(participant_id)
        await participant.rollback()

    def _write_log(self, record: str):
        """写日志（模拟）"""
        self.logger.info(f"LOG: {record}")


class Participant:

    def __init__(self, participant_id: str):
```

### 4.2 三阶段提交 (3PC)

**定义 17.2.18** (三阶段提交协议)：

3PC 在2PC基础上增加预提交阶段，减少阻塞：

1. **CanCommit**：协调者询问参与者是否可以执行
2. **PreCommit**：参与者预执行，锁定资源
3. **DoCommit**：最终提交

**优点**：协调者在PreCommit后崩溃，参与者可以超时后继续提交

### 4.3 Paxos 共识协议

**定义 17.2.19** (Paxos)：

Paxos 解决分布式系统中的共识问题：

- **Proposer**：提出提案
- **Acceptor**：接受/拒绝提案
- **Learner**：学习已确定的值

**Paxos 两个阶段**：

**Phase 1 (Prepare)**：

- Proposer 发送 $\text{Prepare}(n)$ 给多数 Acceptor
- Acceptor 回复 $\text{Promise}(n, v)$ 或拒绝

**Phase 2 (Accept)**：

- Proposer 收到多数 Promise 后，发送 $\text{Accept}(n, v)$
- Acceptor 接受并广播

**定理 17.2.5** (Paxos 安全性)：

Paxos 保证：

1. 只有一个值被选定
2. 只有被提出的值能被选定
3. 只有被选定的值能被学习

**证明概要**：

**唯一性**：若两个不同值 $v_1, v_2$ 都被选定：

- 存在多数集 $Q_1$ 接受 $v_1$，多数集 $Q_2$ 接受 $v_2$
- $Q_1 \cap Q_2 \neq \emptyset$
- 交集中的 Acceptor 必须接受两个值
- 与协议规则矛盾

$\square$

```python
import random
from typing import Set, Optional, Tuple

class PaxosNode:
    """
    Paxos节点（可同时作为Proposer和Acceptor）
    """

    def __init__(self, node_id: str, all_nodes: Set[str]):
        self.node_id = node_id
        self.all_nodes = all_nodes
        self.majority = len(all_nodes) // 2 + 1

        # Acceptor状态
        self.promised_n = 0  # 已承诺的最高提案号
        self.accepted_n = 0  # 已接受的提案号
        self.accepted_v = None  # 已接受的值

        # Proposer状态
        self.proposal_n = node_id  # 提案号前缀

    def generate_proposal_number(self) -> str:
        """生成唯一递增的提案号"""
        self.proposal_n += len(self.all_nodes)
        return f"{self.proposal_n}-{self.node_id}"

    def handle_prepare(self, proposal_n: str) -> Optional[Tuple[str, any]]:
        """
        处理Prepare请求

        Returns:
            None if 拒绝
            (accepted_n, accepted_v) if 已接受过提案
        """
        if proposal_n > self.promised_n:
            self.promised_n = proposal_n
            if self.accepted_n > 0:
                return (self.accepted_n, self.accepted_v)
            return (0, None)
        return None

    def handle_accept(self, proposal_n: str, value: any) -> bool:

        if proposal_n >= self.promised_n:
            self.promised_n = proposal_n
            self.accepted_n = proposal_n
            self.accepted_v = value
            return True
        return False
```

## 5. 形式化可串行化理论

### 5.1 冲突可串行化

**定义 17.2.20** (冲突操作)：

操作 $op_i$ 和 $op_j$ 冲突当且仅当：

1. 属于不同事务 ($i \neq j$)
2. 操作同一数据项
3. 至少一个是写操作

冲突操作对：
$$\text{Conflicts} = \{(r_i[x], w_j[x]), (w_i[x], r_j[x]), (w_i[x], w_j[x]) | i \neq j\}$$

**定义 17.2.21** (冲突可串行化)：

调度 $S$ 是冲突可串行化的，如果其冲突图无环：
$$CG(S) = (V, E), \text{ 其中 } (T_i, T_j) \in E \text{ if } op_i[k] <_S op_j[k] \text{ and } op_i, op_j \text{ 冲突}$$

**定理 17.2.6** (冲突可串行化判定)：

调度 $S$ 冲突可串行化 $\iff$ $CG(S)$ 无环。

**证明**：

**充分性**：若 $CG(S)$ 无环，则存在拓扑序 $\prec$。
构造串行调度 $S'$ 按 $\prec$ 顺序执行事务。
对于任意冲突操作对 $(op_i, op_j)$：

- 若 $T_i \prec T_j$，则 $op_i <_{S'} op_j$
- 在 $S$ 中 $op_i <_S op_j$（因为 $(T_i, T_j) \in E$）
- 因此 $S$ 和 $S'$ 冲突等价

**必要性**：若 $S$ 冲突可串行化于串行调度 $S_\sigma$：

- 对于任意 $(T_i, T_j) \in E$，有 $T_i$ 在 $S_\sigma$ 中先于 $T_j$
- 若 $CG(S)$ 有环，则存在 $T_1 \prec ... \prec T_n \prec T_1$
- 与串行调度的全序矛盾

$\square$

```python
class SerializableChecker:
    """
    可串行化检查器
    """

    def __init__(self):
        self.operations = []  # [(tx_id, op_type, data_item, timestamp)]

    def add_operation(self, tx_id: str, op_type: str,
                     data_item: str, timestamp: int):
        """记录操作"""
        self.operations.append((tx_id, op_type, data_item, timestamp))

    def build_conflict_graph(self) -> Dict[str, Set[str]]:
        """
        构建冲突图

        Returns:
            {tx_id: {conflicting_tx_ids}}
        """
        graph = {}

        # 按时间排序
        sorted_ops = sorted(self.operations, key=lambda x: x[3])

        # 对每个数据项，检查操作冲突
        data_item_ops = {}
        for tx_id, op_type, data_item, ts in sorted_ops:
            if data_item not in data_item_ops:
                data_item_ops[data_item] = []
            data_item_ops[data_item].append((tx_id, op_type, ts))

        # 构建冲突边
        for data_item, ops in data_item_ops.items():
            for i, (tx_i, op_i, ts_i) in enumerate(ops):
                for tx_j, op_j, ts_j in ops[i+1:]:
                    if tx_i != tx_j and self._is_conflict(op_i, op_j):
                        # 添加边 tx_i -> tx_j
                        if tx_i not in graph:
                            graph[tx_i] = set()
                        graph[tx_i].add(tx_j)
                        break  # 只添加第一条边

        return graph

    def _is_conflict(self, op1: str, op2: str) -> bool:
        """检查操作是否冲突"""
        # 至少一个是写操作
        return 'w' in op1 or 'w' in op2

    def is_conflict_serializable(self) -> Tuple[bool, Optional[List[str]]]:
        """

        """

        # 拓扑排序检测环
        in_degree = {tx: 0 for tx in graph}
        for tx, neighbors in graph.items():
            for neighbor in neighbors:
                in_degree[neighbor] = in_degree.get(neighbor, 0) + 1

        queue = [tx for tx, deg in in_degree.items() if deg == 0]
        topo_order = []

        while queue:
            tx = queue.pop(0)
            topo_order.append(tx)

            for neighbor in graph.get(tx, []):
                in_degree[neighbor] -= 1
                if in_degree[neighbor] == 0:
                    queue.append(neighbor)

        # 检查所有节点是否都被访问
        all_txs = set(op[0] for op in self.operations)
        is_serializable = len(topo_order) == len(all_txs)

        return is_serializable, topo_order if is_serializable else None
```

### 5.2 视图可串行化

**定义 17.2.22** (视图等价)：

两个调度 $S_1, S_2$ 视图等价当且仅当：

1. **初始读等价**：若 $T_i$ 在 $S_1$ 中读取了初始值，则在 $S_2$ 中也读取初始值
2. **读来源等价**：若 $T_i$ 在 $S_1$ 中读取了 $T_j$ 写入的值，则在 $S_2$ 中也如此
3. **最终写等价**：最后写入每个数据项的事务相同

**定义 17.2.23** (视图可串行化)：

调度 $S$ 是视图可串行化的，如果存在串行调度 $S'$ 使得 $S \equiv_{view} S'$。

**定理 17.2.7** (视图可串行化与冲突可串行化关系)：

$$\text{Conflict-Serializable} \subset \text{View-Serializable}$$

即：所有冲突可串行化调度都是视图可串行化的，但反之不成立。

### 5.3 可恢复性与无级联性

**定义 17.2.24** (可恢复调度)：

调度 $S$ 是可恢复的，如果：
$$\forall T_i, T_j: \text{if } T_j \text{ reads from } T_i \text{ and } c_j \in S, \text{ then } c_i <_S c_j$$

**定义 17.2.25** (无级联调度)：

调度 $S$ 是无级联的，如果：
$$\forall T_i, T_j: \text{if } T_j \text{ reads from } T_i, \text{ then } c_i <_S r_j[x]$$

```python
class ScheduleAnalyzer:
    """
    调度分析器
    """

    def __init__(self, schedule: List[Tuple[str, str, str]]):
        """
        Args:
            schedule: [(tx_id, operation, data_item)]
                   operation: 'r'(read), 'w'(write), 'c'(commit), 'a'(abort)
        """
        self.schedule = schedule

    def is_recoverable(self) -> bool:
        """检查是否可恢复"""
        # 记录读取来源
        read_from = {}  # {(tx_id, data_item): source_tx_id}
        committed = set()

        for i, (tx_id, op, data_item) in enumerate(self.schedule):
            if op == 'r':
                # 查找最近的写入
                for j in range(i-1, -1, -1):
                    prev_tx, prev_op, prev_item = self.schedule[j]
                    if prev_op == 'w' and prev_item == data_item:
                        read_from[(tx_id, data_item)] = prev_tx
                        break

            elif op == 'c':
                committed.add(tx_id)
                # 检查：如果此事务读了其他事务的数据，那些事务必须已提交
                for (reader, item), source in read_from.items():
                    if reader == tx_id and source not in committed:
                        return False

        return True

    def is_cascadeless(self) -> bool:
        """检查是否无级联"""
        read_from = {}
        committed = set()

        for i, (tx_id, op, data_item) in enumerate(self.schedule):
            if op == 'r':
                # 查找写入者
                for j in range(i-1, -1, -1):
                    prev_tx, prev_op, prev_item = self.schedule[j]
                    if prev_op == 'w' and prev_item == data_item:
                        if prev_tx not in committed:
                            return False
                        read_from[(tx_id, data_item)] = prev_tx
                        break

            elif op == 'c':
                committed.add(tx_id)

        return True

    def is_strict(self) -> bool:
        """检查是否严格"""
        # 严格：不能读/写未提交的写
        uncommitted_writes = {}  # {data_item: tx_id}
        committed = set()

        for tx_id, op, data_item in self.schedule:
            if op == 'w':
                uncommitted_writes[data_item] = tx_id

            elif op in ('r', 'w'):
                # 检查是否有未提交的写入
                if data_item in uncommitted_writes:
                    writer = uncommitted_writes[data_item]
                    if writer != tx_id and writer not in committed:
                        return False

            elif op == 'c':
                committed.add(tx_id)

            elif op == 'a':
                # 中止时，清理写入
                for item, writer in list(uncommitted_writes.items()):
                    if writer == tx_id:
                        del uncommitted_writes[item]

        return True
```

## 7. 事务性能调优

### 7.1 性能指标与监控

```python
import time
from dataclasses import dataclass
from typing import Dict, List
import statistics

@dataclass
class TransactionMetrics:
    """事务性能指标"""
    tx_id: str
    start_time: float
    commit_time: float = None
    operations: int = 0
    locks_held: int = 0
    wait_time: float = 0

    @property
    def duration(self) -> float:
        if self.commit_time:
            return self.commit_time - self.start_time
        return time.time() - self.start_time


class TransactionMonitor:
    """事务性能监控器"""

    def __init__(self):
        self.active_transactions: Dict[str, TransactionMetrics] = {}
        self.completed_transactions: List[TransactionMetrics] = []
        self.conflicts = 0
        self.deadlocks = 0

    def start_transaction(self, tx_id: str):
        """记录事务开始"""
        self.active_transactions[tx_id] = TransactionMetrics(
            tx_id=tx_id,
            start_time=time.time()
        )

    def record_operation(self, tx_id: str):
        """记录操作"""
        if tx_id in self.active_transactions:
            self.active_transactions[tx_id].operations += 1

    def record_wait(self, tx_id: str, wait_duration: float):
        """记录等待时间"""
        if tx_id in self.active_transactions:
            self.active_transactions[tx_id].wait_time += wait_duration

    def commit_transaction(self, tx_id: str):
        """记录事务提交"""
        if tx_id in self.active_transactions:
            metric = self.active_transactions[tx_id]
            metric.commit_time = time.time()
            self.completed_transactions.append(metric)
            del self.active_transactions[tx_id]

    def get_statistics(self) -> Dict:
        """获取统计信息"""
        if not self.completed_transactions:
            return {}

        durations = [t.duration for t in self.completed_transactions]
        wait_times = [t.wait_time for t in self.completed_transactions]

        return {
            'total_transactions': len(self.completed_transactions),
            'avg_duration': statistics.mean(durations),
            'p99_duration': sorted(durations)[int(len(durations) * 0.99)],
            'max_duration': max(durations),
            'avg_wait_time': statistics.mean(wait_times),
            'conflict_rate': self.conflicts / len(self.completed_transactions),
            'deadlock_rate': self.deadlocks / len(self.completed_transactions),
            'throughput': len(self.completed_transactions) / sum(durations)
        }
```

### 7.2 锁调优策略

```python
class LockTuner:
    """锁调优建议器"""

    def __init__(self, monitor: TransactionMonitor):
        self.monitor = monitor

    def analyze_and_recommend(self) -> List[str]:
        """分析并给出调优建议"""
        recommendations = []
        stats = self.monitor.get_statistics()

        if not stats:
            return ["No data available for analysis"]

        # 检查等待时间
        if stats['avg_wait_time'] / stats['avg_duration'] > 0.3:
            recommendations.append(
                "High lock wait ratio detected (>30%). "
                "Consider: 1) Reducing transaction size "
                "2) Using more granular locks "
                "3) Optimizing access patterns"
            )

        # 检查冲突率
        if stats['conflict_rate'] > 0.1:
            recommendations.append(
                f"High conflict rate ({stats['conflict_rate']:.2%}). "
                "Consider: 1) Using optimistic locking "
                "2) Implementing exponential backoff "
                "3) Partitioning hot data"
            )

        # 检查死锁率
        if stats['deadlock_rate'] > 0.01:
            recommendations.append(
                f"Deadlock rate is {stats['deadlock_rate']:.2%}. "
                "Consider: 1) Enforcing consistent lock ordering "
                "2) Using try-lock with timeout "
                "3) Reducing transaction isolation level"
            )

        # 检查事务大小
        avg_ops = statistics.mean([
            t.operations for t in self.monitor.completed_transactions
        ])
        if avg_ops > 1000:
            recommendations.append(
                f"Large transactions detected (avg {avg_ops:.0f} ops). "
                "Consider splitting into smaller transactions"
            )

        return recommendations

    def recommend_isolation_level(self, workload_type: str) -> str:
        """
        推荐隔离级别

        Args:
            workload_type: 'read-heavy' | 'write-heavy' | 'mixed'
        """
        if workload_type == 'read-heavy':
            return "READ COMMITTED with MVCC for best read performance"
        elif workload_type == 'write-heavy':
            return "READ COMMITTED to minimize lock contention"
        else:
            return "REPEATABLE READ as balanced choice"
```

### 7.3 优化实践总结

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 锁等待高 | 大事务、长事务 | 缩小事务范围、批量分批处理 |
| 死锁频繁 | 不一致的加锁顺序 | 统一访问顺序、使用超时 |
| 幻读问题 | 缺乏间隙锁 | 使用 SERIALIZABLE 或 SSI |
| 写偏斜 | 快照隔离局限 | 升级到可串行化快照隔离 |
| 版本膨胀 | MVCC旧版本积累 | 调整 vacuum/autovacuum |
| 热点数据 | 高并发访问同一行 | 分区、缓存、应用层排队 |
