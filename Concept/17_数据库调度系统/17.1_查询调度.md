### 1.1 优化器整体架构

### 1.2 形式化定义：查询优化器

**定义 17.1.1** (查询优化器)：

查询优化器是一个三元组 $\mathcal{O} = (Q, P, C)$，其中：

- $Q$ 是合法查询的集合
- $P$ 是执行计划的集合
- $C: P \rightarrow \mathbb{R}^+$ 是代价函数

优化器的目标是：
$$\text{Given } q \in Q, \text{ find } p^* = \arg\min_{p \in P(q)} C(p)$$

其中 $P(q) \subseteq P$ 是查询 $q$ 的所有等价执行计划。

### 1.3 优化器分类

**定义 17.1.2** (基于规则的优化器 RBO)：

RBO 使用启发式规则进行优化，不依赖数据分布统计信息：
$$\mathcal{O}_{RBO}(q) = \text{apply}(R_n \circ R_{n-1} \circ \cdots \circ R_1, q)$$

其中 $R_i$ 是第 $i$ 个重写规则。

**定义 17.1.3** (基于代价的优化器 CBO)：

CBO 基于统计信息和代价模型选择最优计划：
$$\mathcal{O}_{CBO}(q) = \arg\min_{p \in \text{Enum}(q)} \text{Cost}(p, \mathcal{S})$$

其中 $\mathcal{S}$ 是统计信息，$\text{Enum}(q)$ 是查询 $q$ 的枚举计划空间。

### 1.4 优化器模块详解

#### 1.4.1 逻辑优化器

逻辑优化在关系代数层面进行等价变换：

| 优化规则 | 变换前 | 变换后 |
|---------|--------|--------|
| 谓词下推 | $\sigma_p(R \bowtie S)$ | $\sigma_p(R) \bowtie S$ (若 $p$ 只涉及 $R$) |
| 投影下推 | $\pi_L(R \bowtie S)$ | $\pi_{L_R}(R) \bowtie \pi_{L_S}(S)$ |
| 连接重排 | $R \bowtie (S \bowtie T)$ | $(R \bowtie S) \bowtie T$ |
| 子查询展开 | $\sigma_{a \in (\subquery)}(R)$ | $R \semijoin_{a=b} S$ |

**定理 17.1.1** (逻辑优化保持等价性)：

对于任意关系 $R, S$ 和选择条件 $p$，有：
$$\sigma_p(R \times S) \equiv \sigma_p(R) \times S \quad \text{若 } \text{attrs}(p) \subseteq \text{schema}(R)$$

**证明**：

设 $t \in \sigma_p(R \times S)$，则：

1. $t \in R \times S$ 且 $p(t) = \text{true}$
2. $t = (t_R, t_S)$ 其中 $t_R \in R, t_S \in S$
3. 由于 $\text{attrs}(p) \subseteq \text{schema}(R)$，有 $p(t) = p(t_R)$
4. 故 $p(t_R) = \text{true}$，即 $t_R \in \sigma_p(R)$
5. 因此 $t \in \sigma_p(R) \times S$

反之同理可证。$\square$

#### 1.4.2 物理优化器

物理优化选择具体的执行算法：

```
┌──────────────────────────────────────────────────────────────┐
│                    物理算子选择                                │
├──────────────────────────────────────────────────────────────┤
│  连接算子：                                                   │
│    - Nested Loop Join: O(|R| × |S|)                          │
│    - Hash Join: O(|R| + |S|) 内存需求大                        │
│    - Sort-Merge Join: O(|R|log|R| + |S|log|S|)               │
│    - Index Nested Loop: O(|R| × log|S|) 需索引                │
│                                                              │
│  选择算子：                                                   │
│    - Full Scan: O(|R|)                                       │
│    - Index Scan: O(log|R| + k) k为匹配行数                    │
│    - Bitmap Scan: O(log|R| + m) m为匹配页数                   │
│                                                              │
│  聚合算子：                                                   │
│    - Hash Aggregate: O(|R|) 内存                             │
│    - Sort Aggregate: O(|R|log|R|)                            │
│    - Stream Aggregate: O(|R|) 需有序                          │
└──────────────────────────────────────────────────────────────┘
```

## 2. 查询计划生成算法

### 2.1 动态规划算法

**定义 17.1.4** (最优子结构)：

对于连接树 $T = T_1 \bowtie T_2$，其最优代价满足：
$$\text{OPT}(T) = \min_{\text{plans}(T_1), \text{plans}(T_2)} \left\{ \text{OPT}(T_1) + \text{OPT}(T_2) + \text{CostJoin}(T_1, T_2) \right\}$$

**算法 17.1.1** (DPsize - 基于子集大小的动态规划)：

```python
def DPsize(relations):
    """
    基于子集大小的连接顺序动态规划算法

    Args:
        relations: 关系集合 {R1, R2, ..., Rn}

    Returns:
        最优连接计划
    """
    n = len(relations)
    # best_plan[S] = (cost, plan) 对于子集S
    best_plan = {}

    # 初始化：单表扫描
    for R in relations:
        S = frozenset([R])
        cost = estimate_scan_cost(R)
        best_plan[S] = (cost, Scan(R))

    # 按子集大小递增
    for size in range(2, n + 1):
        # 枚举所有size大小的子集
        for S in subsets_of_size(relations, size):
            best_cost = float('inf')
            best_p = None

            # 枚举所有可能的分割 S1 × S2 = S
            for S1 in proper_subsets(S):
                S2 = S - S1
                if not connected(S1, S2):
                    continue

                cost1, plan1 = best_plan[S1]
                cost2, plan2 = best_plan[S2]

                # 尝试所有连接算法
                for join_op in [HashJoin, MergeJoin, NLJoin]:
                    join_cost = estimate_join_cost(join_op, S1, S2)
                    total_cost = cost1 + cost2 + join_cost

                    if total_cost < best_cost:
                        best_cost = total_cost
                        best_p = join_op(plan1, plan2)

            best_plan[S] = (best_cost, best_p)

    all_relations = frozenset(relations)
    return best_plan[all_relations]
```

**定理 17.1.2** (DPsize 最优性)：

DPsize 算法返回的连接顺序具有最小执行代价。

**证明**：

对子集大小 $k$ 进行归纳：

**基础**：$k=1$ 时，单表扫描是唯一选择，显然最优。

**归纳假设**：对于所有大小 $< k$ 的子集，算法已找到最优计划。

**归纳步骤**：对于大小为 $k$ 的子集 $S$：

1. 算法枚举了 $S$ 的所有二分割 $(S_1, S_2)$
2. 由归纳假设，$best\_plan[S_1]$ 和 $best\_plan[S_2]$ 是最优的
3. 对于每个分割，算法尝试了所有物理连接算子
4. 因此 $best\_plan[S]$ 考虑了所有可能的左深/右深/浓密树计划
5. 取最小值保证最优性

$\square$

### 2.2 遗传算法与启发式搜索

对于大规模查询（表数量 > 15），动态规划的复杂度 $O(3^n)$ 过高，需要启发式算法。

**算法 17.1.2** (遗传算法查询优化)：

```python
import random
from typing import List, Tuple

class QueryOptimizerGA:
    """基于遗传算法的查询优化器"""

    def __init__(self, relations: List[str], population_size=100,
                 mutation_rate=0.1, crossover_rate=0.8):
        self.relations = relations
        self.population_size = population_size
        self.mutation_rate = mutation_rate
        self.crossover_rate = crossover_rate
        self.n = len(relations)

    def encode_plan(self, join_order: List[int]) -> str:
        """将连接顺序编码为染色体"""
        return ''.join(chr(ord('A') + i) for i in join_order)

    def decode_plan(self, chromosome: str) -> List[int]:
        """将染色体解码为连接顺序"""
        return [ord(c) - ord('A') for c in chromosome]

    def initialize_population(self) -> List[str]:
        """初始化种群"""
        population = []
        base = list(range(self.n))
        for _ in range(self.population_size):
            perm = base.copy()
            random.shuffle(perm)
            population.append(self.encode_plan(perm))
        return population

    def fitness(self, chromosome: str) -> float:
        """计算适应度（代价的倒数）"""
        join_order = self.decode_plan(chromosome)
        cost = self.estimate_cost(join_order)
        return 1.0 / (1.0 + cost)

    def estimate_cost(self, join_order: List[int]) -> float:
        """估算连接代价"""
        # 简化的代价模型
        total_cost = 0
        for i in range(1, len(join_order)):
            left_size = self.get_relation_size(join_order[i-1])
            right_size = self.get_relation_size(join_order[i])
            total_cost += left_size * right_size * 0.1
        return total_cost

    def get_relation_size(self, idx: int) -> int:
        """获取关系基数"""
        # 模拟不同表的基数
        sizes = [1000, 5000, 2000, 800, 3000, 1500]
        return sizes[idx % len(sizes)]

    def crossover(self, p1: str, p2: str) -> Tuple[str, str]:
        """有序交叉 (OX)"""
        if random.random() > self.crossover_rate:
            return p1, p2

        n = len(p1)
        cut1, cut2 = sorted(random.sample(range(n), 2))

        # 创建子代
        def create_child(parent1, parent2):
            child = [''] * n
            # 复制片段
            child[cut1:cut2] = parent1[cut1:cut2]
            # 填充剩余
            used = set(child[cut1:cut2])
            ptr = cut2 % n
            for c in parent2[cut2:] + parent2[:cut2]:
                if c not in used:
                    child[ptr] = c
                    ptr = (ptr + 1) % n
            return ''.join(child)

        return create_child(p1, p2), create_child(p2, p1)

    def mutate(self, chromosome: str) -> str:
        """交换变异"""
        if random.random() > self.mutation_rate:
            return chromosome

        chars = list(chromosome)
        i, j = random.sample(range(len(chars)), 2)
        chars[i], chars[j] = chars[j], chars[i]
        return ''.join(chars)

    def select(self, population: List[str], fitnesses: List[float]) -> str:
        """轮盘赌选择"""
        total = sum(fitnesses)
        pick = random.uniform(0, total)
        current = 0
        for chrom, fit in zip(population, fitnesses):
            current += fit
            if current >= pick:
                return chrom
        return population[-1]

    def optimize(self, generations=200) -> Tuple[List[int], float]:
        """执行遗传算法优化"""
        population = self.initialize_population()

        best_chrom = None
        best_cost = float('inf')

        for gen in range(generations):
            fitnesses = [self.fitness(c) for c in population]

            # 更新最优
            for chrom, fit in zip(population, fitnesses):
                cost = 1.0 / fit - 1.0
                if cost < best_cost:
                    best_cost = cost
                    best_chrom = chrom

            # 生成新一代
            new_population = []
            while len(new_population) < self.population_size:
                p1 = self.select(population, fitnesses)
                p2 = self.select(population, fitnesses)
                c1, c2 = self.crossover(p1, p2)
                new_population.append(self.mutate(c1))
                if len(new_population) < self.population_size:
                    new_population.append(self.mutate(c2))

            population = new_population

        return self.decode_plan(best_chrom), best_cost
```

### 2.3 爬山算法与模拟退火

**算法 17.1.3** (模拟退火查询优化)：

```python
import math
import random

def simulated_annealing_optimize(relations, initial_temp=1000,
                                  cooling_rate=0.995, iterations=10000):
    """
    模拟退火算法用于查询优化
    """
    # 初始解
    current_order = list(range(len(relations)))
    random.shuffle(current_order)
    current_cost = estimate_total_cost(current_order)

    best_order = current_order.copy()
    best_cost = current_cost

    temperature = initial_temp

    for i in range(iterations):
        # 生成邻居解（交换两个表）
        neighbor = current_order.copy()
        idx1, idx2 = random.sample(range(len(neighbor)), 2)
        neighbor[idx1], neighbor[idx2] = neighbor[idx2], neighbor[idx1]

        neighbor_cost = estimate_total_cost(neighbor)
        delta = neighbor_cost - current_cost

        # 接受准则
        if delta < 0 or random.random() < math.exp(-delta / temperature):
            current_order = neighbor
            current_cost = neighbor_cost

            if current_cost < best_cost:
                best_cost = current_cost
                best_order = current_order.copy()

        # 降温
        temperature *= cooling_rate

        if temperature < 0.001:
            break

    return best_order, best_cost

def estimate_total_cost(join_order):
    """估算连接代价"""
    # 使用简单的左深树代价模型
    total_cost = 0
    cardinality = get_cardinality(join_order[0])

    for i in range(1, len(join_order)):
        next_card = get_cardinality(join_order[i])
        selectivity = estimate_selectivity(join_order[i-1], join_order[i])

        # 连接代价 = 左表大小 + 输出大小
        join_cost = cardinality + cardinality * next_card * selectivity
        total_cost += join_cost

        # 更新基数
        cardinality = cardinality * next_card * selectivity

    return total_cost

def get_cardinality(table_idx):
    """获取表基数"""
    base_cardinalities = [10000, 50000, 20000, 8000, 30000]
    return base_cardinalities[table_idx % len(base_cardinalities)]

def estimate_selectivity(t1, t2):
    """估计连接选择性"""
    return 0.1
```

## 3. 代价模型

### 3.1 CBO 代价模型形式化

**定义 17.1.5** (代价模型)：

代价模型是一个函数 $C: \mathcal{P} \times \mathcal{S} \rightarrow \mathbb{R}^+$，其中：

- $\mathcal{P}$ 是物理计划空间
- $\mathcal{S}$ 是统计信息空间

$$C(p, \mathcal{S}) = \alpha \cdot \text{IOCost}(p, \mathcal{S}) + \beta \cdot \text{CPUCost}(p, \mathcal{S}) + \gamma \cdot \text{MemCost}(p, \mathcal{S})$$

其中 $\alpha, \beta, \gamma$ 是系统特定的权重系数。

### 3.2 I/O 代价计算

**定理 17.1.3** (顺序扫描 I/O 代价)：

对于关系 $R$，顺序扫描的 I/O 代价为：
$$\text{IO}_{seq}(R) = \lceil |R| / B \rceil$$

其中 $|R|$ 是 $R$ 的页数，$B$ 是每页可处理的页数（缓冲池大小）。

**证明**：

顺序读取需要遍历所有页，每次 I/O 可读取连续的多页。设总页数为 $N = |R|$，每次可读取 $B$ 页，则最少需要 $\lceil N/B \rceil$ 次 I/O。$\square$

**定义 17.1.6** (随机 I/O 代价)：

对于索引扫描，随机读取的代价为：
$$\text{IO}_{rand}(R, k) = h + \lceil k / f \rceil + k$$

其中：

- $h$：索引树高度
- $k$：匹配的记录数
- $f$：每页的指针数

### 3.3 连接代价详细公式

**定义 17.1.7** (哈希连接代价)：

哈希连接 $R \bowtie_H S$ 的代价为：

$$
\text{Cost}_{HJ}(R, S) = \begin{cases}
2(|R| + |S|) + |R \bowtie S| & \text{若 } |R| \leq M - 2 \\
2(|R| + |S|) \cdot \lceil |R| / (M - 2) \rceil + |R \bowtie S| & \text{否则}
\end{cases}
$$

其中 $M$ 是可用缓冲页数，第一阶段构建哈希表，第二阶段探查。

**定义 17.1.8** (归并连接代价)：

归并连接 $R \bowtie_M S$ 的代价为：

$$
\text{Cost}_{MJ}(R, S) = \begin{cases}
|R|\log|R| + |S|\log|S| + |R| + |S| + |R \bowtie S| & \text{若需排序} \\
|R| + |S| + |R \bowtie S| & \text{若已有序}
\end{cases}
$$

**定义 17.1.9** (嵌套循环连接代价)：

嵌套循环连接 $R \bowtie_{NL} S$ 的代价为：

$$
\text{Cost}_{NL}(R, S) = \begin{cases}
|R| + |R| \cdot |S| & \text{朴素 NLJ} \\
|R| + |R| \cdot (h_S + 1) & \text{索引 NLJ}
\end{cases}
$$

其中 $h_S$ 是 $S$ 上索引的高度。

### 3.4 基数估计

**定义 17.1.10** (选择性估计)：

对于选择条件 $\sigma_{A=\theta}(R)$，选择性为：
$$\text{sel}(A = \theta) = \frac{1}{|\text{Dom}(A)|} \text{ (无直方图)}$$

或使用直方图：
$$\text{sel}(A = \theta) = \frac{\text{bin}(\theta)}{|R|}$$

**定义 17.1.11** (连接结果估计)：

对于等值连接 $R \bowtie_{R.A = S.B} S$：

$$|R \bowtie S| = \frac{|R| \cdot |S|}{\max(|\pi_A(R)|, |\pi_B(S)|)}$$

假设均匀分布，使用最大不同值数作为分母。

### 3.5 直方图统计

```python
class Histogram:
    """等深直方图实现"""

    def __init__(self, num_buckets=100):
        self.num_buckets = num_buckets
        self.buckets = []  # [(min_val, max_val, count), ...]
        self.total_count = 0

    def build(self, values):
        """从数据构建等深直方图"""
        sorted_vals = sorted(values)
        n = len(sorted_vals)
        self.total_count = n

        bucket_size = n // self.num_buckets
        self.buckets = []

        for i in range(self.num_buckets):
            start_idx = i * bucket_size
            end_idx = (i + 1) * bucket_size if i < self.num_buckets - 1 else n

            bucket_vals = sorted_vals[start_idx:end_idx]
            self.buckets.append({
                'min': bucket_vals[0],
                'max': bucket_vals[-1],
                'count': len(bucket_vals),
                'distinct': len(set(bucket_vals))
            })

    def estimate_selectivity(self, op, value):
        """估算选择性"""
        if op == '=':
            return self._estimate_equality(value)
        elif op == '<':
            return self._estimate_range(None, value)
        elif op == '>':
            return self._estimate_range(value, None)
        elif op == 'range':
            return self._estimate_range(value[0], value[1])

    def _estimate_equality(self, value):
        """等值查询估计"""
        for bucket in self.buckets:
            if bucket['min'] <= value <= bucket['max']:
                # 假设桶内均匀分布
                if bucket['distinct'] > 0:
                    return (bucket['count'] / bucket['distinct']) / self.total_count
                return 0
        return 0

    def _estimate_range(self, low, high):
        """范围查询估计"""
        count = 0
        for bucket in self.buckets:
            if high is not None and bucket['min'] > high:
                break
            if low is not None and bucket['max'] < low:
                continue

            # 计算重叠部分
            overlap_low = max(low, bucket['min']) if low else bucket['min']
            overlap_high = min(high, bucket['max']) if high else bucket['max']

            bucket_range = bucket['max'] - bucket['min']
            if bucket_range > 0:
                overlap_ratio = (overlap_high - overlap_low) / bucket_range
                count += bucket['count'] * overlap_ratio
            else:
                count += bucket['count']

        return count / self.total_count
```

```python
import asyncio
from typing import List, Dict, Any, Callable
from concurrent.futures import ProcessPoolExecutor
import multiprocessing as mp

class ParallelQueryExecutor:
    """并行查询执行器"""

    def __init__(self, num_workers=None):
        self.num_workers = num_workers or mp.cpu_count()
        self.executor = ProcessPoolExecutor(max_workers=self.num_workers)

    async def execute_parallel(self, plan: Dict, data_partitions: List) -> List:
        """
        并行执行查询计划

        Args:
            plan: 物理执行计划
            data_partitions: 数据分区列表

        Returns:
            各分区结果
        """
        loop = asyncio.get_event_loop()

        # 创建各worker的任务
        tasks = []
        for partition_id, partition_data in enumerate(data_partitions):
            task = loop.run_in_executor(
                self.executor,
                self._execute_partition,
                plan,
                partition_data,
                partition_id
            )
            tasks.append(task)

        # 等待所有任务完成
        results = await asyncio.gather(*tasks)
        return results

    def _execute_partition(self, plan: Dict, data: Any, partition_id: int) -> Any:
        """在单个分区上执行计划"""
        operator = plan['operator']

        if operator == 'scan':
            return self._exec_scan(plan, data)
        elif operator == 'filter':
            child_result = self._execute_partition(plan['child'], data, partition_id)
            return self._exec_filter(plan, child_result)
        elif operator == 'project':
            child_result = self._execute_partition(plan['child'], data, partition_id)
            return self._exec_project(plan, child_result)
        elif operator == 'hash_join':
            return self._exec_hash_join_parallel(plan, data, partition_id)
        elif operator == 'aggregate':
            child_result = self._execute_partition(plan['child'], data, partition_id)
            return self._exec_aggregate(plan, child_result)

        raise ValueError(f"Unknown operator: {operator}")

    def _exec_scan(self, plan: Dict, data: Any) -> Any:
        """扫描算子"""
        return data

    def _exec_filter(self, plan: Dict, data: Any) -> Any:
        """过滤算子"""
        predicate = plan['predicate']
        return [row for row in data if predicate(row)]

    def _exec_project(self, plan: Dict, data: Any) -> Any:
        """投影算子"""
        columns = plan['columns']
        return [{col: row[col] for col in columns} for row in data]

    def _exec_hash_join_parallel(self, plan: Dict, data: Any, partition_id: int) -> Any:
        """并行哈希连接"""
        # 本地哈希表构建
        build_side = plan['build_side']
        probe_side = data if plan['probe_table'] == partition_id else None

        # 构建哈希表
        hash_table = {}
        for row in build_side:
            key = row[plan['join_key']]
            if key not in hash_table:
                hash_table[key] = []
            hash_table[key].append(row)

        # 探查阶段
        result = []
        if probe_side:
            for row in probe_side:
                key = row[plan['join_key']]
                if key in hash_table:
                    for build_row in hash_table[key]:
                        merged = {**build_row, **row}
                        result.append(merged)

        return result

    def _exec_aggregate(self, plan: Dict, data: Any) -> Any:
        """聚合算子（本地预聚合）"""
        group_keys = plan.get('group_by', [])
        aggregations = plan['aggregations']

        groups = {}
        for row in data:
            key = tuple(row[k] for k in group_keys) if group_keys else ()
            if key not in groups:
                groups[key] = {agg['output']: agg['init']() for agg in aggregations}

            for agg in aggregations:
                groups[key][agg['output']] = agg['update'](
                    groups[key][agg['output']],
                    row[agg['input']]
                )

        return [{'group': k, **v} for k, v in groups.items()]


class DataShuffleManager:
    """数据重分布管理器"""

    def __init__(self, num_partitions: int):
        self.num_partitions = num_partitions

    def hash_shuffle(self, data: List, key_func: Callable) -> List[List]:
        """
        哈希重分布数据

        Args:
            data: 输入数据
            key_func: 提取分区键的函数

        Returns:
            按分区组织的数据
        """
        partitions = [[] for _ in range(self.num_partitions)]

        for row in data:
            key = key_func(row)
            partition_id = hash(key) % self.num_partitions
            partitions[partition_id].append(row)

        return partitions

    def range_shuffle(self, data: List, key_func: Callable,
                      boundaries: List) -> List[List]:
        """
        范围重分布数据

        Args:
            data: 输入数据
            key_func: 提取分区键的函数
            boundaries: 分区边界值列表

        Returns:
            按分区组织的数据
        """
        partitions = [[] for _ in range(len(boundaries) + 1)]

        for row in data:
            key = key_func(row)
            partition_id = self._find_partition(key, boundaries)
            partitions[partition_id].append(row)

        return partitions

    def _find_partition(self, key, boundaries: List) -> int:
        """二分查找确定分区"""
        left, right = 0, len(boundaries)

        while left < right:
            mid = (left + right) // 2
            if key < boundaries[mid]:
                right = mid
            else:
                left = mid + 1

        return left
```

```python
        mem_pressure = query_stats.get('memory_pressure', 0.3)
        io_wait = query_stats.get('io_wait', 0.1)
        data_skew = query_stats.get('data_skew', 1.0)

        # 数据倾斜严重时降低并行度

            self.current_parallelism = max(2, self.current_parallelism // 2)

        elif cpu_util < 0.7 and io_wait > 0.3 and mem_pressure < 0.8:
            self.current_parallelism = min(32, self.current_parallelism + 2)

        # CPU饱和时保持或减少
        elif cpu_util > 0.9:
            self.current_parallelism = max(2, self.current_parallelism - 1)

        return self.current_parallelism
```

### 5.2 查询队列管理

```python
import heapq
from dataclasses import dataclass, field
from typing import List, Dict
import time

@dataclass
class QueryRequest:
    """查询请求"""
    query_id: str
    sql: str
    estimated_cost: float
    cpu_requirement: float
    memory_requirement: float
    priority: int = 5
    submit_time: float = field(default_factory=time.time)
    deadline: float = None

    def __lt__(self, other):
        # 优先级高的先执行，相同优先级按提交时间
        if self.priority != other.priority:
            return self.priority > other.priority
        return self.submit_time < other.submit_time


class ResourceAwareScheduler:
    """资源感知查询调度器"""

    def __init__(self, total_cpu: float, total_memory: float):
        self.total_cpu = total_cpu
        self.total_memory = total_memory

        self.available_cpu = total_cpu
        self.available_memory = total_memory

        self.waiting_queue = []  # 优先级队列
        self.running_queries: Dict[str, QueryRequest] = {}
        self.completed_queries: List[QueryRequest] = []

    def submit_query(self, request: QueryRequest) -> bool:
        """
        提交查询请求

        Returns:
            True if 立即执行, False if 进入等待队列
        """
        if self._can_schedule(request):
            self._start_query(request)
            return True
        else:
            heapq.heappush(self.waiting_queue, request)
            return False

    def _can_schedule(self, request: QueryRequest) -> bool:
        """检查是否可以调度"""
        return (request.cpu_requirement <= self.available_cpu and
                request.memory_requirement <= self.available_memory)

    def _start_query(self, request: QueryRequest):
        """开始执行查询"""
        self.available_cpu -= request.cpu_requirement
        self.available_memory -= request.memory_requirement
        self.running_queries[request.query_id] = request
        print(f"Starting query {request.query_id} "
              f"(CPU: {request.cpu_requirement}, MEM: {request.memory_requirement})")

    def complete_query(self, query_id: str):
        """查询完成回调"""
        if query_id in self.running_queries:
            query = self.running_queries.pop(query_id)
            self.available_cpu += query.cpu_requirement
            self.available_memory += query.memory_requirement
            self.completed_queries.append(query)
            print(f"Completed query {query_id}")

            # 尝试调度等待队列中的查询
            self._try_schedule_waiting()

    def _try_schedule_waiting(self):
        """尝试从等待队列调度查询"""
        # 收集所有可调度的查询
        schedulable = []
        temp_queue = []

        while self.waiting_queue:
            query = heapq.heappop(self.waiting_queue)
            if self._can_schedule(query):
                schedulable.append(query)
            else:
                temp_queue.append(query)

        # 恢复不可调度的查询
        for q in temp_queue:
            heapq.heappush(self.waiting_queue, q)

        # 按优先级和资源效率排序
        schedulable.sort(key=lambda q: (
            -q.priority,
            q.estimated_cost / (q.cpu_requirement + q.memory_requirement + 0.001)
        ))

        # 调度查询
        for query in schedulable:
            if self._can_schedule(query):
                self._start_query(query)

    def get_stats(self) -> Dict:
        """获取调度器统计信息"""
        return {
            'available_cpu': self.available_cpu,
            'available_memory': self.available_memory,
            'cpu_utilization': 1 - self.available_cpu / self.total_cpu,
            'memory_utilization': 1 - self.available_memory / self.total_memory,
            'waiting_queries': len(self.waiting_queue),
            'running_queries': len(self.running_queries),
            'completed_queries': len(self.completed_queries)
        }
```

```python
            self.used_memory -= self.query_memory[query_id]
            del self.query_memory[query_id]

        per_partition = query_budget // num_partitions

            num_partitions *= 2
            per_partition = query_budget // num_partitions
```

## 6. 形式化查询优化问题

### 6.1 查询优化作为搜索问题

**定义 17.1.17** (查询优化问题)：

给定：

- 查询 $Q$ 的关系代数表达式
- 等价变换规则集 $\mathcal{R}$
- 物理实现算法集 $\mathcal{A}$
- 代价模型 $C$

求：

- 执行计划 $p^* \in \text{Plans}(Q)$ 使得 $C(p^*) = \min_{p \in \text{Plans}(Q)} C(p)$

**定理 17.1.4** (查询优化是NP-hard)：

对于无限制连接查询，找到最优连接顺序是NP-hard问题。

**证明概要**：

通过从最小集合覆盖问题归约：

1. 给定集合覆盖实例 $(U, \mathcal{S}, k)$
2. 构造查询 $Q$ 使得每个连接对应一个集合选择
3. 最优计划对应最小覆盖
4. 若可在多项式时间找到最优计划，则可解NP-complete问题

$\square$

### 6.2 形式化语义等价

**定义 17.1.18** (关系代数表达式等价)：

两个表达式 $E_1, E_2$ 是语义等价的（记为 $E_1 \equiv E_2$），当且仅当：
$$\forall \text{数据库实例 } I: E_1(I) = E_2(I)$$

其中相等考虑：

- 集合语义：作为集合相等
- 包语义：作为多重集相等（含重复次数）

### 6.3 优化空间边界

**定理 17.1.5** (左深树计划数量)：

对于 $n$ 个关系的连接，不同左深树计划数量为：
$$|\text{LeftDeep}(n)| = n! \cdot C_{n-1}$$

其中 $C_{n-1}$ 是Catalan数，物理算子选择进一步乘以 $|\mathcal{A}|^{n-1}$。

**证明**：

- $n!$：关系的排列顺序
- $C_{n-1} = \frac{1}{n}\binom{2(n-1)}{n-1}$：二叉树形状数
- 对于左深树，形状由连接顺序唯一确定
- 物理算子选择：每个内部节点选择一种连接算法

$\square$

```c
{

    }

    path->startup_cost = startup_cost;
    path->total_cost = startup_cost + run_cost;
    path->rows = baserel->rows;

    /* 并行度估算 */
    if (baserel->consider_parallel)
        path->parallel_workers = compute_parallel_worker(baserel,
            baserel->pages, -1, max_parallel_workers_per_gather);
}

/*
 * cost_hashjoin - 哈希连接代价估算
 */
void
cost_hashjoin(JoinPath *path, PlannerInfo *root,
              List *hashclauses, bool semi_join_done)
{
    Cost        startup_cost = 0;
    Cost        run_cost = 0;
    Cost        cpu_per_tuple;

    Path       *outer_path = path->outerjoinpath;
    Path       *inner_path = path->innerjoinpath;

    /* 子路径代价 */
    startup_cost += outer_path->startup_cost;
    run_cost += outer_path->total_cost - outer_path->startup_cost;

    /* 构建哈希表代价 */
    startup_cost += inner_path->total_cost;

    /* 哈希构建CPU代价 */
    startup_cost += (cpu_operator_cost * inner_path->rows);

    /* 探查代价 */
    cpu_per_tuple = cpu_tuple_cost + cpu_operator_cost;
    run_cost += cpu_per_tuple * path->path.rows;

    path->path.startup_cost = startup_cost;
    path->path.total_cost = startup_cost + run_cost;
}
```

```cpp
/*
 * 计算表连接顺序的代价
 */
double JOIN::estimate_cost(POSITION *positions, uint n_tables)
{
    double total_cost = 0;
    ha_rows row_estimate = 1;

    for (uint i = 0; i < n_tables; i++) {
        POSITION *pos = &positions[i];

        // 当前表的访问代价
        double access_cost = pos->table->file->scan_time();

        // 连接代价
        double join_cost = row_estimate * access_cost;

        // 选择性和条件过滤
        double selectivity = calculate_selectivity(pos->table);
        row_estimate *= selectivity;

        total_cost += join_cost;
    }

    return total_cost;
}

/*
 * 贪婪搜索算法（MySQL默认连接优化算法）
 */
bool JOIN::greedy_search(uint search_depth)
{
    uint current_depth = 0;

    while (current_depth < tables) {
        // 剩余未优化的表
        TABLE_LIST **remaining = &table_list[current_depth];
        uint remaining_count = tables - current_depth;

        // 搜索局部最优
        POSITION best_pos;
        double best_cost = DBL_MAX;

        for (uint i = 0; i < remaining_count; i++) {
            // 尝试将remaining[i]加入当前连接顺序
            double cost = estimate_partial_plan(positions,
                                                current_depth + 1);

            if (cost < best_cost) {
                best_cost = cost;
                best_pos = positions[current_depth];
            }
        }

        // 固定当前最优选择
        positions[current_depth] = best_pos;
        current_depth++;
    }

    return true;
}
```

```rust
/// 比较谓词

    column: String,
    op: ComparisonOp,
    value: ScalarValue,

# [derive(Debug, Clone)]

    Eq, Ne, Lt, Le, Gt, Ge,

# [derive(Debug, Clone)]

    Int64(i64),
    Float64(f64),
    String(String),
    Boolean(bool),
    Null,

impl Predicate for ComparisonPredicate {
    fn evaluate(&self, batch: &RecordBatch) -> arrow::error::Result<BooleanArray> {
        let column = batch.column_by_name(&self.column)
            .ok_or_else(|| arrow::error::ArrowError::InvalidArgumentError(
                format!("Column {} not found", self.column)
            ))?;

        match column.data_type() {
            DataType::Int64 => {
                let array = column.as_any()
                    .downcast_ref::<Int64Array>()
                    .ok_or_else(|| arrow::error::ArrowError::InvalidArgumentError(
                        "Type mismatch".to_string()
                    ))?;

                let target = match &self.value {
                    ScalarValue::Int64(v) => *v,
                    _ => return Ok(BooleanArray::from(vec![false; array.len()])),
                };

                let result: Vec<bool> = (0..array.len())
                    .map(|i| {
                        if array.is_null(i) {
                            false
                        } else {
                            let val = array.value(i);
                            match self.op {
                                ComparisonOp::Eq => val == target,
                                ComparisonOp::Ne => val != target,
                                ComparisonOp::Lt => val < target,
                                ComparisonOp::Le => val <= target,
                                ComparisonOp::Gt => val > target,
                                ComparisonOp::Ge => val >= target,
                            }
                        }
                    })
                    .collect();

                Ok(BooleanArray::from(result))
            }
            _ => Ok(BooleanArray::from(vec![false; column.len()])),
        }
    }

    fn boxes_clone(&self) -> Box<dyn Predicate> {
        Box::new(Self {
            column: self.column.clone(),
            op: self.op.clone(),
            value: self.value.clone(),
        })
    }

pub struct FilterNode {
    child: Box<dyn ExecutionNode>,
    predicate: Box<dyn Predicate>,

# [async_trait]
impl ExecutionNode for FilterNode {
    fn schema(&self) -> Arc<Schema> {
        self.child.schema()
    }

    async fn execute(&self) -> Result<SendableRecordBatchStream> {
        let child_stream = self.child.execute().await?;
        let predicate = self.predicate.clone();

        let filtered = child_stream.filter_map(move |batch_result| {
            let pred = predicate.clone();
            async move {
                match batch_result {
                    Ok(batch) => {
                        match pred.evaluate(&batch) {
                            Ok(selection) => {
                                match arrow::compute::filter_record_batch(&batch, &selection) {
                                    Ok(filtered) if filtered.num_rows() > 0 => Some(Ok(filtered)),
                                    _ => None,
                                }
                            }
                            Err(e) => Some(Err(e.into())),
                        }
                    }
                    Err(e) => Some(Err(e)),
                }
            }
        });

        Ok(Box::pin(filtered))
    }

    fn children(&self) -> Vec<&dyn ExecutionNode> {
        vec![self.child.as_ref()]
    }
}

# [async_trait]
impl ExecutionNode for HashJoinNode {
    fn schema(&self) -> Arc<Schema> {
        // 合并左右schema
        let left_schema = self.left.schema();
        let right_schema = self.right.schema();

        let mut fields: Vec<Field> = left_schema.fields().iter().map(|f| f.as_ref().clone()).collect();
        fields.extend(right_schema.fields().iter().map(|f| f.as_ref().clone()));

        Arc::new(Schema::new(fields))
    }

    async fn execute(&self) -> Result<SendableRecordBatchStream> {
        // 构建哈希表
        let right_batches: Vec<RecordBatch> = self.right.execute()
            .await?
            .collect::<Vec<_>>()
            .await
            .into_iter()
            .filter_map(Result::ok)
            .collect();

        let mut hash_table: HashMap<Vec<String>, Vec<usize>> = HashMap::new();

        // 建立哈希表（简化实现）
        for (batch_idx, batch) in right_batches.iter().enumerate() {
            for row_idx in 0..batch.num_rows() {
                let key: Vec<String> = self.right_keys.iter()
                    .map(|k| format!("{:?}", batch.column_by_name(k).unwrap()))
                    .collect();
                hash_table.entry(key).or_default().push(batch_idx * 10000 + row_idx);
            }
        }

        // 探查左表
        let left_stream = self.left.execute().await?;
        let left_keys = self.left_keys.clone();
        let right_batches = Arc::new(right_batches);

        let joined = left_stream.filter_map(move |batch_result| {
            let hash_table = hash_table.clone();
            let right_batches = right_batches.clone();
            let left_keys = left_keys.clone();

            async move {
                match batch_result {
                    Ok(left_batch) => {
                        // 简化的连接实现
                        let result_rows = left_batch.num_rows();
                        if result_rows > 0 {
                            Some(Ok(left_batch))  // 简化返回
                        } else {
                            None
                        }
                    }
                    Err(e) => Some(Err(e)),
                }
            }
        });

        Ok(Box::pin(joined))
    }

    fn children(&self) -> Vec<&dyn ExecutionNode> {
        vec![self.left.as_ref(), self.right.as_ref()]
    }
}

/// 向量化执行器
pub struct VectorizedExecutor {
    batch_size: usize,
}

impl VectorizedExecutor {
    pub fn new(batch_size: usize) -> Self {
        Self { batch_size }
    }

    /// 执行查询计划
    pub async fn execute(&self, plan: &dyn ExecutionNode) -> Result<Vec<RecordBatch>> {
        let mut results = Vec::new();
        let mut stream = plan.execute().await?;

        while let Some(batch_result) = stream.next().await {
            results.push(batch_result?);
        }

        Ok(results)
    }

    /// 代价估算
    pub fn estimate_cost(&self, plan: &dyn ExecutionNode) -> CostEstimate {
        let mut cost = CostEstimate::default();
        self.estimate_node_cost(plan, &mut cost);
        cost
    }

    fn estimate_node_cost(&self, node: &dyn ExecutionNode, cost: &mut CostEstimate) {
        // 递归估算子节点
        for child in node.children() {
            self.estimate_node_cost(child, cost);
        }

        // 估算当前节点代价（简化）
        cost.total_cost += 1.0;
    }
}
```
