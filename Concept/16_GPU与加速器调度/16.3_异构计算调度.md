## 概述

异构计算调度涉及将计算任务分配到不同类型的处理单元（CPU、GPU、FPGA、TPU、NPU等），以实现最优性能和能效。本章深入探讨异构系统的调度策略和优化技术。

### 形式化定义 16.3.1 (异构计算系统)

**定义**：异构计算系统 $\mathcal{H}$ 是一个七元组：

$$\mathcal{H} = (\mathcal{P}, \mathcal{T}, \mathcal{M}, \mathcal{K}, \mathcal{S}, \mathcal{C}, \mathcal{E})$$

其中：

- $\mathcal{P} = \{p_1, p_2, ..., p_n\}$：异构处理器集合（CPU/GPU/FPGA/TPU）
- $\mathcal{T} = \{t_1, t_2, ..., t_m\}$：任务集合
- $\mathcal{M}$：异构内存系统
- $\mathcal{K}: \mathcal{T} \times \mathcal{P} \rightarrow \mathbb{R}^+$：执行时间核函数
- $\mathcal{S}: \mathcal{T} \rightarrow \mathcal{P}$：调度映射函数
- $\mathcal{C}: \mathcal{T} \rightarrow \mathbb{R}^+$：通信成本函数
- $\mathcal{E}: \mathcal{T} \times \mathcal{P} \rightarrow \mathbb{R}^+$：能耗函数

## CPU-GPU协同调度

### 16.3.1 异构调度模型

#### 形式化定义 16.3.2 (CPU-GPU调度)

**定义**：CPU-GPU调度问题是一个优化问题：

$$\min_{\mathcal{S}} \sum_{t \in \mathcal{T}} \mathcal{K}(t, \mathcal{S}(t)) + \mathcal{C}(t)$$

约束条件：
$$\forall p \in \mathcal{P}: \sum_{t: \mathcal{S}(t)=p} \text{load}(t) \leq \text{capacity}(p)$$

#### 任务特性分析

| 任务特性 | 适合处理器 | 原因 |
|----------|-----------|------|
| 高度并行 | GPU | SIMT执行模型 |
| 复杂控制流 | CPU | 分支预测、乱序执行 |
| 小数据量 | CPU | 低启动开销 |
| 大数据量 | GPU | 高内存带宽 |
| 自定义逻辑 | FPGA | 可重构硬件 |
| 定点运算 | TPU/NPU | 专用矩阵单元 |

### 16.3.2 动态任务分配

#### 形式化定义 16.3.3 (动态调度)

**定义**：动态调度 $\mathcal{S}_{dynamic}$ 根据运行时信息调整映射：

$$\mathcal{S}_{dynamic}(t, p) = f(\mathcal{K}(t,p), \mathcal{C}(t), \mathcal{L}(p), \tau)$$

其中 $\mathcal{L}(p)$ 是处理器 $p$ 的当前负载，$\tau$ 是时间。

#### Heterogeneous Earliest Finish Time (HEFT)算法

```python
import heapq
from dataclasses import dataclass
from typing import List, Dict, Tuple
import numpy as np

@dataclass
class Task:
    task_id: int
    computation_costs: Dict[str, float]  # processor -> time
    dependencies: List[int]
    data_size: float  # 输出数据大小

@dataclass
class Processor:
    proc_id: str
    type: str  # CPU/GPU/FPGA
    speed_factor: float
    availability_time: float = 0.0

def heft_schedule(tasks: List[Task], processors: List[Processor],
                  bandwidth: float = 1.0) -> Dict[int, str]:
    """
    HEFT算法实现
    最小化完工时间(makespan)
    """
    # 1. 计算平均执行时间（用于排序）
    avg_cost = {}
    for task in tasks:
        avg_cost[task.task_id] = np.mean(list(task.computation_costs.values()))

    # 2. 计算任务优先级（向上秩）
    def upward_rank(task_id: int, memo: Dict) -> float:
        if task_id in memo:
            return memo[task_id]

        task = next(t for t in tasks if t.task_id == task_id)
        if not task.dependencies:
            rank = avg_cost[task_id]
        else:
            max_succ_rank = max(
                upward_rank(succ, memo) + avg_cost[succ]
                for succ in task.dependencies
            )
            rank = avg_cost[task_id] + max_succ_rank

        memo[task_id] = rank
        return rank

    rank_memo = {}
    task_priorities = [(upward_rank(t.task_id, rank_memo), t.task_id)
                       for t in tasks]
    task_priorities.sort(reverse=True)

    # 3. 调度任务
    schedule = {}
    task_finish_times = {}

    for _, task_id in task_priorities:
        task = next(t for t in tasks if t.task_id == task_id)

        # 计算每个处理器的最早完成时间
        best_processor = None
        best_finish_time = float('inf')

        for proc in processors:
            # 计算就绪时间（考虑依赖）
            ready_time = proc.availability_time

            for dep_id in task.dependencies:
                dep_finish = task_finish_times[dep_id]
                dep_task = next(t for t in tasks if t.task_id == dep_id)

                # 通信开销
                if schedule[dep_id] != proc.proc_id:
                    comm_cost = dep_task.data_size / bandwidth
                else:
                    comm_cost = 0

                ready_time = max(ready_time, dep_finish + comm_cost)

            # 执行时间
            exec_time = task.computation_costs.get(proc.proc_id, float('inf'))
            finish_time = ready_time + exec_time

            if finish_time < best_finish_time:
                best_finish_time = finish_time
                best_processor = proc

        # 分配任务
        schedule[task_id] = best_processor.proc_id
        task_finish_times[task_id] = best_finish_time
        best_processor.availability_time = best_finish_time

    return schedule

# 示例
if __name__ == "__main__":
    tasks = [
        Task(1, {"CPU": 10, "GPU": 5}, [], 100),
        Task(2, {"CPU": 15, "GPU": 8}, [1], 200),
        Task(3, {"CPU": 12, "GPU": 6}, [1], 150),
        Task(4, {"CPU": 20, "GPU": 10}, [2, 3], 300),
    ]

    processors = [
        Processor("CPU", "CPU", 1.0),
        Processor("GPU", "GPU", 2.0)
    ]

    schedule = heft_schedule(tasks, processors, bandwidth=10.0)
    print("Task Schedule:", schedule)
```

### 16.3.3 负载均衡定理

#### 形式化定理 16.3.1 (最优负载均衡)

**定理**：对于 $n$ 个任务和 $m$ 个异构处理器，最优负载均衡满足：

$$\min \max_{p \in \mathcal{P}} \sum_{t: \mathcal{S}(t)=p} \mathcal{K}(t,p)$$

**近似比**：贪心算法的近似比为 $2 - \frac{1}{m}$。

**证明概要**：

设最优解的完工时间为 $OPT$。

贪心算法每次将任务分配给最早可用的处理器。

考虑最后完成的任务 $t_{last}$，其开始时间为 $S_{last}$：
$$C_{max} = S_{last} + \mathcal{K}(t_{last}, p_{last})$$

由于贪心选择，$S_{last}$ 不大于任何处理器的负载。
所有处理器在 $S_{last}$ 时刻都在忙碌：
$$m \cdot S_{last} \leq \sum_{t \neq t_{last}} \mathcal{K}(t, \mathcal{S}(t))$$

因此：
$$C_{max} \leq \frac{1}{m} \sum_{t} \mathcal{K}(t, \mathcal{S}(t)) + \mathcal{K}(t_{last}, p_{last})$$

$$\leq \frac{1}{m} \cdot m \cdot OPT + OPT = (2 - \frac{1}{m}) \cdot OPT$$

**证毕** □

### 16.3.5 高层次综合调度

#### 形式化定义 16.3.5 (HLS调度)

**定义**：高层次综合（HLS）调度将C/C++代码映射到硬件：

$$\mathcal{HLS}: \text{CDAG} \rightarrow (\mathcal{S}, \mathcal{B}, \mathcal{R})$$

其中：

- $CDAG$：控制数据流图
- $\mathcal{S}$：操作调度表
- $\mathcal{B}$：绑定的功能单元
- $\mathcal{R}$：资源分配

#### 调度约束

**资源约束**：
$$\forall c, \forall r: \sum_{op \in Ops(c)} \text{bind}(op, r) \leq \text{capacity}(r)$$

**依赖约束**：
$$\forall (op_i, op_j) \in E: \mathcal{S}(op_j) \geq \mathcal{S}(op_i) + \text{latency}(op_i)$$

#### HLS调度算法

```python
class HLSScheduler:
    """
    基于列表调度的高层次综合调度器
    """

    def __init__(self, cdfg, resources):
        self.cdfg = cdfg  # 控制数据流图
        self.resources = resources
        self.schedule = {}
        self.binding = {}

    def asap_schedule(self):
        """ASAP (As Soon As Possible) 调度"""
        # 拓扑排序
        sorted_ops = self._topological_sort()

        for op in sorted_ops:
            # 计算最早开始时间
            earliest = 0
            for pred in self.cdfg.predecessors(op):
                earliest = max(earliest,
                    self.schedule[pred] + self.cdfg.latency(pred))

            self.schedule[op] = earliest

    def alap_schedule(self, deadline):
        """ALAP (As Late As Possible) 调度"""
        sorted_ops = self._topological_sort(reverse=True)

        for op in sorted_ops:
            latest = deadline - self.cdfg.latency(op)
            for succ in self.cdfg.successors(op):
                latest = min(latest,
                    self.schedule[succ] - self.cdfg.latency(op))

            self.schedule[op] = latest

    def list_schedule(self, priority_fn):
        """列表调度（考虑资源约束）"""
        sorted_ops = self._topological_sort()
        resource_usage = {r: [] for r in self.resources}

        for op in sorted_ops:
            # 找到满足依赖的最早时间槽
            earliest = 0
            for pred in self.cdfg.predecessors(op):
                earliest = max(earliest,
                    self.schedule[pred] + self.cdfg.latency(pred))

            # 找到可用资源
            required = self.cdfg.resource_requirement(op)
            start_time = self._find_resource_slot(
                resource_usage, required, earliest)

            self.schedule[op] = start_time
            self._allocate_resource(resource_usage, required,
                                   start_time,
                                   start_time + self.cdfg.latency(op))

    def _topological_sort(self, reverse=False):
        """拓扑排序"""
        # Kahn算法实现
        in_degree = {op: 0 for op in self.cdfg.nodes()}
        for op in self.cdfg.nodes():
            for succ in self.cdfg.successors(op):
                in_degree[succ] += 1

        queue = [op for op, deg in in_degree.items() if deg == 0]
        result = []

        while queue:
            # 按优先级排序
            queue.sort(key=priority_fn, reverse=reverse)
            op = queue.pop(0)
            result.append(op)

            for succ in self.cdfg.successors(op):
                in_degree[succ] -= 1
                if in_degree[succ] == 0:
                    queue.append(succ)

        return result
```

#### 形式化定义 16.3.6 (TPU调度)

**定义**：TPU调度 $\mathcal{S}_{TPU}$ 优化矩阵乘法执行：

$$\min \sum_{mm} \mathcal{T}(mm)$$

其中矩阵乘法 $mm$ 被映射到MXU（矩阵单元）：
$$\mathcal{T}(mm) = \left\lceil \frac{M \cdot N \cdot K}{128 \cdot 128 \cdot P} \right\rceil \cdot t_{cycle}$$

### 16.3.7 NPU调度策略

#### 形式化定义 16.3.7 (NPU调度)

**定义**：NPU调度 $\mathcal{S}_{NPU}$ 是一个映射：

$$\mathcal{S}_{NPU}: \text{Layer} \rightarrow (Core, Memory, Scheduling)$$

**调度策略**：

1. **权重静止**：权重保留在片上，输入数据流式处理
2. **输出静止**：部分和保留在片上
3. **行静止**：整行权重和激活保留

```python
class NPUScheduler:
    """
    NPU调度器（模拟NVDLA/昇腾架构）
    """

    def __init__(self, num_cores, onchip_memory, bandwidth):
        self.num_cores = num_cores
        self.onchip_memory = onchip_memory
        self.bandwidth = bandwidth

    def schedule_conv2d(self, layer_config):
        """
        调度卷积层
        """
        N, C, H, W = layer_config['input_shape']
        K, _, R, S = layer_config['weight_shape']

        # 计算数据移动策略
        best_strategy = None
        best_time = float('inf')

        for strategy in ['weight_stationary', 'output_stationary', 'row_stationary']:
            time = self._estimate_time(layer_config, strategy)
            if time < best_time:
                best_time = time
                best_strategy = strategy

        return {
            'strategy': best_strategy,
            'estimated_time': best_time,
            'tiling_config': self._compute_tiling(layer_config, best_strategy)
        }

    def _estimate_time(self, layer_config, strategy):
        """估计执行时间"""
        # 计算内存访问和计算量
        if strategy == 'weight_stationary':
            # 权重加载一次，输入分块
            data_movement = self._weight_stationary_cost(layer_config)
        elif strategy == 'output_stationary':
            data_movement = self._output_stationary_cost(layer_config)
        else:
            data_movement = self._row_stationary_cost(layer_config)

        compute_cycles = self._compute_cycles(layer_config)
        memory_cycles = data_movement / self.bandwidth

        return max(compute_cycles, memory_cycles)

    def _compute_tiling(self, layer_config, strategy):
        """计算分块参数"""
        # 基于片上内存大小计算最优分块
        memory_per_core = self.onchip_memory / self.num_cores

        # 求解最优tiling size
        # 约束: feature_map_tile + weight_tile + output_tile <= memory_per_core

        return {
```

### 16.3.8 异构任务划分

#### 形式化定义 16.3.8 (任务划分)

**定义**：任务划分 $\mathcal{P}$ 将计算图 $G = (V, E)$ 划分为子图：

$$\mathcal{P}: V \rightarrow \mathcal{P}_{type}$$

其中 $\mathcal{P}_{type} \in \{CPU, GPU, FPGA, TPU\}$。

**优化目标**：
$$\min \max_{p \in \mathcal{P}} \left( T_{compute}(p) + T_{comm}(p) \right)$$

#### 图划分算法

```python
import networkx as nx
from typing import Set, Tuple

class HeterogeneousPartitioner:
    """
    异构任务划分器
    基于KL算法和模拟退火
    """

    def __init__(self, graph, processor_types, edge_weights=None):
        self.graph = graph
        self.processor_types = processor_types
        self.edge_weights = edge_weights or {}

    def partition_kl(self, num_partitions):
        """
        Kernighan-Lin图划分算法
        """
        # 初始化随机划分
        partition = {}
        nodes = list(self.graph.nodes())
        random.shuffle(nodes)

        for i, node in enumerate(nodes):
            partition[node] = i % num_partitions

        # 迭代优化
        improved = True
        while improved:
            improved = False

            for node in nodes:
                current_part = partition[node]

                # 计算移动到每个分区的增益
                best_gain = 0
                best_part = current_part

                for part in range(num_partitions):
                    if part == current_part:
                        continue

                    gain = self._compute_gain(node, current_part, part, partition)
                    if gain > best_gain:
                        best_gain = gain
                        best_part = part

                if best_part != current_part:
                    partition[node] = best_part
                    improved = True

        return partition

    def _compute_gain(self, node, from_part, to_part, partition):
        """计算移动节点的增益"""
        # 外部边 - 内部边
        external = 0
        internal = 0

        for neighbor in self.graph.neighbors(node):
            weight = self.edge_weights.get((node, neighbor), 1)
            if partition[neighbor] == from_part:
                internal += weight
            elif partition[neighbor] == to_part:
                external += weight

        return external - internal

    def partition_metis(self, num_partitions):
        """使用METIS库进行划分"""
        try:
            import metis

            # 转换为METIS格式
            adjacency = {}
            for node in self.graph.nodes():
                neighbors = list(self.graph.neighbors(node))
                adjacency[node] = neighbors

            # 调用METIS
            cuts, partition = metis.part_graph(
                adjacency,
                num_partitions,
                objtype='cut'
            )

            return {node: part for node, part in enumerate(partition)}
        except ImportError:
            return self.partition_kl(num_partitions)
```

### 16.3.9 动态负载均衡定理

#### 定理 16.3.2 (动态负载均衡收敛)

**定理**：采用工作窃取（Work Stealing）的动态负载均衡，期望完成时间为：

$$\mathbb{E}[T] \leq T_{opt} + O\left(\frac{T_{seq}}{p}\right) + O(T_{\infty})$$

其中：

- $T_{opt}$：最优调度时间
- $T_{seq}$：串行执行时间
- $p$：处理器数量
- $T_{\infty}$：关键路径长度

### 16.3.14 内存迁移策略

#### 形式化定理 16.3.3 (内存迁移最优策略)

**定理**：对于访问模式已知的内存页，最优迁移策略的页错误次数上界为：

$$\text{Page Faults} \leq 2 \cdot |\text{Pages}| + \sum_{p} \min(\text{CPU accesses}, \text{GPU accesses})$$

#### 内存迁移代码

```cpp
#include <cuda_runtime.h>
#include <vector>
#include <map>

class AdaptiveMigrationManager {
private:
    struct PageInfo {
        void* addr;
        size_t size;
        int cpu_accesses;
        int gpu_accesses;
        int current_location;  // -1: CPU, 0+: GPU device
    };

    std::map<void*, PageInfo> page_table;
    int migration_threshold;

public:
    AdaptiveMigrationManager(int threshold = 3)
        : migration_threshold(threshold) {}

    void registerPage(void* addr, size_t size) {
        page_table[addr] = {addr, size, 0, 0, -1};
    }

    void recordCPUAccess(void* addr) {
        if (page_table.find(addr) != page_table.end()) {
            page_table[addr].cpu_accesses++;

            // 检查是否需要迁移
            checkMigration(addr);
        }
    }

    void recordGPUAccess(void* addr, int device_id) {
        if (page_table.find(addr) != page_table.end()) {
            page_table[addr].gpu_accesses++;

            checkMigration(addr, device_id);
        }
    }

    void checkMigration(void* addr, int preferred_gpu = -1) {
        PageInfo& info = page_table[addr];

        int total_accesses = info.cpu_accesses + info.gpu_accesses;
        if (total_accesses < migration_threshold) {
            return;  // 数据不足
        }

        // 决定最佳位置
        float cpu_ratio = static_cast<float>(info.cpu_accesses) / total_accesses;
        float gpu_ratio = static_cast<float>(info.gpu_accesses) / total_accesses;

        if (gpu_ratio > 0.8 && info.current_location == -1) {
            // 迁移到GPU
            migrateToGPU(addr, preferred_gpu >= 0 ? preferred_gpu : 0);
        } else if (cpu_ratio > 0.8 && info.current_location >= 0) {
            // 迁移到CPU
            migrateToCPU(addr);
        }
    }

    void migrateToGPU(void* addr, int device_id) {
        cudaMemPrefetchAsync(addr, page_table[addr].size, device_id);
        page_table[addr].current_location = device_id;

        // 重置计数器
        page_table[addr].cpu_accesses = 0;
        page_table[addr].gpu_accesses = 0;
    }

    void migrateToCPU(void* addr) {
        cudaMemPrefetchAsync(addr, page_table[addr].size, cudaCpuDeviceId);
        page_table[addr].current_location = -1;

        page_table[addr].cpu_accesses = 0;
        page_table[addr].gpu_accesses = 0;
    }
};
```

```python
class HybridInferenceScheduler:
    """
    异构推理调度器
    根据模型特性和SLA要求选择处理器
    """

    def __init__(self):
        self.cpu_pool = ThreadPoolExecutor(max_workers=16)
        self.gpu_pool = GPUPool(num_gpus=4)
        self.fpga_pool = FPGAPool(num_devices=2)

        # 模型到处理器的映射
        self.model_profiles = {
            'bert-tiny': {'cpu_latency': 10, 'gpu_latency': 5, 'fpga_latency': 8},
            'resnet50': {'cpu_latency': 50, 'gpu_latency': 5, 'fpga_latency': 10},
            'gpt2': {'cpu_latency': 200, 'gpu_latency': 20, 'fpga_latency': 100},
        }

    def schedule(self, request):
        model = request.model_name
        sla = request.sla_ms

        if model not in self.model_profiles:
            # 默认使用GPU
            return self.gpu_pool.submit(request)

        profile = self.model_profiles[model]

        # 检查SLA要求
        if profile['fpga_latency'] < sla and self.fpga_pool.available():
            # FPGA能效比最优
            return self.fpga_pool.submit(request)
        elif profile['gpu_latency'] < sla and self.gpu_pool.available():
            # GPU性能最优
            return self.gpu_pool.submit(request)
        elif profile['cpu_latency'] < sla:
            # CPU兜底
            return self.cpu_pool.submit(self._cpu_infer, request)
        else:
            # SLA无法满足
            raise SLAViolation(f"SLA {sla}ms cannot be met for {model}")
```

## 总结

本章探讨了异构计算调度的核心概念：

1. **CPU-GPU协同**：HEFT算法、动态任务分配
2. **FPGA调度**：布局布线、高层次综合
3. **TPU/NPU**：矩阵单元调度、数据流优化
4. **任务划分**：图划分、负载均衡
5. **编程模型**：OpenCL、SYCL、OneAPI
6. **内存管理**：统一内存、自适应迁移
7. **性能建模**：Roofline模型、ML预测

关键定理：

- **定理16.3.1**：贪心负载均衡近似比 $2 - \frac{1}{m}$
- **定理16.3.2**：工作窃取收敛界
- **定理16.3.3**：内存迁移页错误上界

实际设计原则：

- 小任务/控制流 → CPU
- 大数据并行 → GPU
- 定制逻辑/低延迟 → FPGA
- 定点矩阵运算 → TPU/NPU
