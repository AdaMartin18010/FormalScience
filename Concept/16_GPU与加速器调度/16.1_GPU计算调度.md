## 概述

GPU（Graphics Processing Unit）计算调度是现代高性能计算的核心组成部分。
与传统的CPU调度不同，GPU调度需要处理大规模并行线程、复杂的内存层次结构以及专门的执行模型。
本章深入探讨GPU计算调度的理论基础、实现机制和优化策略。

### 形式化定义 16.1.1 (GPU调度系统)

**定义**：GPU调度系统 $\mathcal{G}$ 是一个六元组：

$$\mathcal{G} = (\mathcal{T}, \mathcal{SM}, \mathcal{M}, \mathcal{S}, \mathcal{C}, \mathcal{P})$$

其中：

- $\mathcal{T} = \{t_1, t_2, ..., t_n\}$：线程集合，$n$ 可达数百万
- $\mathcal{SM} = \{sm_1, sm_2, ..., sm_m\}$：流式多处理器集合
- $\mathcal{M} = (M_{global}, M_{shared}, M_{constant}, M_{texture}, M_{register})$：内存层次结构
- $\mathcal{S}: \mathcal{T} \times \mathbb{R}^+ \rightarrow \mathcal{SM}$：调度函数
- $\mathcal{C}: \mathcal{T} \times \mathcal{M} \rightarrow \mathbb{R}^+$：计算成本函数
- $\mathcal{P}: \mathcal{T} \rightarrow \mathbb{N}$：优先级函数

## CUDA执行模型

### 16.1.1 执行模型架构

CUDA（Compute Unified Device Architecture）是NVIDIA推出的并行计算平台和编程模型。其核心执行模型基于单指令多线程（SIMT）架构。

#### 形式化定义 16.1.2 (CUDA执行模型)

**定义**：CUDA执行模型 $\mathcal{E}_{CUDA}$ 是一个七元组：

$$\mathcal{E}_{CUDA} = (\mathcal{K}, \mathcal{G}, \mathcal{B}, \mathcal{W}, \mathcal{T}, \Phi, \Psi)$$

其中：

- $\mathcal{K}$：内核函数集合
- $\mathcal{G} = (G_x, G_y, G_z)$：网格维度（Grid）
- $\mathcal{B} = (B_x, B_y, B_z)$：块维度（Block），$B_x \times B_y \times B_z \leq 1024$（计算能力7.0+）
- $\mathcal{W}$：Warp集合，每个Warp包含32个线程
- $\mathcal{T} = \{t_{i,j} | 0 \leq i < |\mathcal{G}|, 0 \leq j < |\mathcal{B}|\}$：线程集合
- $\Phi: \mathcal{K} \rightarrow \mathcal{G} \times \mathcal{B}$：内核启动映射
- $\Psi: \mathcal{W} \rightarrow \mathcal{SM}$：Warp到SM的映射

**总线程数计算**：
$$N_{threads} = G_x \cdot G_y \cdot G_z \cdot B_x \cdot B_y \cdot B_z$$

### 16.1.2 Warp执行与分支发散

#### 形式化定义 16.1.3 (Warp执行模型)

**定义**：Warp $w$ 的执行状态 $\sigma(w)$ 定义为：

$$\sigma(w) = (PC_w, A_w, M_w, T_w)$$

其中：

- $PC_w$：程序计数器（所有线程共享）
- $A_w \subseteq \{0, 1, ..., 31\}$：活跃线程掩码
- $M_w$：谓词寄存器状态
- $T_w$：执行时间戳

**分支发散定理 (Theorem 16.1.1)**

**定理**：对于Warp $w$ 中的条件分支，若线程执行不同路径，则总执行时间满足：

$$T_{divergent} = \sum_{p \in P} T_p$$

其中 $P$ 是所有执行路径的集合，$T_p$ 是路径 $p$ 的执行时间。

**证明**：

考虑Warp $w$ 包含32个线程，遇到条件分支：

- 条件为真的线程集合：$T_{true} = \{t_i | predicate(t_i) = true\}$
- 条件为假的线程集合：$T_{false} = \{t_i | predicate(t_i) = false\}$

**情况1**：无发散（$T_{true} = \emptyset$ 或 $T_{false} = \emptyset$）
所有线程走同一路径，执行时间 $T_{uniform} = T_{path}$

**情况2**：有发散（$T_{true} \neq \emptyset$ 且 $T_{false} \neq \emptyset$）
根据SIMT执行模型，硬件需要序列化执行：

1. 首先执行 $T_{true}$ 路径，掩码 $A_w = T_{true}$
2. 然后执行 $T_{false}$ 路径，掩码 $A_w = T_{false}$

因此总执行时间：
$$T_{divergent} = T_{true\_path} + T_{false\_path} = \sum_{p \in \{true, false\}} T_p$$

**证毕** □

#### 发散优化策略

```cuda
// 未优化：严重发散
__global__ void unoptimized_kernel(float* data, int n) {
    int tid = blockIdx.x * blockDim.x + threadIdx.x;
    if (tid % 2 == 0) {
        // 偶数线程路径
        data[tid] = sqrtf(data[tid]);
    } else {
        // 奇数线程路径
        data[tid] = expf(data[tid]);
    }
}

// 优化后：线程重排减少发散
__global__ void optimized_kernel(float* data, int n) {
    int tid = blockIdx.x * blockDim.x + threadIdx.x;
    int lane_id = tid % 32;
    int warp_id = tid / 32;

    // 将同类型数据连续排列
    int new_tid = warp_id * 32 + lane_id;

    // 此时warp内线程大概率走相同路径
    if (data[new_tid] > 0) {
        data[new_tid] = sqrtf(data[new_tid]);
    } else {
        data[new_tid] = expf(data[new_tid]);
    }
}
```

### 16.1.4 Warp调度器详解

#### 形式化定义 16.1.5 (Warp调度器)

**定义**：Warp调度器 $\mathcal{WS}$ 是一个三元组：

$$\mathcal{WS} = (\mathcal{W}_{ready}, \mathcal{W}_{wait}, \Pi)$$

其中：

- $\mathcal{W}_{ready}$：就绪Warp集合
- $\mathcal{W}_{wait}$：等待Warp集合
- $\Pi: \mathcal{W}_{ready} \rightarrow [0, 1]$：调度策略函数

#### 双发射调度机制

现代GPU每个周期可以从同一个Warp调度器发射两条独立指令：

```
周期n:   Warp A - 指令1  →  执行单元
         Warp A - 指令2  →  执行单元（若独立）

周期n+1: Warp B - 指令1  →  执行单元
         Warp C - 指令1  →  执行单元
```

## 线程调度机制

### 16.1.5 GigaThread引擎

GigaThread引擎是NVIDIA GPU的全局线程调度管理器，负责管理数以万计的并发线程。

#### 形式化定义 16.1.6 (GigaThread调度)

**定义**：GigaThread调度 $\mathcal{GT}$ 是一个四元组：

$$\mathcal{GT} = (\mathcal{Q}_{global}, \mathcal{M}_{distribute}, \mathcal{H}_{migration}, \mathcal{P}_{priority})$$

其中：

- $\mathcal{Q}_{global}$：全局线程块队列
- $\mathcal{M}_{distribute}: \mathcal{B} \rightarrow \mathcal{SM}$：块分发映射
- $\mathcal{H}_{migration}$：线程迁移历史
- $\mathcal{P}_{priority}: \mathcal{B} \rightarrow \mathbb{N}$：块优先级

#### 调度延迟定理 (Theorem 16.1.2)

**定理**：GigaThread引擎的块调度延迟 $L_{sched}$ 满足：

$$L_{sched} \leq \frac{|\mathcal{B}|}{|\mathcal{SM}|} \cdot L_{ctx} + L_{dispatch}$$

其中：

- $|\mathcal{B}|$：待调度块总数
- $|\mathcal{SM}|$：可用SM数量
- $L_{ctx}$：上下文切换时间
- $L_{dispatch}$：分发固定开销

**证明**：

考虑最坏情况：所有块都需要被调度到SM上。

**步骤1**：块分发采用轮询（Round-robin）策略
每个SM接收的块数上界：$\lceil |\mathcal{B}| / |\mathcal{SM}| \rceil$

**步骤2**：每个块的调度包含：

- 资源分配（寄存器、共享内存）
- 上下文建立
- Warp初始化

总时间 = 资源分配时间 + 上下文建立时间

**步骤3**：由于调度是流水线化的，多个SM可以并行接收块

因此总调度延迟：
$$L_{sched} = \max_{sm_i \in \mathcal{SM}} \left( \sum_{b \in B_i} L_{ctx}(b) \right) + L_{dispatch}$$

其中 $B_i$ 是分配到 $sm_i$ 的块集合，$|B_i| \leq \lceil |\mathcal{B}| / |\mathcal{SM}| \rceil$

简化后得：
$$L_{sched} \leq \frac{|\mathcal{B}|}{|\mathcal{SM}|} \cdot L_{ctx} + L_{dispatch}$$

**证毕** □

### 16.1.6 Zero-Overhead线程调度

GPU实现Zero-Overhead线程切换的关键技术：

1. **寄存器分区**：每个线程拥有专用寄存器集，无需保存/恢复
2. **独立程序计数器**：每个Warp维护独立PC
3. **硬件调度器**：完全硬件实现的调度逻辑

**形式化描述**：

线程切换开销：
$$
C_{switch} = \begin{cases}
0 & \text{如果同Warp内切换} \\
L_{pipeline\_flush} & \text{如果Warp间切换}
end{cases}
$$

其中 $L_{pipeline\_flush}$ 通常为0-2个周期。

#### 内存访问调度定理 (Theorem 16.1.3)

**定理**：对于 $n$ 个并发内存请求，最优调度策略的完成时间 $T_{opt}$ 满足：

$$T_{opt} \geq \max\left( \frac{\sum_{i=1}^{n} s_i}{B}, \max_{i} L_i \right)$$

其中：

- $s_i$：第 $i$ 个请求的数据大小
- $B$：内存总带宽
- $L_i$：第 $i$ 个请求的访问延迟

**证明**：

**下界1（带宽限制）**：
所有请求传输的总数据量为 $\sum_{i=1}^{n} s_i$
即使带宽完全利用，传输时间至少为：
$$T_{bandwidth} = \frac{\sum_{i=1}^{n} s_i}{B}$$

**下界2（延迟限制）**：
即使有无限带宽，每个请求仍需经历访问延迟
因此：
$$T_{latency} = \max_{i} L_i$$

**综合**：
完成时间必须同时满足两个下界：
$$T_{opt} \geq \max(T_{bandwidth}, T_{latency}) = \max\left( \frac{\sum_{i=1}^{n} s_i}{B}, \max_{i} L_i \right)$$

**证毕** □

### 16.1.8 共享内存与Bank Conflict

#### Bank Conflict分析

共享内存被划分为32个bank（计算能力3.0+），每个bank每周期可服务一个请求。

**形式化定义**：
对于线程 $t_i$ 访问地址 $a_i$，bank索引计算：
$$\text{bank}(a_i) = \left\lfloor \frac{a_i}{4} \right\rfloor \mod 32$$

**冲突检测**：
对于Warp内32个线程的访问模式 $A = \{a_0, a_1, ..., a_{31}\}$，冲突度：
$$\text{conflict}(A) = \max_{b \in [0,31]} |\{i | \text{bank}(a_i) = b\}| - 1$$

总访问周期数 = $1 + \text{conflict}(A)$

#### Bank Conflict优化示例

```cuda
// 冲突版本：步长为32，所有线程访问同一bank
__shared__ float shared_mem[32][32];
int tid = threadIdx.x;
float val = shared_mem[tid][0];  // bank(0) = 0, bank(32) = 0, 冲突！

// 优化版本：步长为33，无冲突
__shared__ float shared_mem[32][33];
int tid = threadIdx.x;
float val = shared_mem[tid][0];  // bank(0) = 0, bank(33) = 1, 无冲突！
```

## 任务划分策略

### 16.1.9 形式化任务划分

#### 形式化定义 16.1.8 (任务划分)

**定义**：任务划分 $\mathcal{P}$ 是从计算任务到线程块的双射：

$$\mathcal{P}: \mathcal{T}_{compute} \rightarrow \mathcal{B}$$

满足：

1. **完整性**：$\bigcup_{b \in \mathcal{B}} \mathcal{P}^{-1}(b) = \mathcal{T}_{compute}$
2. **互斥性**：$\forall b_i, b_j \in \mathcal{B}, i \neq j: \mathcal{P}^{-1}(b_i) \cap \mathcal{P}^{-1}(b_j) = \emptyset$
3. **负载均衡**：$\forall b_i, b_j: |\mathcal{P}^{-1}(b_i)| \approx |\mathcal{P}^{-1}(b_j)|$

#### 任务粒度定理 (Theorem 16.1.4)

**定理**：最优任务粒度 $G_{opt}$ 满足：

$$G_{opt} = \arg\max_{G} \left( \frac{G}{T_{compute}(G) + T_{overhead}(G)} \right)$$

其中：

- $T_{compute}(G)$：粒度为 $G$ 时的计算时间
- $T_{overhead}(G)$：调度开销

**证明概要**：

设总工作量为 $W$，分为 $n = W/G$ 个任务。

计算时间：$T_{compute}(G) = n \cdot t_{comp}(G) = \frac{W}{G} \cdot t_{comp}(G)$

调度开销：$T_{overhead}(G) = n \cdot t_{overhead} = \frac{W}{G} \cdot t_{overhead}$

吞吐量：
$$Throughput(G) = \frac{W}{T_{compute}(G) + T_{overhead}(G)} = \frac{G}{t_{comp}(G) + t_{overhead}}$$

最优粒度使吞吐量最大化。

**证毕** □

## 调度算法

### 16.1.11 经典调度算法

#### Round-Robin调度

**算法描述**：

```
输入：Warp集合 W = {w_1, w_2, ..., w_n}
输出：调度序列 S

for cycle = 1, 2, 3, ... do
    for each warp w_i in W do
        if w_i is ready then
            schedule w_i for execution
        end if
    end for
end for
```

**形式化**：
调度函数：
$$
\mathcal{S}_{RR}(w_i, t) = \begin{cases}
1 & \text{if } t \mod n = i \text{ and } w_i \in \mathcal{W}_{ready} \\
0 & \text{otherwise}
\end{cases}
$$

#### Greedy调度

**算法描述**：

```
输入：Warp集合 W，优先级函数 P
输出：调度序列 S

for cycle = 1, 2, 3, ... do
    W_ready = {w ∈ W | w is ready}
    if W_ready ≠ ∅ then
        w* = argmax_{w ∈ W_ready} P(w)
        schedule w* for execution
    end if
end for
```

#### Oldest-First调度（实际GPU采用）

现代GPU采用Oldest-First策略，优先调度等待时间最长的Warp。

**形式化**：
$$
\mathcal{S}_{OF}(w_i, t) = \begin{cases}
1 & \text{if } waiting\_time(w_i) = \max_{w \in \mathcal{W}_{ready}} waiting\_time(w) \\
0 & \text{otherwise}
\end{cases}
$$

### 16.1.12 优先级调度

#### 形式化定义 16.1.9 (优先级调度)

**定义**：优先级调度 $\mathcal{S}_{prio}$ 是一个映射：

$$\mathcal{S}_{prio}: \mathcal{W} \times \mathbb{R}^+ \rightarrow \{0, 1\}$$

满足：
$$\mathcal{S}_{prio}(w_i, t) = 1 \Rightarrow \forall w_j: P(w_i) \geq P(w_j) \text{ 或 } w_j \notin \mathcal{W}_{ready}$$

#### CUDA流优先级


## 性能优化

### 16.1.13 内存访问优化

#### 合并访问（Coalesced Access）

**形式化定义 16.1.10 (合并访问)**

**定义**：Warp的内存访问 $A = \{a_0, a_1, ..., a_{31}\}$ 是合并的，当且仅当：

$$\forall i, j \in [0, 31]: \left\lfloor \frac{a_i}{128} \right\rfloor = \left\lfloor \frac{a_j}{128} \right\rfloor$$

即所有线程访问同一128字节对齐段。

**性能影响**：

- 完全合并：1次事务，128字节传输
- 完全分散：32次事务，最多4096字节传输
- 性能比：最高32倍

#### 优化示例

```cuda
// 未优化：非合并访问
__global__ void uncoalesced_kernel(float* output, float* input, int width) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;
    // 按行访问，但存储是列优先
    output[y * width + x] = input[x * width + y];
}

// 优化后：合并访问
__global__ void coalesced_kernel(float* output, float* input, int width) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;

    // 使用共享内存转置
    __shared__ float tile[32][33];  // 33避免bank conflict

    tile[threadIdx.y][threadIdx.x] = input[y * width + x];
    __syncthreads();

    int x_out = blockIdx.y * blockDim.y + threadIdx.x;
    int y_out = blockIdx.x * blockDim.x + threadIdx.y;
    output[y_out * width + x_out] = tile[threadIdx.x][threadIdx.y];
}
```

### 16.1.14 Occupancy优化

#### Occupancy计算

**定义**：Occupancy是活跃Warp数与最大Warp数的比值：

$$\text{Occupancy} = \frac{N_{active\_warps}}{N_{max\_warps}} \times 100\%$$

对于NVIDIA A100（计算能力8.0）：

- 每SM最大Warp数：64
- 每SM最大线程数：2048
- 每SM最大块数：32

2. 共享内存使用量

### 16.1.16 Warp调度器实现分析

#### 调度循环伪代码

```c
// 基于NVIDIA架构文档和逆向工程
void warp_scheduler_loop(struct sm_unit *sm) {
    while (true) {
        // 1. 检查所有Warp状态
        for (int i = 0; i < MAX_WARPS_PER_SM; i++) {
            struct warp *w = &sm->warps[i];

            if (w->status == WARP_WAITING) {
                // 检查等待条件是否满足
                if (check_dependencies(w)) {
                    w->status = WARP_READY;
                }
            }
        }

        // 2. 选择可执行Warp（Oldest-First）
        struct warp *selected = NULL;
        uint32_t max_wait_cycles = 0;

        for (int i = 0; i < MAX_WARPS_PER_SM; i++) {
            struct warp *w = &sm->warps[i];
            if (w->status == WARP_READY && w->wait_cycles > max_wait_cycles) {
                selected = w;
                max_wait_cycles = w->wait_cycles;
            }
        }

        // 3. 发射指令
        if (selected != NULL) {
            emit_instruction(selected);
            selected->wait_cycles = 0;
        }

        // 4. 更新等待计数
        for (int i = 0; i < MAX_WARPS_PER_SM; i++) {
            if (sm->warps[i].status == WARP_READY) {
                sm->warps[i].wait_cycles++;
            }
        }
    }
}
```

## CUDA调度模拟代码

### 16.1.17 GPU调度模拟器

```cpp
/**
 * GPU调度模拟器
 * 模拟CUDA执行模型和调度行为
 */

# include <iostream>
# include <vector>
# include <queue>
# include <memory>
# include <random>
# include <chrono>
# include <algorithm>
# include <mutex>
# include <condition_variable>
# include <thread>

namespace gpu_sim {

// 常量定义
constexpr int WARP_SIZE = 32;
constexpr int MAX_WARPS_PER_SM = 64;
constexpr int MAX_BLOCKS_PER_SM = 32;
constexpr int NUM_BANKS = 32;
constexpr int BANK_WIDTH = 4;  // bytes

// 执行状态枚举
enum class WarpStatus {
    EMPTY,          // 未使用
    READY,          // 就绪
    RUNNING,        // 执行中
    WAITING_MEM,    // 等待内存
    WAITING_BARRIER,// 等待同步
    FINISHED        // 完成
};

enum class InstructionType {
    ALU,            // 算术逻辑
    LOAD,           // 内存加载
    STORE,          // 内存存储
    BARRIER,        // 同步屏障
    BRANCH,         // 分支
    SPECIAL         // 特殊指令
};

// 指令结构
struct Instruction {
    InstructionType type;
    int latency;            // 执行周期
    int target_bank;        // 对于内存指令
    bool is_predicated;     // 是否谓词执行
    int dependency_mask;    // 依赖掩码

    Instruction(InstructionType t, int lat)
        : type(t), latency(lat), target_bank(-1),
          is_predicated(false), dependency_mask(0) {}
};

// Warp结构
class Warp {
public:
    int warp_id;
    int block_id;
    WarpStatus status;
    int pc;                 // 程序计数器
    int active_mask;        // 活跃线程掩码（32位）
    std::vector<Instruction> instructions;
    int current_inst;
    int wait_cycles;
    int execution_cycles;

    Warp(int wid, int bid)
        : warp_id(wid), block_id(bid), status(WarpStatus::EMPTY),
          pc(0), active_mask(0xFFFFFFFF), current_inst(0),
          wait_cycles(0), execution_cycles(0) {}

    bool hasNextInstruction() const {
        return current_inst < instructions.size();
    }

    Instruction& getCurrentInstruction() {
        return instructions[current_inst];
    }

    void advancePC() {
        current_inst++;
        pc++;
    }

    float getActivity() const {
        // 计算活跃线程比例
        int active = __builtin_popcount(active_mask);
        return static_cast<float>(active) / WARP_SIZE;
    }
};

// 内存请求
struct MemoryRequest {
    int warp_id;
    int address;
    int size;
    int issue_cycle;
    bool is_write;
};

// 内存子系统
class MemorySubsystem {
private:
    struct Bank {
        int busy_until_cycle;
        std::queue<MemoryRequest> queue;

        Bank() : busy_until_cycle(0) {}
    };

    std::vector<Bank> banks;
    int current_cycle;
    int latency;
    int bandwidth;

    int getBankIndex(int address) const {
        return ((address / BANK_WIDTH) % NUM_BANKS);
    }

public:
    int total_requests;
    int conflict_stalls;

    MemorySubsystem(int lat = 100, int bw = 128)
        : banks(NUM_BANKS), current_cycle(0),
          latency(lat), bandwidth(bw),
          total_requests(0), conflict_stalls(0) {}

    int requestAccess(const MemoryRequest& req) {
        int bank_idx = getBankIndex(req.address);
        Bank& bank = banks[bank_idx];

        total_requests++;

        // 检查bank conflict
        int ready_cycle = std::max(current_cycle, bank.busy_until_cycle);
        if (bank.busy_until_cycle > current_cycle) {
            conflict_stalls++;
        }

        bank.busy_until_cycle = ready_cycle + latency;
        bank.queue.push(req);

        return ready_cycle + latency;
    }

    void cycle() {
        current_cycle++;
    }

    float getConflictRate() const {
        if (total_requests == 0) return 0.0f;
        return static_cast<float>(conflict_stalls) / total_requests;
    }
};

// SM（流式多处理器）
class StreamingMultiprocessor {
private:
    int sm_id;
    std::vector<std::unique_ptr<Warp>> warps;
    std::vector<int> block_warp_count;
    MemorySubsystem memory;

    int register_file_size;
    int shared_memory_size;
    int used_registers;
    int used_shared_memory;

    int cycle_count;
    int active_warps;
    int issued_instructions;
    int stalled_cycles;

    // 调度统计
    struct SchedulerStats {
        int total_issues;
        int alu_issues;
        int mem_issues;
        int barrier_issues;
        int no_ready_warps;

        SchedulerStats() : total_issues(0), alu_issues(0),
                          mem_issues(0), barrier_issues(0),
                          no_ready_warps(0) {}
    } stats;

public:
    StreamingMultiprocessor(int id, int reg_size = 256*1024,
                           int smem_size = 164*1024)
        : sm_id(id), warps(MAX_WARPS_PER_SM),
          register_file_size(reg_size),
          shared_memory_size(smem_size),
          used_registers(0), used_shared_memory(0),
          cycle_count(0), active_warps(0),
          issued_instructions(0), stalled_cycles(0) {
        for (int i = 0; i < MAX_WARPS_PER_SM; i++) {
            warps[i] = nullptr;
        }
    }

    bool canAcceptBlock(int threads_per_block, int registers_per_thread,
                       int shared_mem_per_block) const {
        // 检查资源限制
        int warps_needed = (threads_per_block + WARP_SIZE - 1) / WARP_SIZE;
        int registers_needed = warps_needed * WARP_SIZE * registers_per_thread * 4;

        if (active_warps + warps_needed > MAX_WARPS_PER_SM) return false;
        if (used_registers + registers_needed > register_file_size) return false;
        if (used_shared_memory + shared_mem_per_block > shared_memory_size) return false;
        if (static_cast<int>(block_warp_count.size()) >= MAX_BLOCKS_PER_SM) return false;

        return true;
    }

    void allocateBlock(int block_id, int threads_per_block,
                      int registers_per_thread, int shared_mem_per_block) {
        int warps_needed = (threads_per_block + WARP_SIZE - 1) / WARP_SIZE;
        int warp_start = -1;

        // 找到连续的空槽
        for (int i = 0; i <= MAX_WARPS_PER_SM - warps_needed; i++) {
            bool found = true;
            for (int j = 0; j < warps_needed; j++) {
                if (warps[i + j] != nullptr) {
                    found = false;
                    break;
                }
            }
            if (found) {
                warp_start = i;
                break;
            }
        }

        if (warp_start == -1) return;  // 无法分配

        // 创建Warp
        for (int i = 0; i < warps_needed; i++) {
            int warp_id = warp_start + i;
            warps[warp_id] = std::make_unique<Warp>(warp_id, block_id);
            warps[warp_id]->status = WarpStatus::READY;

            // 设置活跃掩码（最后一个warp可能不满）
            if (i == warps_needed - 1) {
                int remaining = threads_per_block - i * WARP_SIZE;
                warps[warp_id]->active_mask = (1 << remaining) - 1;
            }
        }

        block_warp_count.push_back(warps_needed);
        active_warps += warps_needed;
        used_registers += warps_needed * WARP_SIZE * registers_per_thread * 4;
        used_shared_memory += shared_mem_per_block;
    }

    // Oldest-First调度
    Warp* selectWarpOldestFirst() {
        Warp* selected = nullptr;
        int max_wait = -1;

        for (auto& warp_ptr : warps) {
            if (warp_ptr && warp_ptr->status == WarpStatus::READY) {
                if (warp_ptr->wait_cycles > max_wait) {
                    max_wait = warp_ptr->wait_cycles;
                    selected = warp_ptr.get();
                }
            }
        }

        return selected;
    }

    // Round-Robin调度
    static int rr_index;
    Warp* selectWarpRoundRobin() {
        int start_idx = rr_index;
        do {
            rr_index = (rr_index + 1) % MAX_WARPS_PER_SM;
            if (warps[rr_index] && warps[rr_index]->status == WarpStatus::READY) {
                return warps[rr_index].get();
            }
        } while (rr_index != start_idx);

        return nullptr;
    }

    // 执行一个周期
    void cycle() {
        cycle_count++;

        // 1. 更新内存子系统
        memory.cycle();

        // 2. 检查等待内存的Warp
        for (auto& warp_ptr : warps) {
            if (warp_ptr && warp_ptr->status == WarpStatus::WAITING_MEM) {
                // 简化：假设内存请求已完成
                warp_ptr->status = WarpStatus::READY;
            }
        }

        // 3. 调度Warp
        Warp* selected = selectWarpOldestFirst();

        if (selected == nullptr) {
            stats.no_ready_warps++;
            stalled_cycles++;

            // 更新等待计数
            for (auto& warp_ptr : warps) {
                if (warp_ptr && warp_ptr->status == WarpStatus::READY) {
                    warp_ptr->wait_cycles++;
                }
            }
            return;
        }

        // 4. 发射指令
        if (selected->hasNextInstruction()) {
            Instruction& inst = selected->getCurrentInstruction();

            switch (inst.type) {
                case InstructionType::ALU:
                    selected->execution_cycles += inst.latency;
                    stats.alu_issues++;
                    break;

                case InstructionType::LOAD:
                case InstructionType::STORE: {
                    MemoryRequest req;
                    req.warp_id = selected->warp_id;
                    req.address = inst.target_bank * 128;  // 简化
                    req.size = 128;
                    req.issue_cycle = cycle_count;
                    req.is_write = (inst.type == InstructionType::STORE);

                    memory.requestAccess(req);
                    selected->status = WarpStatus::WAITING_MEM;
                    stats.mem_issues++;
                    break;
                }

                case InstructionType::BARRIER:
                    selected->status = WarpStatus::WAITING_BARRIER;
                    stats.barrier_issues++;
                    break;

                default:
                    break;
            }

            stats.total_issues++;
            issued_instructions++;
            selected->advancePC();

            // 如果指令序列完成
            if (!selected->hasNextInstruction()) {
                selected->status = WarpStatus::FINISHED;
            }
        }

        // 5. 更新等待计数
        for (auto& warp_ptr : warps) {
            if (warp_ptr && warp_ptr->status == WarpStatus::READY) {
                warp_ptr->wait_cycles++;
            }
        }
    }

    void generateWorkload(int num_blocks, int warps_per_block,
                         int insts_per_warp) {
        std::random_device rd;
        std::mt19937 gen(rd());
        std::uniform_int_distribution<> inst_dist(0, 4);
        std::uniform_int_distribution<> latency_dist(1, 100);

        for (int b = 0; b < num_blocks; b++) {
            if (!canAcceptBlock(warps_per_block * WARP_SIZE, 32, 4096)) {
                break;
            }

            allocateBlock(b, warps_per_block * WARP_SIZE, 32, 4096);
        }

        // 为每个Warp生成指令
        for (auto& warp_ptr : warps) {
            if (warp_ptr && warp_ptr->status != WarpStatus::EMPTY) {
                for (int i = 0; i < insts_per_warp; i++) {
                    InstructionType type = static_cast<InstructionType>(
                        inst_dist(gen) % 5);
                    int latency = latency_dist(gen);

                    Instruction inst(type, latency);
                    inst.target_bank = (i * 4) % NUM_BANKS;

                    warp_ptr->instructions.push_back(inst);
                }
            }
        }
    }

    bool isComplete() const {
        for (const auto& warp_ptr : warps) {
            if (warp_ptr && warp_ptr->status != WarpStatus::EMPTY &&
                warp_ptr->status != WarpStatus::FINISHED) {
                return false;
            }
        }
        return true;
    }

    void printStats() const {
        std::cout << "=== SM " << sm_id << " Statistics ===" << std::endl;
        std::cout << "Total cycles: " << cycle_count << std::endl;
        std::cout << "Issued instructions: " << issued_instructions << std::endl;
        std::cout << "IPC: " << static_cast<float>(issued_instructions) / cycle_count << std::endl;
        std::cout << "Stalled cycles: " << stalled_cycles << std::endl;
        std::cout << "Active warps: " << active_warps << std::endl;
        std::cout << "Occupancy: " << (100.0f * active_warps / MAX_WARPS_PER_SM) << "%" << std::endl;
        std::cout << "ALU issues: " << stats.alu_issues << std::endl;
        std::cout << "Memory issues: " << stats.mem_issues << std::endl;
        std::cout << "Barrier issues: " << stats.barrier_issues << std::endl;
        std::cout << "No ready warp cycles: " << stats.no_ready_warps << std::endl;
        std::cout << "Memory conflict rate: " << (memory.getConflictRate() * 100) << "%" << std::endl;
        std::cout << std::endl;
    }
};

int StreamingMultiprocessor::rr_index = 0;

class GPUSimulator {

    std::vector<std::unique_ptr<StreamingMultiprocessor>> sms;

            for (auto& sm : sms) {
                sm->cycle();
            }
        }

    void runUntilComplete() {
        int max_cycles = 1000000;
        int cycle = 0;

        while (cycle < max_cycles) {
            bool all_complete = true;

            for (auto& sm : sms) {
                sm->cycle();
                if (!sm->isComplete()) {
                    all_complete = false;
                }
            }

            cycle++;

            if (all_complete) break;
        }

        std::cout << "Simulation completed in " << cycle << " cycles" << std::endl;
    }

    void generateWorkload(int blocks_per_sm, int warps_per_block,
                         int insts_per_warp) {

            sm->generateWorkload(blocks_per_sm, warps_per_block, insts_per_warp);
        }
    }

    void printStats() const {
        std::cout << "========== GPU Simulation Statistics ==========" << std::endl;
        std::cout << "Number of SMs: " << num_sm << std::endl;
        std::cout << std::endl;

        for (const auto& sm : sms) {
            sm->printStats();
        }
    }
```

```cpp
/**
 * GPU内存访问模式分析器
 * 分析内存访问的合并度和bank conflict
 */

# include <iostream>
# include <vector>
# include <algorithm>
# include <cmath>

class MemoryAccessAnalyzer {
private:
    static constexpr int WARP_SIZE = 32;
    static constexpr int BANK_WIDTH = 4;
    static constexpr int NUM_BANKS = 32;
    static constexpr int CACHE_LINE_SIZE = 128;

public:
    struct AccessPattern {
        std::vector<int> addresses;
        bool is_coalesced;
        int num_transactions;
        int bank_conflict_degree;
        float efficiency;
    };

    // 分析内存访问合并度
    static AccessPattern analyzeCoalescing(const std::vector<int>& addresses) {
        AccessPattern pattern;
        pattern.addresses = addresses;

        if (addresses.empty()) {
            pattern.is_coalesced = true;
            pattern.num_transactions = 0;
            pattern.efficiency = 1.0f;
            return pattern;
        }

        // 计算需要的事务数
        std::vector<int> cache_lines;
        for (int addr : addresses) {
            int line = addr / CACHE_LINE_SIZE;
            cache_lines.push_back(line);
        }

        std::sort(cache_lines.begin(), cache_lines.end());
        cache_lines.erase(std::unique(cache_lines.begin(), cache_lines.end()),
                         cache_lines.end());

        pattern.num_transactions = cache_lines.size();
        pattern.is_coalesced = (cache_lines.size() == 1);

        // 计算效率
        int total_bytes = addresses.size() * 4;  // 假设每个访问4字节
        int transferred_bytes = cache_lines.size() * CACHE_LINE_SIZE;
        pattern.efficiency = static_cast<float>(total_bytes) / transferred_bytes;

        return pattern;
    }

    // 分析bank conflict
    static AccessPattern analyzeBankConflict(const std::vector<int>& addresses) {
        AccessPattern pattern;
        pattern.addresses = addresses;

        if (addresses.empty()) {
            pattern.bank_conflict_degree = 0;
            return pattern;
        }

        // 计算每个bank的访问次数
        std::vector<int> bank_counts(NUM_BANKS, 0);
        for (int addr : addresses) {
            int bank = (addr / BANK_WIDTH) % NUM_BANKS;
            bank_counts[bank]++;
        }

        // 最大冲突度
        pattern.bank_conflict_degree = *std::max_element(
            bank_counts.begin(), bank_counts.end()) - 1;

        return pattern;
    }

    // 打印访问模式
    static void printPattern(const AccessPattern& pattern) {
        std::cout << "Addresses: ";
        for (size_t i = 0; i < std::min(size_t(8), pattern.addresses.size()); i++) {
            std::cout << pattern.addresses[i] << " ";
        }
        if (pattern.addresses.size() > 8) {
            std::cout << "... (" << pattern.addresses.size() << " total)";
        }
        std::cout << std::endl;

        std::cout << "Coalesced: " << (pattern.is_coalesced ? "Yes" : "No") << std::endl;
        std::cout << "Transactions: " << pattern.num_transactions << std::endl;
        std::cout << "Bank conflict degree: " << pattern.bank_conflict_degree << std::endl;
        std::cout << "Efficiency: " << (pattern.efficiency * 100) << "%" << std::endl;
        std::cout << std::endl;
    }

    // 生成常见访问模式
    static std::vector<int> generateSequentialAccess(int base, int stride, int count) {
        std::vector<int> addresses;
        for (int i = 0; i < count; i++) {
            addresses.push_back(base + i * stride);
        }
        return addresses;
    }

    static std::vector<int> generateStridedAccess(int base, int stride, int count) {
        std::vector<int> addresses;
        for (int i = 0; i < count; i++) {
            addresses.push_back(base + i * stride * 4);  // 每个线程4字节
        }
        return addresses;
    }

    static std::vector<int> generateRandomAccess(int min, int max, int count) {
        std::vector<int> addresses;
        for (int i = 0; i < count; i++) {
            addresses.push_back(min + (rand() % (max - min + 1)));
        }
        return addresses;
    }
};

// 测试函数
void testAccessPatterns() {

    // 测试1：完全合并访问
```
