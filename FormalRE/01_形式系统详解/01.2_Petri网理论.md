## 1. Petri网基础 {#petri网基础} {#petri网基础-petri网基础}

### 1 1 精确定义 {#1-精确定义} {#1-精确定义-1-精确定义}

- *定义1.1 (Petri网)**:

一个Petri网 $N$ 是一个四元组：

$$N = (P, T, F, M_0)$$

其中：

- **$P$**: 库所 (Places) 的有限集
- **$T$**: 变迁 (Transitions) 的有限集（$P \cap T = \emptyset$）
- **$F$**: $(P \times T) \cup (T \times P) \to \mathbb{N}$ — 流关系（弧权重）
- **$M_0$**: $P \to \mathbb{N}$ — 初始标识 (Marking)

## 1. 0 概念分析：Petri网 {#0-概念分析petri网} {#0-概念分析petri网-0-概念分析petri网}

### 1 0.2 属性分析 {#02-属性分析} {#02-属性分析-02-属性分析}

- _必要属性_* (Necessary Properties):

1. **二分性**: 库所和变迁是分离的
2. **有限性**: 库所和变迁都是有限集
3. **并发性**: 可以表示并发行为

- _充分属性_* (Sufficient Properties):

1. **流关系**: 定义了库所和变迁之间的连接
2. **标识**: 定义了系统的状态
3. **触发规则**: 定义了状态转换规则

- _本质属性_* (Essential Properties):

1. **并发建模**: 可以建模并发系统
2. **状态转换**: 通过标识和触发建模状态转换
3. **图灵完备性**: 带抑止弧的Petri网是图灵完备的

- _偶然属性_* (Accidental Properties):

1. **具体实现**: 基本Petri网、带抑止弧的Petri网等
2. **应用领域**: 工作流、微服务、区块链等
3. **分析技术**: 可达性分析、死锁检测等

### 1 0.3 外延分析 {#03-外延分析} {#03-外延分析-03-外延分析}

### 1 0.4 内涵分析 {#04-内涵分析} {#04-内涵分析-04-内涵分析}

### 1 2 图形表示 {#2-图形表示} {#2-图形表示-2-图形表示}

- _符号_*:

- ○ 库所 (Place)
- ▭ 变迁 (Transition)
- ● 令牌 (Token)
- → 弧 (Arc)

- _例子_*: 生产者-消费者

```text
    (buffer)
  ○─────○─────○
  ↑     ●     ↓
  │           │
[produce]  [consume]
  ▭           ▭
  ↑           ↓
```

### 1 3 标识与变迁触发 {#3-标识与变迁触发} {#3-标识与变迁触发-3-标识与变迁触发}

- *标识 (Marking)**: M: P → ℕ

- M(p) = 库所 p 中的令牌数

- *使能 (Enabled)**: 变迁 t 在标识 M 下使能，如果：

```text
∀p ∈ P. M(p) ≥ F(p, t)
```

- *触发 (Firing)**: M →ᵗ M'

```text
M'(p) = M(p) - F(p,t) + F(t,p)
```

## 2. 动态行为与可达性 {#动态行为与可达性} {#动态行为与可达性-动态行为与可达性}

### 2 1 可达性问题 {#1-可达性问题} {#1-可达性问题-1-可达性问题}

- _定义2.1_*: 标识 M' 从 M₀ **可达**，如果：

```text
∃t₁, ..., tₙ. M₀ →^t₁ M₁ →^t₂ ··· →^tₙ M'
```

- _可达集_*: Reach(N, M₀) = {M | M₀ →* M}

- _可达性问题_*: 给定 M'，判定 M' ∈ Reach(N, M₀)?

- _定理2.1 (Mayr 1984, Kosaraju 1982)**:
基本Petri网的可达性问题**可判定_*！

- _但_*: 复杂度极高 —— Ackermann函数级别

### 2 2 向量加法系统 (VASS) {#2-向量加法系统-vass} {#2-向量加法系统-vass-2-向量加法系统-vass}

- _等价_*: 基本Petri网 ≡ VASS

- _VASS定义_*: (ℕᵏ, V, v₀)

- 状态 = k维向量
- 转移 = 向量加法
- v → v + δ （当 v + δ ≥ 0）

- _Petri网 → VASS_*:

- 库所 → 向量维度
- 标识 → 向量
- 变迁 → 向量加法

## 3. 带抑止弧的Petri网 {#带抑止弧的petri网} {#带抑止弧的petri网-带抑止弧的petri网}

### 3 1 抑止弧 (Inhibitor Arc) {#1-抑止弧-inhibitor-arc} {#1-抑止弧-inhibitor-arc-1-抑止弧-inhi}

- _新元素_*: 从库所 p 到变迁 t 的**抑止弧**

- _语义_*: t 使能当且仅当：

```text
M(p) = 0  (p 中没有令牌)
```

- _图形_*: ○──⊸ ▭ (圆圈箭头)

### 3 2 图灵完备性 {#2-图灵完备性} {#2-图灵完备性-2-图灵完备性}

- _定理3.1_*: 带抑止弧的Petri网 ≡ 图灵机

- _证明思路_*: 模拟2计数器机 (Minsky机)

- _2计数器机_*:

- 2个计数器 c₁, c₂ ∈ ℕ
- 操作: INC(cᵢ), DEC(cᵢ), if cᵢ=0 then goto

- _Petri网模拟_*:

- 库所 p₁, p₂ 分别存储 c₁, c₂ 的值
- INC(c₁) → 变迁向 p₁ 增加令牌
- DEC(c₁) → 变迁从 p₁ 移除令牌
- if c₁=0 → 用抑止弧检测！

- _结论_*:

- **无抑止弧**: 可判定（VASS）
- **有抑止弧**: 图灵完备（不可判定）

## 4. 图灵完备性 {#图灵完备性} {#图灵完备性-图灵完备性}

### 4 1 从2计数器机到图灵机 {#1-从2计数器机到图灵机} {#1-从2计数器机到图灵机-1-从2计数器机到图灵机}

- *定理4.1 (Minsky 1961)**: 2计数器机 ≡ 图灵机

- _编码思路_*:

```text
磁带内容: ...010110...
          ↑
          头位置

编码为两个数:
  c₁ = 左边的二进制数
  c₂ = 右边的二进制数
```

- _操作_*:

- 右移: c₁ ← 2c₁ + b, c₂ ← ⌊c₂/2⌋
- 左移: c₁ ← ⌊c₁/2⌋, c₂ ← 2c₂ + b'

### 4 2 实践含义 {#2-实践含义} {#2-实践含义-2-实践含义}

- _带抑止弧的Petri网_*:

- ✅ 表达力强（图灵完备）
- ❌ 可达性不可判定（停机问题）
- ❌ 所有分析都不可判定

- *基本Petri网 (VASS)**:

- ✅ 可达性可判定
- ✅ 有界性可判定
- ❌ 表达力受限

```lean4
-- Petri网的定义
structure PetriNet where
  places : Type*
  transitions : Type*
  inputArcs : transitions → places → Nat
  outputArcs : transitions → places → Nat
  initialMarking : places → Nat

-- 变迁使能条件
def enabled (M : Marking N) (t : N.transitions) : Prop :=
  ∀ p, M p ≥ N.inputArcs t p

-- 变迁触发
def fire (M : Marking N) (t : N.transitions) (h : enabled M t) : Marking N :=
  fun p => M p - N.inputArcs t p + N.outputArcs t p

-- 可达性关系
inductive FiringSeq : Marking N → Marking N → Type
  | empty : ∀ M, FiringSeq M M
  | step : ∀ {M₁ M₂ M₃} (t : N.transitions) (h : enabled M₁ t),
           FiringSeq (fire M₁ t h) M₂ → FiringSeq M₁ M₂

-- 有界性定义
def isBounded (N : PetriNet) (k : Nat) : Prop :=
  ∀ (M : Marking N), reachable N.initialMarking M → ∀ p, M p ≤ k

-- 活性定义
def isLive (N : PetriNet) : Prop :=
  ∀ t : N.transitions, liveTransition t
```
