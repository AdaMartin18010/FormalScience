## 1 主题概述

本主题探讨**类型系统**与**基础设施资源**之间的核心映射关系，揭示编程语言中的类型概念如何对应到虚拟化、容器化和沙盒化技术中的资源抽象。

### 1.0 核心概念映射思维导图

```mermaid
mindmap
  root((01 核心概念映射))
    基本类型单元
      基本类型
        int → 基础镜像层
        string → 配置文件层
        bool → 开关配置
      类型实例化
        类型 → 镜像
        实例 → 容器
        值 → 运行时状态
      OCI镜像层
        只读层
        可写层
        层组合
    复合类型与组合运算符
      乘积类型
        struct → Pod
        tuple → 多容器Pod
        record → 配置对象
      和类型
        union → Deployment策略
        enum → 资源类型
        variant → 可选配置
      函数类型
        function → Service
        lambda → Ingress
        closure → 配置函数
      组合运算符
        × → 组合
        + → 选择
        → → 映射
    类型层级与继承
      类型继承
        基类型 → 基础镜像
        派生类型 → 应用镜像
        继承链 → 镜像分层
      子类型
        子类型 → 容器变体
        多态 → 容器替换
        协变/逆变 → 类型转换
      递归类型
        递归 → Kata Containers
        嵌套 → VM in Container
        不动点 → 稳定状态
```

**可视化文档**: 查看 [思维导图与知识矩阵](../思维导图与知识矩阵.md#31-01-核心概念映射) 获取更详细的思维导图。
