## 1 Petri网的历史与动机

**Petri网**由德国数学家Carl Adam Petri于1962年在其博士论文中首次提出，用于建模和分析并发系统。

**核心动机**：

1. **并发性难题**：传统的状态机难以自然表示并发行为
   - 状态空间爆炸：n个并发进程需要2^n个状态
   - 同步复杂：需要复杂的同步原语

2. **Petri网的优势**：
   - **自然表示并发**：多个变迁可以同时使能
   - **直观图形化**：库所-变迁图直观易懂
   - **形式化基础**：严格的数学定义，支持形式化分析

**历史里程碑**：

```text
1962 - Carl Petri博士论文，提出Petri网
1972 - 着色Petri网（Colored Petri Nets, CPN）
1980 - 时间Petri网（Timed Petri Nets, TPN）
1984 - 随机Petri网（Stochastic Petri Nets, SPN）
1990 - 工作流网（Workflow Nets, WF-net）
2000+ - Petri网在工业界广泛应用
2020+ - Petri网与AI结合（PetriRL）
```

## 2 Petri网的形式化定义

### 2.1 基本定义

**定义1.1（Petri网）**：

一个Petri网是一个五元组：

$$
PN = (P, T, F, W, M_0)
$$

其中：

- $P = \{p_1, p_2, \ldots, p_m\}$ 是**库所（Places）**的有限集合
- $T = \{t_1, t_2, \ldots, t_n\}$ 是**变迁（Transitions）**的有限集合
- $F \subseteq (P \times T) \cup (T \times P)$ 是**弧（Arcs）**的集合
- $W: F \to \mathbb{N}^+$ 是**权重函数**（如果未指定，默认为1）
- $M_0: P \to \mathbb{N}$ 是**初始标记（Initial Marking）**

**约束条件**：

1. $P \cap T = \emptyset$ （库所和变迁互不相交）
2. $P \cup T \neq \emptyset$ （至少有一个库所或变迁）
3. 弧只能连接库所和变迁，不能连接同类节点

**记号说明**：

- $\bullet t = \{p \in P \mid (p, t) \in F\}$ ：变迁 $t$ 的**输入库所**
- $t\bullet = \{p \in P \mid (t, p) \in F\}$ ：变迁 $t$ 的**输出库所**
- $\bullet p = \{t \in T \mid (t, p) \in F\}$ ：库所 $p$ 的**输入变迁**
- $p\bullet = \{t \in T \mid (p, t) \in F\}$ ：库所 $p$ 的**输出变迁**

### 2.2 标记与使能

**定义1.2（标记）**：

标记 $M: P \to \mathbb{N}$ 是从库所到非负整数的函数，$M(p)$ 表示库所 $p$ 中的**标记数量**（tokens）。

**定义1.3（使能）**：

变迁 $t \in T$ 在标记 $M$ 下**使能（Enabled）**，记为 $M[t\rangle$，当且仅当：

$$
\forall p \in \bullet t: M(p) \geq W(p, t)
$$

即：所有输入库所的标记数量都大于等于相应弧的权重。

**直观理解**：变迁使能意味着它具备了"点火"的条件，就像调度系统中任务满足了执行的资源需求。

### 2.3 变迁点火

**定义1.4（点火）**：

如果变迁 $t$ 在标记 $M$ 下使能，那么 $t$ 可以**点火（Fire）**，产生新标记 $M'$，记为 $M[t\rangle M'$，其中：

$$
M'(p) = \begin{cases}
M(p) - W(p, t) + W(t, p) & \text{if } p \in \bullet t \cup t\bullet \\
M(p) & \text{otherwise}
\end{cases}
$$

**点火规则的简洁表示**：

$$
M'(p) = M(p) - W(p, t) + W(t, p)
$$

其中未定义的权重默认为0。

**性质1.1（点火的原子性）**：

变迁点火是**原子操作**，不可中断，要么完全执行，要么不执行。

## 3 Petri网的图形表示

Petri网有标准的图形表示法：

```text
图形元素：
- 库所（Place）：   ○ 圆圈
- 变迁（Transition）：▯ 矩形或竖线
- 弧（Arc）：        → 有向箭头
- 标记（Token）：    ● 黑点（在库所内）
- 权重：            数字（在弧旁边，默认为1）
```

**示例：简单的生产者-消费者系统**

```text
    [p_buffer]
       ●●●
      ↗   ↘
  [t_produce]  [t_consume]
      ↑           ↓
   [p_producer] [p_consumer]
      ●           ●
```

**对应形式化表示**：

```text
P = {p_producer, p_buffer, p_consumer}
T = {t_produce, t_consume}
F = {(p_producer, t_produce), (t_produce, p_buffer),
     (p_buffer, t_consume), (t_consume, p_consumer)}
W = 所有弧权重为1（默认）
M_0 = {p_producer: 1, p_buffer: 3, p_consumer: 1}
```

## 4 Petri网的基本性质

### 4.1 有界性（Boundedness）

**定义1.5（k-有界）**：

Petri网 $(PN, M_0)$ 是**k-有界**的，如果对于所有可达标记 $M \in R(M_0)$ 和所有库所 $p \in P$：

$$
M(p) \leq k
$$

**定义1.6（有界）**：

如果存在 $k \in \mathbb{N}$ 使得Petri网是k-有界的，则称其为**有界的**。

**调度系统意义**：有界性保证资源数量有限，不会无限增长。

### 4.2 安全性（Safeness）

**定义1.7（安全）**：

Petri网是**安全的**，如果它是1-有界的，即：

$$
\forall M \in R(M_0), \forall p \in P: M(p) \leq 1
$$

**调度系统意义**：安全性保证每个库所最多一个标记，适合建模二值状态（如互斥锁）。

### 4.3 活性（Liveness）

**定义1.8（死变迁）**：

变迁 $t$ 在 $(PN, M_0)$ 中是**死的**，如果不存在可达标记使其使能：

$$
\not\exists M \in R(M_0): M[t\rangle
$$

**定义1.9（L4-活性）**：

变迁 $t$ 是**L4-活（Live）**的，如果对于所有可达标记 $M \in R(M_0)$，存在 $M' \in R(M)$ 使得 $t$ 在 $M'$ 下使能：

$$
\forall M \in R(M_0), \exists M' \in R(M): M'[t\rangle
$$

**Petri网的活性**：如果所有变迁都是L4-活的，则Petri网是活的。

**调度系统意义**：活性保证系统无死锁，所有任务最终都有机会执行。

### 4.4 可达性（Reachability）

**定义1.10（可达标记集）**：

从初始标记 $M_0$ **可达的标记集**定义为：

$$
R(M_0) = \{M \mid \exists \sigma \in T^*: M_0[\sigma\rangle M\}
$$

其中 $\sigma$ 是变迁点火序列。

**可达性问题**：给定标记 $M$，判断 $M \in R(M_0)$？

**复杂度**：可达性问题是**EXPSPACE-完全**的（Lipton, 1976）。

### 4.5 可逆性（Reversibility）

**定义1.11（可逆）**：

Petri网 $(PN, M_0)$ 是**可逆的**，如果对于所有可达标记 $M \in R(M_0)$，初始标记 $M_0$ 也是从 $M$ 可达的：

$$
\forall M \in R(M_0): M_0 \in R(M)
$$

**调度系统意义**：可逆性保证系统可以回到初始状态，适合循环调度系统。

## 5 可达性图与状态空间

**定义1.12（可达性图）**：

可达性图（Reachability Graph）是一个有向图 $RG = (V, E)$，其中：

- $V = R(M_0)$ ：可达标记集
- $E = \{(M, M') \mid \exists t \in T: M[t\rangle M'\}$ ：标记转换

**构造算法**：

```python
def construct_reachability_graph(PN, M0):
    """
    构造可达性图（广度优先搜索）
    """
    RG = nx.DiGraph()  # 有向图
    queue = [M0]
    visited = {M0}
    RG.add_node(M0)

    while queue:
        M = queue.pop(0)

        # 检查所有变迁
        for t in PN.transitions:
            if is_enabled(PN, M, t):
                M_new = fire_transition(PN, M, t)

                if M_new not in visited:
                    visited.add(M_new)
                    queue.append(M_new)
                    RG.add_node(M_new)

                RG.add_edge(M, M_new, label=t)

    return RG
```

**状态空间爆炸问题**：

- 可达标记数可能指数级增长
- 解决方法：
  1. 有界性分析（限制状态空间）
  2. 符号化方法（BDD）
  3. 局部分析（不变量分析）

## 6 Python实现：基础Petri网建模

### 6.1 使用NetworkX

```python
import networkx as nx
import matplotlib.pyplot as plt
from collections import defaultdict

class PetriNet:
    """基础Petri网实现"""

    def __init__(self):
        self.places = set()
        self.transitions = set()
        self.arcs = {}  # (source, target) -> weight
        self.marking = defaultdict(int)

    def add_place(self, name, tokens=0):
        """添加库所"""
        self.places.add(name)
        self.marking[name] = tokens

    def add_transition(self, name):
        """添加变迁"""
        self.transitions.add(name)

    def add_arc(self, source, target, weight=1):
        """添加弧"""
        self.arcs[(source, target)] = weight

    def is_enabled(self, transition):
        """检查变迁是否使能"""
        for place in self.places:
            if (place, transition) in self.arcs:
                required = self.arcs[(place, transition)]
                if self.marking[place] < required:
                    return False
        return True

    def fire(self, transition):
        """点火变迁"""
        if not self.is_enabled(transition):
            raise ValueError(f"Transition {transition} is not enabled")

        # 消耗输入库所的标记
        for place in self.places:
            if (place, transition) in self.arcs:
                weight = self.arcs[(place, transition)]
                self.marking[place] -= weight

        # 产生输出库所的标记
        for place in self.places:
            if (transition, place) in self.arcs:
                weight = self.arcs[(transition, place)]
                self.marking[place] += weight

    def visualize(self):
        """可视化Petri网"""
        G = nx.DiGraph()

        # 添加节点
        for p in self.places:
            G.add_node(p, node_type='place', tokens=self.marking[p])
        for t in self.transitions:
            G.add_node(t, node_type='transition')

        # 添加边
        for (source, target), weight in self.arcs.items():
            G.add_edge(source, target, weight=weight)

        # 布局
        pos = nx.spring_layout(G)

        # 绘制库所（圆形）
        place_nodes = [n for n, d in G.nodes(data=True) if d['node_type'] == 'place']
        nx.draw_networkx_nodes(G, pos, nodelist=place_nodes,
                                node_color='lightblue', node_shape='o', node_size=800)

        # 绘制变迁（方形）
        trans_nodes = [n for n, d in G.nodes(data=True) if d['node_type'] == 'transition']
        nx.draw_networkx_nodes(G, pos, nodelist=trans_nodes,
                                node_color='lightgreen', node_shape='s', node_size=800)

        # 绘制边
        nx.draw_networkx_edges(G, pos, arrowsize=20)
        nx.draw_networkx_labels(G, pos)

        # 显示标记数
        for p in place_nodes:
            tokens = G.nodes[p]['tokens']
            plt.text(pos[p][0], pos[p][1]-0.1, f"●"*tokens,
                     ha='center', fontsize=10, color='red')

        plt.axis('off')
        plt.show()

# 示例：生产者-消费者系统
pn = PetriNet()

# 添加库所
pn.add_place('p_producer', tokens=1)
pn.add_place('p_buffer', tokens=3)
pn.add_place('p_consumer', tokens=1)

# 添加变迁
pn.add_transition('t_produce')
pn.add_transition('t_consume')

# 添加弧
pn.add_arc('p_producer', 't_produce')
pn.add_arc('t_produce', 'p_buffer')
pn.add_arc('p_buffer', 't_consume')
pn.add_arc('t_consume', 'p_consumer')

# 可视化
pn.visualize()

# 模拟执行
print(f"Initial marking: {dict(pn.marking)}")
pn.fire('t_produce')
print(f"After t_produce: {dict(pn.marking)}")
pn.fire('t_consume')
print(f"After t_consume: {dict(pn.marking)}")
```

### 6.2 使用SNAKES库

```python
import snakes.plugins
snakes.plugins.load('gv', 'snakes.nets', 'nets')
from nets import *

# 创建Petri网
net = PetriNet('Producer-Consumer')

# 添加库所
net.add_place(Place('p_producer', [1]))  # 初始1个标记
net.add_place(Place('p_buffer', [1, 2, 3]))  # 初始3个标记
net.add_place(Place('p_consumer', [1]))

# 添加变迁
net.add_transition(Transition('t_produce'))
net.add_transition(Transition('t_consume'))

# 添加弧
net.add_input('p_producer', 't_produce', Variable('x'))
net.add_output('p_buffer', 't_produce', Variable('x'))
net.add_input('p_buffer', 't_consume', Variable('y'))
net.add_output('p_consumer', 't_consume', Variable('y'))

# 可视化
net.draw('producer_consumer.png')

# 模拟执行
print(f"Initial marking: {net.get_marking()}")
trans = net.transition('t_produce')
if trans.enabled(net.get_marking()):
    net.set_marking(trans.fire(Substitution()))
    print(f"After t_produce: {net.get_marking()}")
```

## 7 实际案例：生产者-消费者系统

**系统描述**：

- 生产者产生数据放入缓冲区
- 消费者从缓冲区取出数据消费
- 缓冲区容量有限（例如：5个位置）

**Petri网建模**：

```python
def producer_consumer_petri_net(buffer_size=5):
    """
    生产者-消费者系统的Petri网模型
    """
    pn = PetriNet()

    # 库所
    pn.add_place('p_producer_ready', tokens=1)  # 生产者准备就绪
    pn.add_place('p_empty_slots', tokens=buffer_size)  # 空槽位
    pn.add_place('p_full_slots', tokens=0)  # 满槽位
    pn.add_place('p_consumer_ready', tokens=1)  # 消费者准备就绪

    # 变迁
    pn.add_transition('t_produce')  # 生产
    pn.add_transition('t_consume')  # 消费

    # 弧（生产）
    pn.add_arc('p_producer_ready', 't_produce')
    pn.add_arc('t_produce', 'p_producer_ready')  # 循环
    pn.add_arc('p_empty_slots', 't_produce')
    pn.add_arc('t_produce', 'p_full_slots')

    # 弧（消费）
    pn.add_arc('p_consumer_ready', 't_consume')
    pn.add_arc('t_consume', 'p_consumer_ready')  # 循环
    pn.add_arc('p_full_slots', 't_consume')
    pn.add_arc('t_consume', 'p_empty_slots')

    return pn

# 运行模拟
pn = producer_consumer_petri_net(buffer_size=5)
pn.visualize()

# 模拟生产-消费循环
for i in range(10):
    if pn.is_enabled('t_produce'):
        pn.fire('t_produce')
        print(f"Step {i+1}: Produced. Buffer: {pn.marking['p_full_slots']}/{pn.marking['p_empty_slots']+pn.marking['p_full_slots']}")

    if pn.is_enabled('t_consume') and i % 2 == 0:  # 每两步消费一次
        pn.fire('t_consume')
        print(f"Step {i+1}: Consumed. Buffer: {pn.marking['p_full_slots']}/{pn.marking['p_empty_slots']+pn.marking['p_full_slots']}")
```

**分析**：

1. **有界性**：缓冲区大小固定，系统是有界的
2. **活性**：生产和消费可以持续进行，系统是活的
3. **死锁自由**：设计保证不会死锁
