## 1 概述

### 1.1 核心洞察

KV-Cache是LLM推理中最关键的内存管理问题。在自回归生成过程中，每个Token的注意力计算需要访问之前所有Token的Key和Value向量，将这些向量缓存起来可以避免重复计算，但会带来巨大的内存开销。

### 1.2 KV-Cache挑战

| 挑战 | 描述 | 影响 |
|------|------|------|
| **内存增长** | 随序列长度线性增长 | 限制最大序列长度 |
| **碎片化** | 请求长度不一导致碎片 | 内存利用率低 |
| **动态性** | 序列长度运行时确定 | 预分配困难 |
| **共享需求** | 前缀共享、Beam Search | 复制开销大 |

### 1.3 KV-Cache内存计算

```text
单请求KV-Cache大小:
  Size = 2 × num_layers × seq_len × num_heads × head_dim × dtype_size

示例 (LLaMA-70B, FP16, 2048 tokens):
  Size = 2 × 80 × 2048 × 64 × 128 × 2 bytes
       = 2 × 80 × 2048 × 8192 × 2 bytes
       ≈ 5.24 GB (单请求!)

批大小=32时:
  Total = 32 × 5.24 GB ≈ 168 GB > 单GPU容量
```

## 4 PagedAttention

### 4.2 数据结构

```python
# PagedAttention 核心数据结构

class Block:
    """物理内存块"""
    def __init__(self, block_size: int, num_layers: int):
        self.block_size = block_size  # Token数量
        self.num_layers = num_layers
        # 每层的K和V张量
        self.k_cache: Tensor  # [num_layers, block_size, num_heads, head_dim]
        self.v_cache: Tensor  # [num_layers, block_size, num_heads, head_dim]
        self.ref_count: int = 0  # 引用计数（用于共享）

class BlockTable:
    """逻辑块到物理块的映射表"""
    def __init__(self):
        self.table: List[int] = []  # table[i] = 物理块索引

    def append_block(self, physical_block_id: int):
        self.table.append(physical_block_id)

    def get_physical_block(self, logical_idx: int) -> int:
        return self.table[logical_idx]

class KVCacheManager:
    """KV-Cache管理器"""
    def __init__(self, num_blocks: int, block_size: int):
        self.num_blocks = num_blocks
        self.block_size = block_size
        self.free_blocks: List[int] = list(range(num_blocks))
        self.block_tables: Dict[int, BlockTable] = {}  # request_id -> BlockTable

    def allocate(self, request_id: int) -> int:
        """为请求分配一个新块"""
        if not self.free_blocks:
            raise MemoryError("No free blocks")
        block_id = self.free_blocks.pop()
        if request_id not in self.block_tables:
            self.block_tables[request_id] = BlockTable()
        self.block_tables[request_id].append_block(block_id)
        return block_id

    def free(self, request_id: int):
        """释放请求的所有块"""
        if request_id in self.block_tables:
            for block_id in self.block_tables[request_id].table:
                self.free_blocks.append(block_id)
            del self.block_tables[request_id]
```

### 4.3 PagedAttention计算

```python
def paged_attention(
    query: Tensor,           # [batch, 1, num_heads, head_dim]
    key_cache: Tensor,       # [num_blocks, block_size, num_heads, head_dim]
    value_cache: Tensor,     # [num_blocks, block_size, num_heads, head_dim]
    block_tables: Tensor,    # [batch, max_num_blocks]
    context_lens: Tensor,    # [batch]
) -> Tensor:
    """
    分页注意力计算
    """
    batch_size = query.shape[0]
    outputs = []

    for b in range(batch_size):
        ctx_len = context_lens[b]
        num_blocks = (ctx_len + block_size - 1) // block_size

        # 收集该请求的所有K,V块
        keys = []
        values = []
        for i in range(num_blocks):
            physical_block = block_tables[b, i]
            keys.append(key_cache[physical_block])
            values.append(value_cache[physical_block])

        # 拼接并计算注意力
        k = torch.cat(keys, dim=0)[:ctx_len]
        v = torch.cat(values, dim=0)[:ctx_len]

        attn_output = scaled_dot_product_attention(query[b], k, v)
        outputs.append(attn_output)

    return torch.stack(outputs)
```

### 5.2 抢占与换出策略

```python
class PreemptionPolicy:
    """抢占策略"""

    @staticmethod
    def select_victim(requests: List[Request], needed_blocks: int) -> List[Request]:
        """选择被抢占的请求"""
        # 策略1: FCFS逆序（最后到达的先抢占）
        # 策略2: 最短剩余优先（接近完成的保留）
        # 策略3: 优先级（低优先级先抢占）

        candidates = sorted(requests, key=lambda r: r.priority)
        victims = []
        freed_blocks = 0

        for req in candidates:
            if freed_blocks >= needed_blocks:
                break
            victims.append(req)
            freed_blocks += req.num_blocks

        return victims

class SwapPolicy:
    """换出到CPU/磁盘策略"""

    def swap_out(self, request: Request):
        """将请求的KV-Cache换出到CPU"""
        for block_id in request.block_table.table:
            gpu_block = self.gpu_blocks[block_id]
            cpu_block = self.cpu_allocate()
            # 异步复制
            cpu_block.copy_from_async(gpu_block)
            request.cpu_block_table.append(cpu_block)
        # 释放GPU块
        self.free_gpu_blocks(request.block_table)

    def swap_in(self, request: Request):
        """将请求的KV-Cache换入GPU"""
        for cpu_block_id in request.cpu_block_table.table:
            gpu_block_id = self.gpu_allocate()
            gpu_block = self.gpu_blocks[gpu_block_id]
            cpu_block = self.cpu_blocks[cpu_block_id]
            gpu_block.copy_from_async(cpu_block)
            request.block_table.append(gpu_block_id)
```

## 7 形式化模型

### 7.1 PagedAttention形式化

```text
定义: 分页KV-Cache系统

结构 PagedKVCache:
  PhysicalMemory = Array<Block>  // 物理块数组
  BlockTable = Map<LogicalIdx, PhysicalIdx>  // 块表
  FreeList = Set<PhysicalIdx>  // 空闲块集合

类型:
  Block = { k: Tensor[block_size, d], v: Tensor[block_size, d] }

操作:
  allocate: FreeList → Option<PhysicalIdx>
  free: PhysicalIdx → FreeList
  read: BlockTable × LogicalIdx → Block
  write: BlockTable × LogicalIdx × TokenKV → ()

不变量:
  ∀ bt ∈ BlockTables: ∀ idx ∈ bt.values: idx ∉ FreeList
  |PhysicalMemory| = |FreeList| + Σ(|bt| for bt in BlockTables)
```

### 7.2 内存利用率证明

```text
定理: PagedAttention达到最优内存利用率

证明:
设总物理块数为 N，块大小为 B
设请求集合 R，每个请求 r 的实际长度为 len(r)

传统预分配利用率:
  U_trad = Σlen(r) / (|R| × max_len)

PagedAttention利用率:
  需要块数 = Σ⌈len(r)/B⌉
  U_paged = Σlen(r) / (Σ⌈len(r)/B⌉ × B)

由于 ⌈len(r)/B⌉ × B ≤ len(r) + B - 1
所以 U_paged ≥ Σlen(r) / (Σlen(r) + |R|×(B-1))

当 Σlen(r) >> |R|×B 时，U_paged → 1

∎
```

### 7.3 抢占正确性

```text
定理: 抢占策略保证进度

给定:
  - 有限请求集 R
  - 有限内存 M
  - 抢占策略选择ref_count最低的块

证明:
1. 若无阻塞: 直接进度
2. 若阻塞但存在可抢占请求:
   - 选择受害者释放块
   - 受害者可换出到CPU
   - 当前请求获得进度
3. 若所有请求都无法抢占:
   - 说明所有请求都是高优先级
   - 按优先级顺序完成
4. 最终: 所有请求完成或被显式取消

∎
```

```python
class RadixAttention:
    """基数树注意力管理"""

    def __init__(self):
        self.radix_tree = RadixTree()

    def insert_sequence(self, tokens: List[int], kv_cache: KVCache):
        """插入序列，自动共享前缀"""
        node = self.radix_tree.root

        for i, token in enumerate(tokens):
            if token in node.children:
                # 前缀匹配，共享现有块
                node = node.children[token]
                node.ref_count += 1
            else:
                # 新分支，分配新块
                new_node = RadixNode(token)
                new_node.kv_block = self.allocate_block()
                node.children[token] = new_node
                node = new_node

    def get_shared_prefix_length(self, tokens: List[int]) -> int:
        """获取可共享的前缀长度"""
        node = self.radix_tree.root
        length = 0

        for token in tokens:
            if token in node.children:
                node = node.children[token]
                length += 1
            else:
                break

        return length
```

### 8.4 vLLM Block Manager 完整实现

vLLM的Block Manager不仅需要处理GPU块分配，还需要协调CPU换出、Copy-on-Write和前缀共享。

```python
class vLLMBlockManager:
    """vLLM 生产级 Block Manager 简化模型"""

    def __init__(self,
                 block_size: int,
                 num_gpu_blocks: int,
                 num_cpu_blocks: int,
                 watermark: float = 0.01):
        self.block_size = block_size
        self.watermark = watermark
        self.gpu_allocator = BlockAllocator(num_gpu_blocks)
        self.cpu_allocator = BlockAllocator(num_cpu_blocks)
        self.block_tables: Dict[int, BlockTable] = {}
        self.watermark_blocks = int(watermark * num_gpu_blocks)

    def can_allocate(self, seq_group) -> AllocStatus:
        """判断是否可以分配新序列"""
        num_required_blocks = 0
        for seq in seq_group.get_seqs():
            num_prompt_blocks = (
                len(seq.get_prompt_ids()) + self.block_size - 1
            ) // self.block_size
            num_required_blocks += num_prompt_blocks

        num_free_gpu_blocks = self.gpu_allocator.get_num_free_blocks()
        # 始终保留 watermark 块
        if num_required_blocks > num_free_gpu_blocks - self.watermark_blocks:
            total = self.gpu_allocator.get_total_num_blocks()
            if num_required_blocks > total:
                return AllocStatus.NEVER
            return AllocStatus.LATER
        return AllocStatus.OK

    def can_append_slots(self, seq_group, num_lookahead_slots: int = 0) -> bool:
        """检查解码步是否有足够空间追加token"""
        num_free_gpu_blocks = self.gpu_allocator.get_num_free_blocks()
        num_required_blocks = 0
        for seq in seq_group.get_seqs():
            logical_blocks = self.block_tables[seq.seq_id].num_blocks
            num_tokens = len(seq.get_token_ids())
            # 加上lookahead slots（用于推测解码）
            new_tokens = num_tokens + num_lookahead_slots
            new_blocks = (new_tokens + self.block_size - 1) // self.block_size
            num_required_blocks += max(0, new_blocks - logical_blocks)
        return num_required_blocks <= num_free_gpu_blocks - self.watermark_blocks

    def fork(self, parent_seq: int) -> int:
        """Copy-on-Write 复制序列块表"""
        parent_table = self.block_tables[parent_seq]
        child_seq = parent_seq + 1
        self.block_tables[child_seq] = parent_table.fork()
        for block in self.block_tables[child_seq].blocks:
            block.ref_count += 1
        return child_seq

    def write_on_copy(self, seq_id: int, block_idx: int):
        """执行写时复制"""
        block_table = self.block_tables[seq_id]
        block = block_table.blocks[block_idx]
        if block.ref_count > 1:
            # 分配新块并复制数据
            new_block = self.gpu_allocator.allocate()
            new_block.copy_from(block)
            block.ref_count -= 1
            block_table.blocks[block_idx] = new_block

    def swap_out(self, seq) -> int:
        """将序列换出到CPU，返回交换的块数"""
        gpu_blocks = self.block_tables[seq.seq_id].blocks
        cpu_blocks = []
        for gpu_block in gpu_blocks:
            cpu_block = self.cpu_allocator.allocate()
            cpu_blocks.append(cpu_block)
        self.block_tables[seq.seq_id] = BlockTable(cpu_blocks)
        self.gpu_allocator.free_blocks(gpu_blocks)
        return len(gpu_blocks)

    def swap_in(self, seq) -> int:
        """将序列从CPU换回GPU"""
        cpu_blocks = self.block_tables[seq.seq_id].blocks
        gpu_blocks = []
        for _ in cpu_blocks:
            gpu_block = self.gpu_allocator.allocate()
            gpu_blocks.append(gpu_block)
        self.block_tables[seq.seq_id] = BlockTable(gpu_blocks)
        self.cpu_allocator.free_blocks(cpu_blocks)
        return len(cpu_blocks)
```

### 8.5 TensorRT-LLM KV-Cache Manager 架构

TensorRT-LLM的KV Cache Manager采用**预分配池（Pre-allocated Pool）**策略，与vLLM的动态分配不同：

```python
class TensorRTLLMKVCacheManager:
    """
    TensorRT-LLM KV Cache Manager 概念实现
    特点：
    1. 启动时预分配所有KV缓存内存
    2. 使用位图（bitmap）快速分配/释放块
    3. 支持paged和unpaged两种模式
    """

    def __init__(self,
                 num_layers: int,
                 num_kv_heads: int,
                 head_size: int,
                 block_size: int,
                 max_blocks: int,
                 dtype: torch.dtype = torch.float16):
        self.block_size = block_size
        self.max_blocks = max_blocks
        self.num_layers = num_layers

        # 预分配连续内存池
        self.kv_pool_shape = (
            num_layers, 2,  # K and V
            max_blocks,
            block_size,
            num_kv_heads,
            head_size
        )
        self.kv_pool = torch.empty(self.kv_pool_shape, dtype=dtype, device='cuda')

        # 使用位图管理空闲块
        self.block_bitmap = BitMap(max_blocks)
        self.seq_block_map: Dict[int, List[int]] = {}

    def allocate_blocks(self, seq_id: int, num_tokens: int) -> List[int]:
        """为新序列分配块"""
        num_blocks = (num_tokens + self.block_size - 1) // self.block_size
        block_indices = self.block_bitmap.allocate(num_blocks)
        self.seq_block_map[seq_id] = block_indices
        return block_indices

    def append_tokens(self, seq_id: int, num_new_tokens: int) -> List[int]:
        """追加token，可能需要分配新块"""
        current_blocks = self.seq_block_map[seq_id]
        current_capacity = len(current_blocks) * self.block_size
        current_tokens = self.seq_lengths.get(seq_id, 0)

        if current_tokens + num_new_tokens > current_capacity:
            additional_blocks_needed = (
                current_tokens + num_new_tokens - current_capacity + self.block_size - 1
            ) // self.block_size
            new_blocks = self.block_bitmap.allocate(additional_blocks_needed)
            current_blocks.extend(new_blocks)

        self.seq_lengths[seq_id] = current_tokens + num_new_tokens
        return current_blocks

    def get_kv_cache_address(self, seq_id: int, layer_idx: int, is_v: bool = False):
        """获取序列在预分配池中的张量视图"""
        block_indices = self.seq_block_map[seq_id]
        kv_idx = 1 if is_v else 0
        # 返回 [num_blocks, block_size, num_kv_heads, head_size] 视图
        return self.kv_pool[layer_idx, kv_idx, block_indices]

    def free_sequence(self, seq_id: int):
        """释放序列占用的所有块"""
        if seq_id in self.seq_block_map:
            blocks = self.seq_block_map[seq_id]
            self.block_bitmap.free(blocks)
            del self.seq_block_map[seq_id]
            if seq_id in self.seq_lengths:
                del self.seq_lengths[seq_id]
```

### 8.6 SGLang 结构化生成与KV-Cache调度

SGLang在结构化生成场景下对KV-Cache有特殊调度需求：当约束分支产生多个候选时，需要高效共享公共前缀并隔离分支后缀。

```python
class SGLangStructuredKVScheduler:
    """
    SGLang结构化生成KV调度器
    处理约束解码中的分支与合并
    """

    def __init__(self, base_manager: RadixAttentionManager):
        self.base_manager = base_manager
        self.constraint_branches: Dict[int, List[int]] = {}  # parent -> children

    def branch(self, parent_seq_id: int, num_branches: int) -> List[int]:
        """在约束点创建分支，共享前缀KV"""
        child_ids = []
        for i in range(num_branches):
            child_id = parent_seq_id * 1000 + i
            # 复制块表（COW）
            self.base_manager.block_tables[child_id] = (
                self.base_manager.block_tables[parent_seq_id].copy()
            )
            for block in self.base_manager.block_tables[child_id].blocks:
                block.ref_count += 1
            child_ids.append(child_id)
        self.constraint_branches[parent_seq_id] = child_ids
        return child_ids

    def merge(self, child_seq_ids: List[int], winner_idx: int) -> int:
        """选择获胜分支，释放其他分支独占的块"""
        winner_id = child_seq_ids[winner_idx]
        for idx, child_id in enumerate(child_seq_ids):
            if child_id == winner_id:
                continue
            # 释放失败分支的独占块
            for block in self.base_manager.block_tables[child_id].blocks:
                block.ref_count -= 1
                if block.ref_count == 0:
                    self.base_manager.gpu_allocator.free(block)
            del self.base_manager.block_tables[child_id]
        return winner_id

    def schedule_constrained_batch(self, requests: List[Request]) -> List[List[int]]:
        """将具有相同约束FSM状态的请求分组批处理"""
        groups = defaultdict(list)
        for req in requests:
            fsm_state = req.fsm_state if req.constraint else -1
            groups[fsm_state].append(req.seq_id)
        return list(groups.values())
```
