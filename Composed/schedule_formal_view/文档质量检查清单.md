## 一、结构完整性检查

### 1.1 必需章节

每个文档应包含以下章节：

- [ ] **目录**：完整的目录结构（一级不带点号，二级带点号）
- [ ] **概述部分**：核心挑战、特征描述
- [ ] **技术细节**：算法、策略、机制详细说明
- [ ] **形式化模型**：问题定义、复杂度、定理
- [ ] **跨领域洞察**：关键洞察、类比、权衡
- [ ] **多维度对比**：算法对比、策略对比
- [ ] **相关主题**：6-8个相关主题链接
- [ ] **文档状态**：最后更新日期、完成状态

### 1.2 章节编号规范

- [ ] 一级标题：使用数字编号（1, 2, 3...）
- [ ] 二级标题：使用数字+点号（1.1, 1.2, 2.1...）
- [ ] 三级标题：使用数字+点号（1.1.1, 1.1.2...）
- [ ] 目录中的链接应与实际标题一致

## 二、内容质量检查

### 2.1 概述部分

- [ ] **核心挑战**：清晰描述该领域的核心挑战
- [ ] **特征描述**：准确描述技术特征和特点
- [ ] **应用场景**：说明适用场景和用例

### 2.2 技术细节

- [ ] **算法描述**：算法步骤清晰、逻辑完整
- [ ] **代码示例**：代码示例正确、可理解
- [ ] **流程图**：流程图清晰、逻辑正确
- [ ] **技术术语**：术语使用准确、一致

### 2.3 跨领域洞察

- [ ] **关键洞察**：至少2-3个关键洞察
- [ ] **类比分析**：与其他领域的类比
- [ ] **权衡分析**：性能、复杂度、成本的权衡

### 2.4 多维度对比

- [ ] **对比表格**：至少2-3个对比表格
- [ ] **对比维度**：多维度对比（性能、复杂度、适用场景等）
- [ ] **对比准确性**：对比数据准确、有依据

---

## 三、形式化内容检查

### 3.1 问题定义

- [ ] **形式化定义**：使用数学符号明确定义问题
- [ ] **符号说明**：所有符号都有清晰说明
- [ ] **约束条件**：约束条件完整、准确

### 3.2 复杂度分析

- [ ] **时间复杂度**：明确给出时间复杂度
- [ ] **空间复杂度**：明确给出空间复杂度
- [ ] **复杂度证明**：复杂度分析有依据

### 3.3 定理和证明

- [ ] **定理陈述**：定理陈述清晰、准确
- [ ] **证明过程**：证明过程完整、逻辑正确
- [ ] **定理编号**：定理编号规范（如：定理14.1）

---

## 四、交叉引用检查

### 4.1 相关主题链接

- [ ] **链接数量**：至少6-8个相关主题链接
- [ ] **链接准确性**：所有链接路径正确
- [ ] **链接描述**：每个链接有简短描述
- [ ] **双向链接**：重要关系建立双向链接

### 4.2 链接有效性

- [ ] **路径正确**：相对路径正确
- [ ] **文件存在**：链接的文件确实存在
- [ ] **锚点正确**：锚点链接正确（如有）

## 五、格式一致性检查

### 5.1 Markdown格式

- [ ] **标题格式**：标题使用正确的Markdown语法
- [ ] **列表格式**：列表使用统一的格式
- [ ] **代码块**：代码块使用正确的语言标记
- [ ] **表格格式**：表格格式正确、对齐

### 5.2 数学公式

- [ ] **LaTeX语法**：数学公式使用正确的LaTeX语法
- [ ] **公式编号**：重要公式有编号（如需要）
- [ ] **公式显示**：公式在渲染后正确显示

### 5.3 术语和符号

- [ ] **术语统一**：术语使用统一、一致
- [ ] **符号统一**：数学符号使用统一
- [ ] **缩写说明**：首次出现的缩写有说明
