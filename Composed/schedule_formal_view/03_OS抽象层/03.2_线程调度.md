## 1 线程模型对比

### 1.1 1:1模型（内核线程）

**特点**：

```
每个用户线程对应一个内核线程
优点：真正的并行、阻塞系统调用不阻塞整个进程
缺点：线程数量受限、上下文切换开销大
```

| 特性 | 值 |
|------|-----|
| 最大线程数 | 受限于内核资源 |
| 上下文切换 | 2-5μs |
| 阻塞处理 | 独立 |
| 并行性 | 真正并行 |

### 1.2 N:1模型（用户线程）

**特点**：

```
多个用户线程映射到一个内核线程
优点：线程数量大、切换快
缺点：无法真正并行、阻塞调用阻塞全部
```

| 特性 | 值 |
|------|-----|
| 最大线程数 | 数十万 |
| 上下文切换 | 纳秒级 |
| 阻塞处理 | 全部阻塞 |
| 并行性 | 无 |

### 1.3 M:N模型（混合线程）

**特点**：

```
M个用户线程映射到N个内核线程（M >= N）
优点：结合两者优势
缺点：实现复杂
```

| 特性 | 值 |
|------|-----|
| 最大线程数 | 较大 |
| 上下文切换 | 用户态：ns，内核态：μs |
| 阻塞处理 | 部分阻塞 |
| 并行性 | 真正并行 |

## 3 N:1模型实现

### 3.1 用户态调度器

**协程实现**：

```c
// 用户态线程（协程）结构
typedef struct {
    void *stack;
    void *context;  // 保存的寄存器状态
    enum { RUNNING, READY, BLOCKED } state;
    struct uthread *next;
} uthread_t;

// 用户态调度器
void uthread_schedule() {
    uthread_t *current = get_current_uthread();
    uthread_t *next = pick_next_uthread();

    if (current != next) {
        switch_context(&current->context, &next->context);
    }
}
```

### 3.2 协作式调度

**yield让出CPU**：

```c
void yield() {
    uthread_schedule();
}

// 使用示例
void task() {
    while (1) {
        do_work();
        yield();  // 主动让出CPU
    }
}
```

### 3.3 抢占式用户态调度

**信号驱动的抢占**：

```c
// 设置定时器信号
void setup_preemption() {
    struct sigaction sa;
    sa.sa_handler = preemption_handler;
    sigaction(SIGALRM, &sa, NULL);

    // 设置100Hz的抢占频率
    struct itimerval timer = {
        .it_interval = { 0, 10000 },  // 10ms
        .it_value = { 0, 10000 }
    };
    setitimer(ITIMER_REAL, &timer, NULL);
}

void preemption_handler(int sig) {
    uthread_schedule();
}
```

## 4 M:N模型实现

### 4.1 调度器激活

**概念**：内核通知用户态调度器事件

```
1. 用户线程阻塞（如I/O）
2. 内核切换到其他用户线程
3. 内核通知用户态调度器（upcall）
4. 用户态调度器选择新线程运行
```

### 4.2 两级调度

**用户态+内核态协同**：

```c
// 用户态调度
void user_schedule() {
    // 选择用户线程
    uthread_t *next = select_uthread();

    // 如果没有就绪线程，让出CPU
    if (next == NULL) {
        syscall(SYS_sched_yield);
    }
}

// 内核态调度
void kernel_schedule() {
    // 选择内核线程（LWP）
    task_t *next = pick_next_task();
    context_switch(next);
}
```

### 4.3 现代实现

**Go Goroutine调度器（GMP模型）**：

```
G: Goroutine（用户线程）
M: Machine（内核线程）
P: Processor（逻辑处理器）

调度流程：
1. G在P的本地队列中运行
2. 如果P的队列为空，从全局队列偷取
3. 如果M阻塞，P可以绑定到其他M
```

## 5 线程同步与调度

### 5.1 互斥锁调度

**futex实现**：

```c
// futex快速路径
int futex_lock(int *futex) {
    int c;
    // 尝试原子获取锁
    if ((c = cmpxchg(futex, 0, 1)) != 0) {
        // 慢路径：进入内核等待
        if (c != 2) c = xchg(futex, 2);
        while (c != 0) {
            syscall(SYS_futex, futex, FUTEX_WAIT, 2);
            c = xchg(futex, 2);
        }
    }
    return 0;
}
```

### 5.2 条件变量调度

**等待/通知机制**：

```c
// 条件变量等待
void cond_wait(cond_t *cond, mutex_t *mutex) {
    // 释放锁并加入等待队列
    mutex_unlock(mutex);

    // 原子操作：检查条件并睡眠
    futex_wait(&cond->futex);

    // 重新获取锁
    mutex_lock(mutex);
}

// 条件变量通知
void cond_signal(cond_t *cond) {
    futex_wake(&cond->futex, 1);  // 唤醒一个
}
```

### 5.3 读写锁调度

**读优先 vs 写优先**：

| 策略 | 特点 | 适用场景 |
|------|------|----------|
| 读优先 | 高吞吐，写饥饿 | 读多写少 |
| 写优先 | 低延迟，读等待 | 写频繁 |
| 公平 | 无饥饿 | 通用 |

## 6 形式化定义

**线程调度问题**：

给定：

- 线程集合$T = \{t_1, \ldots, t_n\}$
- 处理器集合$P = \{p_1, \ldots, p_m\}$
- 就绪时间$r: T \rightarrow \mathbb{R}^+$
- 执行时间$e: T \rightarrow \mathbb{R}^+$
- 截止时间$d: T \rightarrow \mathbb{R}^+$

寻找调度$S: T \times \mathbb{R}^+ \rightarrow P \cup \{\bot\}$

**公平性定义**：

$$
\forall t_i, t_j: \lim_{\tau\to\infty} \frac{\text{cpu\_time}(t_i, \tau)}{\text{cpu\_time}(t_j, \tau)} = \frac{w_i}{w_j}
$$
