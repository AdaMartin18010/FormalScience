## 1 页表管理

### 1.1 多级页表

**x86-64 5级页表**：

```
CR3 → PML4 → PDPT → PD → PT → Page
  │     │      │     │    │
  │     │      │     │    └── 4KB页
  │     │      │     │
  │     │      │     └── 2MB大页（可选）
  │     │      │
  │     │      └── 1GB大页（可选）
  │     │
  │     └── 512GB页（可选，5级）
  │
  └── 256TB地址空间
```

**页表项结构**：

| 位 | 名称 | 描述 |
|----|------|------|
| 0 | P | Present |
| 1 | R/W | Read/Write |
| 2 | U/S | User/Supervisor |
| 3 | PWT | Write Through |
| 4 | PCD | Cache Disable |
| 5 | A | Accessed |
| 6 | D | Dirty |
| 7 | PS | Page Size |
| 12-51 | PFN | Page Frame Number |
| 63 | XD | Execute Disable |

### 1.2 页表遍历

**硬件页表遍历器（PTW）**：

```c
// 页表遍历过程
uint64_t walk_page_table(uint64_t cr3, uint64_t vaddr) {
    uint64_t pml4_idx = (vaddr >> 39) & 0x1FF;
    uint64_t pdpt_idx = (vaddr >> 30) & 0x1FF;
    uint64_t pd_idx   = (vaddr >> 21) & 0x1FF;
    uint64_t pt_idx   = (vaddr >> 12) & 0x1FF;

    // 读取PML4
    uint64_t *pml4 = phys_to_virt(cr3);
    uint64_t pdpt_entry = pml4[pml4_idx];
    if (!(pdpt_entry & 1)) return PAGE_FAULT;

    // 读取PDPT
    uint64_t *pdpt = phys_to_virt(pdpt_entry & PFN_MASK);
    uint64_t pd_entry = pdpt[pdpt_idx];
    if (!(pd_entry & 1)) return PAGE_FAULT;

    // 检查1GB大页
    if (pd_entry & PAGE_SIZE) {
        return (pd_entry & PFN_MASK_1G) | (vaddr & OFFSET_MASK_1G);
    }

    // 继续遍历...
}
```

### 1.3 页表缓存

**页表缓存优化**：

```
PML4缓存：1条目（最近使用）
PDPT缓存：4条目
PD缓存：32条目
PT缓存：128条目
```

## 2 TLB调度

### 2.1 TLB结构

**多级TLB**（Intel Skylake）：

| TLB | 条目数 | 页大小 | 延迟 |
|-----|--------|--------|------|
| L1 DTLB | 64 | 4KB | 1周期 |
| L1 DTLB | 32 | 2MB/4MB | 1周期 |
| L1 ITLB | 128 | 4KB | 1周期 |
| L2 STLB | 1536 | 4KB/2MB | 5周期 |

### 2.2 TLB替换策略

**伪LRU实现**：

```c
// TLB替换
int tlb_replace_entry(tlb_t *tlb, uint64_t vaddr, uint64_t paddr) {
    // 查找空闲条目
    for (int i = 0; i < TLB_SIZE; i++) {
        if (!tlb->entries[i].valid) {
            return i;
        }
    }

    // 使用PLRU选择牺牲者
    int victim = plru_select(tlb->plru_bits);

    // 驱逐旧条目
    if (tlb->entries[victim].dirty) {
        tlb_writeback(&tlb->entries[victim]);
    }

    return victim;
}
```

### 2.3 超级页支持

**大页TLB条目**：

```
4KB页：1个TLB条目覆盖4KB
2MB页：1个TLB条目覆盖2MB（512x）
1GB页：1个TLB条目覆盖1GB（262144x）
```

### 2.4 TLB shootdown

**跨核TLB刷新**：

```c
// TLB shootdown流程
void tlb_shootdown(uint64_t vaddr) {
    // 1. 获取cpu掩码
    cpumask_t mask;
    mm_cpumask(mm, &mask);

    // 2. 发送IPI到其他CPU
    for_each_cpu(cpu, &mask) {
        if (cpu != smp_processor_id()) {
            apic_send_ipi(cpu, TLB_FLUSH_VECTOR);
        }
    }

    // 3. 本地刷新
    invlpg(vaddr);

    // 4. 等待其他CPU完成
    for_each_cpu(cpu, &mask) {
        while (per_cpu(tlb_flush_pending, cpu));
    }
}
```

## 3 大页支持

### 3.1 大页类型

| 页大小 | x86-64 | ARM64 | 覆盖率 |
|--------|--------|-------|--------|
| 4KB | 支持 | 支持 | 1x |
| 16KB | - | 支持 | 4x |
| 64KB | - | 支持 | 16x |
| 2MB | 支持 | 支持 | 512x |
| 1GB | 支持 | 支持 | 262144x |
| 16GB | - | 支持 | 4194304x |

### 3.2 透明大页

**Linux THP机制**：

```
1. 应用程序分配内存（4KB页）
2. khugepaged扫描合并机会
3. 找到连续的4KB页
4. 合并为2MB大页
5. 更新页表
```

**配置**：

```bash
# 启用/禁用THP
echo always > /sys/kernel/mm/transparent_hugepage/enabled
echo madvise > /sys/kernel/mm/transparent_hugepage/enabled
echo never > /sys/kernel/mm/transparent_hugepage/enabled
```

### 3.3 大页分配策略

**hugetlbfs**：

```bash
# 预留大页
echo 1024 > /sys/kernel/mm/hugepages/hugepages-2048kB/nr_hugepages

# 使用大页
mount -t hugetlbfs none /mnt/huge
```

## 4 内存分配策略

### 4.1 物理内存分配

**Buddy系统**：

```c
// Buddy分配
struct page *alloc_pages(gfp_t gfp_mask, unsigned int order) {
    struct zone *zone;
    struct page *page;

    // 选择合适的zone
    zone = zonelist_zone(zonelist, classzone_idx);

    // 从Buddy系统分配
    page = rmqueue(zone, order, gfp_mask);

    return page;
}
```

### 4.2 虚拟内存分配

**VMA管理**：

```c
// 虚拟内存区域
struct vm_area_struct {
```

### 4.3 NUMA内存分配

**NUMA策略**：

```c
// NUMA内存分配
struct page *alloc_pages_numa(gfp_t gfp, int nid, unsigned int order) {
    struct zone *zone = &NODE_DATA(nid)->node_zones[ZONE_NORMAL];
    return rmqueue(zone, order, gfp);
}

// 交错分配
struct page *alloc_pages_interleaved(gfp_t gfp, unsigned int order) {
    static int last_nid = 0;
    int nid = next_node_in_mask(last_nid++, &node_online_map);
    return alloc_pages_numa(gfp, nid, order);
}
```

## 5 形式化定义

**页表遍历延迟**：

$$
T_{walk} = \sum_{i=1}^{d} (T_{mem} + T_{lookup})
$$

其中$d$是页表深度。

**TLB命中率**：

$$
H_{TLB} = \frac{N_{hit}}{N_{hit} + N_{miss}}
$$

**有效内存访问时间**：

$$
T_{effective} = H_{TLB} \cdot T_{TLB} + (1 - H_{TLB}) \cdot (T_{TLB} + T_{walk} + T_{mem})
$$

## 6 Linux内核实现分析

**页分配快速路径**：

```c
// 快速路径分配
struct page *get_page_from_freelist(gfp_t gfp_mask,
                                     unsigned int order,
                                     int alloc_flags,
                                     const struct alloc_context *ac) {
    struct zoneref *z;
    struct zone *zone;
    struct page *page;

    // 遍历zonelist
    for_each_zone_zonelist_nodemask(zone, z, ac->zonelist,
                                     ac->high_zoneidx, ac->nodemask) {
        // 尝试分配
        page = rmqueue(zone, order, gfp_mask);
        if (page) {
            // 成功
            return page;
        }

        // 检查是否需要唤醒kswapd
        if (zone_watermark_ok(zone, order, ...)) {
            wake_all_kswapds(order, ac);
        }
    }

    return NULL;
}
```
