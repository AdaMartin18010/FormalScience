## 1 TEE概述

### 1.1 什么是TEE

**Trusted Execution Environment (TEE)** 是一个独立的安全执行环境，即使在操作系统被攻破的情况下，也能保护敏感数据和代码的机密性和完整性。

**TEE的核心特性**:

```text
TEE = (隔离执行, 内存加密, 远程认证, 安全存储)
```

**主流TEE技术对比**:

| TEE技术 | 厂商 | 隔离粒度 | 内存加密 | 性能开销 | 应用场景 |
|---------|------|----------|----------|----------|----------|
| **Intel SGX** | Intel | 进程级 (Enclave) | ✅ | 10-50% | 云计算、区块链 |
| **AMD SEV** | AMD | VM级 | ✅ | 5-15% | 机密虚拟机 |
| **Intel TDX** | Intel | VM级 | ✅ | 5-10% | 机密容器 |
| **ARM TrustZone** | ARM | CPU模式 | ✅ | <5% | 移动设备、IoT |

### 1.2 TEE的核心安全属性

**定义27.11（TEE安全属性）**:

TEE必须提供以下安全保证:

1. **机密性（Confidentiality）**:
   $$\forall d \in \text{Data}_{TEE}, \text{Accessible}(d, \text{Attacker}) = \bot$$

2. **完整性（Integrity）**:
   $$\forall c \in \text{Code}_{TEE}, \text{Modifiable}(c, \text{Attacker}) = \bot$$

3. **认证性（Authenticity）**:
   $$\text{RemoteAttestation}(TEE) \to \text{VerifiedState}(TEE)$$

### 1.3 TEE的应用场景

**典型应用**:

1. **云计算**: 机密虚拟机（Confidential Computing）
2. **区块链**: 隐私智能合约
3. **金融**: 安全支付、密钥管理
4. **医疗**: 隐私数据分析
5. **AI**: 机密ML训练和推理

## 2 Intel SGX调度模型

### 2.1 SGX架构

**Intel SGX (Software Guard Extensions)** 提供进程级TEE，通过Enclave隔离敏感代码和数据。

**SGX架构图**:

```text
┌────────────────────────────────────────────┐
│         User Application                    │
├────────────────────────────────────────────┤
│  Untrusted Part  │   Enclave (Trusted)     │
│                  │   ┌──────────────────┐  │
│  ┌────────┐      │   │  Trusted Code    │  │
│  │Untrust │ ECALL│   │  Trusted Data    │  │
│  │ Code   │─────>│   │  (Encrypted)     │  │
│  │        │<─────│   │                  │  │
│  └────────┘ OCALL│   └──────────────────┘  │
├────────────────────────────────────────────┤
│              Operating System              │
├────────────────────────────────────────────┤
│         Hardware (CPU + Memory)            │
│   ┌────────────────────────────────────┐   │
│   │  EPC (Enclave Page Cache)          │   │
│   │  ┌──────────┬──────────┬────────┐  │   │
│   │  │ Enclave1 │ Enclave2 │ ...    │  │   │
│   │  └──────────┴──────────┴────────┘  │   │
│   └────────────────────────────────────┘   │
└────────────────────────────────────────────┘
```

**关键组件**:

- **Enclave**: 受保护的执行区域
- **EPC**: Enclave Page Cache，加密内存区域（默认128MB）
- **ECALL**: 从非信任区域进入Enclave
- **OCALL**: 从Enclave调用非信任代码

### 2.2 Enclave生命周期

**生命周期状态机**:

```text
[Initialized] --ECREATE--> [Created]
                              |
                           EADD (添加页面)
                              |
                          EINIT (初始化)
                              |
                           [Active]
                              |
                        EENTER (进入)
                              |
                          [Running]
                              |
                       EEXIT (退出)
                              |
                         [Active]
                              |
                      EREMOVE (销毁)
                              |
                        [Destroyed]
```

**Python模拟**:

```python
from enum import Enum
from dataclasses import dataclass
from typing import List, Optional

class EnclaveState(Enum):
    INITIALIZED = "initialized"
    CREATED = "created"
    ACTIVE = "active"
    RUNNING = "running"
    DESTROYED = "destroyed"

@dataclass
class Enclave:
    enclave_id: int
    state: EnclaveState
    epc_pages: List[int]  # EPC页面列表
    code_size: int  # KB
    data_size: int  # KB

    def create(self):
        """ECREATE: 创建Enclave"""
        if self.state == EnclaveState.INITIALIZED:
            self.state = EnclaveState.CREATED
            print(f"Enclave {self.enclave_id}: Created")

    def add_pages(self, pages: List[int]):
        """EADD: 添加页面到Enclave"""
        if self.state == EnclaveState.CREATED:
            self.epc_pages.extend(pages)
            print(f"Enclave {self.enclave_id}: Added {len(pages)} pages")

    def init(self):
        """EINIT: 初始化Enclave（度量并签名）"""
        if self.state == EnclaveState.CREATED:
            self.state = EnclaveState.ACTIVE
            print(f"Enclave {self.enclave_id}: Initialized and Active")

    def enter(self):
        """EENTER: 进入Enclave"""
        if self.state == EnclaveState.ACTIVE:
            self.state = EnclaveState.RUNNING
            print(f"Enclave {self.enclave_id}: Entered (Running)")

    def exit(self):
        """EEXIT: 退出Enclave"""
        if self.state == EnclaveState.RUNNING:
            self.state = EnclaveState.ACTIVE
            print(f"Enclave {self.enclave_id}: Exited (Active)")

    def destroy(self):
        """EREMOVE: 销毁Enclave"""
        if self.state in [EnclaveState.CREATED, EnclaveState.ACTIVE]:
            self.state = EnclaveState.DESTROYED
            self.epc_pages.clear()
            print(f"Enclave {self.enclave_id}: Destroyed")

# 使用示例
enclave = Enclave(
    enclave_id=1,
    state=EnclaveState.INITIALIZED,
    epc_pages=[],
    code_size=1024,  # 1MB
    data_size=512    # 512KB
)

# 生命周期演示
enclave.create()
enclave.add_pages([1, 2, 3, 4, 5])
enclave.init()
enclave.enter()
# ... 执行敏感计算 ...
enclave.exit()
enclave.destroy()
```

### 2.3 SGX调度开销

**性能开销来源**:

1. **Enclave进入/退出**: 2,000-8,000个CPU周期
2. **内存加密**: 5-15%性能损失
3. **EPC分页**: 如果EPC不足，换页开销巨大（>100x）

**定理27.12（SGX调度开销）**:

给定Enclave调用频率$f$和单次进入/退出开销$c$，总开销为:

$$
\text{Overhead}_{SGX} = f \cdot c + \alpha \cdot \text{EncryptionCost} + \beta \cdot \text{PagingCost}
$$

其中:

- $\alpha$: 内存加密开销系数（5-15%）
- $\beta$: EPC分页开销系数（0如果EPC足够，否则>100）

### 2.4 EPC（Enclave Page Cache）管理

**EPC管理策略**:

```python
class EPCManager:
    def __init__(self, total_epc_size_mb=128):
        self.total_size = total_epc_size_mb * 1024  # KB
        self.used_size = 0
        self.enclaves = {}

    def allocate(self, enclave_id: int, size_kb: int) -> bool:
        """为Enclave分配EPC"""
        if self.used_size + size_kb <= self.total_size:
            self.enclaves[enclave_id] = size_kb
            self.used_size += size_kb
            print(f"EPC allocated {size_kb}KB for Enclave {enclave_id}")
            return True
        else:
            print(f"EPC allocation failed: insufficient space")
            return False

    def deallocate(self, enclave_id: int):
        """释放Enclave的EPC"""
        if enclave_id in self.enclaves:
            size = self.enclaves[enclave_id]
            del self.enclaves[enclave_id]
            self.used_size -= size
            print(f"EPC deallocated {size}KB from Enclave {enclave_id}")

    def get_fragmentation(self) -> float:
        """计算EPC碎片率"""
        return (self.total_size - self.used_size) / self.total_size

# 使用示例
epc_manager = EPCManager(total_epc_size_mb=128)
epc_manager.allocate(enclave_id=1, size_kb=40960)  # 40MB
epc_manager.allocate(enclave_id=2, size_kb=20480)  # 20MB
print(f"EPC Fragmentation: {epc_manager.get_fragmentation():.2%}")
```

## 3 AMD SEV调度机制

### 3.1 SEV-SNP架构

**AMD SEV (Secure Encrypted Virtualization)** 提供虚拟机级TEE，支持VM内存加密。

**SEV架构**:

```text
┌────────────────────────────────────────────┐
│      Confidential VM 1    │    VM 2        │
│  ┌──────────────────┐     │                │
│  │ Guest OS         │     │                │
│  │ (Encrypted Mem)  │     │                │
│  └──────────────────┘     │                │
├────────────────────────────────────────────┤
│           Hypervisor (KVM/Xen)             │
├────────────────────────────────────────────┤
│       Hardware (AMD EPYC CPU)              │
│   ┌────────────────────────────────────┐   │
│   │ SEV Memory Encryption Engine       │   │
│   │  - VM Key (per-VM)                 │   │
│   │  - AES-128 Encryption              │   │
│   └────────────────────────────────────┘   │
└────────────────────────────────────────────┘
```

**SEV演进**:

- **SEV**: VM内存加密
- **SEV-ES**: + 寄存器加密
- **SEV-SNP** (Secure Nested Paging): + 完整性保护

### 3.2 虚拟机加密调度

**VM加密生命周期**:

```python
from enum import Enum
from dataclasses import dataclass

class SEVState(Enum):
    UNENCRYPTED = "unencrypted"
    LAUNCH_START = "launch_start"
    LAUNCH_UPDATE = "launch_update"
    LAUNCH_FINISH = "launch_finish"
    RUNNING = "running"
    SEND_START = "send_start"
    RECEIVE_START = "receive_start"

@dataclass
class SEV_VM:
    vm_id: int
    state: SEVState
    vm_key: bytes  # 128-bit AES key
    memory_size_gb: int

    def launch_start(self):
        """SEV_LAUNCH_START: 开始VM加密启动"""
        if self.state == SEVState.UNENCRYPTED:
            # 生成VM专属密钥
            import os
            self.vm_key = os.urandom(16)  # 128-bit key
            self.state = SEVState.LAUNCH_START
            print(f"SEV VM {self.vm_id}: Launch started")

    def launch_update(self, data: bytes):
        """SEV_LAUNCH_UPDATE: 更新VM内存（加密）"""
        if self.state == SEVState.LAUNCH_START:
            # 使用VM密钥加密内存
            encrypted_data = self._encrypt(data, self.vm_key)
            self.state = SEVState.LAUNCH_UPDATE
            print(f"SEV VM {self.vm_id}: Memory encrypted")
            return encrypted_data

    def launch_finish(self):
        """SEV_LAUNCH_FINISH: 完成VM启动"""
        if self.state == SEVState.LAUNCH_UPDATE:
            self.state = SEVState.RUNNING
            print(f"SEV VM {self.vm_id}: Running (encrypted)")

    def _encrypt(self, data: bytes, key: bytes) -> bytes:
        """AES-128加密（简化）"""
        from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
        from cryptography.hazmat.backends import default_backend

        cipher = Cipher(algorithms.AES(key), modes.ECB(), backend=default_backend())
        encryptor = cipher.encryptor()
        return encryptor.update(data) + encryptor.finalize()

# 使用示例
sev_vm = SEV_VM(
    vm_id=1,
    state=SEVState.UNENCRYPTED,
    vm_key=b'',
    memory_size_gb=8
)

sev_vm.launch_start()
encrypted_mem = sev_vm.launch_update(b'sensitive data' * 1000)
sev_vm.launch_finish()
```

### 3.3 SEV性能开销

**定理27.13（SEV性能开销）**:

SEV的性能开销主要来自内存加密：

$$
\text{Overhead}_{SEV} = \alpha \cdot \text{MemoryBandwidth} + \beta \cdot \text{CPUOverhead}
$$

其中:

- $\alpha \approx 5-10\%$: 内存带宽开销
- $\beta \approx 2-5\%$: CPU开销

**实测数据（AMD EPYC 7003）**:

- 内存密集型应用: 8-12%性能损失
- 计算密集型应用: 2-5%性能损失
- I/O密集型应用: <3%性能损失

## 4 Intel TDX调度架构

### 4.1 TDX与SGX的区别

**Intel TDX (Trust Domain Extensions)** 是VM级TEE，与SGX互补。

| 特性 | SGX | TDX |
|------|-----|-----|
| **隔离粒度** | 进程级（Enclave） | VM级（Trust Domain） |
| **内存限制** | 128MB (EPC) | 无限制（全部内存） |
| **OS支持** | 需要SDK修改 | 无需修改Guest OS |
| **应用场景** | 云函数、边缘计算 | 机密容器、Kubernetes |

### 4.2 Trust Domain调度

**TDX架构**:

```text
┌────────────────────────────────────────────┐
│   Trust Domain (TD)                        │
│  ┌──────────────────────────────────────┐  │
│  │   Guest OS (Unmodified Linux)        │  │
│  │   Application Containers             │  │
│  └──────────────────────────────────────┘  │
├────────────────────────────────────────────┤
│    TDX Module (CPU微代码)                  │
│   - Memory Encryption (MKTME)             │
│   - Integrity Protection                   │
│   - Remote Attestation                     │
├────────────────────────────────────────────┤
│           VMM (KVM/Qemu)                   │
└────────────────────────────────────────────┘
```

### 4.3 TDX性能优化

**优化策略**:

1. **减少TD退出**: 使用virtio-vsock而非传统I/O
2. **批量操作**: 聚合系统调用
3. **直接赋值**: 使用SR-IOV避免虚拟化开销

**性能对比（相对原生VM）**:

- 计算性能: 95-98%
- 内存带宽: 90-95%
- 网络吞吐: 85-92%
- 存储I/O: 88-93%

## 5 ARM TrustZone调度

### 5.1 TrustZone架构

**ARM TrustZone** 将CPU划分为两个世界：安全世界（Secure World）和正常世界（Normal World）。

```text
┌────────────────────────────────────────────┐
│         Normal World                       │
│  ┌──────────────────────────────────────┐  │
│  │  Rich OS (Android/Linux)             │  │
│  │  Applications                        │  │
│  └──────────────────────────────────────┘  │
├────────────────────────────────────────────┤
│         Secure Monitor                     │
│   SMC (Secure Monitor Call)                │
├────────────────────────────────────────────┤
│         Secure World                       │
│  ┌──────────────────────────────────────┐  │
│  │  Secure OS (OP-TEE)                  │  │
│  │  Trusted Applications (TAs)          │  │
│  └──────────────────────────────────────┘  │
└────────────────────────────────────────────┘
```

### 5.2 安全世界与正常世界切换

**切换机制**:

```c
// ARM TrustZone SMC调用（简化示例）
#include <stdint.h>

// SMC指令
#define SMC_CALL(func_id, arg1, arg2, arg3) \
    asm volatile( \
        "mov x0, %0\n" \
        "mov x1, %1\n" \
        "mov x2, %2\n" \
        "mov x3, %3\n" \
        "smc #0\n" \
        : \
        : "r"((uint64_t)(func_id)), \
          "r"((uint64_t)(arg1)), \
          "r"((uint64_t)(arg2)), \
          "r"((uint64_t)(arg3)) \
        : "x0", "x1", "x2", "x3" \
    )

// 调用安全服务
void call_secure_service(uint32_t service_id, uint64_t param) {
    SMC_CALL(service_id, param, 0, 0);
}

// 示例：请求安全密钥
void request_secure_key() {
    call_secure_service(0x1000, 0);  // 服务ID: 0x1000
}
```

## 6 TEE调度的形式化模型

### 6.1 安全调度模型

**定义27.14（TEE安全调度模型）**:

TEE调度系统$\mathcal{S} = (T, R, \phi, \psi)$，其中:

- $T = \{t_1, \ldots, t_n\}$: 任务集合
- $R = \{r_1, \ldots, r_m\}$: TEE资源集合（Enclave、TD、TrustZone）
- $\phi: T \to R$: 调度函数
- $\psi: T \times R \to \{0, 1\}$: 安全性谓词

**安全性保证**:

$$
\forall t \in T, \psi(t, \phi(t)) = 1 \implies \text{Secure}(t)
$$

### 6.2 性能-安全权衡

**多目标优化**:

$$
\min_{\phi} \left( w_1 \cdot \text{Latency}(\phi) - w_2 \cdot \text{Security}(\phi) \right)
$$

### 6.3 TLA+规约

```tla
----------------------- MODULE TEEScheduling -----------------------
EXTENDS Naturals, Sequences

CONSTANTS Tasks, TEEs, MaxLatency

VARIABLES scheduled, running, completed

vars == <<scheduled, running, completed>>

Init ==
    /\ scheduled = {}
    /\ running = [t \in Tasks |-> FALSE]
    /\ completed = {}

ScheduleTask(t, tee) ==
    /\ t \in Tasks
    /\ t \notin scheduled
    /\ tee \in TEEs
    /\ scheduled' = scheduled \cup {t}
    /\ running' = [running EXCEPT ![t] = TRUE]
    /\ UNCHANGED <<completed>>

CompleteTask(t) ==
    /\ t \in scheduled
    /\ running[t]
    /\ running' = [running EXCEPT ![t] = FALSE]
    /\ completed' = completed \cup {t}
    /\ UNCHANGED <<scheduled>>

Next ==
    \/ \E t \in Tasks, tee \in TEEs : ScheduleTask(t, tee)
    \/ \E t \in Tasks : CompleteTask(t)

Spec == Init /\ [][Next]_vars

TypeOK ==
    /\ scheduled \subseteq Tasks
    /\ completed \subseteq Tasks

====================================================================
```
