## 1 多核架构模型

### 1.1 SMP架构

**对称多处理（SMP）**：

```
     ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
     │ CPU0│ │ CPU1│ │ CPU2│ │ CPU3│
     └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘
        └───────┴───────┴───────┘
                │
          共享内存总线
                │
           ┌────┴────┐
           │  内存   │
           └─────────┘
```

**特点**：

- 所有CPU共享统一内存空间
- 访问延迟相同（UMA）
- 扩展性受限（<16核）

### 1.2 NUMA架构

**非统一内存访问（NUMA）**：

```
  Node 0              Node 1
┌─────────┐         ┌─────────┐
│ CPU0-3  │←──UPI──→│ CPU4-7  │
│ Memory0 │         │ Memory1 │
└────┬────┘         └────┬────┘
     │                   │
     └─────────┬─────────┘
               │
           互联网络
```

**访问延迟**：

| 访问类型 | 延迟 | 延迟比 |
|---------|------|--------|
| 本地访问 | 80ns | 1.0x |
| 跨Socket（1跳） | 150ns | 1.9x |
| 跨Socket（2跳） | 220ns | 2.8x |

### 1.3 混合架构

**异构多核（2025年）**：

| 核心类型 | 数量 | 频率 | 用途 |
|---------|------|------|------|
| **P-core** | 8 | 5.5GHz | 高性能计算 |
| **E-core** | 16 | 4.0GHz | 能效任务 |
| **NPU** | 1 | 2.0GHz | AI加速 |

## 2 NUMA调度

### 2.1 NUMA拓扑感知

**距离矩阵**：

```
        Node0   Node1   Node2   Node3
Node0     10      21      31      41
Node1     21      10      41      31
Node2     31      41      10      21
Node3     41      31      21      10
```

**拓扑发现**：

```c
// NUMA拓扑结构
typedef struct {
    int node_id;
    cpumask_t cpumask;          // 节点包含的CPU
    unsigned long total_memory; // 总内存
    unsigned long free_memory;  // 空闲内存
    int distance[MAX_NUMNODES]; // 到其他节点的距离
} numa_node_t;

// 计算NUMA距离
int node_distance(int from, int to) {
    return numa_nodes[from].distance[to];
}
```

### 2.2 本地性调度

**目标**：最大化本地内存访问比例

**策略**：

```c
// NUMA感知调度
int numa_schedule_task(struct task_struct *p) {
    int preferred_node = p->numa_preferred_node;
    int best_cpu = -1;
    int min_load = INT_MAX;

    // 优先选择本地节点
    for_each_cpu(cpu, &numa_nodes[preferred_node].cpumask) {
        int load = cpu_rq(cpu)->nr_running;
        if (load < min_load) {
            min_load = load;
            best_cpu = cpu;
        }
    }

    // 如果本地节点负载过高，考虑相邻节点
    if (min_load > LOAD_THRESHOLD) {
        best_cpu = find_best_cpu_nearby(preferred_node);
    }

    return best_cpu;
}
```

### 2.3 跨NUMA访问优化

**复制策略**：

```
只读数据：在多个NUMA节点复制
读写数据：仅在单一节点存储
```

**交错分配**：

```c
// 跨NUMA交错分配
void *numa_alloc_interleaved(size_t size) {
    void *ptr = mmap(NULL, size, PROT_READ | PROT_WRITE,
                     MAP_PRIVATE | MAP_ANONYMOUS, -1, 0);

    // 设置交错策略
    int mode = MPOL_INTERLEAVE;
    unsigned long nodemask = (1 << numa_num_nodes()) - 1;
    mbind(ptr, size, mode, &nodemask, numa_num_nodes(), 0);

    return ptr;
}
```

## 3 核心亲和性

### 3.1 亲和性类型

| 类型 | 描述 | 实现方式 |
|------|------|----------|
| **硬亲和性** | 强制绑定到指定核心 | cpuset, sched_setaffinity |
| **软亲和性** | 优先选择指定核心 | NUMA hint, 调度提示 |
| **自然亲和性** | 保持上次运行的核心 | 默认行为 |

### 3.2 硬亲和性实现

**API接口**：

```c
// 设置CPU亲和性
#define _GNU_SOURCE
#include <sched.h>

int sched_setaffinity(pid_t pid, size_t cpusetsize, const cpu_set_t *mask);
int sched_getaffinity(pid_t pid, size_t cpusetsize, cpu_set_t *mask);

// 使用示例
cpu_set_t mask;
CPU_ZERO(&mask);
CPU_SET(2, &mask);  // 绑定到CPU 2
CPU_SET(3, &mask);  // 绑定到CPU 3
sched_setaffinity(0, sizeof(mask), &mask);
```

**内核实现**：

```c
// Linux内核亲和性检查
static inline int cpuset_cpus_allowed(struct task_struct *p,
                                       struct cpumask *mask)
{
    rcu_read_lock();
    *mask = p->cpus_allowed;
    rcu_read_unlock();
    return 0;
}

// 调度时检查亲和性
bool cpu_allowed(struct task_struct *p, int cpu)
{
    return cpumask_test_cpu(cpu, &p->cpus_allowed);
}
```

### 3.3 软亲和性策略

**缓存亲和性**：

```
策略：尽量将任务调度到上次运行的CPU
原因：利用缓存热数据
实现：task->last_cpu
```

**内存亲和性**：

```c
// 自动NUMA平衡
void task_numa_fault(int node, struct page *page, int flags)
{
    struct task_struct *p = current;

    // 统计远程访问次数
    if (node != p->numa_preferred_node) {
        p->numa_faults[node]++;

        // 考虑迁移任务或页面
        if (should_migrate_task(p)) {
            migrate_task_to_node(p, node);
        }
    }
}
```

## 4 负载均衡

### 4.1 负载度量

**负载指标**：

| 指标 | 描述 | 计算方式 |
|------|------|----------|
| **运行队列长度** | 就绪任务数 | rq->nr_running |
| **CPU利用率** | 非空闲时间占比 | 1 - idle_time/total_time |
| **负载权重** | 加权任务数 | sum(task->se.load.weight) |
| **平均负载** | 指数衰减平均 | load_avg = load_avg * decay + load |

### 4.2 迁移决策

**不平衡阈值**：

```
imbalance = max(0, src_load - dst_load - threshold)

default_threshold = 25% * capacity
```

**迁移成本评估**：

```c
// 迁移成本计算
unsigned long migration_cost(struct task_struct *p, int src, int dst)
{
    unsigned long cost = 0;

    // 缓存热度损失
    cost += cache_warmth_cost(p);

    // 跨NUMA访问成本
    if (cpu_to_node(src) != cpu_to_node(dst)) {
        cost += NUMA_MIGRATION_COST;
    }

    // 上下文切换开销
    cost += CONTEXT_SWITCH_COST;

    return cost;
}
```

### 4.3 调度域层次

**调度域结构**：

```
MC (Multi-Core) 域：超线程核心
  ↓
DIE 域：同一Die的核心
  ↓
NUMA 域：同一NUMA节点的核心
  ↓
ALL 域：所有核心
```

**负载均衡频率**：

| 调度域 | 平衡间隔 | 说明 |
|--------|---------|------|
| MC | 2ms | 超线程间快速平衡 |
| DIE | 4ms | 核心间平衡 |
| NUMA | 100ms | 节点间谨慎平衡 |
| ALL | 1s | 全局平衡 |

## 5 线程迁移

### 5.1 迁移开销

**开销组成**：

| 操作 | 延迟 | 说明 |
|------|------|------|
| 保存寄存器 | 100-200ns | 16-32个寄存器 |
| TLB刷新 | 500ns-1μs | 无PCID时 |
| 缓存冷启动 | 2-10μs | 取决于工作集大小 |
| 跨NUMA迁移 | 10-50μs | 内存访问模式改变 |

**总开销**：

- 同核迁移：~200ns
- 同Socket迁移：~2μs
- 跨Socket迁移：~10μs

### 5.2 迁移策略

**主动迁移**：

```c
// 负载均衡触发的迁移
void load_balance(int this_cpu, struct rq *this_rq)
{
    struct sched_domain *sd;

    rcu_read_lock();
    for_each_domain(this_cpu, sd) {
        if (sd->flags & SD_LOAD_BALANCE) {
            // 查找最忙的运行队列
            struct rq *busiest = find_busiest_queue(sd, this_cpu);

            if (busiest && imbalance > threshold) {
                // 迁移任务
                detach_task(task, busiest);
                attach_task(task, this_rq);
            }
        }
    }
    rcu_read_unlock();
}
```

**被动迁移**：

```
场景：CPU离线、电源管理
策略：将所有任务迁移到其他CPU
```

### 5.3 热插拔支持

**CPU热插拔流程**：

```
1. 通知调度器：CPU即将离线
2. 停止新任务分配
3. 迁移现有任务
4. 等待CPU空闲
5. 下线CPU
```

## 6 形式化定义

### 6.1 NUMA调度形式化

**定义4.1（NUMA调度问题）**：

给定：

- 任务集合$T = \{t_1, t_2, \ldots, t_n\}$
- NUMA节点集合$N = \{n_1, n_2, \ldots, n_m\}$
- CPU集合$C = \{c_1, c_2, \ldots, c_k\}$
- 内存访问模式$A: T \times N \rightarrow \mathbb{R}^+$

寻找映射$S: T \rightarrow C$，使得：

$$
\min_{S} \sum_{t \in T} \sum_{n \in N} A(t, n) \cdot d(n, \text{node}(S(t)))
$$

其中$d(n_1, n_2)$是节点间距离。

### 6.2 亲和性约束

**定义4.2（亲和性约束）**：

任务$t$的亲和性约束是一个CPU掩码$M_t \subseteq C$，调度必须满足：

$$
S(t) \in M_t
$$

**硬约束**：必须满足，否则任务不能运行

**软约束**：优先满足，但可违反（带惩罚）

## 7 Linux内核实现分析

### 7.1 调度器架构

**CFS调度器核心**：

```c
// 完全公平调度器
struct cfs_rq {
    struct load_weight load;
    unsigned long nr_running;

    struct rb_root_cached tasks_timeline;  // 红黑树
    struct sched_entity *curr;              // 当前任务
    struct sched_entity *next;              // 下一个任务
};

// 任务选择
static struct sched_entity *pick_next_entity(struct cfs_rq *cfs_rq)
{
    // 选择vruntime最小的任务
    struct sched_entity *se = __pick_first_entity(cfs_rq);

    // 检查唤醒抢占
    if (cfs_rq->next && wakeup_preempt_entity(cfs_rq->next, se) < 0)
        return cfs_rq->next;

    return se;
}
```

### 7.2 NUMA Balancing

**自动NUMA平衡**：

```c
// NUMA缺页处理
void task_numa_work(struct callback_head *work)
{
    struct task_struct *p = current;
    struct mm_struct *mm = p->mm;

    // 采样内存访问
    walk_page_range(mm, start, end, &numa_walk_ops, &data);

    // 分析访问模式
    if (data.local_ratio < NUMA_BALANCING_THRESHOLD) {
        // 本地访问率低，考虑迁移
        if (should_migrate_memory(p, data.target_node)) {
            migrate_misplaced_page(p, data.target_node);
        }
    }
}
```

**迁移决策**：

```
条件：
1. 远程访问率 > 50%
2. 任务运行时间 > 5秒
3. 目标节点有足够内存
4. 迁移收益 > 迁移成本
```
