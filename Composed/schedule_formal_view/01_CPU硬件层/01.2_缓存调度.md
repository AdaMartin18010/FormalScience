## 1 缓存层次调度模型

### 1.1 多级缓存架构

**缓存层次参数**（2025年典型值）：

| 层级 | 容量 | 延迟 | 带宽 | 关联度 | 位置 |
|------|------|------|------|--------|------|
| **L1I** | 32-64KB | 4周期 | 2TB/s | 8路 | 核心内 |
| **L1D** | 32-64KB | 4周期 | 2TB/s | 8路 | 核心内 |
| **L2** | 256KB-1MB | 12周期 | 500GB/s | 16路 | 核心内 |
| **L3** | 8-64MB | 40周期 | 200GB/s | 16路 | 共享 |
| **L4/eDRAM** | 64-256MB | 100周期 | 100GB/s | 8路 | 片外/封装 |

**缓存调度目标**：

$$
\min_{S} \left( \alpha \cdot \text{miss\_rate} + \beta \cdot \text{coherence\_traffic} + \gamma \cdot \text{energy} \right)
$$

### 1.2 缓存访问调度

**访问流程**：

```
CPU请求 → L1检查 → L2检查 → L3检查 → 内存访问
            ↓命中    ↓命中    ↓命中
          返回    返回    返回
```

**并行访问优化**：

```c
// 并行缓存访问调度
typedef struct {
    int level;
    uint64_t tag;
    int set_index;
    bool hit;
    uint8_t *data;
} cache_access_t;

// 并行查询L1和L2
cache_access_t access_cache_parallel(uint64_t addr) {
    cache_access_t result;

    // 并行检查各级缓存
    #pragma omp parallel sections
    {
        #pragma omp section
        result.l1_hit = check_l1_cache(addr);

        #pragma omp section
        result.l2_hit = check_l2_cache(addr);
    }

    return result;
}
```

## 2 缓存一致性协议调度

### 2.1 MESI协议调度

**状态转换**：

```
        +--------+         +--------+
   +--->|   E    |<------->|   S    |
   |    |(独占) |         |(共享)  |
   |    +--------+         +--------+
   |         ↑                ↑
   |         |                |
   |    +--------+         +--------+
   +----|   M    |         |   I    |
        |(修改)  |<------->|(无效)  |
        +--------+         +--------+
```

**状态转换表**：

| 当前状态 | 本地读 | 本地写 | 远程读 | 远程写 |
|---------|--------|--------|--------|--------|
| **I** | E/S | M | I | I |
| **E** | E | M | S | I |
| **S** | S | M | S | I |
| **M** | M | M | S+I→WB | I+I→WB |

**形式化定义**：

$$
\delta: S \times E \rightarrow S'
$$

其中$S = \{M, E, S, I\}$，$E = \{\text{PrRd}, \text{PrWr}, \text{BusRd}, \text{BusRdX}\}$

```c
// MESI状态机实现
typedef enum { M, E, S, I } mesi_state_t;
typedef enum { PR_RD, PR_WR, BUS_RD, BUS_RDX } mesi_event_t;

mesi_state_t mesi_transition(mesi_state_t state, mesi_event_t event) {
    switch (state) {
        case I:
            switch (event) {
                case PR_RD: return E;  // 从内存读取，独占
                case PR_WR: return M;  // 写入，修改
                default: return I;
            }
        case E:
            switch (event) {
                case PR_RD: return E;
                case PR_WR: return M;
                case BUS_RD: return S;  // 其他核心读取，变为共享
                case BUS_RDX: return I; // 其他核心写入，失效
            }
        case S:
            switch (event) {
                case PR_RD: return S;
                case PR_WR: return M;   // 需要使其他副本失效
                case BUS_RD: return S;
                case BUS_RDX: return I;
            }
        case M:
            switch (event) {
                case PR_RD: return M;
                case PR_WR: return M;
                case BUS_RD: /* WB */ return S;
                case BUS_RDX: /* WB */ return I;
            }
    }
    return I;
}
```

### 2.2 MOESI协议调度

**新增O状态（Owned）**：允许共享脏数据，减少写回操作

**状态转换**：

| 当前状态 | 本地读 | 本地写 | 远程读 | 远程写 |
|---------|--------|--------|--------|--------|
| **O** | O | M | O | I |

**优势**：

- 减少写回次数：~30%
- 降低内存带宽消耗
- 提高多核共享数据性能

### 2.3 MESIF协议调度

**新增F状态（Forward）**：指定唯一转发者，减少总线竞争

**调度策略**：

```
当多个S状态副本收到BusRd请求时：
- MESI：所有S状态同时响应，总线冲突
- MESIF：仅F状态响应，其他保持静默
```

**Intel实现**：使用LRU选择F状态持有者

### 2.4 目录协议调度

**适用场景**：大规模NUMA系统（>64核）

**目录结构**：

```c
typedef struct {
    uint64_t tag;
    mesi_state_t state;
    uint64_t sharers_bitmap;  // 哪些核心拥有副本
    bool dirty;               // 是否有修改
} directory_entry_t;

#define DIR_ENTRIES 65536
```

**消息类型**：

| 消息类型 | 描述 | 延迟 |
|---------|------|------|
| **GetS** | 获取共享副本 | ~20ns |
| **GetM** | 获取独占副本 | ~25ns |
| **PutS** | 释放共享副本 | ~10ns |
| **PutM** | 写回修改数据 | ~30ns |

## 3 缓存预取策略

### 3.1 顺序预取

**原理**：检测到顺序访问模式时，预取后续缓存行

```
访问地址: A, A+64, A+128, A+192...
预取地址: A+256, A+320...
```

**实现**：

```c
// 顺序预取器
typedef struct {
    uint64_t last_addr;
    int stride;
    int confidence;  // 置信度计数器
} stream_prefetcher_t;

void stream_prefetch(stream_prefetcher_t *pf, uint64_t addr) {
    if (pf->last_addr + 64 == addr) {
        pf->confidence++;
        if (pf->confidence >= CONFIDENCE_THRESHOLD) {
            // 触发预取
            prefetch_to_l1(addr + 64 * pf->prefetch_distance);
        }
    } else {
        pf->confidence = 0;
    }
    pf->last_addr = addr;
}
```

### 3.2 跨步预取

**原理**：检测固定步长的访问模式

```
访问地址: A, A+256, A+512, A+768...
步长: 256字节（64个元素数组访问）
```

**适用场景**：数组遍历、矩阵运算

### 3.3 关联预取

**原理**：利用访问历史，预测下一步可能访问的地址

```
历史: A → B → C
预测: 访问A后，预取B和C
```

**实现方式**：

- **全局历史缓冲区（GHB）**：记录最近访问序列
- **Delta关联**：记录地址差值序列

### 3.4 机器学习预取

**2025年技术**：使用轻量级神经网络预测访问模式

```python
# 简化的ML预取器概念
class ML_Prefetcher:
    def __init__(self):
        self.model = LightweightNN(input_size=10, hidden_size=64)

    def predict(self, access_history):
        # 输入：最近10个访问地址
        # 输出：预测的下一个地址
        prediction = self.model.predict(access_history)
        return prediction

    def train(self, access_trace):
        # 在线学习更新模型
        self.model.update(access_trace)
```

**性能对比**（2025年）：

| 预取器类型 | 覆盖率 | 准确率 | 性能提升 |
|-----------|--------|--------|----------|
| 顺序预取 | 45% | 85% | +8% |
| 跨步预取 | 30% | 90% | +5% |

## 4 缓存替换算法

### 4.1 LRU算法

**Least Recently Used**：替换最久未访问的缓存行

**实现**：

```c
// 真LRU实现（成本高）
typedef struct {
    cache_line_t lines[SET_SIZE][WAY_COUNT];
    uint64_t access_counter;
    uint64_t lru_bits[WAY_COUNT];
} cache_set_t;

int lru_select_victim(cache_set_t *set) {
    int victim = 0;
    uint64_t min_lru = set->lru_bits[0];

    for (int i = 1; i < WAY_COUNT; i++) {
        if (set->lru_bits[i] < min_lru) {
            min_lru = set->lru_bits[i];
            victim = i;
        }
    }
    return victim;
}
```

**复杂度**：O(Way)，硬件成本高

### 4.2 伪LRU算法

**树形PLRU**：使用二叉树近似LRU

```
        [0]
       /   \
     [0]   [1]
     / \   / \
    A  B  C  D
```

**实现**：

```c
// PLRU实现
int plru_select_victim(uint8_t *tree, int ways) {
    int victim = 0;
    for (int i = 0; i < log2(ways); i++) {
        int bit = tree[victim];
        victim = 2 * victim + 1 + bit;
    }
    return victim - (ways - 1);
}
```

**优势**：O(log Way)，硬件实现简单

### 4.3 自适应替换算法

**ARC（Adaptive Replacement Cache）**：平衡最近使用和频繁使用

```
ARC = LRU_list + LFU_list
```

**Dueling Clock**：多个候选算法竞争，动态选择最优

## 5 形式化定义

### 5.1 缓存一致性形式化

**定义2.1（缓存一致性）**：

对于多处理器系统，缓存一致性要求满足以下条件：

1. **写串行化**：对同一地址的写操作，所有处理器看到的顺序一致
2. **写传播**：一个处理器对地址A的写操作，最终对所有处理器可见
3. **事务串行化**：每个处理器看到的内存操作序列符合程序序

**形式化表示**：

$$
\forall p_i, p_j, \forall a: \text{Write}_i(a, v) \rightarrow \lozenge \text{Read}_j(a) = v
$$

### 5.2 预取效果模型

**预取准确率定义**：

$$
\text{Accuracy} = \frac{\text{Used Prefetches}}{\text{Total Prefetches}}
$$

**覆盖率定义**：

$$
\text{Coverage} = \frac{\text{Prefetched Misses}}{\text{Total Misses}}
$$

**预取性能增益**：

$$
\text{Speedup} = \frac{1}{(1 - \text{Coverage}) + \frac{\text{Coverage}}{\text{Prefetch Accuracy}} \cdot \frac{\text{Prefetch Latency}}{\text{Miss Latency}}}
$$
