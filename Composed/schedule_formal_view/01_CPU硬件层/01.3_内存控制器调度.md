## 1 内存控制器架构

### 1.1 IMC集成架构

**现代IMC架构**（2025年）：

```
CPU核心 → L3缓存 → IMC → 内存通道 → DIMM
                     ↓
              调度队列 → 命令调度器 → 时序控制器
```

**IMC功能模块**：

| 模块 | 功能 | 延迟 |
|------|------|------|
| **请求队列** | 缓冲内存请求 | ~5ns |
| **命令调度器** | FR-FCFS调度 | ~10ns |
| **时序控制器** | 管理DRAM时序 | 可变 |
| **刷新控制器** | 定期刷新管理 | 64ms周期 |

### 1.2 通道组织

**多通道配置**：

| 配置 | 通道数 | 带宽 | 延迟 | 典型应用 |
|------|--------|------|------|----------|
| **双通道** | 2 | 50GB/s | 80ns | 消费级 |
| **四通道** | 4 | 100GB/s | 80ns | 工作站 |
| **八通道** | 8 | 200GB/s | 80ns | 服务器 |

## 2 内存交错调度

### 2.1 交错模式

**概念**：将连续地址分布在多个通道/Bank上，提高并行度

**交错粒度**：

```
地址A → 通道 (A / granularity) % num_channels
```

| 粒度 | 描述 | 适用场景 |
|------|------|----------|
| **64B** | 缓存行级交错 | 随机访问 |
| **4KB** | 页级交错 | 顺序访问 |
| **1MB** | 大块交错 | 大内存分配 |

### 2.2 地址映射

**典型地址映射**（双通道）：

```
物理地址：[Tag | Channel | Bank | Row | Column]

位分配：
- Column: 0-5 (64字节缓存行)
- Channel: 6 (2通道)
- Bank: 7-10 (16 Bank)
- Row: 11+ (行地址)
- Tag: 高地址位
```

**优化映射**：

```c
// 地址解码函数
void decode_address(uint64_t phys_addr,
                    int *channel, int *bank, int *row, int *col) {
    *col = phys_addr & 0x3F;           // 低6位
    *channel = (phys_addr >> 6) & 0x1; // 第6位
    *bank = (phys_addr >> 7) & 0xF;    // 7-10位
    *row = (phys_addr >> 11);          // 11位以上
}
```

### 2.3 交错粒度优化

**动态粒度选择**：

```
顺序访问模式 → 大粒度交错（4KB）
随机访问模式 → 小粒度交错（64B）
混合访问模式 → 中等粒度（1KB）
```

## 3 Bank调度

### 3.1 Bank结构

**DDR5 Bank组织**（2025年）：

```
DIMM
├── Rank 0
│   ├── Bank Group 0
│   │   ├── Bank 0, 1, 2, 3
│   ├── Bank Group 1
│   │   ├── Bank 4, 5, 6, 7
│   ├── Bank Group 2
│   │   ├── Bank 8, 9, 10, 11
│   └── Bank Group 3
│       ├── Bank 12, 13, 14, 15
└── Rank 1 (类似)
```

**Bank状态机**：

```
Idle → Activate → Active → Read/Write → Precharge → Idle
```

### 3.2 Bank冲突处理

**Bank冲突**：两个请求访问同一Bank的不同Row

**解决方法**：

1. **预充电调度**：尽早预充电，为后续访问准备
2. **Bank并行**：同时激活不同Bank
3. **重排序**：调整请求顺序，减少冲突

```c
// Bank冲突检测
typedef struct {
    int bank_id;
    int row_id;
    bool active;
    uint64_t active_row;
} bank_state_t;

bool check_bank_conflict(bank_state_t *bank, uint64_t row) {
    if (bank->active && bank->active_row != row) {
        return true;  // 需要预充电和重新激活
    }
    return false;
}
```

### 3.3 Bank并行调度

**目标**：最大化Bank级并行，同时访问多个Bank

**调度策略**：

```c
// Bank并行调度器
typedef struct {
    request_queue_t queues[NUM_BANKS];  // 每个Bank的请求队列
    bank_state_t banks[NUM_BANKS];
} bank_scheduler_t;

void schedule_bank_requests(bank_scheduler_t *sched) {
    for (int i = 0; i < NUM_BANKS; i++) {
        if (!sched->banks[i].active ||
            sched->queues[i].head->row == sched->banks[i].active_row) {
            // Bank空闲或行命中，可以服务请求
            issue_request(&sched->queues[i]);
        }
    }
}
```

## 4 命令调度算法

### 4.1 FR-FCFS算法

**First-Ready First-Come-First-Served**：优先调度就绪的命令

**就绪条件**：

```
1. 行缓冲命中（Row Buffer Hit）
2. 满足时序约束
3. 无更高优先级命令
```

**算法实现**：

```c
// FR-FCFS调度器
typedef struct {
    dram_command_t commands[MAX_QUEUE_DEPTH];
    int head, tail;
    int count;
} command_queue_t;

dram_command_t *fr_fcfs_schedule(command_queue_t *cq) {
    dram_command_t *best_cmd = NULL;
    int best_priority = INT_MIN;

    for (int i = cq->head; i != cq->tail; i = (i + 1) % MAX_QUEUE_DEPTH) {
        dram_command_t *cmd = &cq->commands[i];

        if (!is_ready(cmd)) continue;

        int priority = calculate_priority(cmd);

        if (priority > best_priority) {
            best_priority = priority;
            best_cmd = cmd;
        }
    }

    return best_cmd;
}

int calculate_priority(dram_command_t *cmd) {
    int priority = 0;

    // 行缓冲命中优先
    if (is_row_buffer_hit(cmd)) {
        priority += 1000;
    }

    // 等待时间
    priority += cmd->wait_time;

    // 请求类型优先级
    if (cmd->type == READ) {
        priority += 10;  // 读请求优先
    }

    return priority;
}
```

### 4.2 调度约束

**DRAM时序约束**：

| 参数 | 描述 | DDR5-5600典型值 |
|------|------|----------------|
| **tCL** | CAS延迟 | 48周期（8.6ns） |
| **tRCD** | RAS到CAS延迟 | 48周期 |
| **tRP** | 行预充电时间 | 48周期 |
| **tRAS** | 行激活时间 | 100周期 |
| **tRRD** | 行到行延迟 | 8周期 |
| **tFAW** | 四激活窗口 | 40周期 |

**调度约束检查**：

```c
bool check_timing_constraints(dram_command_t *cmd) {
    switch (cmd->type) {
        case READ:
            return (current_cycle - cmd->activate_time) >= tCL;
        case WRITE:
            return (current_cycle - cmd->activate_time) >= tCL;
        case ACTIVATE:
            return (current_cycle - cmd->precharge_time) >= tRP &&
                   (current_cycle - last_activate) >= tRRD;
        case PRECHARGE:
            return (current_cycle - cmd->last_access) >= tRAS;
    }
    return false;
}
```

### 4.3 优先级调度

**QoS支持**：为不同类型请求设置优先级

```c
typedef enum {
    PRIORITY_REALTIME = 0,   // 实时：最高优先级
    PRIORITY_HIGH = 1,       // 高优先级
    PRIORITY_NORMAL = 2,     // 普通
    PRIORITY_LOW = 3         // 低优先级
} request_priority_t;

// 优先级调度
dram_command_t *priority_schedule(command_queue_t *cq) {
    for (int p = PRIORITY_REALTIME; p <= PRIORITY_LOW; p++) {
        dram_command_t *cmd = find_first_ready(cq, p);
        if (cmd != NULL) return cmd;
    }
    return NULL;
}
```

## 5 刷新调度

### 5.1 刷新机制

**DRAM刷新要求**：

- **刷新周期**：64ms内所有行必须刷新一次
- **刷新间隔**：$t_{REFI}$ = 64ms / 行数（DDR5: ~7.8μs）
- **刷新命令**：REFRESH，刷新多行（通常8行）

**刷新开销**：

| DDR类型 | 刷新命令数 | 刷新占用时间 | 性能损失 |
|---------|-----------|-------------|----------|
| DDR4 | 8192 | ~3% | 2-5% |
| DDR5 | 8192 | ~2% | 1-3% |

### 5.2 刷新调度策略

**集中刷新 vs 分布式刷新**：

```
集中刷新: [活跃] [刷新x8] [活跃] [刷新x8] ...
分布式刷新: [活跃] [刷新] [活跃] [刷新] [活跃] ...
```

**自适应刷新调度**：

```c
// 自适应刷新调度器
typedef struct {
    uint64_t last_refresh;
    int refresh_counter;
    bool urgent_mode;
} refresh_scheduler_t;

void schedule_refresh(refresh_scheduler_t *rs) {
    if (rs->refresh_counter >= REFRESH_THRESHOLD) {
        // 必须进行刷新
        issue_refresh();
        rs->refresh_counter = 0;
    } else if (command_queue_empty()) {
        // 空闲时提前刷新
        issue_refresh();
    }
}
```

### 5.3 温度感知刷新

**温度与刷新率**：

| 温度范围 | 刷新率 | 说明 |
|---------|--------|------|
| <45°C | 1x（标准） | 正常刷新 |
| 45-85°C | 2x | 双倍刷新 |
| >85°C | 4x | 四倍刷新 |

## 6 形式化定义

### 6.1 调度问题形式化

**定义3.1（内存调度问题）**：

给定：

- 请求集合$R = \{r_1, r_2, \ldots, r_n\}$
- Bank集合$B = \{b_1, b_2, \ldots, b_m\}$
- 时序约束集合$C$

寻找调度$S: R \rightarrow \mathbb{N} \times B$，使得：

1. **时序约束**：$\forall r_i \in R$，$S(r_i)$满足$C$中的所有时序要求
2. **Bank约束**：同一Bank同时只能服务一个请求
3. **目标**：最小化$\sum_{r_i \in R} (S(r_i) - arrival(r_i))$

### 6.2 优化目标

**多目标优化**：

$$
\min_{S} \left( \alpha \cdot \text{avg\_latency}(S) + \beta \cdot \text{max\_latency}(S) + \gamma \cdot \text{fairness}(S) \right)
$$

**Bank利用率**：

$$
\text{Bank Utilization} = \frac{\sum_{b \in B} \text{active\_time}(b)}{|B| \cdot \text{total\_time}}
$$
