### 1.2 调度算法详解

**预选阶段（Predicates）**：

```python
# 伪代码：预选阶段
def predicate_filter(pod, nodes):
    feasible_nodes = []

    for node in nodes:
        # 1. 资源检查
        if not check_resource_fit(pod, node):
            continue

        # 2. 节点亲和性
        if not check_node_affinity(pod, node):
            continue

        # 3. Pod亲和性/反亲和性
        if not check_pod_affinity(pod, node):
            continue

        # 4. 污点容忍
        if not check_tolerations(pod, node):
            continue

        # 5. 卷绑定检查
        if not check_volume_binding(pod, node):
            continue

        feasible_nodes.append(node)

    return feasible_nodes
```

**优选阶段（Priorities）**：

```python
# 伪代码：优选阶段
def priority_score(pod, feasible_nodes):
    scores = {}

    for node in feasible_nodes:
        score = 0

        # 1. 资源匹配度 (权重: 1)
        score += 1 * score_resource_allocation(pod, node)

        # 2. 节点亲和性 (权重: 1)
        score += 1 * score_node_affinity(pod, node)

        # 3. Pod亲和性 (权重: 1)
        score += 1 * score_pod_affinity(pod, node)

        # 4. 负载均衡 (权重: 1)
        score += 1 * score_balanced_resource_allocation(pod, node)

        # 5. 镜像本地性 (权重: 1)
        score += 1 * score_image_locality(pod, node)

        scores[node] = score

    return max(scores, key=scores.get)
```

### 1.3 调度性能优化案例

**案例：大规模集群调度优化**

**背景**：某互联网公司Kubernetes集群规模达10万节点，调度延迟高达500ms。

**优化措施**：

| **优化项** | **实现** | **效果** |
|-----------|---------|---------|
| **批处理调度** | 一次处理100个Pod | 延迟-60% |
| **节点信息缓存** | 本地缓存节点状态 | 延迟-40% |
| **并行预选** | 多线程预选检查 | 延迟-30% |
| **Snapshot共享** | 共享集群状态快照 | 延迟-50% |

**优化结果**：

- P50调度延迟：500ms → 25ms
- P99调度延迟：1200ms → 80ms
- 调度吞吐量：200 Pod/s → 3000 Pod/s

### 1.4 自定义调度器实践

**场景：GPU调度**

```yaml
# GPU调度插件配置
apiVersion: kubescheduler.config.k8s.io/v1
kind: KubeSchedulerConfiguration
profiles:
  - schedulerName: gpu-scheduler
    plugins:
      filter:
        enabled:
          - name: NodeResourcesFit
          - name: GpuTopologyMatch  # 自定义插件
      score:
        enabled:
          - name: GpuTopologyMatch
            weight: 100
```

**自定义调度算法**：

```python
# GPU拓扑感知调度
class GpuTopologyScheduler:
    def filter(self, pod, node):
        """检查GPU拓扑匹配"""
        required_gpus = pod.spec.containers[0].resources.limits['nvidia.com/gpu']
        available_gpus = node.status.allocatable['nvidia.com/gpu']

        if available_gpus < required_gpus:
            return False

        # 检查NVLink拓扑
        if pod.annotations.get('gpu-topology') == 'nvlink':
            return self._check_nvlink_topology(node, required_gpus)

        return True

    def score(self, pod, node):
        """GPU拓扑打分"""
        # NVLink连接的GPU得分更高
        gpu_score = self._calculate_gpu_locality(pod, node)
        return gpu_score * 100
```

### 4.2 边缘调度策略

**场景：智能工厂边缘调度**

| **应用类型** | **调度策略** | **延迟要求** | **部署位置** |
|-------------|-------------|-------------|-------------|
| 质量检测 | 本地优先 | <10ms | 产线边缘节点 |
| 预测维护 | 区域优先 | <50ms | 工厂边缘云 |
| 库存管理 | 云端优先 | <200ms | 公有云 |

**调度决策公式**：

```
Score = α × (1/latency) + β × (resource_fit) + γ × (data_locality)

其中：
- latency: 网络延迟
- resource_fit: 资源匹配度
- data_locality: 数据本地性
- α, β, γ: 权重系数
```
