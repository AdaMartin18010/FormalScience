## 1 概述

### 1.1 核心洞察

使用Petri网建模调度系统，可以**形式化表示调度策略、资源分配和任务执行**。Petri网的自然并发语义使其成为调度系统建模的理想工具，能够清晰地表示任务依赖、资源竞争和调度决策。

### 1.2 调度系统建模优势

**Petri网在调度系统中的优势**：

1. **并发建模**：自然表示任务的并发执行
2. **资源竞争**：清晰表示资源竞争和分配
3. **依赖关系**：直观表示任务间的依赖关系
4. **形式化验证**：可以形式化验证调度策略的正确性

### 1.3 设计原理与动机

**为什么使用Petri网建模调度系统？**

1. **形式化基础**：提供严格的数学基础
2. **可视化**：直观的图形表示
3. **分析工具**：丰富的分析工具（可达性、死锁检测等）
4. **验证能力**：可以验证调度策略的正确性

## 2 调度Petri网定义

### 2.1 调度Petri网

**定义 2.1**（调度Petri网）：

```text
调度Petri网SPN = (P, T, F, W, M₀, R, σ, τ)

其中：
  P: 库所集合
    - P_task: 任务状态库所（待执行、执行中、已完成）
    - P_resource: 资源状态库所（可用、已分配）
    - P_queue: 队列库所（任务队列）

  T: 变迁集合
    - T_schedule: 调度变迁（任务调度）
    - T_execute: 执行变迁（任务执行）
    - T_complete: 完成变迁（任务完成）
    - T_allocate: 分配变迁（资源分配）
    - T_release: 释放变迁（资源释放）

  F: 流关系（库所与变迁之间的连接）
  W: 权重函数（弧的权重）
  M₀: 初始标记（初始任务和资源状态）
  R: 资源集合（CPU、内存、网络等）
  σ: 调度函数（决定哪个任务被调度）
  τ: 时间函数（任务执行时间）
```

### 2.2 核心概念精确定义

#### 2.2.1 任务状态

**定义 2.2**（任务状态）：

```text
任务状态通过库所表示：
  - p_task_pending: 待执行任务
  - p_task_running: 执行中任务
  - p_task_completed: 已完成任务

状态转换：
  p_task_pending →[t_schedule]→ p_task_running →[t_complete]→ p_task_completed
```

#### 2.2.2 资源状态

**定义 2.3**（资源状态）：

```text
资源状态通过库所表示：
  - p_resource_available: 可用资源
  - p_resource_allocated: 已分配资源

状态转换：
  p_resource_available + p_task →[t_allocate]→ p_resource_allocated
  p_resource_allocated →[t_release]→ p_resource_available
```

#### 2.2.3 调度函数

**定义 2.4**（调度函数）：

```text
调度函数σ: M → T决定哪个变迁被选择执行：

  σ(M) = argmax_{t∈T_enabled} priority(t, M)

其中：
  - T_enabled是使能的变迁集合
  - priority(t, M)是变迁t在标记M下的优先级
```

## 3 调度策略Petri网表示

### 3.1 FIFO调度

**定义 3.1**（FIFO调度Petri网）：

```text
FIFO调度Petri网：
  队列库所：p_queue（任务队列，FIFO顺序）
  调度变迁：t_schedule（从队列头部取出任务）
  执行库所：p_executing（执行中任务）

结构：
  p_queue →[t_schedule]→ p_executing →[t_complete]→ p_completed

调度规则：
  - 任务按到达顺序进入队列
  - 调度变迁总是选择队列头部的任务
  - 使用队列数据结构维护FIFO顺序
```

**FIFO调度性质**：

```text
1. 公平性：
   - 所有任务最终都会被调度
   - 先到达的任务先执行

2. 无饥饿：
   - 不存在任务永远等待的情况

3. 简单性：
   - 实现简单，开销小
```

### 3.2 优先级调度

**定义 3.2**（优先级调度Petri网）：

```text
优先级调度Petri网：
  使用守卫函数控制变迁使能：
    t_high_priority: 高优先级任务
    t_medium_priority: 中优先级任务
    t_low_priority: 低优先级任务

结构：
  p_task_high →[t_high_priority]→ p_executing
  p_task_medium →[t_medium_priority]→ p_executing
  p_task_low →[t_low_priority]→ p_executing

守卫函数：
  G(t_high_priority) = priority(t) > HIGH_THRESHOLD
  G(t_medium_priority) = MEDIUM_THRESHOLD < priority(t) ≤ HIGH_THRESHOLD
  G(t_low_priority) = priority(t) ≤ MEDIUM_THRESHOLD

调度规则：
  - 高优先级任务优先调度
  - 同优先级任务按FIFO调度
```

**优先级调度性质**：

```text
1. 优先级保证：
   - 高优先级任务总是优先执行
   - 低优先级任务可能饥饿

2. 响应性：
   - 高优先级任务响应快
   - 低优先级任务响应慢

3. 公平性：
   - 同优先级任务公平调度
   - 不同优先级任务不公平
```

### 3.3 时间片轮转调度

**定义 3.3**（时间片轮转调度Petri网）：

```text
时间片轮转调度Petri网：
  时间片库所：p_timeslice（时间片剩余）
  执行库所：p_executing（执行中任务）
  就绪库所：p_ready（就绪任务）

结构：
  p_ready + p_timeslice →[t_schedule]→ p_executing
  p_executing →[t_timeslice_expire]→ p_ready（时间片用完）
  p_executing →[t_complete]→ p_completed（任务完成）

时间约束：
  - 时间片大小：TIME_SLICE
  - 时间片到期：t_timeslice_expire在TIME_SLICE后自动触发
```

### 3.4 最短作业优先调度

**定义 3.4**（最短作业优先调度Petri网）：

```text
最短作业优先调度Petri网：
  使用估计执行时间作为优先级：
    priority(t) = -estimated_time(t)

结构：
  p_task →[t_schedule]→ p_executing

调度函数：
  σ(M) = argmin_{t∈T_enabled} estimated_time(t)

守卫函数：
  G(t) = estimated_time(t) = min{estimated_time(t') | t' ∈ T_enabled}
```

## 4 资源调度Petri网

### 4.1 资源分配

**定义 4.1**（资源分配Petri网）：

```text
资源分配Petri网：
  资源库所：p_resource（可用资源，标记数表示资源数量）
  任务库所：p_task（待调度任务）
  分配变迁：t_allocate（资源分配）

结构：
  p_resource + p_task →[t_allocate]→ p_allocated

权重：
  W(p_resource, t_allocate) = resource_required（所需资源数）
  W(p_task, t_allocate) = 1
  W(t_allocate, p_allocated) = 1

使能条件：
  M(p_resource) ≥ resource_required ∧ M(p_task) ≥ 1
```

**资源分配策略**：

```text
1. 预分配策略：
   - 任务开始前分配所有资源
   - 避免资源竞争
   - 可能造成资源浪费

2. 按需分配策略：
   - 任务需要时分配资源
   - 资源利用率高
   - 可能造成资源竞争

3. 部分分配策略：
   - 任务开始时分配部分资源
   - 执行过程中动态分配
   - 平衡资源利用率和竞争
```

### 4.2 资源竞争

**定义 4.2**（资源竞争Petri网）：

```text
资源竞争Petri网：
  多个任务竞争同一资源：
    p_resource →[t_allocate_1]→ p_allocated_1
    p_resource →[t_allocate_2]→ p_allocated_2
    p_resource →[t_allocate_3]→ p_allocated_3

竞争条件：
  - 如果M(p_resource) < Σ resource_required，则存在竞争
  - 调度函数决定哪个任务获得资源

冲突解决：
  - 优先级：高优先级任务优先
  - FIFO：先请求的任务优先
  - 随机：随机选择任务
```

### 4.3 资源释放

**定义 4.3**（资源释放Petri网）：

```text
资源释放Petri网：
  已分配资源库所：p_allocated
  释放变迁：t_release
  可用资源库所：p_resource

结构：
  p_allocated →[t_release]→ p_resource

释放时机：
  - 任务完成时释放
  - 任务阻塞时释放
  - 超时释放
```

### 4.4 死锁预防

**定义 4.4**（死锁预防Petri网）：

```text
死锁预防策略：

1. 资源排序：
   - 对所有资源排序
   - 按顺序请求资源
   - 避免循环等待

2. 超时机制：
   - 资源请求设置超时
   - 超时后释放资源
   - 避免无限等待

3. 死锁检测：
   - 定期检测死锁
   - 检测到死锁后回滚
   - 释放资源并重试
```

## 5 关系属性与依赖分析

### 5.1 调度策略之间的关系

**调度策略对比**：

| 策略 | 公平性 | 响应性 | 吞吐量 | 复杂度 |
|------|--------|--------|--------|--------|
| **FIFO** | 高 | 低 | 中 | 低 |
| **优先级** | 中 | 高 | 高 | 中 |
| **时间片轮转** | 高 | 中 | 中 | 中 |
| **最短作业优先** | 低 | 中 | 高 | 高 |

### 5.2 资源调度与任务调度的关系

**资源调度依赖任务调度**：

```text
1. 任务调度决定任务执行顺序
2. 资源调度决定资源分配
3. 两者相互影响：
   - 任务调度影响资源需求
   - 资源可用性影响任务调度
```

### 5.3 调度性能指标

**定义 5.1**（调度性能指标）：

```text
1. 吞吐量（Throughput）：
   Throughput = 完成任务数 / 时间

2. 响应时间（Response Time）：
   ResponseTime = 完成时间 - 到达时间

3. 等待时间（Waiting Time）：
   WaitingTime = 开始时间 - 到达时间

4. 周转时间（Turnaround Time）：
   TurnaroundTime = 完成时间 - 到达时间

5. 资源利用率（Resource Utilization）：
   Utilization = 资源使用时间 / 总时间
```
