## 1. GPU架构基础

### 1.1 GPU硬件架构

现代GPU采用大规模并行架构，其核心组成包括：

$$
\text{GPU} = \langle \mathcal{SM}, \mathcal{MEM}, \mathcal{SCH}, \mathcal{BUS} \rangle
$$

其中：

- $\mathcal{SM}$：流式多处理器（Streaming Multiprocessors）集合
- $\mathcal{MEM}$：内存层次结构
- $\mathcal{SCH}$：硬件调度器
- $\mathcal{BUS}$：互连总线

### 1.2 执行模型

**定义 1.1（GPU执行模型）**: GPU执行遵循SIMT（Single Instruction Multiple Threads）模型，线程按Warp（通常32线程）组织：

$$
\text{Warp} = \{t_1, t_2, ..., t_{32}\} \quad \text{其中} \quad \forall t_i, t_j \in \text{Warp}: \text{PC}(t_i) = \text{PC}(t_j)
$$

**定义 1.2（线程层次结构）**: GPU线程按三级层次组织：

$$
\text{Grid} = \bigcup_{b=1}^{B} \text{Block}_b, \quad \text{Block}_b = \bigcup_{w=1}^{W_b} \text{Warp}_w
$$

| 层次 | 范围 | 同步能力 | 内存可见性 |
|------|------|----------|-----------|
| Thread | 单线程 | 无 | 寄存器 |
| Warp | 32线程 | 隐式同步 | 无 |
| Block | ≤1024线程 | `__syncthreads()` | 共享内存 |
| Grid | 所有Block | 全局同步 | 全局内存 |

### 1.3 内存层次结构

**定义 1.3（GPU内存模型）**: GPU内存层次可形式化为多元组：

$$
\mathcal{MEM} = \langle R, S, L_2, G, H \rangle
$$

其中延迟和带宽特征：

| 内存类型 | 延迟 | 带宽 | 容量 | 访问方式 |
|----------|------|------|------|----------|
| 寄存器 (R) | 1 cycle | ~80 TB/s | 256 KB/SM | 线程私有 |
| 共享内存 (S) | ~20 cycles | ~10 TB/s | ≤164 KB/SM | Block共享 |
| L2缓存 (L₂) | ~200 cycles | ~4 TB/s | 几MB-几十MB | GPU共享 |
| 全局内存 (G) | ~400 cycles | ~1-2 TB/s | 几GB-几十GB | 全局可见 |
| 主机内存 (H) | ~10 μs | ~64 GB/s (PCIe) | 系统内存 | 通过PCIe |

## 2. GPU任务调度基础

### 2.1 任务模型

**定义 2.1（GPU任务）**: GPU任务是一个六元组：

$$
T_i = (id_i, \tau_i, \rho_i, \delta_i, \pi_i, \prec_i)
$$

其中：

- $id_i$：任务唯一标识符
- $\tau_i = (\tau^{grid}, \tau^{block})$：线程配置（Grid和Block维度）
- $\rho_i$：资源需求（寄存器数、共享内存大小）
- $\delta_i = (d^{comp}_i, d^{mem}_i)$：计算和内存需求
- $\pi_i$：优先级
- $\prec_i$：前置任务集合

**定义 2.2（资源需求）**: 任务的资源需求为：

$$
\rho_i = (regs_i, smem_i, blocks_i)
$$

其中：

- $regs_i$：每线程寄存器数量
- $smem_i$：每Block共享内存大小
- $blocks_i$：所需Block数量

### 2.2 调度约束

**约束 2.1（资源容量约束）**: 每个SM上同时驻留的任务总资源需求不能超过SM容量：

$$
\forall SM_k: \sum_{T_i \in \text{Active}(SM_k)} \rho_i \leqslant C_{SM_k}
$$

**约束 2.2（依赖约束）**: 任务必须在所有前置任务完成后才能开始：

$$
\forall T_i, T_j: T_j \prec T_i \Rightarrow S_i \geqslant C_j
$$

其中 $S_i$ 和 $C_j$ 分别是任务的开始和完成时间。

**约束 2.3（Occupancy约束）**: 为隐藏内存延迟，需要足够的Warp并行度：

$$
\text{Occupancy}(SM_k) = \frac{\text{Active Warps}}{\text{Max Warps}} \geqslant \theta_{min}
$$

其中 $\theta_{min}$ 通常为0.25-0.5。

## 3. GPU调度算法

### 3.1 轮询调度（Round-Robin）

**算法 3.1（RR-GPU调度）**: 按时间片轮转分配SM资源：

```python
class RRGpuScheduler:
    def __init__(self, num_sms, time_quantum=100):
        self.num_sms = num_sms
        self.time_quantum = time_quantum  # μs
        self.ready_queue = deque()
        self.sm_assignments = [None] * num_sms
        self.remaining_time = {}

    def enqueue(self, task):
        """添加任务到就绪队列"""
        self.ready_queue.append(task)
        self.remaining_time[task.id] = task.duration

    def schedule(self, current_time):
        """执行一轮调度"""
        # 检查完成的任务
        for sm_id in range(self.num_sms):
            if self.sm_assignments[sm_id]:
                task = self.sm_assignments[sm_id]
                if self.remaining_time[task.id] <= 0:
                    self.sm_assignments[sm_id] = None

        # 分配空闲SM
        for sm_id in range(self.num_sms):
            if not self.sm_assignments[sm_id] and self.ready_queue:
                task = self.ready_queue.popleft()
                if self.can_fit_on_sm(task, sm_id):
                    self.sm_assignments[sm_id] = task
                    task.start_time = current_time

        return self.sm_assignments
```

**复杂度分析**：

- 时间复杂度：$O(|\mathcal{T}| + m)$，其中 $m$ 是SM数量
- 空间复杂度：$O(|\mathcal{T}|)$

### 3.2 最短任务优先（STF）

**算法 3.2（STF-GPU调度）**: 优先调度执行时间最短的任务：

$$
\sigma_{STF}(t) = \arg\min_{T_i \in \text{Ready}(t)} \{d^{comp}_i\}
$$

**最优性定理**：

**定理 3.1（STF最优性）**: 对于单SM且无依赖的任务集，STF最小化平均完成时间：

$$
\min \frac{1}{n} \sum_{i=1}^{n} C_i
$$

**证明**：

设有两个任务 $T_1$ 和 $T_2$，执行时间分别为 $p_1$ 和 $p_2$，且 $p_1 \leqslant p_2$。

- 顺序1（STF）：$T_1$ 先执行，$C_1 = p_1$, $C_2 = p_1 + p_2$
  - 总完成时间：$C^{(1)} = p_1 + (p_1 + p_2) = 2p_1 + p_2$

- 顺序2（非STF）：$T_2$ 先执行，$C_2 = p_2$, $C_1 = p_2 + p_1$
  - 总完成时间：$C^{(2)} = p_2 + (p_2 + p_1) = p_1 + 2p_2$

比较：$C^{(1)} - C^{(2)} = (2p_1 + p_2) - (p_1 + 2p_2) = p_1 - p_2 \leqslant 0$

因此STF顺序总完成时间更小。通过归纳法可推广到 $n$ 个任务。∎

### 3.3 优先级调度（Priority-based）

**算法 3.3（优先级GPU调度）**: 根据任务优先级进行调度：

```python
import heapq
from dataclasses import dataclass, field

@dataclass(order=True)
class PriorityTask:
    priority: int
    arrival_time: int = field(compare=False)
    duration: int = field(compare=False)
    task_id: str = field(compare=False)
    resource_req: dict = field(compare=False)

class PriorityGpuScheduler:
    def __init__(self, num_sms):
        self.num_sms = num_sms
        self.ready_queue = []  # 优先队列
        self.running_tasks = {}  # SM_ID -> Task
        self.sm_resources = {i: {'regs': 65536, 'smem': 164000}
                            for i in range(num_sms)}

    def can_schedule(self, task, sm_id):
        """检查任务是否可以在指定SM上运行"""
        sm = self.sm_resources[sm_id]
        req = task.resource_req
        return (sm['regs'] >= req.get('regs', 0) and
                sm['smem'] >= req.get('smem', 0))

    def allocate(self, task, sm_id):
        """分配资源"""
        sm = self.sm_resources[sm_id]
        req = task.resource_req
        sm['regs'] -= req.get('regs', 0)
        sm['smem'] -= req.get('smem', 0)
        self.running_tasks[sm_id] = task

    def release(self, sm_id):
        """释放资源"""
        if sm_id in self.running_tasks:
            task = self.running_tasks[sm_id]
            sm = self.sm_resources[sm_id]
            req = task.resource_req
            sm['regs'] += req.get('regs', 0)
            sm['smem'] += req.get('smem', 0)
            del self.running_tasks[sm_id]

    def schedule(self, current_time):
        """执行优先级调度"""
        scheduled = []

        # 尝试在空闲SM上调度高优先级任务
        for sm_id in range(self.num_sms):
            if sm_id not in self.running_tasks and self.ready_queue:
                # 找到最高优先级且能适配的任务
                temp_queue = []
                selected = None

                while self.ready_queue and not selected:
                    task = heapq.heappop(self.ready_queue)
                    if self.can_schedule(task, sm_id):
                        selected = task
                    else:
                        temp_queue.append(task)

                # 恢复未选中的任务
                for t in temp_queue:
                    heapq.heappush(self.ready_queue, t)

                if selected:
                    self.allocate(selected, sm_id)
                    scheduled.append((sm_id, selected))

        return scheduled
```

## 4. 形式化模型与证明

### 4.1 GPU调度问题形式化定义

**定义 4.1（GPU调度问题）**: GPU调度问题是一个六元组：

$$
\Pi_{GPU} = (\mathcal{T}, \mathcal{SM}, \mathcal{R}, \mathcal{C}, \mathcal{O}, \mathcal{D})
$$

其中：

- $\mathcal{T} = \{T_1, T_2, ..., T_n\}$：任务集合
- $\mathcal{SM} = \{SM_1, SM_2, ..., SM_m\}$：SM集合
- $\mathcal{R}$：资源类型集合
- $\mathcal{C}$：约束条件集合
- $\mathcal{O}$：目标函数
- $\mathcal{D}$：依赖关系图

**定义 4.2（可行调度）**: 调度方案 $\sigma: \mathcal{T} \times \mathcal{SM} \times \mathbb{R}^+ \to \{0, 1\}$ 是可行的，当且仅当满足：

$$
\begin{aligned}
&\forall t, SM_k: \sum_{T_i \in \mathcal{T}} \sigma(T_i, SM_k, t) \cdot \rho_i \leqslant C_{SM_k} \\
&\forall T_i, T_j: T_j \prec T_i \Rightarrow S_i \geqslant C_j \\
&\forall T_i: \int_{0}^{\infty} \sum_{SM_k} \sigma(T_i, SM_k, t) \, dt = d^{comp}_i
\end{aligned}
$$

### 4.2 最优性理论

**定理 4.1（GPU调度NP难性）**: 考虑资源约束和任务依赖的GPU调度问题是NP难的。

**证明**：

通过将**装箱问题**（Bin Packing）规约到GPU调度问题：

给定装箱问题实例：物品大小集合 $\{s_1, s_2, ..., s_n\}$，箱子容量 $B$。

构造GPU调度实例：

- 每个物品对应一个任务 $T_i$，$d^{comp}_i = 1$，$\rho_i = s_i$
- 每个SM对应一个箱子，容量为 $B$
- 目标：最小化使用的SM数量

如果存在多项式时间算法解决GPU调度，则可解决装箱问题。由于装箱问题是NP难的，故GPU调度也是NP难的。∎

**定理 4.2（在线GPU调度的竞争比下界）**: 对于在线GPU调度问题，任何确定性算法的竞争比至少为2。

**证明概要**：

构造对抗实例：

1. 第一个任务需要一半SM资源
2. 若算法将其分配到特定SM，则后续任务只能与该任务并行或串行
3. 最优离线算法可以将任务完美打包

通过适当构造任务序列，可以证明竞争比下界。∎

## 5. 高级调度技术

### 5.1 动态并行调度

CUDA Dynamic Parallelism允许GPU kernel启动其他kernel：

```cuda
// 父kernel
__global__ void parent_kernel(int *data, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        // 动态启动子kernel
        child_kernel<<<4, 256>>>(data + idx * 256);
        cudaDeviceSynchronize();
    }
}

// 子kernel
__global__ void child_kernel(int *sub_data) {
    int tid = threadIdx.x;
    // 处理子任务
    sub_data[tid] *= 2;
}
```

**调度挑战**：

- 嵌套kernel的资源分配
- 父子kernel的依赖管理
- 避免资源死锁

### 5.2 多流并发

CUDA流允许并行执行多个kernel：

```cuda
// 多流并发示例
cudaStream_t streams[NUM_STREAMS];
for (int i = 0; i < NUM_STREAMS; i++) {

for (int i = 0; i < NUM_STREAMS; i++) {
```

## 6. 性能优化策略

### 6.1 Occupancy优化

**定义 6.1（Occupancy）**: Occupancy是活跃Warp数与最大可能Warp数的比值：

$$
\text{Occupancy} = \frac{\text{Active Warps per SM}}{\text{Max Warps per SM}}
$$

**优化策略**：

```python
def calculate_occupancy(block_size, regs_per_thread, smem_per_block,
                       sm_resources):
    """
    计算kernel的occupancy

    Args:
        block_size: 每Block线程数
        regs_per_thread: 每线程寄存器数
        smem_per_block: 每Block共享内存
        sm_resources: SM资源容量
    """
    # 寄存器限制下的Block数
    blocks_per_sm_regs = sm_resources['max_regs'] // (regs_per_thread * block_size)

    # 共享内存限制下的Block数
    blocks_per_sm_smem = sm_resources['max_smem'] // smem_per_block

    # Warp限制下的Block数
    warps_per_block = (block_size + 31) // 32
    blocks_per_sm_warps = sm_resources['max_warps'] // warps_per_block

    # 实际Block数取最小值
    blocks_per_sm = min(blocks_per_sm_regs, blocks_per_sm_smem,
                       blocks_per_sm_warps, sm_resources['max_blocks'])

    # 计算occupancy
    active_warps = blocks_per_sm * warps_per_block
    occupancy = active_warps / sm_resources['max_warps']

    return {
        'blocks_per_sm': blocks_per_sm,
        'active_warps': active_warps,
        'occupancy': occupancy,
        'limiting_factor': get_limiting_factor(...)
    }

# 示例：A100 SM资源
a100_sm = {
    'max_regs': 65536,
    'max_smem': 164000,
    'max_blocks': 32,
    'max_warps': 64
}

# 计算一个典型kernel的occupancy
result = calculate_occupancy(
    block_size=256,
    regs_per_thread=64,
    smem_per_block=48000,
    sm_resources=a100_sm
)
print(f"Occupancy: {result['occupancy']:.2%}")
```

### 6.2 内存访问优化

**全局内存合并访问**：

```cuda
// 差的访问模式（非合并）
__global__ void uncoalesced_access(float *input, float *output, int n) {
    int idx = threadIdx.x;
    // 线程访问不连续的地址
    output[idx] = input[idx * 2];
}

// 好的访问模式（合并）
__global__ void coalesced_access(float *input, float *output, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    // 线程访问连续地址
    if (idx < n) {
        output[idx] = input[idx];
    }
}
```

**共享内存Bank Conflict**：

```cuda
// 有Bank Conflict的访问
```

```cuda
// 分块矩阵乘法
#define BLOCK_SIZE 32

__global__ void matrix_multiply(float *A, float *B, float *C, int N) {
    __shared__ float As[BLOCK_SIZE][BLOCK_SIZE];
    __shared__ float Bs[BLOCK_SIZE][BLOCK_SIZE];

    int bx = blockIdx.x, by = blockIdx.y;
    int tx = threadIdx.x, ty = threadIdx.y;

    int row = by * BLOCK_SIZE + ty;
    int col = bx * BLOCK_SIZE + tx;

    float sum = 0.0f;

    for (int m = 0; m < (N + BLOCK_SIZE - 1) / BLOCK_SIZE; ++m) {
        // 加载A和B到共享内存
        if (row < N && m * BLOCK_SIZE + tx < N)
            As[ty][tx] = A[row * N + m * BLOCK_SIZE + tx];
        else
            As[ty][tx] = 0.0f;

        if (col < N && m * BLOCK_SIZE + ty < N)
            Bs[ty][tx] = B[(m * BLOCK_SIZE + ty) * N + col];
        else
            Bs[ty][tx] = 0.0f;

        __syncthreads();

        // 计算部分积
        for (int k = 0; k < BLOCK_SIZE; ++k)
            sum += As[ty][k] * Bs[k][tx];

        __syncthreads();
    }

    if (row < N && col < N)
        C[row * N + col] = sum;
}

// 调度配置
void schedule_matmul(int N) {
    dim3 block(BLOCK_SIZE, BLOCK_SIZE);
    dim3 grid((N + BLOCK_SIZE - 1) / BLOCK_SIZE,
              (N + BLOCK_SIZE - 1) / BLOCK_SIZE);

    int regs_per_thread = 32;  // 估算
```

### 7.2 深度学习算子调度

```python
import torch
import torch.nn as nn

class GpuScheduler:
    """GPU任务调度器，优化深度学习训练"""

    def __init__(self, device='cuda'):
        self.device = device
        self.stream_pool = [torch.cuda.Stream(device=device)
                           for _ in range(4)]
        self.current_stream = 0

    def schedule_forward(self, layers, x):
        """调度前向传播"""
        results = []
        for i, layer in enumerate(layers):
            stream = self.stream_pool[i % len(self.stream_pool)]
            with torch.cuda.stream(stream):
                x = layer(x)
                results.append(x)
        torch.cuda.synchronize()
        return x

    def schedule_backward(self, loss, optimizer):
        """调度反向传播"""
        # 使用默认流进行反向传播
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    def benchmark_kernel(self, kernel_fn, *args, warmup=10, iterations=100):
        """测量kernel执行时间"""
        # Warmup
        for _ in range(warmup):
            kernel_fn(*args)
        torch.cuda.synchronize()

        # 测量
        start = torch.cuda.Event(enable_timing=True)
        end = torch.cuda.Event(enable_timing=True)

        start.record()
        for _ in range(iterations):
            kernel_fn(*args)
        end.record()
        torch.cuda.synchronize()

        elapsed_ms = start.elapsed_time(end) / iterations
        return elapsed_ms
```
