## 1. 图形渲染管线概述

### 1.1 渲染管线架构

现代GPU图形渲染管线遵循可编程管线模型：

$$
\text{Pipeline} = \langle IA, VS, HS, TS, DS, GS, RS, PS, OM \rangle
$$

其中各阶段定义：

- **IA** (Input Assembler)：顶点数据组装
- **VS** (Vertex Shader)：顶点变换
- **HS** (Hull Shader)：细分控制
- **TS** (Tessellator)：细分生成
- **DS** (Domain Shader)：细分后顶点处理
- **GS** (Geometry Shader)：几何处理
- **RS** (Rasterizer)：光栅化
- **PS** (Pixel Shader)：像素着色
- **OM** (Output Merger)：输出合并

```mermaid
flowchart LR
    IA["IA<br/>输入组装"] --> VS["VS<br/>顶点着色"]
    VS --> HS["HS<br/>外壳着色"]
    HS --> TS["TS<br/>细分器"]
    TS --> DS["DS<br/>域着色"]
    DS --> GS["GS<br/>几何着色"]
    GS --> RS["RS<br/>光栅化"]
    RS --> PS["PS<br/>像素着色"]
    PS --> OM["OM<br/>输出合并"]
    OM --> FB["Framebuffer"]
```

### 1.2 渲染工作负载特征

**定义 1.1（渲染任务）**: 渲染任务可形式化为：

$$
R_i = (G_i, S_i, B_i, D_i, \delta_i)
$$

其中：

- $G_i = (V_i, E_i)$：场景图，顶点集和边集
- $S_i$：着色器程序集合
- $B_i$：渲染批次列表
- $D_i$：绘制调用集合
- $\delta_i$：截止时间（帧时间预算）

**渲染工作负载统计特征**：

| 指标 | 典型值 | 波动范围 | 影响因素 |
|------|--------|----------|----------|
| 顶点数/帧 | 1M-10M | 0.1x-10x | LOD, 场景复杂度 |
| 绘制调用/帧 | 500-5000 | 0.5x-5x | 合批效率 |
| 着色器指令 | 100-1000 | 1x-10x | 材质复杂度 |
| 显存带宽 | 50-200 GB/s | 0.5x-2x | 纹理分辨率 |

### 1.3 帧时间预算

**定义 1.2（帧时间预算）**: 给定目标帧率 $FPS_{target}$，帧时间预算为：

$$
\Delta t_{frame} = \frac{1000}{FPS_{target}} \text{ ms}
$$

常见帧率目标：

| 应用场景 | 目标FPS | 帧时间预算 |
|----------|---------|-----------|
| 手机游戏 | 30-60 | 16.7-33.3 ms |
| 主机游戏 | 60 | 16.7 ms |
| 电竞游戏 | 120-240 | 4.2-8.3 ms |
| VR应用 | 90+ | ≤11.1 ms |

## 2. 渲染调度基础

### 2.1 渲染命令调度

**定义 2.1（渲染命令）**: 渲染命令是一个四元组：

$$
C_i = (type_i, res_i, deps_i, prio_i)
$$

其中：

- $type_i \in \{DRAW, COMPUTE, COPY, BARRIER\}$：命令类型
- $res_i$：资源需求（PSO, 纹理, 缓冲区）
- $deps_i$：依赖命令集合
- $prio_i$：优先级

### 2.2 管线状态对象（PSO）

**定义 2.2（PSO）**: 管线状态对象是渲染管线的完整配置：

$$
\text{PSO} = \langle VS, PS, BLEND, DEPTH, RASTER, INPUT \rangle
$$

## 3. 帧调度算法

### 3.1 双缓冲与三缓冲

**定义 3.1（缓冲策略）**: 帧缓冲策略决定渲染与显示的并行度：

$$
\text{Buffering}(n) = \begin{cases}
\text{Double} & n = 2 \\
\text{Triple} & n = 3
\end{cases}
$$

**双缓冲**：

```
时间轴:  0    16.7  33.3  50.0  66.7  ms
Front:   [F0]  F1    F0    F1    F0    (显示)
Back:     B0   [B1]  [B0]  [B1]  [B0]  (渲染)
                ↑     ↑     ↑     ↑
              VSync交换
```

**三缓冲优势**：

```
时间轴:  0    16.7  33.3  50.0  66.7  83.3  ms
Display:  F0    F1    F1    F2    F2    F3
Render:   [B0]  [B1]  [B2]  [B0]  [B1]  [B2]
Pending:        P0    P0,P1  P1   P2
```

延迟与流畅度的权衡：

| 策略 | 显示延迟 | 卡顿保护 | 显存占用 |
|------|----------|----------|----------|
| 双缓冲 | 16.7ms | 无 | 2x |
| 三缓冲 | 33.3ms | 有 | 3x |
| 自适应 | 可变 | 有 | 2-3x |

### 3.2 基于优先级的帧调度

**算法 3.1（优先级帧调度）**: 根据渲染阶段优先级调度：

```python
from enum import IntEnum
from dataclasses import dataclass
from typing import List, Set
import heapq

class RenderPriority(IntEnum):
    CRITICAL = 0      # UI, HUD
    HIGH = 1          # 主场景
    MEDIUM = 2        # 特效
    LOW = 3           # 后处理
    BACKGROUND = 4    # 异步加载

@dataclass(order=True)
class RenderCommand:
    priority: RenderPriority
    estimated_cost: float  # μs
    command_id: int
    dependencies: Set[int] = None

    def __post_init__(self):
        if self.dependencies is None:
            self.dependencies = set()

class FrameScheduler:
    def __init__(self, frame_budget_ms=16.67):
        self.frame_budget_ms = frame_budget_ms
        self.command_queue = []
        self.completed = set()
        self.frame_stats = []

    def submit(self, command: RenderCommand):
        """提交渲染命令"""
        heapq.heappush(self.command_queue, command)

    def schedule_frame(self) -> List[RenderCommand]:
        """
        调度一帧的渲染命令

        返回可执行命令列表，保证依赖满足且不超过帧预算
        """
        scheduled = []
        remaining_budget = self.frame_budget_ms * 1000  # 转换为μs
        temp_queue = []

        while self.command_queue and remaining_budget > 0:
            cmd = heapq.heappop(self.command_queue)

            # 检查依赖是否满足
            if cmd.dependencies.issubset(self.completed):
                # 检查预算
                if cmd.estimated_cost <= remaining_budget:
                    scheduled.append(cmd)
                    remaining_budget -= cmd.estimated_cost
                    self.completed.add(cmd.command_id)
                else:
                    # 预算不足，放回队列
                    temp_queue.append(cmd)
                    break
            else:
                # 依赖未满足，暂存
                temp_queue.append(cmd)

        # 恢复未调度的命令
        for cmd in temp_queue:
            heapq.heappush(self.command_queue, cmd)

        self.frame_stats.append({
            'commands': len(scheduled),
            'remaining_budget': remaining_budget / 1000,  # ms
            'utilization': 1 - remaining_budget / (self.frame_budget_ms * 1000)
        })

        return scheduled

    def should_drop_frame(self, pending_cost: float) -> bool:
        """
        判断是否应跳过当前帧

        当剩余工作超过预算时，考虑跳过以保证帧率
        """
        if pending_cost > self.frame_budget_ms * 1000 * 1.5:
            return True
        return False
```

### 3.3 动态分辨率缩放

**定义 3.2（动态分辨率）**: 根据GPU负载动态调整渲染分辨率：

$$
\text{Scale}(t) = \min\left(1.0, \frac{\Delta t_{target}}{\Delta t_{actual}(t-1)} \cdot \alpha + (1-\alpha) \cdot \text{Scale}(t-1)\right)
$$

其中 $\alpha$ 是平滑因子。

```python
class DynamicResolutionScaler:
    def __init__(self, target_frame_time_ms=16.67,
                 min_scale=0.5, max_scale=1.0,
                 alpha=0.3):
        self.target_frame_time = target_frame_time_ms
        self.min_scale = min_scale
        self.max_scale = max_scale
        self.alpha = alpha
        self.current_scale = 1.0
        self.history = []

    def update(self, actual_frame_time_ms: float) -> float:
        """
        根据实际帧时间更新缩放比例

        Args:
            actual_frame_time_ms: 上一帧实际耗时

        Returns:
            新的渲染分辨率缩放比例
        """
        # 计算目标缩放
        target_scale = self.target_frame_time / actual_frame_time_ms

        # 平滑处理
        self.current_scale = (self.alpha * target_scale +
                             (1 - self.alpha) * self.current_scale)

        # 限制范围
        self.current_scale = max(self.min_scale,
                                min(self.max_scale, self.current_scale))

        self.history.append({
            'frame_time': actual_frame_time_ms,
            'scale': self.current_scale
        })

        return self.current_scale

    def get_resolution(self, base_width: int, base_height: int) -> tuple:
        """获取实际渲染分辨率"""
        width = int(base_width * self.current_scale)
        height = int(base_height * self.current_scale)
        # 对齐到8像素（GPU优化）
        width = (width // 8) * 8
        height = (height // 8) * 8
        return width, height
```

## 4. 形式化模型与证明

### 4.1 渲染调度问题形式化定义

**定义 4.1（渲染调度问题）**: 渲染调度问题是一个七元组：

$$
\Pi_{render} = (\mathcal{F}, \mathcal{C}, \mathcal{R}, \mathcal{P}, \mathcal{B}, \mathcal{D}, \mathcal{O})
$$

其中：

- $\mathcal{F} = \{F_1, F_2, ..., F_n\}$：帧序列
- $\mathcal{C}$：渲染命令集合
- $\mathcal{R}$：资源集合（纹理, 缓冲区, PSO）
- $\mathcal{P}$：处理单元集合（VS, PS, CS）
- $\mathcal{B}$：带宽约束
- $\mathcal{D}$：依赖关系
- $\mathcal{O}$：优化目标

**定义 4.2（帧调度方案）**: 帧 $F_k$ 的调度方案为：

$$
\sigma_k: \mathcal{C} \times \mathcal{P} \times \mathbb{R}^+ \to \{0, 1\}
$$

约束条件：

$$
\begin{aligned}
&\text{(资源独占)} \quad \forall r \in \mathcal{R}, t: \sum_{c \in \mathcal{C}} \sigma_k(c, r, t) \leqslant 1 \\
&\text{(依赖满足)} \quad \forall c_i, c_j: c_j \prec c_i \Rightarrow S(c_i) \geqslant C(c_j) \\
&\text{(帧预算)} \quad \forall k: C_{max}(F_k) - S_{min}(F_k) \leqslant \Delta t_{frame}
\end{aligned}
$$

### 4.2 实时性保证理论

**定义 4.3（硬实时渲染）**: 渲染任务满足硬实时约束，当且仅当：

$$
\forall F_k: C_k - S_k \leqslant D_k \quad \text{且} \quad C_k \leqslant k \cdot \Delta t_{frame}
$$

**定理 4.1（帧率稳定性条件）**: 若第 $k$ 帧的渲染工作量 $W_k$ 服从分布 $W_k \sim \mathcal{D}(\mu, \sigma^2)$，则维持目标帧率的条件为：

$$
\mu + 2\sigma \leqslant \Delta t_{frame}
$$

**证明**：

由切比雪夫不等式：

$$P(|W_k - \mu| \geqslant 2\sigma) \leqslant \frac{\sigma^2}{(2\sigma)^2} = 0.25$$

因此：

$$P(W_k \leqslant \mu + 2\sigma) \geqslant 0.75$$

若 $\mu + 2\sigma \leqslant \Delta t_{frame}$，则至少有75%的帧能在预算内完成。

为满足99.7%的帧（近似3σ原则），需要：

$$\mu + 3\sigma \leqslant \Delta t_{frame}$$ ∎

**定理 4.2（PSO预缓存最优性）**: 给定 $n$ 个渲染批次，使用 $k$ 种不同的PSO，按PSO排序调度的切换次数为 $k-1$，是最小可能的。

**证明**：

设渲染批次序列为 $B_1, B_2, ..., B_n$，每个批次使用PSO $p(B_i) \in \{1, ..., k\}$。

- **上界**：按PSO排序后，相同PSO的批次连续，切换次数为 $k-1$
- **下界**：若 $k$ 种PSO都被使用，至少需要 $k-1$ 次切换

因此排序调度达到最优。∎

### 5.1 异步计算

现代GPU支持图形与计算并行执行：

```cpp
// Vulkan异步计算示例
class AsyncComputeScheduler {
public:
    void Initialize(VkDevice device) {
        // 创建两个队列族：图形队列和计算队列
        VkDeviceQueueCreateInfo queueInfos[2] = {};

        // 图形队列
        float graphicsPriority = 1.0f;
        queueInfos[0].sType = VK_STRUCTURE_TYPE_DEVICE_QUEUE_CREATE_INFO;
        queueInfos[0].queueFamilyIndex = graphicsFamily;
        queueInfos[0].queueCount = 1;
        queueInfos[0].pQueuePriorities = &graphicsPriority;

        // 计算队列（可能来自不同队列族）
        float computePriority = 0.8f;
        queueInfos[1].sType = VK_STRUCTURE_TYPE_DEVICE_QUEUE_CREATE_INFO;
        queueInfos[1].queueFamilyIndex = computeFamily;
        queueInfos[1].queueCount = 1;
        queueInfos[1].pQueuePriorities = &computePriority;

        // 创建设备
        VkDeviceCreateInfo createInfo = {};
        createInfo.sType = VK_STRUCTURE_TYPE_DEVICE_CREATE_INFO;
        createInfo.queueCreateInfoCount = 2;
        createInfo.pQueueCreateInfos = queueInfos;
        vkCreateDevice(physicalDevice, &createInfo, nullptr, &device);

        // 获取队列
        vkGetDeviceQueue(device, graphicsFamily, 0, &graphicsQueue);
        vkGetDeviceQueue(device, computeFamily, 0, &computeQueue);
    }

    void SubmitAsyncWork() {
        // 图形工作
        VkSubmitInfo graphicsSubmit = {};
        graphicsSubmit.sType = VK_STRUCTURE_TYPE_SUBMIT_INFO;
        graphicsSubmit.commandBufferCount = 1;
        graphicsSubmit.pCommandBuffers = &graphicsCmdBuffer;

        // 计算工作（可与图形并行）
        VkSubmitInfo computeSubmit = {};
        computeSubmit.sType = VK_STRUCTURE_TYPE_SUBMIT_INFO;
        computeSubmit.commandBufferCount = 1;
        computeSubmit.pCommandBuffers = &computeCmdBuffer;

        // 并行提交到不同队列
        vkQueueSubmit(graphicsQueue, 1, &graphicsSubmit, VK_NULL_HANDLE);
        vkQueueSubmit(computeQueue, 1, &computeSubmit, VK_NULL_HANDLE);
    }
};
```

### 5.2 时间性抗锯齿（TAA）调度

**定义 5.1（TAA抖动模式）**: TAA使用子像素抖动生成历史帧：

$$
J_k = (j_x(k), j_y(k)) \in \left[-\frac{1}{2}, \frac{1}{2}\right] \times \left[-\frac{1}{2}, \frac{1}{2}\right]
$$

常用抖动序列：

```cpp
// Halton序列抖动
class TAAJitterGenerator {
public:
    glm::vec2 GetJitter(int frameIndex, int width, int height) {
        // Halton(2, 3)序列
        float x = Halton(frameIndex % 8, 2) - 0.5f;
        float y = Halton(frameIndex % 8, 3) - 0.5f;

        // 转换为像素偏移
        return glm::vec2(x / width, y / height);
    }

private:
    float Halton(int index, int base) {
        float result = 0.0f;
        float f = 1.0f / base;
        int i = index;
        while (i > 0) {
            result += f * (i % base);
            i = i / base;
            f = f / base;
        }
        return result;
    }
};
```

```cpp
// Unreal Engine风格渲染调度器
class FRenderScheduler {
public:
    void ScheduleFrame(FSceneView& View) {
        // 1. 可见性计算（异步）
        EnqueueAsyncTask([this, &View]() {
            ComputeVisibility(View);
        });

        // 2. 阴影渲染
        if (ShouldRenderShadows()) {
            ScheduleShadowPasses();
        }

        // 3. 基础Pass
        ScheduleBasePass();

        // 4. 光照
        ScheduleLighting();

        // 5. 后处理
        SchedulePostProcess();

        // 6. UI渲染（最高优先级）
        ScheduleUI();
    }

private:
    void ScheduleBasePass() {
        // 按PSO排序渲染
        TArray<FMeshBatch> SortedMeshes = SortMeshesByPSO(VisibleMeshes);

        for (auto& Mesh : SortedMeshes) {
            FRHIRenderPassInfo PassInfo;
            PassInfo.ColorTargets[0] = GBufferA;
            PassInfo.ColorTargets[1] = GBufferB;
            PassInfo.DepthStencilTarget = DepthBuffer;

            RHIBeginRenderPass(PassInfo);
            Mesh.Draw(RHICmdList);
            RHIEndRenderPass();
        }
    }

    void SchedulePostProcess() {
        // 动态调整后处理质量
        float FrameTime = GetLastFrameTime();
        if (FrameTime > TargetFrameTime * 1.1f) {
            // 帧时间超标，降低后处理质量
            PostProcessSettings.BloomQuality =
                FMath::Max(0, PostProcessSettings.BloomQuality - 1);
            PostProcessSettings.SSAOQuality =
                FMath::Max(0, PostProcessSettings.SSAOQuality - 1);
        }

        // 调度后处理阶段
        AddPostProcessPass("Tonemapping", Priority_High);
        AddPostProcessPass("Bloom", Priority_Medium);
        AddPostProcessPass("SSAO", Priority_Low);
    }
};
```

### 6.2 VR渲染调度

```cpp
// VR特定渲染优化
class VRRenderScheduler {
public:
    void Initialize() {
        // VR需要更严格的帧时间预算
        FrameBudgetMs = 1000.0f / 90.0f;  // 90 FPS = 11.1ms

        // 启用固定注视点渲染
        EnableFixedFoveatedRendering();
    }

    void RenderFrame(const FPose& LeftEyePose, const FPose& RightEyePose) {
        // 预测姿态（降低运动到光子延迟）
        FPredictedPose PredictedLeft = PredictPose(LeftEyePose, PredictionTimeMs);
        FPredictedPose PredictedRight = PredictPose(RightEyePose, PredictionTimeMs);

        // 并行渲染双眼
        std::future<void> LeftEyeTask = std::async(std::launch::async, [&]() {
            RenderEye(EEye::Left, PredictedLeft);
        });

        RenderEye(EEye::Right, PredictedRight);
        LeftEyeTask.wait();

        // 提交到合成器
        SubmitToCompositor(LeftEyeTexture, RightEyeTexture);
    }

    void RenderEye(EEye Eye, const FPredictedPose& Pose) {
        // 应用注视点渲染
        FVector2D FoveationCenter = GetGazeDirection(Eye);
        SetFoveationPattern(FoveationCenter, HighResRadius);

        // 渲染场景
        RenderScene(Pose);

        // 应用透镜畸变校正
        ApplyLensDistortion(Eye);
    }

    void ManageFrameTiming() {
        // 等待合成器VSync
        WaitForCompositorSync();

        // 如果CPU过快，添加适当延迟以匹配GPU
        float CpuFrameTime = GetCpuFrameTime();
        float GpuFrameTime = GetGpuFrameTime();

        if (CpuFrameTime < GpuFrameTime * 0.8f) {
            // CPU等待GPU，添加一点延迟平衡负载
            Sleep(GpuFrameTime - CpuFrameTime);
        }
    }
};
```

GPU图形渲染调度是一个多目标优化问题，需要在视觉质量、帧率稳定性和系统延迟之间取得平衡。本文档涵盖了：

- 帧时间预算是渲染调度的核心约束

- 动态分辨率是应对GPU负载波动的有效手段
- 异步计算可提升GPU利用率
