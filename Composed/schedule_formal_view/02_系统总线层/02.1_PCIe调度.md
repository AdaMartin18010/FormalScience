## 1 PCIe协议层次调度

### 1.1 事务层调度

**TLP（Transaction Layer Packet）类型**：

| 类型 | 用途 | 优先级 | 延迟要求 |
|------|------|--------|----------|
| **MRd** | 内存读 | 高 | <10μs |
| **MWr** | 内存写 | 中 | <50μs |
| **Cpl** | 完成包 | 高 | - |
| **Msg** | 消息 | 低 | <100μs |

**TLP调度策略**：

```c
// TLP调度器
typedef struct {
    tlp_queue_t queues[NUM_PRIORITY_LEVELS];
    uint64_t bandwidth_limit;
    uint64_t current_usage;
} tlp_scheduler_t;

tlp_t *schedule_tlp(tlp_scheduler_t *sched) {
    // 优先级调度
    for (int p = 0; p < NUM_PRIORITY_LEVELS; p++) {
        if (!tlp_queue_empty(&sched->queues[p])) {
            return tlp_dequeue(&sched->queues[p]);
        }
    }
    return NULL;
}
```

### 1.2 数据链路层调度

**ACK/NAK机制**：

```
发送TLP → 等待ACK → 重传（如果NAK）
```

**重传缓冲区管理**：

```c
// 重传缓冲区
typedef struct {
    tlp_t buffer[REPLAY_BUFFER_SIZE];
    uint32_t seq_num;
    uint32_t ack_num;
    timer_t replay_timer;
} replay_buffer_t;

void handle_ack(replay_buffer_t *rb, uint32_t ack_seq) {
    // 释放已确认的TLP
    while (rb->ack_num < ack_seq) {
        free_tlp(&rb->buffer[rb->ack_num % REPLAY_BUFFER_SIZE]);
        rb->ack_num++;
    }
}
```

### 1.3 物理层调度

**Lane调度**：

| 配置 | Lane数 | 带宽 | 适用场景 |
|------|--------|------|----------|
| x1 | 1 | 4GB/s | 低速设备 |
| x4 | 4 | 16GB/s | 网卡、SSD |
| x8 | 8 | 32GB/s | GPU、HBA |

## 2 SR-IOV调度

### 2.1 SR-IOV架构

**单根IO虚拟化（SR-IOV）**：

```
Physical Function (PF)
├── Virtual Function 0
├── Virtual Function 1
├── Virtual Function 2
└── Virtual Function N
```

**功能特性**：

| 特性 | PF | VF |
|------|----|----|
| 配置空间 | 完整 | 受限 |
| MSI-X中断 | 支持 | 支持 |
| DMA | 支持 | 支持（通过IOMMU）|
| 资源管理 | 主控 | 受限 |

### 2.2 VF调度策略

**VF隔离**：

```c
// VF调度器
typedef struct {
    uint16_t vf_id;
    uint64_t tx_quota;      // 发送配额
    uint64_t rx_quota;      // 接收配额
    uint64_t bandwidth_limit;
} vf_sched_entry_t;

bool can_vf_transmit(vf_sched_entry_t *vf, size_t size) {
    if (vf->tx_quota < size) {
        return false;  // 配额不足
    }
    vf->tx_quota -= size;
    return true;
}
```

**VF带宽分配**：

| VF类型 | 带宽分配 | 优先级 |
|--------|---------|--------|
| 管理VF | 10% | 高 |
| 数据VF | 80% | 中 |
| 预留 | 10% | 低 |

### 2.3 资源分配

**BAR空间分配**：

```
PF BAR: 16MB
VF BAR: 256KB x 64 VF = 16MB
Total: 32MB
```

**中断分配**：

```
PF: 32 MSI-X向量
每VF: 8 MSI-X向量
Total: 32 + 64*8 = 544向量
```

## 3 DMA调度

### 3.1 DMA引擎调度

**DMA操作类型**：

| 类型 | 描述 | 性能 |
|------|------|------|
| **线性DMA** | 连续地址传输 | 最优 |
| **分散-聚集** | 多段地址传输 | 中 |
| **环形DMA** | 循环缓冲区 | 适合流式 |

**DMA调度算法**：

```c
// DMA调度器
typedef struct {
    dma_desc_t descriptors[RING_SIZE];
    uint32_t head;  // 硬件当前位置
    uint32_t tail;  // 软件提交位置
} dma_ring_t;

int submit_dma_request(dma_ring_t *ring, dma_request_t *req) {
    uint32_t next = (ring->tail + 1) % RING_SIZE;

    if (next == ring->head) {
        return -ENOMEM;  // 队列满
    }

    // 填充描述符
    ring->descriptors[ring->tail].src = req->src_addr;
    ring->descriptors[ring->tail].dst = req->dst_addr;
    ring->descriptors[ring->tail].len = req->length;
    ring->descriptors[ring->tail].flags = req->flags;

    ring->tail = next;

    // 通知硬件
    write_doorbell(ring->tail);

    return 0;
}
```

### 3.2 描述符队列管理

**队列深度优化**：

| 队列深度 | 延迟 | 吞吐量 | CPU开销 |
|---------|------|--------|---------|
| 64 | 低 | 中 | 高 |
| 256 | 中 | 高 | 中 |
| 1024 | 高 | 最高 | 低 |

**批量提交**：

```c
// 批量提交优化
void submit_batch(dma_ring_t *ring, dma_request_t *reqs, int count) {
    for (int i = 0; i < count; i++) {
        fill_descriptor(&ring->descriptors[ring->tail], &reqs[i]);
        ring->tail = (ring->tail + 1) % RING_SIZE;
    }

    // 只写一次doorbell
    write_doorbell(ring->tail);
}
```

### 3.3 带宽分配

**加权公平队列（WFQ）**：

```c
// WFQ调度
typedef struct {
    uint64_t weight;
    uint64_t deficit;
    dma_queue_t queue;
} wfq_queue_t;

dma_request_t *wfq_schedule(wfq_queue_t *queues, int n) {
    static int last = 0;

    for (int i = 0; i < n; i++) {
        int idx = (last + i) % n;

        if (!dma_queue_empty(&queues[idx].queue)) {
            dma_request_t *req = dma_dequeue(&queues[idx].queue);

            if (req->length <= queues[idx].deficit) {
                queues[idx].deficit -= req->length;
                last = idx;
                return req;
            } else {
                queues[idx].deficit += queues[idx].weight;
                dma_enqueue_front(&queues[idx].queue, req);
            }
        }
    }

    return NULL;
}
```

## 4 虚拟通道调度

### 4.1 VC仲裁

**虚拟通道（VC）**：

| VC | 用途 | 优先级 |
|----|------|--------|
| VC0 | 默认流量 | 低 |
| VC1 | 高优先级 | 高 |
| VC2 | 等时流量 | 最高 |
| VC3-7 | 预留 | - |

**轮询仲裁**：

```c
// 轮询仲裁器
typedef struct {
    tlp_queue_t vc_queues[8];
    int last_vc;
} vc_arbiter_t;

tlp_t *round_robin_arbitrate(vc_arbiter_t *arb) {
    for (int i = 0; i < 8; i++) {
        int vc = (arb->last_vc + i) % 8;

        if (!tlp_queue_empty(&arb->vc_queues[vc])) {
            arb->last_vc = vc;
            return tlp_dequeue(&arb->vc_queues[vc]);
        }
    }
    return NULL;
}
```

**加权轮询仲裁（WRR）**：

```c
// WRR仲裁
int vc_weights[8] = {1, 2, 4, 1, 1, 1, 1, 1};  // VC2权重最高

tlp_t *wrr_arbitrate(vc_arbiter_t *arb) {
    static int credits[8] = {0};

    for (int i = 0; i < 8; i++) {
        int vc = (arb->last_vc + i) % 8;

        if (credits[vc] > 0 && !tlp_queue_empty(&arb->vc_queues[vc])) {
            credits[vc]--;
            arb->last_vc = vc;
            return tlp_dequeue(&arb->vc_queues[vc]);
        }
    }

    // 重置信用
    for (int i = 0; i < 8; i++) {
        credits[i] = vc_weights[i];
    }

    return NULL;
}
```

### 4.2 端口仲裁

**端口仲裁算法**：

| 算法 | 描述 | 公平性 |
|------|------|--------|
| **Fixed** | 固定优先级 | 差 |
| **WRR** | 加权轮询 | 好 |

### 4.3 流量控制

**Credit-Based流控**：

```
发送方维护每个VC的Credit计数
Credit > 0时才能发送TLP
收到UpdateFC DLLP后增加Credit
```

```c
// Credit管理
typedef struct {
    int credit_limit;
    int credit_consumed;
} vc_credit_t;

bool can_transmit(vc_credit_t *credit, int tlp_size) {
    return (credit->credit_limit - credit->credit_consumed) >= tlp_size;
}

void update_credit(vc_credit_t *credit, int new_limit) {
    credit->credit_limit = new_limit;
}
```

### 5.2 带宽保证

**最小带宽保证**：

```c
// 带宽保证调度器
typedef struct {
    uint64_t min_bandwidth;   // 最小带宽（MB/s）
    uint64_t max_bandwidth;   // 最大带宽（MB/s）
    uint64_t current_usage;   // 当前使用量
    uint64_t period_start;    // 周期开始时间
} bandwidth_guarantee_t;

bool check_bandwidth(bandwidth_guarantee_t *bg, size_t size) {
    uint64_t now = get_time();
    uint64_t elapsed = now - bg->period_start;

    if (elapsed >= BANDWIDTH_PERIOD) {
        // 新周期，重置计数
        bg->current_usage = 0;
        bg->period_start = now;
    }

    uint64_t projected = bg->current_usage + size;
    uint64_t max_allowed = bg->max_bandwidth * elapsed / 1000000;

    return projected <= max_allowed;
}
```

## 6 形式化定义

### 6.1 TLP调度模型

**定义5.1（TLP调度问题）**：

给定：

- TLP集合$P = \{p_1, p_2, \ldots, p_n\}$
- VC集合$V = \{v_0, v_1, \ldots, v_7\}$
- 优先级函数$\pi: P \rightarrow \mathbb{N}$
- 截止时间$d: P \rightarrow \mathbb{R}^+$

寻找调度$S: P \rightarrow V \times \mathbb{N}$，使得：

1. **优先级约束**：高优先级TLP优先调度
2. **截止时间约束**：$\forall p \in P, S(p).time \leq d(p)$
3. **VC容量约束**：每个VC同时只能传输一个TLP

### 6.2 资源约束

**带宽约束**：

$$
\sum_{p \in P_v} \frac{\text{size}(p)}{\text{period}} \leq B_v
$$

其中$P_v$是分配给VC $v$的TLP集合，$B_v$是VC带宽。

## 7 Linux内核实现分析

### 7.1 PCIe子系统

**设备枚举**：

```c
// PCIe设备扫描
int pci_scan_slot(struct pci_bus *bus, int devfn)
{
    struct pci_dev *dev;

    dev = pci_alloc_dev(bus);
    dev->devfn = devfn;

    // 读取配置空间
    pci_read_config_word(dev, PCI_VENDOR_ID, &dev->vendor);
    pci_read_config_word(dev, PCI_DEVICE_ID, &dev->device);

    // 配置BAR
    pci_setup_device(dev);

    // 添加到设备树
    pci_device_add(dev, bus);

    return 0;
}
```

### 7.2 SR-IOV驱动

**VF管理**：

```c
// 启用SR-IOV
int pci_enable_sriov(struct pci_dev *dev, int nr_virtfn)
{
    // 分配VF资源
    dev->sriov = kzalloc(sizeof(*dev->sriov), GFP_KERNEL);
    dev->sriov->total = nr_virtfn;
    dev->sriov->nr_virtfn = nr_virtfn;

    // 初始化每个VF
    for (i = 0; i < nr_virtfn; i++) {
        virtfn = pci_alloc_dev(dev->bus);
        virtfn->is_virtfn = 1;
        virtfn->physfn = pci_dev_get(dev);

        // 配置VF
        pci_setup_vf(dev, virtfn, i);
    }

    return 0;
}
```
