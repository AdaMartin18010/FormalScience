## 1 片上网络（NoC）架构

### 1.1 NoC拓扑

**常见拓扑结构**：

| 拓扑 | 延迟 | 带宽 | 复杂度 | 应用 |
|------|------|------|--------|------|
| **Mesh** | O(√n) | 中 | 低 | 通用处理器 |
| **Torus** | O(√n) | 高 | 中 | HPC |
| **Tree** | O(log n) | 低 | 低 | 多核SoC |
| **Fat Tree** | O(log n) | 高 | 中 | 数据中心 |
| **Dragonfly** | O(1) | 很高 | 高 | 超级计算机 |

**2D Mesh示例**：

```
00──01──02──03
│   │   │   │
04──05──06──07
│   │   │   │
08──09──10──11
│   │   │   │
12──13──14──15
```

### 1.2 路由器结构

**5端口路由器**：

```
         [N]
          │
[W]──[Router]──[E]
          │
         [S]
          │
         [Local]
```

**路由器组件**：

| 组件 | 功能 |
|------|------|
| **输入缓冲区** | 存储到达的flit |
| **路由计算** | 确定输出端口 |
| **交换机分配** | 分配交叉开关 |
| **输出缓冲区** | 存储待发flit |

## 2 路由算法

### 2.1 确定性路由

**XY路由（2D Mesh）**：

```c
// XY路由算法
void xy_route(int src_x, int src_y, int dst_x, int dst_y,
              int *next_port) {
    int x = src_x;
    int y = src_y;

    if (x < dst_x) {
        *next_port = EAST;
    } else if (x > dst_x) {
        *next_port = WEST;
    } else if (y < dst_y) {
        *next_port = NORTH;
    } else if (y > dst_y) {
        *next_port = SOUTH;
    } else {
        *next_port = LOCAL;
    }
}
```

### 2.2 自适应路由

**最小自适应路由**：

```c
// 自适应路由
void adaptive_route(router_t *r, flit_t *f, int *next_port) {
    int dst_x = f->dst_x;
    int dst_y = f->dst_y;

    int x_diff = dst_x - r->x;
    int y_diff = dst_y - r->y;

    // 评估可用路径的拥塞
    int best_port = -1;
    int min_congestion = INT_MAX;

    if (x_diff > 0 && !congested(r, EAST)) {
        if (get_congestion(r, EAST) < min_congestion) {
            min_congestion = get_congestion(r, EAST);
            best_port = EAST;
        }
    }
    if (x_diff < 0 && !congested(r, WEST)) {
        if (get_congestion(r, WEST) < min_congestion) {
            best_port = WEST;
        }
    }
    // 类似处理Y方向

    *next_port = best_port;
}
```

### 2.3 无死锁路由

**虚拟通道分离**：

```
VC0: X方向路由
VC1: Y方向路由
```

**转弯模型**：

| 模型 | 禁止转弯 | 适应性 |
|------|---------|--------|
| **XY** | Y→X | 无 |
| **West-First** | N→W, S→W, E→W | 部分 |
| **North-Last** | N→E, N→W, N→S | 部分 |
| **Odd-Even** | 根据坐标奇偶 | 完全 |

## 3 流量控制

### 3.1 存储转发

**完整包接收后再转发**：

```
延迟 = 包大小 × 跳数
```

### 3.2 虚拟通道

**多虚拟通道减少阻塞**：

```c
// 虚拟通道分配
int allocate_vc(router_t *r, int port, flit_t *f) {
    for (int vc = 0; vc < NUM_VCS; vc++) {
        if (r->vc_state[port][vc] == IDLE) {
            r->vc_state[port][vc] = ACTIVE;
            return vc;
        }
    }
    return -1;  // 无可用VC
}
```

### 3.3 信令流量控制

```

```

## 4 交换机调度

### 4.1 输入队列调度

**输入队列路由器**：

```c
// 输入队列调度
typedef struct {
    flit_queue_t queues[NUM_PORTS][NUM_VCS];
} input_buffer_t;

void input_queue_schedule(router_t *r) {
    for (int in_port = 0; in_port < NUM_PORTS; in_port++) {
        for (int vc = 0; vc < NUM_VCS; vc++) {
            if (!queue_empty(&r->input_buffer[in_port][vc])) {
                flit_t *f = queue_peek(&r->input_buffer[in_port][vc]);
                int out_port = route_compute(f);

                // 请求交叉开关
                request_switch(r, in_port, out_port, vc);
            }
        }
    }
}
```

### 4.2 输出队列调度

**输出队列避免队头阻塞**：

| 队列类型 | 优点 | 缺点 |
|---------|------|------|
| 输入队列 | 简单 | 队头阻塞 |
| 输出队列 | 无阻塞 | 速度快要求高 |
| 组合队列 | 平衡 | 复杂 |

### 4.3 交叉开关调度

**iSLIP算法**：

```c
// iSLIP调度算法
void islip_schedule(router_t *r) {
    int grant[NUM_PORTS] = {-1};
    int accept[NUM_PORTS] = {-1};

    // 第1轮：请求
    for (int in = 0; in < NUM_PORTS; in++) {
        for (int out = 0; out < NUM_PORTS; out++) {
            if (r->request[in][out]) {
                // 发送请求
            }
        }
    }

    // 第2轮：授权
    for (int out = 0; out < NUM_PORTS; out++) {
        // 轮询选择请求者
        grant[out] = select_requester(r, out);
    }

    // 第3轮：接受
    for (int in = 0; in < NUM_PORTS; in++) {
        if (grant[in] >= 0) {
            accept[in] = grant[in];
        }
    }
}
```

## 5 形式化定义

**NoC路由问题**：

给定拓扑图$G = (V, E)$，源节点$s$，目的节点$d$，寻找路径$P = (v_0, v_1, \ldots, v_k)$，其中$v_0 = s, v_k = d$，最小化：

$$
\text{cost}(P) = \sum_{i=0}^{k-1} \left( delay(v_i, v_{i+1}) + congestion(v_{i+1}) \right)
$$
