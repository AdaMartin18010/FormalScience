## 1 内存总线架构

### 1.1 共享总线模型

**传统共享总线**：

```
CPU0 ──┐
CPU1 ──┼──[仲裁器]──[内存总线]──[内存]
CPU2 ──┘
```

**仲裁需求**：

- 多主设备竞争总线访问权
- 需要公平、高效的仲裁机制
- 保证实时性要求

### 1.2 交叉开关架构

**现代交叉开关**：

```
       ┌───┬───┬───┐
CPU0 ──┤   │   │   ├── 内存通道0
CPU1 ──┼───┼───┼───┼── 内存通道1
CPU2 ──┤   │   │   ├── 内存通道2
CPU3 ──┴───┴───┴───┘    内存通道3
```

## 2 仲裁算法

### 2.1 固定优先级仲裁

**实现简单，但可能导致饥饿**：

```c
// 固定优先级仲裁
int fixed_priority_arbitrate(request_t **requests, int n, int *priorities) {
    int highest = -1;
    int max_priority = -1;

    for (int i = 0; i < n; i++) {
        if (requests[i] && priorities[i] > max_priority) {
            max_priority = priorities[i];
            highest = i;
        }
    }

    return highest;
}
```

### 2.2 轮询仲裁

**公平但延迟高**：

```c
// 轮询仲裁器
typedef struct {
    int last_grant;
} round_robin_arbiter_t;

int round_robin_arbitrate(round_robin_arbiter_t *arb,
                          request_t **requests, int n) {
    for (int i = 1; i <= n; i++) {
        int idx = (arb->last_grant + i) % n;
        if (requests[idx]) {
            arb->last_grant = idx;
            return idx;
        }
    }
    return -1;
}
```

### 2.3 加权轮询仲裁

**根据权重分配带宽**：

```c
// 加权轮询仲裁
typedef struct {
    int weight;
    int credit;
} weighted_request_t;

int weighted_round_robin(weighted_request_t *requests, int n) {
    for (int i = 0; i < n; i++) {
        if (requests[i].credit > 0 && requests[i].active) {
            requests[i].credit--;
            return i;
        }
    }

    // 重置信用
    for (int i = 0; i < n; i++) {
        requests[i].credit = requests[i].weight;
    }

    return -1;
}
```

### 2.4 公平仲裁算法

**deficit round robin (DRR)**：

```c
// DRR仲裁器
typedef struct {
    int quantum;      // 量子大小
    int deficit;      // 赤字计数器
    request_queue_t queue;
} drr_class_t;

int drr_arbitrate(drr_class_t *classes, int n) {
    for (int i = 0; i < n; i++) {
        if (!queue_empty(&classes[i].queue)) {
            request_t *req = queue_peek(&classes[i].queue);

            if (req->size <= classes[i].deficit) {
                classes[i].deficit -= req->size;
                return i;
            }
        }
    }

    // 增加赤字
    for (int i = 0; i < n; i++) {
        classes[i].deficit += classes[i].quantum;
    }

    return -1;
}
```

## 3 优先级调度

### 3.1 动态优先级

**基于等待时间调整优先级**：

```
priority = base_priority + waiting_time / time_slice
```

### 3.2 优先级继承

**避免优先级反转**：

```
低优先级任务持有锁 → 高优先级任务等待
→ 提升低优先级任务优先级到高优先级
```

### 3.3 老化机制

**防止饥饿**：

```c
// 老化机制
void aging_update(task_t *tasks, int n) {
    for (int i = 0; i < n; i++) {
        if (!tasks[i].granted) {
            tasks[i].priority += AGING_INCREMENT;
        } else {
            tasks[i].priority = tasks[i].base_priority;
        }
    }
}
```

## 5 形式化定义

**定义（仲裁问题）**：

给定请求集合$R = \{r_1, \ldots, r_n\}$，寻找调度$S: \mathbb{N} \rightarrow R$，满足：

1. **互斥**：同一时刻只有一个请求被服务
2. **公平性**：$\forall r_i, r_j: \lim_{t\to\infty} \frac{services(r_i)}{services(r_j)} = \frac{w_i}{w_j}$
3. **无饥饿**：每个请求最终会被服务

## 6 性能数据

**不同仲裁算法对比**：

| 算法 | 公平性 | 延迟 | 复杂度 | 适用场景 |
|------|--------|------|--------|----------|
| 固定优先级 | 差 | 低 | O(1) | 实时系统 |
| 轮询 | 优 | 中 | O(n) | 通用系统 |
| 加权轮询 | 可调 | 中 | O(n) | QoS系统 |
| DRR | 优 | 高 | O(n) | 网络系统 |
