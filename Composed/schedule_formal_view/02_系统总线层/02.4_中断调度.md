## 1 中断架构

### 1.1 APIC架构

**本地APIC（LAPIC）**：

```
每个CPU核心一个LAPIC
- IRR：中断请求寄存器
- ISR：服务中寄存器
- TMR：触发模式寄存器
- EOI：中断结束寄存器
```

**I/O APIC**：

```
系统级中断控制器
- 24个中断输入
- 可编程中断路由
- 支持多核分发
```

### 1.2 中断传递机制

**中断类型对比**：

| 类型 | 延迟 | 可扩展性 | 适用场景 |
|------|------|---------|----------|
| **INTx** | 5μs | 差（共享） | 遗留设备 |
| **MSI** | 2μs | 中（32向量） | 现代设备 |
| **MSI-X** | 1μs | 优（2048向量） | 高性能设备 |

## 2 MSI/X机制

### 2.1 MSI优势

**消息信号中断**：

```
传统INTx：物理引脚 → 电平触发 → APIC
MSI：内存写操作 → 直接写入APIC寄存器
```

**优势**：

- 无需共享引脚
- 支持更多中断向量
- 延迟更低

### 2.2 MSI-X扩展

**MSI-X特性**：

```c
// MSI-X配置
#define MSI_X_TABLE_SIZE  2048
#define MSI_X_TABLE_BIR   0  // BAR 0
#define MSI_X_PBA_BIR     0  // BAR 0

typedef struct {
    uint32_t msg_addr_lo;
    uint32_t msg_addr_hi;
    uint32_t msg_data;
    uint32_t vector_ctrl;
} msix_table_entry_t;
```

**每个向量独立配置**：

| 字段 | 描述 |
|------|------|
| Msg Address | 目标APIC地址 |
| Msg Data | 中断向量号 |
| Vector Control | 屏蔽位 |

### 2.3 中断向量管理

**向量分配**：

```c
// 分配MSI-X向量
int pci_alloc_irq_vectors(struct pci_dev *dev, int min_vecs, int max_vecs,
                          unsigned int flags) {
    int nvec = max_vecs;

    // 尝试MSI-X
    if (flags & PCI_IRQ_MSIX) {
        nvec = pci_enable_msix_range(dev, NULL, min_vecs, max_vecs);
        if (nvec > 0) return nvec;
    }

    // 回退到MSI
    if (flags & PCI_IRQ_MSI) {
        nvec = pci_enable_msi_range(dev, min_vecs, max_vecs);
        if (nvec > 0) return nvec;
    }

    // 回退到INTx
    return 1;
}
```

## 3 中断亲和性

### 3.1 亲和性设置

**设置中断CPU亲和性**：

```bash
# 查看中断亲和性
cat /proc/interrupts

# 设置亲和性（绑定到CPU 2）
echo 4 > /proc/irq/24/smp_affinity

# 使用smp_affinity_list
echo 2 > /proc/irq/24/smp_affinity_list
```

**内核实现**：

```c
// 中断亲和性设置
int irq_set_affinity(unsigned int irq, const struct cpumask *cpumask) {
    struct irq_desc *desc = irq_to_desc(irq);

    if (!desc) return -EINVAL;

    // 更新亲和性掩码
    cpumask_copy(desc->irq_common_data.affinity, cpumask);

    // 重新分发中断
    irq_set_affinity_deactivated(desc, cpumask);

    return 0;
}
```

### 3.2 负载均衡策略

**中断分布算法**：

```c
// 自动中断负载均衡
void irq_balance(struct irq_desc *desc) {
    int min_cpu = -1;
    int min_load = INT_MAX;

    // 找到负载最轻的CPU
    for_each_online_cpu(cpu) {
        int load = per_cpu(irq_load, cpu);
        if (load < min_load) {
            min_load = load;
            min_cpu = cpu;
        }
    }

    if (min_cpu >= 0) {
        cpumask_t mask;
        cpumask_clear(&mask);
        cpumask_set_cpu(min_cpu, &mask);
        irq_set_affinity(desc->irq_data.irq, &mask);
    }
}
```

## 4 中断负载均衡

### 4.1 静态分配

**手工配置**：

```bash
# 网卡中断绑定
#!/bin/bash
IRQs=$(cat /proc/interrupts | grep eth0 | awk '{print $1}' | tr -d ':')
CPU=0
for IRQ in $IRQs; do
    echo $((1 << CPU)) > /proc/irq/$IRQ/smp_affinity
    CPU=$((CPU + 1))
done
```

### 4.2 动态均衡

**irqbalance守护进程**：

```
监控中断频率
自动调整中断亲和性
支持NUMA感知
```

### 4.3 自适应算法

**基于中断频率的调整**：

```c
// 自适应中断均衡
void adaptive_irq_balance(void) {
    for_each_irq(irq) {
        struct irq_stat *stat = &irq_stats[irq];

        // 计算中断频率
        uint64_t rate = stat->count - stat->last_count;
        stat->last_count = stat->count;

        if (rate > HIGH_THRESHOLD) {
            // 高频率中断，需要负载均衡
            spread_irq_to_multiple_cpus(irq);
        } else if (rate < LOW_THRESHOLD) {
            // 低频率中断，集中到单个CPU
            consolidate_irq_to_one_cpu(irq);
        }
    }
}
```

## 5 虚拟化中断

### 5.1 中断注入

**VM中断注入**：

```c
// KVM中断注入
int kvm_set_irq(struct kvm_kernel_irq_routing_entry *e,
                struct kvm *kvm, int irq_source_id, int level) {
    struct kvm_vcpu *vcpu = e->vcpu;

    // 设置中断状态
    kvm_vcpu_ioctl(vcpu, KVM_INTERRUPT, &e->irq);

    // 唤醒vCPU
    kvm_vcpu_kick(vcpu);

    return 0;
}
```

### 5.2 Posted中断

**硬件辅助虚拟化**：

```
传统：设备中断 → Host → 注入 → Guest
Posted：设备中断 → Posted Interrupt Descriptor → 直接唤醒Guest vCPU
```

**优势**：延迟从10μs降至1μs

## 6 形式化定义

**中断调度问题**：

给定：

- 中断源集合$I = \{i_1, \ldots, i_n\}$
- CPU集合$C = \{c_1, \ldots, c_m\}$
- 中断频率$f: I \rightarrow \mathbb{R}^+$
- 处理时间$t: I \rightarrow \mathbb{R}^+$

寻找映射$S: I \rightarrow C$，最小化：

$$
\max_{c \in C} \sum_{i: S(i)=c} f(i) \cdot t(i)
$$

约束：$\forall i \in I: S(i) \in allowed(i)$
