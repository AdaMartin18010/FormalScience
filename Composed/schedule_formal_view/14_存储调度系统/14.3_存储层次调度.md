## 1 存储层次调度概述

### 1.1 存储层次结构

**完整存储层次**：

```
层次          延迟          容量           带宽           技术
─────────────────────────────────────────────────────────────────────
寄存器        0.3 ns        <1KB          10 TB/s        SRAM
  ↓
L1缓存        1 ns          32-64KB       1 TB/s         SRAM
  ↓
L2缓存        3-4 ns        256-512KB     500 GB/s       SRAM
  ↓
L3缓存        10-20 ns      8-32MB        200 GB/s       SRAM
  ↓
内存 (DRAM)   80-100 ns     8-512GB       50 GB/s        DRAM
  ↓
持久内存      200-300 ns    128GB-1TB     10 GB/s        3D XPoint
  ↓
NVMe SSD      10-100 μs     1TB-8TB       3-7 GB/s       NAND
  ↓
SATA SSD      100-500 μs    1TB-4TB       500 MB/s       NAND
  ↓
HDD           5-10 ms       10-20TB       200 MB/s       磁性
  ↓
网络存储      1-50 ms       PB级          1-100 Gbps     网络
  ↓
云存储        10-200 ms     EB级          1-10 Gbps      互联网

总跨度: 延迟差异 10^12 倍 (0.3ns → 200ms)
```

**存储层次特征**：

| 特性 | 上层（缓存） | 下层（存储） |
|------|-----------|------------|
| 速度 | 快 | 慢 |
| 容量 | 小 | 大 |
| 成本/GB | 高 | 低 |
| 易失性 | 易失 | 非易失 |
| 管理 | 硬件 | 软件 |

### 1.2 存储层次调度的核心挑战

**四大挑战**：

```
1. 命中率优化
   目标: 最大化缓存命中率
   方法: 智能替换算法、预取

2. 延迟隐藏
   目标: 掩盖下层访问延迟
   方法: 预取、多线程、异步IO

3. 一致性保证
   目标: 多级缓存数据一致
   方法: MESI/MOESI协议

4. 带宽优化
   目标: 最大化数据传输效率
   方法: 批量传输、流水化
```

### 1.3 局部性原理

**时间局部性**：

```
概念: 被访问的数据很可能在不久的将来再次被访问

示例:
for (int i = 0; i < n; i++) {
    sum += arr[i];  // arr[i] 在短时间内多次访问
}

利用: 缓存保留最近访问的数据
```

**空间局部性**：

```
概念: 被访问数据附近的数据很可能被访问

示例:
for (int i = 0; i < n; i++) {
    sum += arr[i];  // 访问arr[0]后，arr[1], arr[2]...很可能被访问
}

利用: 预取相邻数据到缓存
```

### 2.2 缓存一致性协议

**MESI协议**：

| 状态 | 描述 | 允许操作 |
|------|------|---------|
| **M** (Modified) | 已修改，独占 | 读写，需写回 |
| **E** (Exclusive) | 独占，未修改 | 读写 |
| **S** (Shared) | 共享，只读 | 读 |
| **I** (Invalid) | 无效 | 无 |

**状态转换**：

```
                读命中                写命中
              ┌─────┐              ┌─────┐
              ↓     │              ↓     │
    ┌───┐   ┌───┐   │    ┌───┐   ┌───┐   │
    │ I │ ← │ S │ ──┘    │ I │ ← │ E │ ──┘
    └─┬─┘   └───┘        └─┬─┘   └───┘
      │     ↑              │     ↑
      │     │              │     │
      └─────┘              └─────┘
      其他CPU读             其他CPU读

    ┌───┐                  ┌───┐
    │ M │ ──写回──→ I      │ M │ ←──本地写
    └───┘                  └───┘
```

**伪代码实现**：

```python
class MESICache:
    """
    MESI缓存一致性协议实现
    """

    def __init__(self, cache_id):
        self.cache_id = cache_id
        self.cache_lines = {}  # tag -> (data, state)
        self.bus = None  # 总线接口

    def read(self, address):
        tag = self.get_tag(address)

        if tag in self.cache_lines:
            data, state = self.cache_lines[tag]
            if state != 'I':  # 有效状态
                return data  # 命中

        # 未命中，发送总线请求
        return self.handle_read_miss(address, tag)

    def handle_read_miss(self, address, tag):
        # 检查其他缓存
        responses = self.bus.snoop_read(address, self.cache_id)

        if 'M' in responses:  # 其他缓存有修改版本
            data = responses['M']
            self.cache_lines[tag] = (data, 'S')
            # 写回内存，状态变为S
            self.bus.writeback(address, data)
        elif 'E' in responses or 'S' in responses:
            data = responses.get('E') or responses.get('S')
            self.cache_lines[tag] = (data, 'S')
        else:  # 内存读取
            data = self.bus.read_memory(address)
            self.cache_lines[tag] = (data, 'E')

        return data

    def write(self, address, data):
        tag = self.get_tag(address)

        if tag in self.cache_lines:
            _, state = self.cache_lines[tag]
            if state in ('M', 'E'):
                # 独占状态，直接写入
                self.cache_lines[tag] = (data, 'M')
                return
            elif state == 'S':
                # 共享状态，需要使其他缓存失效
                self.bus.invalidate(address, self.cache_id)
                self.cache_lines[tag] = (data, 'M')
                return

        # 未命中
        self.handle_write_miss(address, tag, data)

    def handle_write_miss(self, address, tag, data):
        # 使其他缓存失效
        self.bus.invalidate(address, self.cache_id)
        # 写入
        self.cache_lines[tag] = (data, 'M')
```

### 2.3 缓存替换算法

**LRU（Least Recently Used）**：

```python
class LRUCache:
    """
    LRU缓存实现

    使用哈希表 + 双向链表实现O(1)操作
    """

    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = {}  # key -> Node
        self.head = Node(0, 0)  # 哑头节点
        self.tail = Node(0, 0)  # 哑尾节点
        self.head.next = self.tail
        self.tail.prev = self.head

    def get(self, key):
        if key in self.cache:
            node = self.cache[key]
            self._remove(node)
            self._add(node)
            return node.value
        return None

    def put(self, key, value):
        if key in self.cache:
            self._remove(self.cache[key])

        node = Node(key, value)
        self._add(node)
        self.cache[key] = node

        if len(self.cache) > self.capacity:
            # 移除最少使用的（tail.prev）
            lru = self.tail.prev
            self._remove(lru)
            del self.cache[lru.key]

    def _add(self, node):
        """添加到头部（最近使用）"""
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node

    def _remove(self, node):
        """从链表中移除"""
        node.prev.next = node.next
        node.next.prev = node.prev
```

**LFU（Least Frequently Used）**：

```python
class LFUCache:
    """
    LFU缓存实现

    按访问频率组织数据
    """

    def __init__(self, capacity):
        self.capacity = capacity
        self.min_freq = 0
        self.key_table = {}  # key -> (value, freq)
        self.freq_table = {}  # freq -> OrderedDict(keys)

    def get(self, key):
        if key not in self.key_table:
            return None

        value, freq = self.key_table[key]
        self._increase_freq(key, freq)
        return value

    def put(self, key, value):
        if self.capacity <= 0:
            return

        if key in self.key_table:
            _, freq = self.key_table[key]
            self.key_table[key] = (value, freq)
            self._increase_freq(key, freq)
        else:
            if len(self.key_table) >= self.capacity:
                # 移除最小频率的
                self._evict_lfu()

            self.key_table[key] = (value, 1)
            self.freq_table.setdefault(1, OrderedDict())[key] = None
            self.min_freq = 1

    def _increase_freq(self, key, freq):
        """增加访问频率"""
        del self.freq_table[freq][key]
        if not self.freq_table[freq]:
            del self.freq_table[freq]
            if freq == self.min_freq:
                self.min_freq += 1

        self.key_table[key] = (self.key_table[key][0], freq + 1)
        self.freq_table.setdefault(freq + 1, OrderedDict())[key] = None

    def _evict_lfu(self):
        """驱逐最少使用的"""
        key, _ = self.freq_table[self.min_freq].popitem(last=False)
        del self.key_table[key]
```

**CLOCK算法（近似LRU）**：

```python
class ClockCache:
    """
    CLOCK缓存替换算法

    使用引用位近似LRU，开销更小
    """

    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = {}  # key -> (value, ref_bit)
        self.clock_hand = 0
        self.keys = []

    def get(self, key):
        if key in self.cache:
            value, _ = self.cache[key]
            self.cache[key] = (value, 1)  # 设置引用位
            return value
        return None

    def put(self, key, value):
        if key in self.cache:
            self.cache[key] = (value, 1)
            return

        if len(self.cache) >= self.capacity:
            self._evict()

        self.cache[key] = (value, 1)
        self.keys.append(key)

    def _evict(self):
        """CLOCK驱逐"""
        while True:
            key = self.keys[self.clock_hand]
            _, ref_bit = self.cache[key]

            if ref_bit == 0:
                # 驱逐
                del self.cache[key]
                self.keys[self.clock_hand] = None
                return
            else:
                # 清除引用位，继续扫描
                self.cache[key] = (self.cache[key][0], 0)
                self.clock_hand = (self.clock_hand + 1) % len(self.keys)
```

**替换算法对比**：

| 算法 | 命中率 | 复杂度 | 硬件支持 | 适用场景 |
|------|-------|-------|---------|---------|
| LRU | 高 | O(1) | 需要时间戳 | 通用 |
| LFU | 高 | O(1) | 需要计数器 | 频率敏感 |
| FIFO | 低 | O(1) | 简单 | 资源受限 |
| CLOCK | 中 | O(n) | 简单 | 硬件实现 |
| Random | 低 | O(1) | 无 | 简单场景 |

## 3 预取策略

### 3.1 顺序预取

**顺序预取算法**：

```python
class SequentialPrefetcher:
    """
    顺序预取器

    检测到顺序访问模式后，预取后续数据
    """

    def __init__(self, prefetch_degree=4):
        self.prefetch_degree = prefetch_degree  # 预取深度
        self.streams = {}  # 活跃的顺序流

    def on_access(self, address):
        """处理访问请求"""
        # 查找匹配的流
        for stream_id, stream in list(self.streams.items()):
            if stream.is_next(address):
                # 命中现有流
                stream.advance()
                self.trigger_prefetch(stream)
                return

        # 创建新流
        new_stream = AccessStream(address)
        self.streams[id(new_stream)] = new_stream

    def trigger_prefetch(self, stream):
        """触发预取"""
        for i in range(1, self.prefetch_degree + 1):
            prefetch_addr = stream.current + i * stream.stride
            if not self.in_cache(prefetch_addr):
                self.issue_prefetch(prefetch_addr)

    def is_next(self, address):
        """检查是否是流的下一个地址"""
        return address == self.current + self.stride

class AccessStream:
    def __init__(self, start_addr):
        self.start = start_addr
        self.current = start_addr
        self.stride = 64  # 假设64字节缓存行
        self.confidence = 1

    def advance(self):
        self.current += self.stride
        self.confidence = min(self.confidence + 1, 10)
```

### 3.2 步长预取

**步长预取算法**：

```python
class StridePrefetcher:
    """
    步长预取器

    检测固定步长的访问模式
    """

    def __init__(self):
        self.history = {}  # PC -> 上次访问地址
        self.patterns = {}  # PC -> (stride, confidence)

    def on_access(self, pc, address):
        """处理访问请求"""
        if pc in self.history:
            last_addr = self.history[pc]
            stride = address - last_addr

            # 更新模式
            if pc in self.patterns:
                old_stride, confidence = self.patterns[pc]
                if stride == old_stride:
                    # 步长一致，增加置信度
                    self.patterns[pc] = (stride, min(confidence + 1, 10))
                    if confidence >= 3:
                        self.prefetch(address + stride, pc)
                else:
                    # 步长变化，降低置信度
                    self.patterns[pc] = (stride, confidence - 1)
            else:
                self.patterns[pc] = (stride, 1)

        self.history[pc] = address

    def prefetch(self, address, pc):
        """发出预取请求"""
        if not self.in_cache(address):
            self.issue_prefetch(address, priority='LOW')
```

### 3.3 自适应预取

**自适应预取**：

```python
class AdaptivePrefetcher:
    """
    自适应预取器

    根据预取准确性动态调整策略
    """

    def __init__(self):
        self.prefetchers = [
            SequentialPrefetcher(),
            StridePrefetcher(),
            RegionPrefetcher()
        ]
        self.prefetch_stats = {'issued': 0, 'useful': 0, 'pollution': 0}
        self.aggressiveness = 0.5  # 预取激进程度

    def on_access(self, address, pc):
        """处理访问"""
        # 检查是否是预取命中
        if self.is_prefetched(address):
            self.prefetch_stats['useful'] += 1

        # 运行所有预取器
        for pf in self.prefetchers:
            pf.on_access(address, pc)

        # 调整预取激进程度
        self.adjust_aggressiveness()

    def adjust_aggressiveness(self):
        """根据预取效果调整策略"""
        total = self.prefetch_stats['issued']
        useful = self.prefetch_stats['useful']

        if total > 100:  # 有足够样本
            accuracy = useful / total

            if accuracy > 0.8:
                # 准确率高，增加预取
                self.aggressiveness = min(self.aggressiveness * 1.1, 1.0)
            elif accuracy < 0.5:
                # 准确率低，减少预取
                self.aggressiveness = max(self.aggressiveness * 0.9, 0.1)
```

**文件系统调度策略**：

```python
class VFSScheduler:
    """
    VFS层调度器

    管理文件读写请求的调度
    """

    def __init__(self):
        self.page_cache = PageCache()
        self.writeback_queue = WritebackQueue()
        self.readahead_window = 128  # KB

    def read(self, file, offset, size):
        """调度读请求"""
        pages = []

        for page_offset in range(offset, offset + size, PAGE_SIZE):
            # 1. 检查Page Cache
            page = self.page_cache.get(file, page_offset)
            if page:
                pages.append(page)
                continue

            # 2. 触发预取
            if self.should_readahead(file, page_offset):
                self.trigger_readahead(file, page_offset)

            # 3. 从设备读取
            page = self.read_from_device(file, page_offset)
            self.page_cache.put(file, page_offset, page)
            pages.append(page)

        return pages

    def write(self, file, offset, data):
        """调度写请求"""
        for page_offset in range(offset, offset + len(data), PAGE_SIZE):
            # 写入Page Cache
            self.page_cache.put(file, page_offset, data)

            # 标记为脏页
            self.page_cache.set_dirty(file, page_offset)

            # 加入回写队列
            self.writeback_queue.enqueue(file, page_offset)

    def should_readahead(self, file, offset):
        """判断是否应该预取"""
        # 检查访问模式
        if file.sequential_score > 0.8:
            return True

        # 检查预取命中率
        if file.readahead_hit_rate > 0.7:
            return True

        return False
```

### 4.2 块分配策略

**文件系统块分配**：

```python
class BlockAllocator:
    """
    文件系统块分配器

    目标: 最大化局部性，最小化碎片
    """

    def __init__(self, fs):
        self.fs = fs
        self.free_blocks = FreeSpaceManager()

    def allocate_blocks(self, inode, num_blocks, allocation_strategy='extents'):
        """
        为文件分配块

        策略:
        - extents: 连续分配 (ext4/XFS)
        - bitmap: 位图分配 (传统)
        - delayed: 延迟分配 (XFS/Btrfs)
        """
        if allocation_strategy == 'extents':
            return self.allocate_extents(inode, num_blocks)
        elif allocation_strategy == 'delayed':
            return self.delayed_allocation(inode, num_blocks)
        else:
            return self.allocate_bitmap(inode, num_blocks)

    def allocate_extents(self, inode, num_blocks):
        """
        Extent分配

        优先分配连续块，形成extent
        """
        extents = []
        remaining = num_blocks

        # 尝试在文件末尾附近分配
        if inode.extents:
            last_extent = inode.extents[-1]
            start = self.find_contiguous_blocks(
                last_extent.start + last_extent.length,
                remaining
            )
            if start:
                size = min(remaining, self.get_contiguous_length(start))
                extents.append(Extent(start, size))
                remaining -= size

        # 分配剩余块
        while remaining > 0:
            start = self.find_best_extent(remaining)
            if not start:
                raise NoSpaceError()

            size = min(remaining, self.get_contiguous_length(start))
            extents.append(Extent(start, size))
            remaining -= size

        return extents

    def delayed_allocation(self, inode, num_blocks):
        """
        延迟分配

        不立即分配物理块，记录逻辑需求
        在写回时根据实际布局选择最佳位置
        """
        # 记录预留
        reservation = self.free_blocks.reserve(num_blocks)
        inode.delayed_allocation.append(reservation)
        return reservation

    def find_best_extent(self, min_size):
        """查找最佳extent位置"""
        # 优先选择: 大块连续空间 > 小块空间 > 分散空间
        candidates = self.free_blocks.find_extents(min_size)

        # 按大小排序，优先大块
        candidates.sort(key=lambda e: e.size, reverse=True)

        return candidates[0].start if candidates else None
```

### 4.3 日志与journaling

**日志模式**：

| 模式 | 描述 | 性能 | 安全性 |
|------|------|------|--------|
| writeback | 只记录元数据，不保证数据 | 最高 | 最低 |
| ordered | 元数据前将数据刷盘 | 中 | 中 |
| journal | 数据和元数据都记录 | 最低 | 最高 |

**日志调度**：

```python
class JournalScheduler:
    """
    文件系统日志调度器
    """

    def __init__(self, journal_dev):
        self.journal = Journal(journal_dev)
        self.transaction = None

    def start_transaction(self):
        """开始事务"""
        self.transaction = Transaction()

    def log_metadata(self, block):
        """记录元数据"""
        if self.transaction:
            self.transaction.add_metadata(block)

    def log_data(self, block):
        """记录数据（journal模式）"""
        if self.transaction:
            self.transaction.add_data(block)

    def commit_transaction(self):
        """提交事务"""
        if not self.transaction:
            return

        # 1. 写入日志（有序）
        for block in self.transaction.data_blocks:
            self.journal.write(block)

        for block in self.transaction.metadata_blocks:
            self.journal.write(block)

        # 2. 写入提交记录
        self.journal.write_commit_record(self.transaction.tid)

        # 3. 刷盘
        self.journal.flush()

        # 4. 写回实际位置
        self.writeback_transaction(self.transaction)

        self.transaction = None
```

**缓存配额**：

```python
class CacheQuotaManager:
    """
    缓存配额管理器

    为不同用户/组/进程分配缓存配额
    """

    def __init__(self, total_cache_size):
        self.total_size = total_cache_size
        self.quotas = {}  # cgroup_id -> quota
        self.usage = {}   # cgroup_id -> current_usage

    def set_quota(self, cgroup_id, quota):
        """设置缓存配额"""
        self.quotas[cgroup_id] = quota

    def can_cache(self, cgroup_id, page_size):
        """检查是否可以缓存"""
        if cgroup_id not in self.quotas:
            return True  # 无限制

        current = self.usage.get(cgroup_id, 0)
        quota = self.quotas[cgroup_id]

        if current + page_size <= quota:
            return True

        # 超出配额，需要回收
        return self.reclaim_for_cgroup(cgroup_id, page_size)

    def reclaim_for_cgroup(self, cgroup_id, needed_size):
        """为指定cgroup回收缓存"""
        # 优先回收该cgroup自己的缓存
        reclaimed = self.reclaim_from_cgroup(cgroup_id, needed_size)

        if reclaimed < needed_size:
            # 回收低优先级cgroup的缓存
            reclaimed += self.reclaim_from_low_priority(
                cgroup_id, needed_size - reclaimed
            )

        return reclaimed >= needed_size
```

### 6.1 数据布局优化

**数据布局策略**：

```python
class DataLayoutOptimizer:
    """
    数据布局优化器

    优化数据在存储层次中的分布
    """

    def __init__(self):
        self.hot_data_threshold = 10  # 10次访问为热点
        self.access_history = {}

    def optimize_layout(self, data_id, current_tier):
        """
        优化数据布局

        根据访问频率将数据迁移到合适的存储层次
        """
        access_freq = self.get_access_frequency(data_id)

        # 决定目标层次
        if access_freq > self.hot_data_threshold * 10:
            target_tier = 'L1_CACHE'
        elif access_freq > self.hot_data_threshold:
            target_tier = 'L3_CACHE'
        elif access_freq > 1:
            target_tier = 'DRAM'
        else:
            target_tier = 'SSD'

        if target_tier != current_tier:
            self.migrate_data(data_id, current_tier, target_tier)

    def get_access_frequency(self, data_id):
        """获取访问频率"""
        history = self.access_history.get(data_id, [])
        # 计算最近时间窗口内的访问次数
        window = time.now() - 3600  # 1小时窗口
        return sum(1 for t in history if t > window)
```

```python
class BandwidthOptimizer:
    """
    存储带宽优化器
    """

    def __init__(self):
        self.bandwidth_limits = {
            'L1': 1000,  # GB/s
            'L2': 500,
            'L3': 200,
            'DRAM': 50,
            'SSD': 3
        }

    def optimize_transfer(self, data_size, source_tier, dest_tier):
        """
        优化数据传输

        策略:
        1. 批量传输，减少启动开销
        2. 流水线传输，隐藏延迟
        3. 并行传输，利用多通道
        """
        # 计算最优传输块大小
        block_size = self.calculate_optimal_block_size(
            data_size, source_tier, dest_tier
        )

        # 分块并行传输
        num_parallel = self.bandwidth_limits[dest_tier] // block_size

        return self.parallel_transfer(data_size, block_size, num_parallel)

    def calculate_optimal_block_size(self, data_size, source, dest):
        """计算最优块大小"""
        # 基于延迟和带宽计算
        latency = self.get_latency(source, dest)
        bandwidth = min(self.bandwidth_limits[source],
                       self.bandwidth_limits[dest])

        # 块大小应使得传输时间远大于延迟
        optimal = bandwidth * latency * 10

        return min(optimal, data_size // 4)
```

```c
int add_to_page_cache_locked(struct page *page,
                             struct address_space *mapping,
                             pgoff_t offset, gfp_t gfp_mask)
{

    // 设置页属性

    page->index = offset;

    }
```

## 8 形式化模型

### 8.1 存储层次调度问题定义

$$
\text{存储层次调度问题} = (H, A, P, R, C, O)
$$

其中：

- $H = \{h_1, h_2, \ldots, h_n\}$：存储层次集合
  - $h_i = (\text{capacity}_i, \text{latency}_i, \text{bandwidth}_i)$
- $A = \{a_1, a_2, \ldots, a_m\}$：访问请求序列
  - $a_j = (addr_j, type_j, time_j, size_j)$
- $P$：预取策略
- $R$：替换策略
- $C$：约束条件
  - 容量约束：$\sum_{h_i} \text{data}(h_i) \leq \text{capacity}(h_i)$
  - 一致性约束：所有层次数据一致
- $O$：优化目标
  - 最大化命中率：$\max \frac{\text{hits}}{\text{total}}$
  - 最小化平均延迟：$\min \sum_{a_j} \text{latency}(a_j)$

### 8.2 定理：缓存命中率上界

**定理14.3（缓存命中率上界）**：

对于缓存容量$C$，工作集大小$W$，最优替换策略的命中率上界为：

$$
\text{hit\_rate} \leq \min\left(1, \frac{C}{W}\right)
$$

**证明**：

由鸽巢原理，工作集中最多$C$个不同块可以放入缓存。

如果$W > C$，至少$W-C$个块无法放入缓存，每次访问这些块必然未命中。

因此命中率上限为$C/W$。∎
