## 1 SSD调度概述

### 1.1 SSD物理特性

**NAND Flash组织结构**：

```
SSD 内部结构
├── 控制器 (Controller)
│   ├── 处理器 (ARM/MIPS)
│   ├── DRAM缓存
│   └── 硬件加速器 (ECC/加密)
│
├── 闪存通道 (Channels)
│   ├── Channel 0
│   │   ├── CE0: Die 0, 1
│   │   └── CE1: Die 2, 3
│   ├── Channel 1
│   │   └── ...
│   └── Channel N
│
└── NAND Flash 组织
    ├── Die (芯片)
    │   ├── Plane (2-4个)
    │   │   ├── Block (256-2048个)
    │   │   │   └── Page (256-512个)
    │   │   │       └── Cell (4KB-16KB)
```

**SSD操作特性**：

| 操作 | 粒度 | 延迟 | 特点 |
|------|------|------|------|
| 读 | Page (4-16KB) | 25-100μs | 直接读取 |
| 写 | Page | 200-500μs | 只能写入空页 |
| 擦除 | Block (256KB-2MB) | 1-3ms | 整块擦除 |

**擦写次数限制（P/E Cycles）**：

| NAND类型 | 制程 | P/E Cycles | 应用 |
|---------|------|-----------|------|
| SLC | 1bit/cell | 100K | 企业级/工业 |
| MLC | 2bit/cell | 10K | 企业级/高端消费 |
| TLC | 3bit/cell | 3K | 消费级 |
| QLC | 4bit/cell | 1K | 大容量存储 |
| PLC | 5bit/cell | <500 | 归档存储 |

### 1.2 SSD与HDD调度差异

**调度策略对比**：

| 维度 | HDD | SSD |
|------|-----|-----|
| 寻道延迟 | 4-10ms | 0 |
| 旋转延迟 | 2-4ms | 0 |
| 访问模式优化 | 顺序优先 | 并行优先 |
| 合并策略 | 重要 | 次要 |
| 调度器选择 | Deadline/BFQ | NOOP/NONE |
| 预取策略 | 顺序预取 | 随机预取 |

### 1.3 SSD调度的核心挑战

**四大挑战**：

```
1. 写入放大 (Write Amplification)
   逻辑写入 1GB → 物理写入 1.5-3GB
   来源: GC、磨损均衡、日志结构

2. 磨损均衡 (Wear Leveling)
   均匀分布擦除操作
   目标: 所有块P/E次数均衡

3. 垃圾回收 (Garbage Collection)
   回收无效页占用的空间
   挑战: 平衡GC开销和空间回收

4. 性能一致性 (Performance Consistency)
   避免GC导致的性能抖动
   目标: 99%延迟稳定
```

## 2 FTL调度

### 2.1 地址映射策略

**页级映射（Page-Level Mapping）**：

```python
class PageLevelFTL:
    """
    页级映射FTL

    优点:
    - 灵活性最高，任意页可映射
    - 随机写性能好

    缺点:
    - 映射表大 (4KB页，1TB SSD需要1GB映射表)
    - 需要大量DRAM
    """

    def __init__(self, num_logical_pages, num_physical_pages):
        self.mapping_table = {}  # LPN -> PPN
        self.reverse_table = {}  # PPN -> LPN

    def read(self, lpn):
        """读取逻辑页"""
        if lpn not in self.mapping_table:
            return None  # 未写入，返回空
        ppn = self.mapping_table[lpn]
        return self.read_physical_page(ppn)

    def write(self, lpn, data):
        """写入逻辑页"""
        # 分配新的物理页
        ppn = self.allocate_free_page()

        # 更新映射表
        if lpn in self.mapping_table:
            old_ppn = self.mapping_table[lpn]
            self.invalidate_page(old_ppn)

        self.mapping_table[lpn] = ppn
        self.reverse_table[ppn] = lpn

        # 写入物理页
        self.write_physical_page(ppn, data)
```

**块级映射（Block-Level Mapping）**：

```python
class BlockLevelFTL:
    """
    块级映射FTL

    优点:
    - 映射表小 (1TB SSD只需要4MB映射表)
    - DRAM需求低

    缺点:
    - 随机写性能差
    - 需要日志块处理覆盖写
    """

    def __init__(self, num_logical_blocks, num_physical_blocks, pages_per_block):
        self.block_table = {}  # LBN -> PBN
        self.log_blocks = []   # 日志块池
        self.pages_per_block = pages_per_block

    def write(self, lpn, data):
        """写入逻辑页"""
        lbn = lpn // self.pages_per_block
        offset = lpn % self.pages_per_block

        if lbn not in self.block_table:
            self.block_table[lbn] = self.allocate_free_block()

        pbn = self.block_table[lbn]
        ppn = pbn * self.pages_per_block + offset

        # 如果页已写入，需要日志块处理
        if self.is_page_written(ppn):
            ppn = self.write_to_log_block(lbn, offset, data)
        else:
            self.write_physical_page(ppn, data)
```

**混合映射（Hybrid Mapping）**：

```python
class HybridFTL:
    """
    混合映射FTL (如FAST, LAST)

    数据块: 块级映射
    日志块: 页级映射

    平衡性能和空间开销
    """

    def __init__(self):
        self.data_block_table = {}  # LBN -> PBN (块级)
        self.log_block_table = {}   # LPN -> PPN (页级)
        self.log_blocks_pool = []

    def write(self, lpn, data):
        """写入逻辑页"""
        # 先尝试顺序写入数据块
        lbn = lpn // self.pages_per_block

        if self.can_sequential_write(lbn, lpn):
            # 顺序写，使用块级映射
            self.write_to_data_block(lbn, lpn, data)
        else:
            # 随机写，使用日志块
            self.write_to_log_block(lpn, data)

        # 检查日志块是否满
        if self.log_blocks_full():
            self.merge_log_blocks()
```

**映射策略对比**：

| 策略 | 映射表大小 | 随机写性能 | 顺序写性能 | 适用场景 |
|------|-----------|-----------|-----------|---------|
| 页级映射 | 大 (0.1%容量) | 优秀 | 优秀 | 高端SSD |
| 块级映射 | 小 (0.0004%容量) | 差 | 优秀 | 低端SSD |
| 混合映射 | 中 | 好 | 优秀 | 消费级SSD |

### 2.2 页分配算法

**分配策略**：

```python
class PageAllocator:
    """
    物理页分配器
    """

    def __init__(self):
        self.free_pages = []  # 空闲页列表
        self.block_erasures = {}  # 块擦除次数

    def allocate_sequential(self, num_pages):
        """顺序分配 - 最大化连续页"""
        # 选择擦除次数最少的块，顺序分配页
        block = self.select_block_by_erasures(min)
        pages = self.get_consecutive_pages(block, num_pages)
        return pages

    def allocate_random(self, num_pages):
        """随机分配 - 最大化磨损均衡"""
        pages = []
        for _ in range(num_pages):
            # 优先选择擦除次数少的块
            block = self.select_block_by_wear_leveling()
            page = self.get_free_page_from_block(block)
            pages.append(page)
        return pages

    def allocate_greedy(self, num_pages, io_type):
        """贪心分配 - 根据IO类型优化"""
        if io_type == 'SEQUENTIAL':
            # 顺序IO: 选择干净块，最大化并行
            return self.allocate_for_parallel(num_pages)
        else:
            # 随机IO: 分散到不同die/channel
            return self.allocate_for_random(num_pages)

    def select_block_by_wear_leveling(self):
        """基于磨损均衡选择块"""
        # 选择擦除次数最少的块，但有阈值限制
        min_erasures = min(self.block_erasures.values())
        candidates = [b for b, e in self.block_erasures.items()
                     if e <= min_erasures + WEAR_THRESHOLD]
        return random.choice(candidates)
```

### 2.3 写入放大优化

**写入放大计算**：

```
写入放大因子 (WAF) = 物理写入量 / 逻辑写入量

影响因素:
1. 垃圾回收: 移动有效页产生额外写入
2. 磨损均衡: 数据迁移产生额外写入
3. 预留空间: 预留空间越少，GC越频繁

WAF = 1 + (GC写入 + WL写入) / 用户写入
```

**优化策略**：

```python
class WriteAmplificationOptimizer:
    """
    写入放大优化器
    """

    def __init__(self):
        self.over_provisioning = 0.28  # 28%预留空间
        self.gc_threshold = 0.10  # GC触发阈值

    def optimize_gc_trigger(self, free_space_ratio):
        """
        动态GC触发阈值

        空闲空间多: 延迟GC，减少写入放大
        空闲空间少: 积极GC，避免空间耗尽
        """
        if free_space_ratio > 0.30:
            return 0.05  # 宽松GC
        elif free_space_ratio > 0.15:
            return 0.10  # 标准GC
        else:
            return 0.15  # 积极GC

    def optimize_write_pattern(self, io_stream):
        """
        写模式优化

        顺序写: 直接写入数据块，减少GC
        随机写: 先写入日志块，批量合并
        """
        if self.detect_sequential(io_stream):
            return self.direct_write(io_stream)
        else:
            return self.buffered_write(io_stream)

    def compress_data(self, data):
        """数据压缩 - 减少物理写入量"""
        compressed = self.hardware_compress(data)
        return compressed
```

## 3 垃圾回收调度

### 3.1 GC触发策略

**触发条件**：

| 触发类型 | 条件 | 优先级 |
|---------|------|--------|
| 空闲空间 | 空闲块 < 阈值 (5-10%) | 高 |
| 后台GC | 空闲时主动回收 | 低 |
| 强制GC | 写失败时紧急回收 | 最高 |

**GC调度算法**：

```python
class GCScheduler:
    """
    垃圾回收调度器
    """

    def __init__(self):
        self.gc_state = 'IDLE'
        self.gc_queue = []

    def check_gc_trigger(self, free_blocks):
        """检查是否触发GC"""
        # 前台GC触发
        if free_blocks < FOREGROUND_GC_THRESHOLD:
            return 'FOREGROUND'

        # 后台GC触发
        if free_blocks < BACKGROUND_GC_THRESHOLD:
            return 'BACKGROUND'

        return 'NONE'

    def schedule_gc(self, trigger_type):
        """调度GC任务"""
        if trigger_type == 'FOREGROUND':
            # 前台GC: 阻塞写操作，快速回收
            self.run_foreground_gc()
        elif trigger_type == 'BACKGROUND':
            # 后台GC: 空闲时执行，不影响性能
            self.schedule_background_gc()

    def run_foreground_gc(self):
        """执行前台GC"""
        victim_blocks = self.select_victim_blocks(aggressive=True)

        for block in victim_blocks:
            # 迁移有效页
            valid_pages = self.get_valid_pages(block)
            for page in valid_pages:
                new_page = self.allocate_page()
                self.migrate_page(page, new_page)

            # 擦除块
            self.erase_block(block)

            # 检查是否足够
            if self.free_blocks_sufficient():
                break

    def schedule_background_gc(self):
        """调度后台GC"""
        # 在低负载时执行
        if self.system_idle():
            victim_blocks = self.select_victim_blocks(aggressive=False)
            for block in victim_blocks:
                self.perform_gc_step(block)
                if not self.system_idle():
                    break  # 系统变忙，暂停GC
```

### 3.2 Victim块选择算法

**贪心算法（Greedy）**：

```python
def greedy_selection(blocks):
    """
    贪心算法: 选择无效页最多的块

    优点: 最小化迁移开销
    缺点: 不考虑磨损均衡
    """
    best_block = None
    max_invalid = 0

    for block in blocks:
        invalid_pages = count_invalid_pages(block)
        if invalid_pages > max_invalid:
            max_invalid = invalid_pages
            best_block = block

    return best_block
```

**成本-效益算法（Cost-Benefit）**：

```python
def cost_benefit_selection(blocks):
    """
    成本-效益算法

    成本 = 有效页数 (需要迁移的页)
    效益 = 回收的空间 * 块年龄

    选择效益/成本比最高的块
    """
    best_block = None
    best_ratio = 0

    for block in blocks:
        valid_pages = count_valid_pages(block)
        invalid_pages = total_pages - valid_pages

        if valid_pages == 0:
            return block  # 空块，直接回收

        cost = valid_pages  # 迁移成本
        age = get_block_age(block)  # 块年龄
        benefit = invalid_pages * age  # 回收效益

        ratio = benefit / cost
        if ratio > best_ratio:
            best_ratio = ratio
            best_block = block

    return best_block
```

**磨损感知算法（Wear-Aware）**：

```python
def wear_aware_selection(blocks, age_threshold=10):
    """
    磨损感知算法

    优先回收: 擦除次数高的块
    避免: 擦除次数低的块
    """
    # 按擦除次数分组
    groups = group_by_erasures(blocks)

    # 从高擦除次数组选择
    for erasure_count in sorted(groups.keys(), reverse=True):
        group = groups[erasure_count]

        # 在组内使用贪心算法
        candidate = greedy_selection(group)
        if candidate:
            return candidate

    return None
```

## 4 磨损均衡调度

### 4.1 动态磨损均衡

**动态磨损均衡算法**：

```python
class DynamicWearLeveling:
    """
    动态磨损均衡

    在运行时监控块擦除次数
    动态调整数据分布
    """

    def __init__(self):
        self.erasures = {}  # 块擦除次数
        self.threshold = 100  # 触发阈值

    def monitor_writes(self, block):
        """监控写入操作"""
        # 记录擦除次数
        if block not in self.erasures:
            self.erasures[block] = 0

        # 检查是否需要迁移
        if self.need_wear_leveling():
            self.perform_migration()

    def need_wear_leveling(self):
        """检查是否需要进行磨损均衡"""
        if not self.erasures:
            return False

        max_erasures = max(self.erasures.values())
        min_erasures = min(self.erasures.values())

        return (max_erasures - min_erasures) > self.threshold

    def perform_migration(self):
        """执行磨损均衡迁移"""
        # 找出最热和最冷的块
        hot_block = max(self.erasures, key=self.erasures.get)
        cold_block = min(self.erasures, key=self.erasures.get)

        # 将冷数据迁移到热块
        # 将热数据迁移到冷块
        self.swap_data(hot_block, cold_block)
```

```python
class StaticWearLeveling:
    """
    静态磨损均衡

    处理冷数据（很少访问的数据）
    将冷数据迁移到高擦除次数的块
    """

    def __init__(self):
        self.check_interval = 3600  # 每小时检查一次

    def identify_cold_data(self):
        """识别冷数据"""
        cold_blocks = []
        for block in all_blocks:
            if self.get_access_frequency(block) < COLD_THRESHOLD:
                cold_blocks.append(block)
        return cold_blocks

    def static_relocation(self):
        """静态重定位"""
        cold_data = self.identify_cold_data()
        hot_blocks = self.get_high_erasures_blocks()

        for data_block, target_block in zip(cold_data, hot_blocks):
            # 将冷数据迁移到高擦除次数块
            self.migrate_block(data_block, target_block)
```

### 4.3 磨损均衡算法

**双池算法**：

```python
class DualPoolWearLeveling:
    """
    双池磨损均衡算法

    自由池: 空闲块
    数据池: 已使用块

    分配时优先从自由池选择擦除次数少的块
    """

    def __init__(self):
        self.free_pool = []   # 自由池
        self.data_pool = []   # 数据池

    def allocate_block(self):
        """分配块"""
        # 按擦除次数排序
        self.free_pool.sort(key=lambda b: b.erasures)

        # 选择擦除次数最少的块
        block = self.free_pool.pop(0)
        self.data_pool.append(block)
        return block

    def recycle_block(self, block):
        """回收块"""
        self.data_pool.remove(block)
        block.erasures += 1
        self.free_pool.append(block)
```

## 5 TRIM与UNMAP

### 5.1 TRIM机制

**TRIM流程**：

```
文件系统层
    ↓ 删除文件
发送TRIM命令 (ATA/SCSI/NVMe)
    ↓
FTL层接收
    ↓
标记对应LBA的物理页为无效
    ↓
无需迁移，直接擦除
    ↓
减少GC时的有效页迁移
```

### 5.2 TRIM调度策略

**TRIM调度**：

```python
class TRIMScheduler:
    """
    TRIM调度器

    优化TRIM执行时机，避免影响前台性能
    """

    def __init__(self):
        self.trim_queue = []
        self.batch_size = 64  # 批量处理大小

    def receive_trim(self, lba_ranges):
        """接收TRIM请求"""
        for range in lba_ranges:
            self.trim_queue.append(range)

        # 批量处理
        if len(self.trim_queue) >= self.batch_size:
            self.execute_trim_batch()

    def execute_trim_batch(self):
        """执行批量TRIM"""
        batch = self.trim_queue[:self.batch_size]
        self.trim_queue = self.trim_queue[self.batch_size:]

        # 合并连续范围
        merged = self.merge_ranges(batch)

        # 发送到SSD
        for range in merged:
            self.ssd.trim(range.lba, range.length)

    def schedule_background_trim(self):
        """后台TRIM调度"""
        while self.system_idle() and self.trim_queue:
            self.execute_trim_batch()
```

```python
class NVMeArbitration:
    """
    NVMe仲裁机制

    支持三种仲裁方式:
    1. RR (Round Robin): 轮询
    2. WRR (Weighted RR): 加权轮询
    3. WRR with Urgent: 带紧急优先级的加权轮询
    """

    def __init__(self, num_queues, arbitration_type='WRR'):
        self.queues = [[] for _ in range(num_queues)]
        self.weights = [1] * num_queues
        self.arbitration_type = arbitration_type
        self.credits = [0] * num_queues

    def enqueue(self, queue_id, command):
        """将命令加入队列"""
        self.queues[queue_id].append(command)

    def arbitrate(self):
        """仲裁选择下一个命令"""
        if self.arbitration_type == 'RR':
            return self.round_robin()
        elif self.arbitration_type == 'WRR':
            return self.weighted_round_robin()
        elif self.arbitration_type == 'WRR_URGENT':
            return self.weighted_round_robin_urgent()

    def weighted_round_robin(self):
        """加权轮询仲裁"""
        for i in range(len(self.queues)):
            queue_idx = (self.current_queue + i) % len(self.queues)

            if self.credits[queue_idx] > 0 and self.queues[queue_idx]:
                self.credits[queue_idx] -= 1
                return self.queues[queue_idx].pop(0)

            # 重置信用
            if self.credits[queue_idx] == 0:
                self.credits[queue_idx] = self.weights[queue_idx]

        self.current_queue = (self.current_queue + 1) % len(self.queues)
        return None
```

### 8.2 NVMe驱动实现

**NVMe驱动核心**：

```c
// NVMe队列结构
struct nvme_queue {
    struct nvme_dev *dev;
    spinlock_t sq_lock;
    spinlock_t cq_lock;

    struct nvme_command *sq_cmds;      // 提交队列
    volatile struct nvme_completion *cqes;  // 完成队列

    dma_addr_t sq_dma_addr;
    dma_addr_t cq_dma_addr;

    u16 sq_tail;    // 提交队列尾指针
    u16 cq_head;    // 完成队列头指针
    u16 q_depth;    // 队列深度
    u8 cq_phase;    // 完成阶段标志

    struct nvme_completion_ctx *cmd_ctx;
};

// 提交IO请求
int nvme_submit_sync_cmd(struct nvme_queue *nvmeq,
                         struct nvme_command *cmd,
                         void *buffer, unsigned bufflen)
{
    u16 tail;

    spin_lock_irq(&nvmeq->sq_lock);

    // 获取队列位置
    tail = nvmeq->sq_tail;

    // 填充命令
    memcpy(&nvmeq->sq_cmds[tail], cmd, sizeof(*cmd));

    // 更新尾指针
    tail = (tail + 1) % nvmeq->q_depth;
    nvmeq->sq_tail = tail;

    // 写Doorbell通知SSD
    writel(tail, nvmeq->dev->bar + NVME_REG_SQTDBL(nvmeq->qid));

    spin_unlock_irq(&nvmeq->sq_lock);

    return 0;
}
```

## 9 形式化模型

### 9.1 SSD调度问题定义

**形式化定义**：

$$
\text{SSD调度问题} = (B, P, W, E, C, O)
$$

其中：

- $B = \{b_1, b_2, \ldots, b_n\}$：物理块集合
  - $b_i = (\text{pages}, \text{erasures}, \text{state})$
- $P = \{p_1, p_2, \ldots, p_m\}$：物理页集合
- $W = \{w_1, w_2, \ldots, w_k\}$：写入请求集合
- $E$：擦除操作集合
- $C$：约束条件
  - 擦除限制：$\text{erasures}(b_i) \leq E_{max}$
  - 写入限制：页必须先擦除后写入
  - GC约束：空闲空间必须大于阈值
- $O$：优化目标
  - 最小化写入放大：$\min \frac{\sum \text{physical\_writes}}{\sum \text{logical\_writes}}$
  - 最大化磨损均衡：$\min \sigma(\text{erasures}(b_i))$
  - 最小化GC开销：$\min t_{gc}$

### 9.2 定理：写入放大下界

**定理14.2（写入放大下界）**：

对于SSD，设预留空间比例为 $OPR$（Over-Provisioning Ratio），则写入放大因子的下界为：

$$
\text{WA} \geq 1 + \frac{1}{OPR}
$$

**证明**：

考虑稳态情况，设：

- $C$：物理容量
- $U$：用户可见容量
- $OPR = (C - U) / U$

每次GC需要移动的有效数据比例为 $U/C = 1/(1+OPR)$

因此，每次用户写入触发的总写入为：

$$
\text{WA} = 1 + \frac{U}{C-U} = 1 + \frac{1}{OPR}
$$

∎

|---------|---------|-----------------|-----------------|
| 7% | 15.3 | 8-12 | 2-3 |
| 28% | 4.6 | 3-5 | 1.5-2 |
| 50% | 3.0 | 2-3 | 1.2-1.5 |
