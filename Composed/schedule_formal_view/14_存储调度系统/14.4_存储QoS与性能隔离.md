## 1 存储QoS概述

### 1.1 QoS核心指标

**存储QoS三大核心指标**：

| 指标 | 符号 | 单位 | 典型值范围 | 说明 |
|------|------|------|-----------|------|
| **IOPS** | $I$ | ops/s | 1 - 1,000,000 | 每秒IO操作数 |
| **带宽** | $B$ | MB/s | 1 - 10,000 | 数据传输速率 |
| **延迟** | $L$ | ms/μs | 10μs - 100ms | IO响应时间 |

**延迟分布指标**：

| 指标 | 含义 | 典型要求 |
|------|------|---------|
| P50 (中位数) | 50%请求延迟 | 基准 |
| P90 | 90%请求延迟 | < 2x P50 |
| P99 | 99%请求延迟 | < 5x P50 |
| P99.9 | 99.9%请求延迟 | < 10x P50 |
| 最大延迟 | 最坏情况 | 有界 |

### 1.2 QoS挑战

**主要挑战**：

```
1. 资源争用
   多个租户/进程竞争有限的存储资源
   → 需要公平分配和隔离

2. 性能抖动
   后台活动(GC, 日志)导致延迟波动
   → 需要优先级调度和资源预留

3. 突发流量
   短时间大量IO请求
   → 需要流量整形和缓冲

4. 混合负载
   读/写/顺序/随机混合
   → 需要自适应调度
```

## 2 QoS模型与算法

### 2.1 令牌桶限流

**令牌桶算法**：

```python
class TokenBucket:
    """
    令牌桶限流器

    原理:
    - 桶以固定速率产生令牌
    - 请求需要消费令牌才能执行
    - 桶有容量限制，防止突发
    """

    def __init__(self, rate, capacity):
        """
        rate: 令牌产生速率 (tokens/second)
        capacity: 桶容量 (最大突发令牌数)
        """
        self.rate = rate
        self.capacity = capacity
        self.tokens = capacity
        self.last_update = time.time()
        self.lock = threading.Lock()

    def consume(self, tokens=1):
        """
        尝试消费指定数量的令牌

        返回: (success, wait_time)
        """
        with self.lock:
            now = time.time()
            elapsed = now - self.last_update

            # 补充令牌
            self.tokens = min(self.capacity,
                             self.tokens + elapsed * self.rate)
            self.last_update = now

            if self.tokens >= tokens:
                self.tokens -= tokens
                return True, 0

            # 计算等待时间
            wait_time = (tokens - self.tokens) / self.rate
            return False, wait_time

    def throttle(self, tokens=1):
        """限流，需要时阻塞等待"""
        success, wait_time = self.consume(tokens)
        if not success:
            time.sleep(wait_time)
            return self.consume(tokens)[0]
        return True

# 使用示例
iops_bucket = TokenBucket(rate=1000, capacity=2000)  # 1000 IOPS, 突发2000
bw_bucket = TokenBucket(rate=100*1024*1024, capacity=200*1024*1024)  # 100MB/s

def process_io_request(request):
    # IOPS限制
    if not iops_bucket.throttle(1):
        return False

    # 带宽限制
    if not bw_bucket.throttle(request.size):
        return False

    return execute_io(request)
```

**令牌桶特性**：

| 参数 | 作用 | 配置建议 |
|------|------|---------|
| rate | 平均速率 | 基于SLA设置 |
| capacity | 突发容量 | rate × 允许突发时间 |

### 2.2 漏桶算法

**漏桶算法**：

```python
class LeakyBucket:
    """
    漏桶限流器

    原理:
    - 请求先进入桶（队列）
    - 桶以固定速率漏出（处理）
    - 桶满时新请求被拒绝

    特点: 平滑输出，无突发
    """

    def __init__(self, leak_rate, capacity):
        self.leak_rate = leak_rate  # 漏出速率
        self.capacity = capacity    # 桶容量
        self.water = 0              # 当前水量
        self.last_leak = time.time()
        self.lock = threading.Lock()

    def add(self, amount=1):
        """
        尝试添加水量

        返回: 是否成功
        """
        with self.lock:
            self._leak()

            if self.water + amount <= self.capacity:
                self.water += amount
                return True
            else:
                return False  # 桶满，拒绝

    def _leak(self):
        """漏水"""
        now = time.time()
        elapsed = now - self.last_leak
        leaked = elapsed * self.leak_rate

        self.water = max(0, self.water - leaked)
        self.last_leak = now
```

**令牌桶 vs 漏桶**：

| 特性 | 令牌桶 | 漏桶 |
|------|--------|------|
| 突发处理 | 允许突发 | 平滑输出 |
| 实现复杂度 | 简单 | 中等 |
| 典型应用 | 带宽限制 | 请求整形 |

### 2.3 加权公平队列

**WFQ算法**：

```python
class WeightedFairQueue:
    """
    加权公平队列 (Weighted Fair Queuing)

    原理:
    - 每个流分配一个虚拟时间
    - 按虚拟时间排序服务
    - 权重决定虚拟时间增长速率
    """

    def __init__(self):
        self.queues = {}  # flow_id -> Queue
        self.weights = {}  # flow_id -> weight
        self.virtual_time = 0
        self.finish_times = {}
        self.active_flows = []

    def enqueue(self, flow_id, packet):
        """将包加入队列"""
        if flow_id not in self.queues:
            self.queues[flow_id] = deque()
            self.weights[flow_id] = 1  # 默认权重

        # 计算虚拟开始和完成时间
        weight = self.weights[flow_id]
        start_time = max(self.virtual_time,
                        self.finish_times.get(flow_id, 0))
        finish_time = start_time + packet.length / weight

        packet.vstart = start_time
        packet.vfinish = finish_time

        self.queues[flow_id].append(packet)
        self.finish_times[flow_id] = finish_time

        if flow_id not in self.active_flows:
            self.active_flows.append(flow_id)

    def dequeue(self):
        """选择下一个包"""
        if not self.active_flows:
            return None

        # 选择虚拟完成时间最小的流
        min_flow = min(self.active_flows,
                      key=lambda f: self.queues[f][0].vfinish)

        packet = self.queues[min_flow].popleft()

        if not self.queues[min_flow]:
            self.active_flows.remove(min_flow)

        # 更新虚拟时间
        self.virtual_time = packet.vstart

        return packet
```

## 3 IOPS与带宽控制

### 3.1 IOPS限制实现

**IOPS限制器**：

```python
class IOPSLimiter:
    """
    IOPS限制器

    实现:
    - 时间窗口内计数
    - 超限则延迟或拒绝
    """

    def __init__(self, target_iops, window_ms=100):
        self.target_iops = target_iops
        self.window_ms = window_ms
        self.max_per_window = target_iops * window_ms / 1000
        self.requests = deque()
        self.lock = threading.Lock()

    def allow_request(self):
        """检查是否允许请求"""
        with self.lock:
            now = time.time() * 1000  # ms
            window_start = now - self.window_ms

            # 移除窗口外的请求记录
            while self.requests and self.requests[0] < window_start:
                self.requests.popleft()

            # 检查是否超限
            if len(self.requests) < self.max_per_window:
                self.requests.append(now)
                return True, 0
            else:
                # 计算需要等待的时间
                wait_time = self.requests[0] + self.window_ms - now
                return False, max(wait_time, 0)

    def throttle(self):
        """限流，必要时等待"""
        allowed, wait = self.allow_request()
        if not allowed:
            time.sleep(wait / 1000)  # 转换为秒
            return self.allow_request()[0]
        return True
```

### 3.2 带宽限制实现

**带宽限制器**：

```python
class BandwidthLimiter:
    """
    带宽限制器

    实现:
    - 基于令牌桶
    - 按字节计费
    """

    def __init__(self, target_bps):
        self.target_bps = target_bps
        self.tokens = target_bps  # 初始满桶
        self.max_tokens = target_bps  # 1秒容量
        self.last_update = time.time()
        self.lock = threading.Lock()

    def consume(self, bytes_amount):
        """消费带宽额度"""
        with self.lock:
            now = time.time()
            elapsed = now - self.last_update

            # 补充令牌
            self.tokens = min(self.max_tokens,
                             self.tokens + elapsed * self.target_bps)
            self.last_update = now

            if self.tokens >= bytes_amount:
                self.tokens -= bytes_amount
                return True, 0

            # 计算等待时间
            wait_time = (bytes_amount - self.tokens) / self.target_bps
            return False, wait_time

    def transfer(self, data):
        """执行带宽限制的传输"""
        offset = 0
        while offset < len(data):
            chunk = min(len(data) - offset, 64*1024)  # 64KB chunks

            allowed, wait = self.consume(chunk)
            if not allowed:
                time.sleep(wait)
                continue

            # 实际传输
            yield data[offset:offset+chunk]
            offset += chunk
```

### 3.3 混合限制策略

**混合限制器**：

```python
class MixedLimiter:
    """
    IOPS和带宽混合限制器

    同时满足IOPS和带宽限制
    """

    def __init__(self, max_iops, max_bw):
        self.iops_limiter = IOPSLimiter(max_iops)
        self.bw_limiter = BandwidthLimiter(max_bw)

    def submit_io(self, io_request):
        """
        提交IO请求

        需要同时满足IOPS和带宽限制
        """
        # 1. 检查IOPS
        iops_allowed, iops_wait = self.iops_limiter.allow_request()

        # 2. 检查带宽
        bw_allowed, bw_wait = self.bw_limiter.consume(io_request.size)

        if iops_allowed and bw_allowed:
            return self.execute_io(io_request)

        # 需要等待
        wait_time = max(iops_wait, bw_wait)
        time.sleep(wait_time / 1000 if wait_time > 1 else wait_time)

        return self.submit_io(io_request)
```

### 4.2 实时调度

**EDF（最早截止时间优先）**：

```python
class EDFScheduler:
    """
    最早截止时间优先调度器

    原理:
    - 每个请求有截止时间
    - 优先执行截止时间最近的请求
    - 最优的实时调度算法
    """

    def __init__(self):
        self.ready_queue = []  # (deadline, request)

    def enqueue(self, request, deadline):
        """加入就绪队列"""
        heapq.heappush(self.ready_queue, (deadline, request))

    def dispatch(self, current_time):
        """选择下一个请求"""
        if not self.ready_queue:
            return None

        deadline, request = self.ready_queue[0]

        # 检查是否错过截止时间
        if deadline < current_time:
            # 记录截止时间错过
            self.record_deadline_miss(request)

        heapq.heappop(self.ready_queue)
        return request

    def schedulability_test(self, tasks):
        """
        可调度性测试

        对于周期性任务，检查是否满足:
        Σ(Ci/Ti) ≤ 1

        其中Ci是执行时间，Ti是周期
        """
        utilization = sum(task.wcet / task.period for task in tasks)
        return utilization <= 1.0
```

### 4.3 尾延迟优化

**尾延迟优化策略**：

```python
class TailLatencyOptimizer:
    """
    尾延迟优化器

    目标: 降低P99/P99.9延迟
    """

    def __init__(self, target_p99):
        self.target_p99 = target_p99
        self.latency_histogram = Histogram(buckets=100)

    def on_request_complete(self, latency):
        """记录请求完成"""
        self.latency_histogram.record(latency)

        # 检查尾延迟
        current_p99 = self.latency_histogram.get_percentile(0.99)

        if current_p99 > self.target_p99:
            self.optimize()

    def optimize(self):
        """执行优化"""
        # 1. 识别长尾请求
        tail_requests = self.latency_histogram.get_tail_requests()

        # 2. 分析原因
        causes = self.analyze_tail_causes(tail_requests)

        # 3. 应用优化
        if 'gc' in causes:
            self.throttle_background_gc()

        if 'queue_depth' in causes:
            self.reduce_queue_depth()

        if 'interference' in causes:
            self.enforce_isolation()

    def hedged_request(self, request, hedge_delay_ms):
        """
        对冲请求

        如果原请求超时，发送副本请求
        使用先完成的响应
        """
        primary_future = self.send_request(request)

        try:
            return primary_future.get(timeout=hedge_delay_ms/1000)
        except TimeoutError:
            # 发送对冲请求
            hedge_future = self.send_request(request)

            # 等待任一完成
            done, _ = wait([primary_future, hedge_future],
                          return_when=FIRST_COMPLETED)
            return done[0].result()
```

### 5.1 租户资源配额

**配额模型**：

```python
class TenantQuota:
    """
    租户资源配额
    """

    def __init__(self, tenant_id):
        self.tenant_id = tenant_id

        # 资源限制
        self.limits = {
            'iops_read': 10000,
            'iops_write': 5000,
            'bw_read_mb': 500,
            'bw_write_mb': 250,
            'capacity_gb': 1000,
            'cache_mb': 100
        }

        # 当前使用
        self.usage = {
            'iops_read': 0,
            'iops_write': 0,
            'bw_read_mb': 0,
            'bw_write_mb': 0,
            'capacity_gb': 0,
            'cache_mb': 0
        }

        # 限流器
        self.limiters = {
            'iops_read': TokenBucket(self.limits['iops_read'],
                                    self.limits['iops_read'] * 2),
            'iops_write': TokenBucket(self.limits['iops_write'],
                                     self.limits['iops_write'] * 2),
            'bw_read': TokenBucket(self.limits['bw_read_mb'] * 1024 * 1024,
                                  self.limits['bw_read_mb'] * 1024 * 1024),
            'bw_write': TokenBucket(self.limits['bw_write_mb'] * 1024 * 1024,
                                   self.limits['bw_write_mb'] * 1024 * 1024)
        }

    def check_and_consume(self, operation, size_bytes):
        """
        检查配额并消费

        返回: (allowed, reason)
        """
        # 确定操作类型
        if operation == 'read':
            iops_key = 'iops_read'
            bw_key = 'bw_read'
        else:
            iops_key = 'iops_write'
            bw_key = 'bw_write'

        # 检查IOPS限制
        iops_allowed, _ = self.limiters[iops_key].consume(1)
        if not iops_allowed:
            return False, 'iops_limit_exceeded'

        # 检查带宽限制
        bw_allowed, _ = self.limiters[bw_key].consume(size_bytes)
        if not bw_allowed:
            return False, 'bandwidth_limit_exceeded'

        # 更新使用统计
        self.usage[iops_key] += 1
        self.usage[bw_key] += size_bytes / (1024 * 1024)

        return True, None
```

```python
class CgroupIsolation:

    """

    def __init__(self, cgroup_path):
        self.cgroup_path = cgroup_path

    def set_io_limits(self, device, iops_limit=None, bw_limit=None):
        """
        设置IO限制

        device: 设备号 (major:minor)
        """
        base = f"{self.cgroup_path}/blkio"

        if iops_limit:
            # 设置IOPS限制
            write_file(f"{base}/blkio.throttle.read_iops_device",
                      f"{device} {iops_limit}")
            write_file(f"{base}/blkio.throttle.write_iops_device",
                      f"{device} {iops_limit}")

        if bw_limit:
            # 设置带宽限制 (字节/秒)
            write_file(f"{base}/blkio.throttle.read_bps_device",
                      f"{device} {bw_limit}")
            write_file(f"{base}/blkio.throttle.write_bps_device",
                      f"{device} {bw_limit}")

    def set_weight(self, weight):
        """
        设置IO权重

        weight: 100-1000
        """
        write_file(f"{self.cgroup_path}/blkio/blkio.weight", str(weight))
```

### 5.3 noisy neighbor 防护

**防护策略**：

```python
class NoisyNeighborProtection:
    """
    吵闹邻居防护

    检测并限制影响其他租户的租户
    """

    def __init__(self):
        self.tenant_stats = {}
        self.threshold = 0.8  # 阈值

    def monitor(self):
        """监控所有租户"""
        for tenant_id, stats in self.tenant_stats.items():
            # 检测异常行为
            if self.is_noisy_neighbor(stats):
                self.throttle_tenant(tenant_id)

    def is_noisy_neighbor(self, stats):
        """
        检测是否是吵闹邻居

        指标:
        - IO延迟激增
        - 资源使用率异常高
        - 影响其他租户
        """
        # 检查自身延迟
        if stats['p99_latency'] > stats['baseline_p99'] * 3:
            return True

        # 检查对其他租户的影响
        if stats['impact_score'] > self.threshold:
            return True

        return False

    def throttle_tenant(self, tenant_id):
        """限制吵闹租户"""
        # 临时降低其配额
        new_quota = self.get_current_quota(tenant_id) * 0.5
        self.set_quota(tenant_id, new_quota)

        # 设置恢复定时器
        self.schedule_recovery(tenant_id, delay=60)
```

**实现原理**：

```c
// io.latency核心逻辑
struct io_latency_state {
    u64 target;         // 目标延迟 (ns)
    u64 window;         // 采样窗口 (ns)

    // 统计
    u64 total_samples;
    u64 missed_samples;

    // 限流状态
    bool throttled;
    u64 throttle_until;
};

void io_latency_check(struct request *rq)
{
    struct io_latency_state *state = get_latency_state(rq);
    u64 latency = get_request_latency(rq);

    state->total_samples++;

    if (latency > state->target) {
        state->missed_samples++;
    }

    // 计算错过率
    u64 miss_rate = state->missed_samples * 100 / state->total_samples;

    // 如果错过率过高，触发限流
    if (miss_rate > 10) {  // 10%阈值
        throttle_cgroup(state->cgroup);
    } else if (miss_rate < 5) {
        unthrottle_cgroup(state->cgroup);
    }
}
```

**自适应QoS控制器**：

```python
class AdaptiveQoSController:
    """
    自适应QoS控制器

    根据负载动态调整QoS参数
    """

    def __init__(self):
        self.targets = {
            'p99_latency': 10,  # ms
            'throughput': 1000   # MB/s
        }
        self.current_limits = {}

    def adjust(self, metrics):
        """
        根据指标调整限制

        控制逻辑:
        - 延迟过高: 降低其他租户配额
        - 延迟过低: 可以提高限制
        """
        current_p99 = metrics['p99_latency']
        target_p99 = self.targets['p99_latency']

        error = current_p99 - target_p99

        if error > 0:
            # 延迟过高，需要降低负载
            self.decrease_limits(error)
        elif error < -2:  # 有2ms余量
            # 延迟有富余，可以提高限制
            self.increase_limits(-error)

    def decrease_limits(self, amount):
        """降低限制"""
        for tenant in self.get_low_priority_tenants():
            current = self.current_limits[tenant]
            new_limit = current * 0.9  # 降低10%
            self.set_limit(tenant, new_limit)

    def increase_limits(self, amount):
        """增加限制"""
        for tenant in self.get_throttled_tenants():
            current = self.current_limits[tenant]
            new_limit = min(current * 1.1, tenant.max_limit)
            self.set_limit(tenant, new_limit)
```

## 8 形式化模型

### 8.1 QoS问题定义

$$
\text{存储QoS问题} = (T, R, C, G, O)
$$

其中：

- $T = \{t_1, t_2, \ldots, t_n\}$：租户集合
- $R$：资源类型（IOPS, BW, Capacity）
- $C$：容量约束
  - $\sum_i \text{iops}(t_i) \leq \text{IOPS}_{total}$
  - $\sum_i \text{bw}(t_i) \leq \text{BW}_{total}$
- $G$：SLO保证
  - $\text{latency}(t_i) \leq L_{target,i}$
  - $\text{availability}(t_i) \geq A_{target,i}$
- $O$：优化目标
  - 最大化利用率：$\max \sum_i \text{actual\_usage}(t_i)$
  - 最小化违规：$\min \sum_i \text{violation}(t_i)$

### 8.2 性能隔离定理

**定理14.4（性能隔离下界）**：

对于$n$个租户共享存储设备，最坏情况下的性能干扰满足：

$$
\text{degradation}_i \geq \frac{\sum_{j \neq i} \text{load}_j}{\text{capacity}} \times \text{baseline}_i
$$

**证明**：

考虑设备总容量为$C$，租户$i$的基础性能为$b_i$。

其他租户的总负载为$L_{-i} = \sum_{j \neq i} \text{load}_j$。

当$L_{-i}$接近$C$时，租户$i$可获得的资源比例趋近于0。

因此性能下降至少与$L_{-i}/C$成正比。∎

**定理14.5（令牌桶QoS保证）**：

对于令牌桶限流器，设令牌产生速率为$\lambda$，桶容量为$B$，则在稳态下：

1. **平均速率保证**: $\lim_{T \to \infty} \frac{N(T)}{T} \leq \lambda$
2. **突发容忍**: 最大突发大小为$B$令牌
3. **延迟上界**: 请求等待时间 $W \leq \frac{B}{\lambda}$

**证明**：

1. 令牌产生速率为$\lambda$，长期平均消费速率不可能超过产生速率。

2. 桶最大容量为$B$，因此最大突发为$B$。

3. 最坏情况下桶为空，需要等待$B$个令牌产生，时间为$B/\lambda$。∎

**定理14.6（加权公平队列带宽保证）**：

对于$N$个队列，权重分别为$w_1, w_2, \ldots, w_N$，总带宽为$C$，则队列$i$获得的带宽：

$$
B_i = C \times \frac{w_i}{\sum_{j=1}^{N} w_j}
$$

**证明**：

WFQ按虚拟时间服务，虚拟时间增长速率与权重成反比。

队列$i$在虚拟时间$t$内获得的服务量为 $w_i \times t$。

所有队列总服务量为 $t \times \sum w_j$，等于实际时间内的总带宽$C \times T$。

因此队列$i$的带宽份额与其权重成正比。∎

**形式化QoS保证证明**：

```
QoS保证框架:

输入:
- 租户集合 T = {t₁, t₂, ..., tₙ}
- 每个租户的SLO: SLOᵢ = (IOPSᵢ, BWᵢ, LATENCYᵢ)
- 系统资源: R = (IOPS_max, BW_max)

保证条件:
1. 资源充足性: Σ IOPSᵢ ≤ IOPS_max ∧ Σ BWᵢ ≤ BW_max
2. 调度策略: 使用WFQ或令牌桶
3. 监控机制: 实时检测违规并触发限流

证明QoS可达:

Step 1: 资源分配
为每个租户分配权重: wᵢ = IOPSᵢ / IOPS_max
使用WFQ调度，则租户i获得:
  IOPSᵢ' = IOPS_max × (wᵢ / Σwⱼ) ≥ IOPSᵢ (当Σwⱼ ≤ 1)

Step 2: 延迟保证
使用EDF调度截止时间敏感的请求，确保:
  completion_time ≤ deadline

Step 3: 隔离保证
通过cgroup或硬件隔离，确保:
  performanceᵢ ≥ baselineᵢ × (1 - interference)

结论: 当资源充足且使用适当的调度策略时，QoS保证可达。
```
