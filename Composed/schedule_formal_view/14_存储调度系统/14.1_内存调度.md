# 14.1 内存调度

## 1. 概述

内存调度是存储系统架构中的关键层级，位于CPU硬件层和操作系统之间，负责管理DRAM内存资源的分配、访问优化和数据迁移。高效的内存调度对系统整体性能至关重要。

### 1.1 内存子系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    CPU核心                               │
└─────────────────────────────────────────────────────────┘
                           ↕
┌─────────────────────────────────────────────────────────┐
│              内存调度层 (Memory Scheduler)               │
│  ┌─────────────────────────────────────────────────┐   │
│  │  请求队列 → 调度仲裁 → 地址映射 → 时序控制        │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                           ↕
┌─────────────────────────────────────────────────────────┐
│              内存控制器 (Memory Controller)              │
│  ┌─────────────────────────────────────────────────┐   │
│  │  命令队列 | 行缓冲管理 | 刷新控制 | 电源管理       │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                           ↕
┌─────────────────────────────────────────────────────────┐
│              物理DRAM (DDR4/DDR5/HBM)                    │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐               │
│  │  Channel │ │  Channel │ │  Channel │ ...           │
│  │  DIMM 0  │ │  DIMM 1  │ │  DIMM 2  │               │
│  └──────────┘ └──────────┘ └──────────┘               │
└─────────────────────────────────────────────────────────┘
```

### 1.2 内存调度目标

| 目标 | 描述 | 关键指标 |
|------|------|----------|
| **高带宽** | 最大化数据传输率 | GB/s |
| **低延迟** | 最小化内存访问延迟 | ns |
| **公平性** | 各请求源获得合理份额 | 延迟方差 |
| **能效** | 降低每比特传输能耗 | pJ/bit |

## 2. 内存访问特性

### 2.1 DRAM命令序列

**读取操作**：

```
时钟周期: 0    5    10   15   20   25
          |    |    |    |    |    |
行激活:   ACT  ─────────────────────────→
列读取:        RD   ─────────────────────→
预充电:                  PRE  ────────────→
数据传输:                     D0 D1 D2 D3  (突发传输)
总延迟:   tRCD + CL + 突发周期
```

**关键时序参数**：

| 参数 | 描述 | DDR4-3200 | DDR5-6400 |
|------|------|-----------|-----------|
| tRCD | 行到列延迟 | 14.06 ns | 13.75 ns |
| CL | CAS延迟 | 14.06 ns | 13.75 ns |
| tRP | 行预充电 | 14.06 ns | 13.75 ns |
| tRAS | 行激活时间 | 32 ns | 32 ns |

### 2.2 行缓冲局部性

**行命中（Row Hit）**：访问已激活行的列，延迟最低
**行冲突（Row Conflict）**：需关闭当前行、激活新行，延迟高
**行未命中（Row Miss）**：直接激活新行

```
访问模式影响:

顺序访问 (高行缓冲命中率):
激活 Row0 → 读 Col0 → 读 Col1 → 读 Col2 → ...
延迟:  tACT + n × tBurst

随机访问 (低行缓冲命中率):
激活 Row0 → 读 Col → 激活 Row1 → 读 Col → 激活 Row2
延迟:  n × (tACT + tRCD + CL)
```

## 3. 内存调度算法

### 3.1 先到先服务（FCFS）

**特点**：

- 按请求到达顺序服务
- 实现简单
- 公平性好
- 无法优化行缓冲命中率

### 3.2 FR-FCFS（First Ready, First Come First Serve）

**原理**：

1. 优先调度**行命中**的请求（Ready）
2. 行命中请求间按FCFS顺序

**优势**：

- 最大化行缓冲命中率
- 显著提升内存带宽

**形式化**：
$$\text{优先级}(r) = \begin{cases} 2 & \text{if } \text{row}(r) = \text{row}_{open} \\ 1 & \text{otherwise} \end{cases}$$

**缺点**：

- 可能导致行冲突请求的饥饿
- 公平性问题

### 3.3 并行感知调度（PAR-BS）

**目标**：在多核系统中平衡性能和公平性。

**机制**：

1. **批量处理**：收集一批请求统一调度
2. **最短作业优先**：在批内优先服务短请求
3. **优先级累积**：防止饥饿

### 3.4 公平内存调度（FST）

**原理**：

- 为每个线程维护一个信用值
- 服务请求时消耗信用
- 信用不足时降低优先级

### 3.5 智能调度器

**机器学习优化**：

```
特征提取 → 神经网络 → 调度决策
  ↓           ↓          ↓
行状态    预测收益    选择请求
历史模式    延迟估计    最优命令
```

## 4. 内存控制器优化

### 4.1 命令重排序

**重排序窗口**：在命令队列中重排请求顺序

**优化目标**：

- 最大化行缓冲命中率
- 最小化总线空闲周期
- 满足实时性约束

### 4.2 请求合并

**写合并**：

```
原始请求:
Write Addr=0x1000 Data=A
Write Addr=0x1004 Data=B
Write Addr=0x1008 Data=C

合并后:
Write Burst Addr=0x1000 Data=[A,B,C,...]
```

**读合并**：相邻地址的读请求合并为突发传输

### 4.3 预取策略

**顺序预取**：

- 检测顺序访问模式
- 提前激活下一行

**自适应预取**：

```
if 访问Addr, Addr+1, Addr+2:
    预取 Addr+3, Addr+4
if 预取命中率 < 阈值:
    减少预取深度
```

### 4.4 写缓冲优化

**写合并缓冲**：

- 缓存写请求
- 合并对同一地址的多次写入
- 延迟写回（Write-back）

**写 draining**：

- 读请求到来时，先刷新相关写缓冲
- 保证数据一致性

## 5. 多通道与交织

### 5.1 通道交织

**原理**：将连续地址分布到多个通道，实现并行访问

```
地址映射:
物理地址 → [通道号 | 行号 | 列号 | 块内偏移]
              ↑
         由低位地址决定

示例 (4通道):
Addr 0x0000 → Channel 0
Addr 0x0040 → Channel 1
Addr 0x0080 → Channel 2
Addr 0x00C0 → Channel 3
Addr 0x0100 → Channel 0
```

**优势**：

- 提高带宽利用率
- 降低通道间干扰

### 5.2 Bank交织

**Bank级并行**：

- 同一通道内多个Bank可独立操作
- 交错访问不同Bank隐藏预充电延迟

```
时间线:
Bank0: ACT ── RD ── PRE ──
Bank1:      ACT ── RD ── PRE ──
Bank2:           ACT ── RD ── PRE ──
```

## 6. 3D堆叠内存（HBM）

### 6.1 HBM架构

```
┌─────────────────────────────────────┐
│           逻辑层 (Base Die)          │
│  ┌─────────────────────────────┐   │
│  │      内存控制器 + PHY        │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
                  ↑ TSV (硅通孔)
┌─────────────────────────────────────┐
│           存储层 (DRAM Dies)         │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐  │
│  │Bank0│ │Bank1│ │Bank2│ │Bank3│  │ Die 0
│  └─────┘ └─────┘ └─────┘ └─────┘  │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐  │
│  │Bank0│ │Bank1│ │Bank2│ │Bank3│  │ Die 1
│  └─────┘ └─────┘ └─────┘ └─────┘  │
│  ...                               │
└─────────────────────────────────────┘
```

### 6.2 HBM调度特点

| 特性 | DDR4 | HBM2 | HBM3 |
|------|------|------|------|
| 通道数 | 2 | 8 | 16 |
| 每通道Bank数 | 16 | 16 | 16 |
| 引脚数量 | 288 | 1024 | 2048 |
| 带宽/堆栈 | 25.6 GB/s | 307 GB/s | 819 GB/s |

**调度优化**：

- 充分利用大量Bank的并行性
- 优化TSV带宽分配
- 考虑温度管理（热点避免）

## 7. 异构内存系统

### 7.1 非易失性内存（NVM）

**特性对比**：

| 特性 | DRAM | NVM (Optane) |
|------|------|--------------|
| 读取延迟 | 50-100ns | 100-300ns |
| 写入延迟 | 50-100ns | 100ns-1μs |
| 持久性 | 易失 | 非易失 |
| 容量 | 小 | 大 |
| 字节寻址 | 是 | 是 |

### 7.2 混合内存调度

**数据迁移策略**：

```
热点数据 ──→ DRAM (快速访问)
  ↑↓
冷数据  ←─── NVM (大容量存储)
```

**调度策略**：

- 热数据优先放入DRAM
- 异步迁移冷数据到NVM
- 预取预测数据

## 8. 代码示例

### 8.1 内存访问模式优化

```c
// 差：非连续访问，缓存未命中率高
void bad_access(float **matrix, int n) {
    for (int j = 0; j < n; j++)
        for (int i = 0; i < n; i++)
            matrix[i][j] += 1;  // 列优先，跨行访问
}

// 好：连续访问，高缓存命中率
void good_access(float **matrix, int n) {
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            matrix[i][j] += 1;  // 行优先，连续访问
}
```

### 8.2 内存带宽测试

```c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>
#include <string.h>

#define SIZE (1024 * 1024 * 256)  // 256MB

double benchmark_memcpy(size_t size) {
    char *src = aligned_alloc(64, size);
    char *dst = aligned_alloc(64, size);

    // 预热
    memcpy(dst, src, size);

    struct timespec start, end;
    clock_gettime(CLOCK_MONOTONIC, &start);

    // 执行10次取平均
    for (int i = 0; i < 10; i++)
        memcpy(dst, src, size);

    clock_gettime(CLOCK_MONOTONIC, &end);

    double seconds = (end.tv_sec - start.tv_sec) +
                     (end.tv_nsec - start.tv_nsec) / 1e9;
    double bandwidth = (10.0 * size) / (seconds * 1e9);  // GB/s

    free(src); free(dst);
    return bandwidth;
}

int main() {
    printf("内存带宽测试:\n");
    for (size_t size = 64 * 1024; size <= SIZE; size *= 2) {
        double bw = benchmark_memcpy(size);
        printf("大小: %8zu KB, 带宽: %.2f GB/s\n", size / 1024, bw);
    }
    return 0;
}
```

### 8.3 内存延迟测试

```c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

// 链表遍历，测量随机访问延迟
struct Node {
    struct Node *next;
    char padding[64 - sizeof(struct Node *)];  // 64B缓存行对齐
};

double measure_latency(int num_nodes) {
    struct Node *nodes = aligned_alloc(64, num_nodes * sizeof(struct Node));

    // 创建随机遍历链表（破坏预取）
    int *indices = malloc(num_nodes * sizeof(int));
    for (int i = 0; i < num_nodes; i++) indices[i] = i;

    // Fisher-Yates shuffle
    srand(42);
    for (int i = num_nodes - 1; i > 0; i--) {
        int j = rand() % (i + 1);
        int tmp = indices[i]; indices[i] = indices[j]; indices[j] = tmp;
    }

    // 构建链表
    for (int i = 0; i < num_nodes; i++)
        nodes[indices[i]].next = &nodes[indices[(i+1) % num_nodes]];

    struct Node *p = &nodes[0];

    // 预热缓存
    for (int i = 0; i < num_nodes; i++) p = p->next;

    struct timespec start, end;
    clock_gettime(CLOCK_MONOTONIC, &start);

    // 遍历多次
    int iterations = 10000000 / num_nodes + 1;
    for (int i = 0; i < iterations * num_nodes; i++) p = p->next;

    clock_gettime(CLOCK_MONOTONIC, &end);

    double seconds = (end.tv_sec - start.tv_sec) +
                     (end.tv_nsec - start.tv_nsec) / 1e9;
    double latency_ns = (seconds * 1e9) / (iterations * num_nodes);

    free(nodes); free(indices);
    return latency_ns;
}

int main() {
    printf("内存延迟测试:\n");
    int sizes[] = {1000, 10000, 100000, 1000000, 10000000};
    for (int i = 0; i < 5; i++) {
        double lat = measure_latency(sizes[i]);
        printf("工作集: %8d 节点, 延迟: %.2f ns\n", sizes[i], lat);
    }
    return 0;
}
```
