## 1 概述

### 1.1 目的

本术语表旨在统一 **形式语言视角** 和 **调度视角** 之间的术语使用，确保跨视角文档的一致性和可读性。

### 1.2 使用说明

- **首选术语**：在两个视角中优先使用的统一术语
- **形式语言术语**：类型理论、范畴论相关术语
- **调度术语**：系统调度、资源管理相关术语
- **映射说明**：两个视角间的概念对应关系

## 3 形式语言视角术语

### 3.1 类型系统术语

| 术语 | 英文 | 符号 | 定义 | 示例 |
|------|------|------|------|------|
| **基本类型** | Primitive Type | τ_base | 不可再分的类型 | int, string, bool |
| **复合类型** | Composite Type | τ₁ × τ₂ | 由基本类型组合而成 | struct, tuple |
| **函数类型** | Function Type | τ₁ → τ₂ | 接受输入产生输出的类型 | A → B |
| **泛型** | Generic | ∀α.τ | 参数化的类型 | List\<T\> |
| **依赖类型** | Dependent Type | Π(x:A).B(x) | 依赖值的类型 | Vec(n) |
| **线性类型** | Linear Type | τ⊸ | 必须使用恰好一次的类型 | Owned\<T\> |
| **仿射类型** | Affine Type | τ? | 至多使用一次的类型 | Rust默认 |

### 3.2 范畴论术语

| 术语 | 英文 | 符号 | 定义 | 示例 |
|------|------|------|------|------|
| **范畴** | Category | C | 对象和态射的集合 | Set, Type |
| **函子** | Functor | F: C → D | 范畴间的映射 | List, Option |
| **自然变换** | Natural Transformation | η: F ⇒ G | 函子间的映射 | flatten |
| **单子** | Monad | M | 特殊的函子 | IO, State |
| **幺半群** | Monoid | (M, ⊕, e) | 有单位元的半群 | String, List |

### 3.3 证明论术语

| 术语 | 英文 | 符号 | 定义 |
|------|------|------|------|
| **判断** | Judgment | Γ ⊢ e : τ | 类型断言 |
| **推导** | Derivation | - | 证明过程 |
| **健全性** | Soundness | - | 类型系统的正确性 |
| **完备性** | Completeness | - | 类型系统的完整性 |
| **可判定性** | Decidability | - | 类型检查可终止 |

## 4 调度视角术语

### 4.1 资源管理术语

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|
| **资源** | Resource | 可分配的系统实体 | CPU、内存、GPU |
| **配额** | Quota | 资源的分配上限 | ResourceQuota |
| **请求** | Request | 资源的最低需求 | requests.cpu |
| **限制** | Limit | 资源的使用上限 | limits.memory |
| **分配** | Allocation | 资源的分配动作 | 调度分配 |
| **回收** | Reclaim | 资源的释放动作 | GC、Finalizer |

### 4.2 调度术语

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|
| **调度器** | Scheduler | 决定资源分配的组件 | kube-scheduler |
| **调度策略** | Scheduling Policy | 分配决策的规则 | FIFO、优先级 |
| **亲和性** | Affinity | 调度偏好 | nodeAffinity |
| **抢占** | Preemption | 高优先级替代低优先级 | Pod抢占 |
| **驱逐** | Eviction | 强制终止运行中的实体 | 资源不足驱逐 |

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|

| **互斥** | Mutual Exclusion | 排他访问 | Mutex |
| **死锁** | Deadlock | 循环等待 | 资源竞争 |
| **竞态** | Race Condition | 非确定性行为 | 数据竞争 |
| **原子性** | Atomicity | 不可分割的操作 | CAS |

## 5 跨视角概念对照

### 5.1 类型系统 ↔ 资源管理

```text
映射关系:

类型系统                    资源管理
─────────────────────────────────────────────────
Type τ              ←→     Resource R
  - 基本类型                - 原子资源 (CPU, Memory)
  - 复合类型                - 复合资源 (Pod, Deployment)

Type Checking       ←→     Validation
  - 静态检查                - Admission Control
  - 运行时检查              - Runtime Validation

Type Inference      ←→     Auto-Configuration
  - 推断类型                - 默认值推断
  - 上下文推断              - 环境感知配置

Subtyping          ←→     Resource Hierarchy
  - τ₁ <: τ₂               - 资源继承关系
  - 协变/逆变              - 向上/向下兼容
```

### 5.2 所有权 ↔ 资源生命周期

```text
Rust所有权                  K8s资源管理
─────────────────────────────────────────────────
Owned<T>            ←→     ownerReference
  - 唯一所有者              - 所有者对象
  - 移动语义                - 级联删除

&T (借用)           ←→     共享访问
  - 不可变引用              - 只读ConfigMap
  - 多个借用                - 多Pod引用

&mut T              ←→     独占访问
  - 可变引用                - Leader选举
  - 排他访问                - 分布式锁

Drop                ←→     Finalizer
  - 自动析构                - 清理钩子
  - RAII                    - 资源保护

'a (生命周期)       ←→     Lease/TTL
  - 借用有效期              - 租约时间
  - 子类型关系              - 续约机制
```

### 5.3 效应系统 ↔ 状态管理

```text
效应系统                    状态管理
─────────────────────────────────────────────────
Pure Function       ←→     Stateless Service
  - 无副作用                - 无状态Pod
  - 幂等性                  - 可水平扩展

IO Monad           ←→     StatefulSet
  - 封装副作用              - 有状态工作负载
  - 顺序执行                - 有序部署

State Monad        ←→     Persistent State
  - 状态传递                - PersistentVolume
  - 状态转换                - 状态持久化

Reader Monad       ←→     ConfigMap/Secret
  - 只读环境                - 配置注入
  - 依赖注入                - 环境变量
```

### 5.4 并发模型 ↔ 分布式调度

```text
并发模型                    分布式调度
─────────────────────────────────────────────────
Thread              ←→     Pod
  - 执行单元                - 调度单元
  - 共享内存                - 网络通信

Mutex               ←→     Distributed Lock
  - 互斥锁                  - etcd锁
  - 临界区                  - Leader选举

Channel             ←→     Message Queue
  - 消息传递                - Kafka/SQS
  - 缓冲通道                - 异步通信

Future/Promise      ←→     Job
  - 异步计算                - 批处理任务
  - 等待结果                - 完成状态
```

## 6 符号约定

### 6.1 类型系统符号

| 符号 | 含义 | 示例 |
|------|------|------|
| τ | 类型变量 | τ = int |
| Γ | 类型上下文 | Γ = {x: int, y: bool} |
| ⊢ | 类型判断 | Γ ⊢ e : τ |
| → | 函数类型 | int → bool |
| × | 乘积类型 | A × B |
| + | 和类型 | A + B |
| ∀ | 全称量化 | ∀α. α → α |
| ∃ | 存在量化 | ∃α. α × (α → int) |
| <: | 子类型 | int <: number |

### 6.2 范畴论符号

| 符号 | 含义 | 示例 |
|------|------|------|
| C, D | 范畴 | Set, Type |
| F, G | 函子 | F: C → D |
| η, ε | 自然变换 | η: F ⇒ G |
| ∘ | 态射组合 | g ∘ f |
| id | 恒等态射 | id_A: A → A |
| ≅ | 同构 | A ≅ B |

### 6.3 调度符号

| 符号 | 含义 | 示例 |
|------|------|------|
| R | 资源 | R = {cpu, mem} |
| S | 调度器 | S: Tasks → Nodes |
| λ | 到达率 | λ = 100 req/s |
| μ | 服务率 | μ = 150 req/s |
| ρ | 利用率 | ρ = λ/μ |
| W | 等待时间 | W = ρ/(μ-λ) |
