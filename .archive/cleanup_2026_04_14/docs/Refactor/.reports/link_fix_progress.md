## 修复策略

### 1. 自动路径匹配修复 (721 个)

对于指向不存在的文件但同名文件存在于其他位置的链接，通过以下规则自动重定向：

- **同级目录重定向**: 修正相对路径中的 `../` 和 `./` 错误
- **文件名匹配**: 当完整路径不存在时，在整个 `docs/Refactor/` 目录树中搜索同名文件
- **目录索引补全**: 对于指向目录的链接（如 `../../02_形式语言/`），自动补全为目录下的索引文件（优先 `_index.md`，其次 `README.md` 或 `00_目录与导航.md`）

**典型案例**:

- `01_数学基础/_index.md` 中的 `./02_形式语言/04_范畴论/04.1_范畴基本概念.md` → `../02_形式语言/04_范畴论/04.1_范畴基本概念.md`
- `14_形式语言学/00_目录与导航.md` 中的 `../../02_形式语言/` → `../../02_形式语言/_index.md`
- `07_交叉视角/04_研究前沿.md` 中的 `./03_编程范式/04_函数式编程/04.2_单子与函子.md` → `../03_编程范式/04_函数式编程/04.2_单子与函子.md`

### 2. 移除无法修复的链接 (103 个)

对于完全找不到对应目标文件的链接，采取**保守删除策略**：保留链接文本，仅移除链接语法，避免读者点击后进入 404。

**主要类别**:

- **模板占位符链接**: `.templates/` 中的示例路径（如 `../path/to/doc-a.md`）
- **根目录外部引用**: 指向 `docs/Refactor/` 外部的文件（如 `../LICENSE`、`../tutorials/network_scheduler.md`）
- **迁移/配置文档**: `DEPRECATIONS.md` 中指向不存在的 migration 文档
- **数学公式误识别**: 部分形如 `agent→formula`、`alice→p` 的伪链接语法（多见于 `13_认知科学形式模型/`）

## 发现的模式与问题

1. **大量 `_index.md` / `README.md` 路径错误**: 许多索引文件使用 `./章节/文件.md` 引用其他章节，但正确路径应为 `../章节/文件.md`（因为索引文件通常位于章节根目录，而被引用文件在同级目录中）。

2. **目录链接未补全索引文件**: 导航类文件习惯用 `../../02_形式语言/` 这种目录路径引用，但 Markdown 链接需要指向具体的 `.md` 文件。本次修复统一补全为 `_index.md` 或同级索引文件。

3. **跨模块引用大量指向不存在的章节**: `12_决策与博弈论/`、`07_交叉视角/` 等文件引用了如 `../02_形式化方法/`、`../09_概率论与统计/` 等目录，但 `docs/Refactor/` 中实际不存在这些目录名（实际目录名为 `02_形式语言`、`09_统计学` 等）。这些引用属于早期规划阶段的残留。

4. **数学/逻辑符号被误识别为 Markdown 链接**: `13_认知科学形式模型/` 中的部分文档使用 `agent→formula`、`alice→p` 等伪链接语法表示形式化逻辑表达式，被链接检查器识别为断链。这些属于语法误报，已清理。
