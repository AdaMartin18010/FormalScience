## 简介

`document_quality_checker.py` 是 FormalScience 项目的文档质量自动化检查工具。它能够：

- 自动检测文档中的占位符和待补充内容
- 验证文档结构是否符合标准模板
- 检查交叉引用链接的有效性
- 评估元数据头的完整性
- 生成详细的检查报告

---

## 功能特性

| 功能 | 描述 | 严重程度 |
|------|------|----------|
| 🔴 占位符检测 | 检测 `- |- |- |- `、`待补充`、`TODO` 等占位符 | 警告 |
| 🔴 链接有效性 | 验证 Markdown 内部链接是否指向存在的文件 | 错误 |
| 🟡 元数据检查 | 检查必需的元数据字段是否完整 | 错误/警告 |
| 🟡 章节完整性 | 验证推荐章节是否存在 | 建议 |
| 🟡 导航检查 | 检查文档导航是否完整 | 警告 |
| 🔵 思维表征 | 验证 6 种思维表征图表 | 建议 |
| 🔵 目录检查 | 验证目录结构完整性 | 警告 |

### 环境要求

## 基本使用

### 1. 检查单个文件

```bash
python tools/document_quality_checker.py Concept/README.md
```

### 2. 检查整个目录

```bash
# 非递归检查（仅当前目录）
python tools/document_quality_checker.py Concept/

# 递归检查（包含子目录）
python tools/document_quality_checker.py -r Concept/
```

### 3. 生成不同格式的报告

```bash
# Markdown 格式报告
python tools/document_quality_checker.py -f markdown -o report.md Concept/

# JSON 格式（用于程序处理）
python tools/document_quality_checker.py -f json -o report.json Concept/

# 控制台格式（默认）
python tools/document_quality_checker.py Concept/
```

### 4. 详细输出模式

```bash
python tools/document_quality_checker.py -v Concept/
```

## 高级用法

### 1. 标准化元数据头

自动为所有文档添加标准元数据头：

```bash
# 单个文件
python tools/document_quality_checker.py --normalize Concept/README.md

# 整个目录（递归）
python tools/document_quality_checker.py --normalize -r Concept/
```

标准化后的元数据头格式：

```markdown
# 文档标题

> **文档版本**: v1.0.0
> **创建日期**: 2026-04-12
> **最后更新**: 2026-04-12
> **文档规模**: 估算中 | 待评估
> **主题**: 待补充
> **难度**: ⭐⭐⭐
> **前置知识**: 待补充
> **重要性**: ⭐⭐⭐⭐⭐
> **阅读建议**: 本文档适合有相关基础的读者

---
```

### 2. CI/CD 集成

在 CI/CD 管道中使用，当发现错误时返回非零状态码：

```yaml
# .github/workflows/docs-quality.yml
name: Document Quality Check

on: [push, pull_request]

jobs:
  quality-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      - name: Check document quality
        run: python tools/document_quality_checker.py -r Concept/
```

### 3. 批量修复建议

生成报告后，根据报告批量修复问题：

```bash
# 生成详细报告
python tools/document_quality_checker.py -f markdown -o quality_report.md -r Concept/

# 根据报告中的建议逐个修复
```

## 检查规则说明

### 元数据字段要求

#### 必需字段（🔴 错误级）

| 字段 | 说明 | 示例 |
|------|------|------|
| 文档版本 | 语义化版本号 | v1.0.0 |
| 最后更新 | 日期格式 YYYY-MM-DD | 2026-04-12 |
| 主题 | 一句话描述 | 递归可枚举性理论 |

#### 推荐字段（🟡 警告级）

| 字段 | 说明 | 示例 |
|------|------|------|
| 文档规模 | 行数和描述 | 500行 \| 理论深度 |
| 阅读建议 | 目标读者建议 | 适合有计算理论基础的读者 |
| 创建日期 | 初始创建日期 | 2026-01-15 |
| 难度 | 1-5星 | ⭐⭐⭐ |
| 前置知识 | 需要的前置知识 | Lambda演算、图灵机 |
| 重要性 | 1-5星 | ⭐⭐⭐⭐⭐ |

### 占位符检测规则

以下模式会被识别为占位符：

```
- |- |- |- |          # 表格占位符
待补充                 # 待补充标记
待完善                 # 待完善标记
TODO                   # TODO标记
FIXME                  # FIXME标记
XXX                    # XXX占位符
[待补充]               # 带括号的待补充
[TODO]                 # 带括号的TODO
```

### 章节结构要求

推荐的文档结构（根据 FormalRE 标准）：

```markdown
1. 核心理论章节
   1.1 子章节
       1.1.1 概念分析
2. 理论展开章节
3. 证明/构造章节
4. 应用/实例章节
5. 批判性分析
6. 思维表征
   6.1 概念关系网络图
   6.2 论证逻辑路径图
   6.3 概念属性矩阵
   6.4 外延内涵分析图
   6.5 理论发展脉络图
   6.6 跨模块关联图
7. 权威资源对标
8. 主题-子主题论证逻辑关系图
9. 参考资源
10. 相关主题
```

## 报告解读

### 控制台报告示例

```
============================================================
文档质量检查报告
============================================================

检查文件数: 10
总问题数: 25
  🔴 错误: 3
  🟡 警告: 12
  🔵 建议: 10

------------------------------------------------------------
问题详情:
------------------------------------------------------------

📄 Concept/README.md
  🔴 行15: [元数据缺失] 缺少必需元数据字段: 文档版本
     💡 建议: 在文档头部添加: > **文档版本**: v1.0.0
  🟡 行45: [占位符] 发现占位符: 待补充
     💡 建议: 请补充实际内容
```

### Markdown 报告结构

生成的 Markdown 报告包含：

1. **质量概览** - 问题统计
2. **问题列表** - 按文件分组的问题详情
3. **文档指标** - 每个文件的元数据、章节、链接等统计

### 严重程度说明

| 图标 | 级别 | 说明 | 处理建议 |
|------|------|------|----------|
| 🔴 | 错误 | 必须修复的问题 | 立即修复 |
| 🟡 | 警告 | 建议修复的问题 | 优先修复 |
| 🔵 | 建议 | 可选的改进建议 | 有时间再处理 |

## 常见问题

### Q1: 如何处理大量的"链接失效"错误？

**A**: 链接失效通常是因为：

1. 示例链接（如 `./文件名.md`）需要替换为实际链接
2. 相对路径计算错误，检查 `../` 的数量
3. 目标文件确实不存在，需要创建

修复建议：

```bash
# 使用标准化功能先修复元数据
python tools/document_quality_checker.py --normalize -r Concept/

# 然后手动检查并修复链接
```

### Q2: 如何忽略某些特定的警告？

**A**: 目前工具不支持忽略特定警告。建议在 CI/CD 中只检查错误级别：

```bash
# 检查退出码，只有错误时才会非零
python tools/document_quality_checker.py Concept/ || echo "有错误需要修复"
```

### Q3: 如何自定义检查规则？

**A**: 编辑 `tools/document_quality_checker.py` 文件：

- `PLACEHOLDER_PATTERNS` - 占位符模式
- `REQUIRED_METADATA_FIELDS` - 必需元数据字段
- `RECOMMENDED_METADATA_FIELDS` - 推荐元数据字段
- `RECOMMENDED_SECTIONS` - 推荐章节

### Q4: 报告中的文档指标如何解读？

**A**:

| 指标 | 说明 | 健康标准 |
|------|------|----------|
| 元数据 | 是否有标准元数据头 | ✅ 必须有 |
| 目录 | 是否有目录章节 | ✅ 建议有 |
| 导航 | 是否有上下节导航 | ✅ 建议有 |
| 章节数 | 一级章节数量 | ≥3 较好 |
| 链接数 | 交叉引用链接数量 | ≥5 较好 |
| 占位符 | 待补充内容数量 | 0 最佳 |
