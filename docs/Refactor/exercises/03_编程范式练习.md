## 一、Rust所有权分析（5题）

### 第1题 ⭐ 所有权基础

**题目**：分析以下Rust代码，指出编译错误并修正：

```rust
fn main() {
    let s = String::from("hello");
    let t = s;
    println!("{}", s);
}
```

<details>
<summary>💡 提示</summary>

String类型在赋值时发生所有权转移（move），原变量失效。
</details>

<details>
<summary>✅ 解答</summary>

**错误分析**：

```rust
let s = String::from("hello");  // s拥有字符串的所有权
let t = s;                       // 所有权从s转移到t，s失效
println!("{}", s);               // 错误：s的值已被移动
```

编译错误：`borrow of moved value: s`

**修正方案1：克隆**（需要两份数据时）

```rust
fn main() {
    let s = String::from("hello");
    let t = s.clone();  // 深拷贝，s和t各自拥有数据
    println!("{} {}", s, t);
}
```

**修正方案2：借用**（只需要读取时）

```rust
fn main() {
    let s = String::from("hello");
    let t = &s;  // t借用s，s仍拥有所有权
    println!("{} {}", s, t);
}
```

**修正方案3：转移后使用新变量**

```rust
fn main() {
    let s = String::from("hello");
    let t = s;
    println!("{}", t);  // 使用t而非s
}
```

> 所有权规则：每个值有且只有一个所有者，当所有者离开作用域，值被丢弃。
</details>

### 第2题 ⭐⭐ 借用与生命周期

**题目**：分析以下代码的问题，并给出两种修正方案：

```rust
fn get_longest(s1: &str, s2: &str) -> &str {
    if s1.len() > s2.len() { s1 } else { s2 }
}

fn main() {
    let s1 = String::from("short");
    let result;
    {
        let s2 = String::from("longer string");
        result = get_longest(&s1, &s2);
    }
    println!("{}", result);
}
```

<details>
<summary>✡️ 提示</summary>

返回值的生命周期必须与输入参数关联。当前实现中result可能引用已释放的s2。
</details>

<details>
<summary>✅ 解答</summary>

**错误分析**：

编译错误：`cannot return reference to function parameter 's2'`（需要显式生命周期注解）

即使添加生命周期，问题依然存在：result引用了s2，但s2在内层作用域结束时被释放。

**修正方案1：显式生命周期（如果s2生命周期足够长）**

```rust
fn get_longest<'a>(s1: &'a str, s2: &'a str) -> &'a str {
    if s1.len() > s2.len() { s1 } else { s2 }
}

fn main() {
    let s1 = String::from("short");
    let s2 = String::from("longer string");
    let result = get_longest(&s1, &s2);
    println!("{}", result);  // s1和s2在同一作用域，安全
}
```

**修正方案2：返回String（所有权转移）**

```rust
fn get_longest(s1: &str, s2: &str) -> String {
    if s1.len() > s2.len() { s1.to_string() } else { s2.to_string() }
}
```

> 生命周期确保引用不会比被引用的数据活得更长。
</details>

### 第3题 ⭐⭐ 可变借用与内部可变性

**题目**：实现一个函数，接收不可变引用但需要在内部修改计数。使用 `RefCell` 实现：

<details>
<summary>💡 提示</summary>

RefCell提供运行时借用检查，允许在不可变引用内部修改数据。
</details>

<details>
<summary>✅ 解答</summary>

```rust
use std::cell::RefCell;

struct Counter {
    count: RefCell<i32>,
}

impl Counter {
    fn new() -> Self {
        Counter { count: RefCell::new(0) }
    }

    fn increment(&self) {
        *self.count.borrow_mut() += 1;
    }

    fn get(&self) -> i32 {
        *self.count.borrow()
    }
}

fn main() {
    let counter = Counter::new();
    counter.increment();
    counter.increment();
    println!("Count: {}", counter.get());  // 输出: 2
}
```

**内部可变性模式对比**：

| 类型 | 运行时检查 | 线程安全 | 适用场景 |
|------|-----------|---------|---------|
| `RefCell<T>` | 是 | 否 | 单线程内部可变性 |
| `Cell<T>` | 否 | 否 | 单线程简单类型 |
| `Mutex<T>` | 是 | 是 | 多线程互斥访问 |
| `RwLock<T>` | 是 | 是 | 多读单写 |

> RefCell将借用检查从编译时推迟到运行时，换取更大的灵活性。
</details>

### 第4题 ⭐⭐⭐ 自引用结构体

**题目**：解释以下代码的问题，并提供使用 `Pin` 的正确实现：

```rust
struct SelfRef {
    data: String,
    ptr_to_data: *const u8,
}
```

<details>
<summary>💡 提示</summary>

当SelfRef被移动时，data的地址改变，但ptr_to_data仍指向旧地址，导致悬垂指针。
</details>

<details>
<summary>✅ 解答</summary>

**问题**：移动SelfRef时data地址改变，但ptr_to_data仍指向旧位置。

**使用Pin的解决方案**：

```rust
use std::pin::Pin;
use std::marker::PhantomPinned;

struct SelfRef {
    data: String,
    ptr_to_data: *const u8,
    _pin: PhantomPinned,
}

impl SelfRef {
    fn new(data: String) -> Pin<Box<Self>> {
        let mut boxed = Box::pin(SelfRef {
            data,
            ptr_to_data: std::ptr::null(),
            _pin: PhantomPinned,
        });

        let ptr = unsafe { boxed.as_mut().get_unchecked_mut() };
        ptr.ptr_to_data = ptr.data.as_ptr();

        boxed
    }
}
```

**更安全的替代方案：使用索引**：

```rust
struct SafeSelfRef {
    data: String,
    start_idx: usize,
    end_idx: usize,
}
// 用索引代替指针，完全安全
```

> 自引用结构体是Rust中最复杂的模式之一，通常应寻求替代设计。
</details>

### 第5题 ⭐⭐⭐ 自定义智能指针

<summary>💡 提示</summary>

<summary>✅ 解答</summary>

```rust
use std::ops::Deref;

    fn new(x: T) -> MyBox<T> {
        println!("MyBox created");
        MyBox(x)

}
```

### 第6题 ⭐ Hindley-Milner推导

**题目**：为以下表达式推导最一般的类型：

```haskell
f x y = (x, y)
g = f True
```

<details>
<summary>💡 提示</summary>

f是一个函数，返回元组。g是f的部分应用。
</details>

<details>
<summary>✅ 解答</summary>

**推导 f**：

```haskell
f x y = (x, y)
```

- `(,)` 的类型：`a -> b -> (a, b)`
- 因此 `f :: a -> b -> (a, b)`

**推导 g**：

```haskell
g = f True
```

- `f :: a -> b -> (a, b)`
- `True :: Bool`，所以 `a = Bool`
- `g :: b -> (Bool, b)`

**最终类型**：

```haskell
f :: a -> b -> (a, b)
g :: b -> (Bool, b)
```

</details>

### 第9题 ⭐⭐⭐ 关联类型

**题目**：解释Rust中关联类型的用途，并实现一个带关联类型的Graph trait。

<details>
<summary>💡 提示</summary>

关联类型允许trait的实现者指定相关类型，每个实现只有一个关联类型映射。
</details>

<details>
<summary>✅ 解答</summary>

**关联类型 vs 泛型参数**：

| 特性 | 关联类型 | 泛型参数 |
|------|---------|---------|
| 实现数量 | 每个类型一个实现 | 多个实现 |
| 类型推断 | 更优 | 需显式指定 |

**Graph trait实现**：

```rust
trait Graph {
    type Node: Clone + Eq;
    type Edge;

    fn nodes(&self) -> Vec<Self::Node>;
    fn has_edge(&self, from: &Self::Node, to: &Self::Node) -> bool;
}

struct AdjListGraph {
    adj: Vec<Vec<usize>>,
}

impl Graph for AdjListGraph {
    type Node = usize;
    type Edge = (usize, usize);

    fn nodes(&self) -> Vec<Self::Node> {
        (0..self.adj.len()).collect()
    }

    fn has_edge(&self, from: &Self::Node, to: &Self::Node) -> bool {
        self.adj.get(*from).map_or(false, |n| n.contains(to))
    }
}
```

> 关联类型在标准库中广泛使用（如Iterator::Item）。
</details>

### 第10题 ⭐⭐⭐ 类型级编程

**题目**：使用Rust的类型系统实现编译时 Peano 自然数，并实现加法。

<details>
<summary>💡 提示</summary>

使用trait和impl在类型层面进行计算。递归处理Succ，基础情况处理Zero。
</details>

<details>
<summary>✅ 解答</summary>

```rust
use std::marker::PhantomData;

// Peano自然数定义
struct Zero;
struct Succ<N>(PhantomData<N>);

// 类型级加法trait
trait Add<RHS> {
    type Result;
}

// 0 + N = N
impl<N> Add<N> for Zero {
    type Result = N;
}

// Succ<M> + N = Succ<M + N>
impl<M, N> Add<N> for Succ<M>
where
    M: Add<N>,
{
    type Result = Succ<<M as Add<N>>::Result>;
}

// 获取类型对应的数值
trait ToUsize {
    const VALUE: usize;
}

impl ToUsize for Zero {
    const VALUE: usize = 0;
}

impl<N: ToUsize> ToUsize for Succ<N> {
    const VALUE: usize = 1 + N::VALUE;
}

fn main() {
    type Two = Succ<Succ<Zero>>;
    type One = Succ<Zero>;
    type Three = <Two as Add<One>>::Result;

    println!("2 + 1 = {}", Three::VALUE);  // 输出: 3
}
```

> 这是Rust类型系统能力的展示，实际项目中通常使用const generics。
</details>

### 第13题 ⭐⭐ 无锁数据结构

**题目**：使用 `AtomicUsize` 实现一个简单的无锁计数器，并解释内存序的选择。

<details>
<summary>💡 提示</summary>

原子操作需要选择内存序：Relaxed、Acquire/Release、SeqCst。
</details>

<details>
<summary>✅ 解答</summary>

```rust
use std::sync::atomic::{AtomicUsize, Ordering};
use std::sync::Arc;
use std::thread;

struct LockFreeCounter {
    count: AtomicUsize,
}

impl LockFreeCounter {
    fn new() -> Self {
        Self { count: AtomicUsize::new(0) }
    }

    fn increment(&self) {
        self.count.fetch_add(1, Ordering::Relaxed);
    }

    fn get(&self) -> usize {
        self.count.load(Ordering::Relaxed)
    }
}

fn main() {
    let counter = Arc::new(LockFreeCounter::new());
    let mut handles = vec![];

    for _ in 0..10 {
        let c = Arc::clone(&counter);
        handles.push(thread::spawn(move || {
            for _ in 0..1000 { c.increment(); }
        }));
    }

    for h in handles { h.join().unwrap(); }
    println!("Final count: {}", counter.get());
}
```

**内存序选择**：

| 内存序 | 用途 | 开销 |
|--------|------|------|
| `Relaxed` | 单纯的原子操作 | 最低 |
| `Acquire/Release` | 线程间同步 | 中等 |
| `SeqCst` | 全局顺序一致性 | 最高 |

> 无锁编程需要仔细考虑内存序，通常Relaxed足够。
</details>

### 第14题 ⭐⭐⭐ 异步编程

**题目**：实现一个异步函数，使用 `tokio` 并发执行多个HTTP请求，并处理第一个完成的响应。

<details>
<summary>💡 提示</summary>

使用 `tokio::select!` 或 `futures::future::select` 等待多个future。
</details>

<details>
<summary>✅ 解答</summary>

```rust
use tokio::time::{sleep, Duration};

async fn fetch_url(url: &str) -> Result<String, String> {
    let delay = match url {
        "https://api.a.com" => 100,
        "https://api.b.com" => 50,
        _ => 150,
    };
    sleep(Duration::from_millis(delay)).await;
    Ok(format!("Response from {}", url))
}

use tokio::task::JoinSet;

async fn fetch_first(urls: Vec<&str>) -> Result<String, String> {
    let mut set = JoinSet::new();

    for url in urls {
        set.spawn(fetch_url(url));
    }

    // 返回第一个成功的结果
    while let Some(result) = set.join_next().await {
        match result {
            Ok(Ok(response)) => return Ok(response),
            Ok(Err(_)) => continue,  // 请求失败，继续等待其他
            Err(_) => continue,       // 任务 panic，继续
        }
    }

    Err("All requests failed".to_string())
}

#[tokio::main]
async fn main() {
    let urls = vec![
        "https://api.a.com",
        "https://api.b.com",
        "https://api.c.com",
    ];

    match fetch_first(urls).await {
        Ok(resp) => println!("First response: {}", resp),
        Err(e) => println!("Error: {}", e),
    }
}
```

**tokio::select!版本**：

```rust
async fn fetch_first_select(urls: Vec<&str>) -> Result<String, String> {
    let mut futures: Vec<_> = urls.iter()
        .map(|&url| fetch_url(url))
        .collect();

    tokio::select! {
        result = futures.remove(0) => result,
        result = futures.remove(0) => result,
        result = futures.remove(0) => result,
    }
}
```

> 异步编程允许高效地处理I/O密集型任务。
</details>

### 第15题 ⭐⭐⭐ 并发模式：生产者-消费者

**题目**：使用Rust实现一个多生产者单消费者（MPSC）队列，使用标准库通道。

<details>
<summary>💡 提示</summary>

使用 `std::sync::mpsc` 或 `tokio::sync::mpsc`，处理多个生产者和一个消费者。
</details>

<details>
<summary>✅ 解答</summary>

```rust
use std::sync::mpsc;
use std::thread;
use std::time::Duration;

fn main() {
    let (tx, rx) = mpsc::channel();

    // 多个生产者
    for i in 0..3 {
        let tx = tx.clone();
        thread::spawn(move || {
            for j in 0..5 {
                let msg = format!("Producer {}: message {}", i, j);
                tx.send(msg).unwrap();
                thread::sleep(Duration::from_millis(100));
            }
        });
    }

    // 关闭原始发送者（克隆的还在）
    drop(tx);

    // 单消费者
    for msg in rx {
        println!("Received: {}", msg);
    }

    println!("All messages consumed");
}
```

**有界通道（避免内存无限增长）**：

```rust
use std::sync::mpsc::sync_channel;

fn main() {
    // 容量为10的有界通道
    let (tx, rx) = sync_channel::<i32>(10);

    thread::spawn(move || {
        for i in 0..100 {
            tx.send(i).unwrap();  // 当通道满时阻塞
        }
    });

    for msg in rx {
        println!("{}", msg);
    }
}
```

**tokio异步版本**：

```rust
use tokio::sync::mpsc;

#[tokio::main]
async fn main() {
    let (tx, mut rx) = mpsc::channel(100);  // 有界通道

    // 生产者任务
    let tx2 = tx.clone();
    tokio::spawn(async move {
        for i in 0..10 {
            tx2.send(i).await.unwrap();
        }
    });

    // 另一个生产者
    tokio::spawn(async move {
        for i in 10..20 {
            tx.send(i).await.unwrap();
        }
    });

    // 消费者
    drop(tx);  // 关闭所有发送者

    while let Some(msg) = rx.recv().await {
        println!("Received: {}", msg);
    }
}
```

> 生产者-消费者是并发编程的基础模式，Rust的通道提供了类型安全的消息传递。
