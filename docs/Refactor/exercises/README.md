## 🎯 难度分级说明

### ⭐ 基础（Beginner）

- **目标**：巩固核心概念，建立基础直觉
- **特点**：直接应用定义和基本定理
- **建议时间**：5-15分钟/题
- **通过标准**：能独立完成80%以上

### ⭐⭐ 进阶（Intermediate）

- **目标**：培养分析能力，理解概念关联
- **特点**：需要综合运用多个知识点，涉及简单推导
- **建议时间**：15-30分钟/题
- **通过标准**：在提示下能完成，理解解答思路

### ⭐⭐⭐ 挑战（Advanced）

- **目标**：培养形式化建模和解决复杂问题的能力
- **特点**：需要深度思考，可能涉及证明或编程实现
- **建议时间**：30-60分钟/题
- **通过标准**：理解解答并能复述核心思路

## 📊 自评标准

### 完成度评估

| 等级 | 完成比例 | 描述 |
|------|----------|------|
| 🥉 入门 | 30% | 了解基本概念，能识别但运用不熟练 |
| 🥈 熟练 | 60% | 能独立解决大部分问题，偶有错误 |
| 🥇 精通 | 90% | 能灵活运用，具备举一反三的能力 |
| 🏆 专家 | 100% | 完全掌握，能创造新的解法或扩展 |

### 各模块掌握标准

#### 数学基础

- **合格**：能正确进行集合运算、理解基本代数结构、完成简单证明
- **优秀**：能独立构造证明、理解不同证明技术的适用场景

#### 形式语言

- **合格**：能设计简单自动机、理解文法转换、阅读类型推导
- **优秀**：能分析复杂语言性质、实现简单的解析器、理解类型系统深层原理

#### 编程范式

- **合格**：能理解Rust所有权错误信息、阅读类型签名、使用并发原语
- **优秀**：能设计无数据竞争的并发程序、实现类型推导、优化所有权使用

### 学习建议

1. **循序渐进**：建议按顺序完成各模块，确保基础扎实
2. **独立思考**：先看题目独立思考，确实困难再看提示
3. **动手实践**：编程类题目务必实际编码运行
4. **总结反思**：完成练习后总结易错点和核心思路
5. **定期复习**：间隔一段时间重做标记为"困难"的题目

## 📝 使用指南

### 练习流程

```
1. 选择难度 → 2. 独立完成 → 3. 对照解答 → 4. 标记难点 → 5. 定期复习
```

### 标记约定

- ✅ 已掌握
- ⚠️ 需要复习
- ❌ 尚未理解
- ⭐ 重点题目

### 时间规划建议

| 学习目标 | 建议时间 | 每日练习量 |
|----------|----------|------------|
| 快速浏览 | 2周 | 5-8题 |
| 系统学习 | 1个月 | 3-5题 |
| 深度掌握 | 2个月 | 2-3题 + 复习 |
