### 第1题 ⭐⭐ 正则语言的代数性质

**题目背景**：
在编译器的词法分析器中，需要处理多种token模式。这些模式可以用正则表达式描述。

**问题**：

1. 证明：正则语言在并、交、补运算下封闭（只需证明并运算）
2. 设 $L_1$ 是标识符正则式 `[a-zA-Z_][a-zA-Z0-9_]*` 描述的语言，$L_2$ 是关键字 `if|then|else` 描述的语言。如何用集合运算表示"合法标识符但不能是关键字"？
3. 用泵引理说明为什么不能用正则表达式匹配嵌套括号

<details>
<summary>💡 提示</summary>

1. 使用乘积自动机构造
2. 集合的差运算
3. 泵引理要求正则语言的所有足够长字符串可被"泵"，但嵌套结构需要计数

</details>

<details>
<summary>✅ 解答</summary>

**1. 正则语言对并运算封闭**

**构造性证明**：

设 $L_1$ 被 $M_1 = (Q_1, \Sigma, \delta_1, q_{01}, F_1)$ 识别，
$L_2$ 被 $M_2 = (Q_2, \Sigma, \delta_2, q_{02}, F_2)$ 识别。

构造新NFA $M = (Q, \Sigma, \delta, q_0, F)$：

- $Q = Q_1 \cup Q_2 \cup \{q_0\}$（新增初始状态）
- $F = F_1 \cup F_2$
- $\delta(q_0, \varepsilon) = \{q_{01}, q_{02}\}$（非确定选择进入哪个自动机）
- 其他转移继承自 $M_1$ 和 $M_2$

则 $L(M) = L_1 \cup L_2$。∎

**2. 集合运算**

```
合法标识符但不能是关键字 = L₁ ∩ L₂的补集 = L₁ \ L₂

用正则表达式：
[a-zA-Z_][a-zA-Z0-9_]*  且不是  (if|then|else)
```

**实际实现**：词法分析器通常先识别最长匹配，再查关键字表。

**3. 嵌套括号非正则**

设 $L = \{a^n b^n \mid n \geq 0\}$（类比嵌套结构）。

由泵引理，若 $L$ 正则，存在泵长度 $p$。取 $w = a^p b^p$。

$w = xyz$，$|xy| \leq p$，$|y| > 0$，则 $y = a^k$（$k > 0$）。

$xy^2z = a^{p+k} b^p \notin L$（$a$ 和 $b$ 数量不等）。

矛盾！因此 $L$ 非正则，嵌套括号需要下推自动机（上下文无关文法）。∎

> 这就是为什么现代词法分析器（正则）和语法分析器（上下文无关）分离的原因。
</details>

### 第2题 ⭐⭐⭐ 类型系统的代数模型

**题目背景**：
在函数式语言中，类型可以看作代数结构。积类型（元组）对应乘法，和类型（枚举）对应加法。

**问题**：

1. 解释为什么 `Option<T>` 对应代数中的 $1 + T$（其中1是单位类型）
2. 证明类型等式 `(A, Either<B, C>)` ≅ `(Either<(A,B), (A,C)>)` 的代数对应
3. 在Rust中实现上述类型转换函数

<details>
<summary>💡 提示</summary>

1. Option有None和Some两个变体，None对应单位类型
2. 代数对应：乘法对加法的分配律
3. 实现两个方向的转换函数

</details>

<details>
<summary>✅ 解答</summary>

**1. Option<T> = 1 + T**

```rust
enum Option<T> {
    None,      // 单位类型 ()，只有一个值
    Some(T),   // 类型 T
}
```

- `None`：恰好1个值，对应代数1
- `Some(T)`：|T| 个值，对应代数 T
- 总共：1 + |T| 个值，对应 1 + T

**2. 代数证明**

左式：`(A, Either<B, C>)` = $A \times (B + C)$

右式：`(Either<(A,B), (A,C)>)` = $A \times B + A \times C$

等式：$A \times (B + C) = A \times B + A \times C$

这是乘法对加法的**分配律**！∎

**3. Rust实现**

```rust
enum Either<L, R> {
    Left(L),
    Right(R),
}

// 左式到右式
fn distribute_left<A, B, C>(
    pair: (A, Either<B, C>)
) -> Either<(A, B), (A, C)> {
    let (a, either) = pair;
    match either {
        Either::Left(b) => Either::Left((a, b)),
        Either::Right(c) => Either::Right((a, c)),
    }
}

// 右式到左式
fn distribute_right<A, B, C>(
    either: Either<(A, B), (A, C)>
) -> (A, Either<B, C>) {
    match either {
        Either::Left((a, b)) => (a, Either::Left(b)),
        Either::Right((a, c)) => (a, Either::Right(c)),
    }
}

// 证明是同构：两个方向互逆
fn test() {
    let left: (i32, Either<bool, char>) =
        (42, Either::Left(true));

    let right = distribute_left(left);
    let back = distribute_right(right);
    // back 应该等于原始 left
}
```

**代数数据类型（ADT）**：

| 类型构造 | 代数运算 | 示例 |
|---------|---------|------|
| 积类型 `(A, B)` | 乘法 $A \times B$ | 同时有A和B |
| 和类型 `Either<A,B>` | 加法 $A + B$ | 有A或有B |
| 函数类型 `A -> B` | 指数 $B^A$ | 从A到B的映射 |
| 单位类型 `()` | 1 | 只有一个值 |
| 空类型 `!` | 0 | 没有值 |

> 这就是**类型代数**，是Curry-Howard同构的基础。
</details>

### 第3题 ⭐⭐⭐ 形式化验证基础

**题目背景**：
在安全性关键的软件（如航空电子、医疗设备）中，需要形式化证明程序的正确性。

**问题**：

1. 用霍尔逻辑（Hoare Logic）描述和证明：交换两个变量的值
2. 解释部分正确性（Partial Correctness）和完全正确性（Total Correctness）的区别
3. 在Rust中，类型系统如何保证内存安全？这与霍尔逻辑的哪个方面对应？

<details>
<summary>💡 提示</summary>

1. 霍尔三元组 {P} C {Q}，使用中间断言
2. 完全正确性 = 部分正确性 + 终止性
3. Rust的所有权和借用系统对应霍尔逻辑中的资源管理

</details>

<details>
<summary>✅ 解答</summary>

**1. 交换两个变量的霍尔逻辑证明**

程序：

```
{t = x; x = y; y = t}
```

霍尔三元组：

```
{x = a ∧ y = b}
t = x; x = y; y = t
{x = b ∧ y = a}
```

**证明**：

```
{x = a ∧ y = b}
t = x        {t = a ∧ x = a ∧ y = b}
x = y        {t = a ∧ x = b ∧ y = b}
y = t        {t = a ∧ x = b ∧ y = a}
```

因此最终满足 $x = b \wedge y = a$。∎

**2. 部分正确性 vs 完全正确性**

| 特性 | 部分正确性 | 完全正确性 |
|------|-----------|-----------|
| 定义 | 若前置条件成立且程序终止，则后置条件成立 | 前置条件成立则程序终止且后置条件成立 |
| 包含终止性 | 否 | 是 |
| 证明难度 | 相对简单 | 需额外证明终止性 |
| 符号 | $\{P\} C \{Q\}$ | $[P] C [Q]$ |

**例子**：

```
// 部分正确但不完全正确
while (x > 0) { }
// 若x>0前置条件，永不终止，部分正确成立，完全正确不成立
```

**3. Rust类型系统与霍尔逻辑**

| Rust概念 | 霍尔逻辑对应 | 作用 |
|---------|-------------|------|
| 所有权 | 资源唯一性 | 确保无use-after-free |
| 借用 | 临时权限转移 | 确保无数据竞争 |
| 生命周期 | 时序逻辑 | 确保引用有效性 |
| Send/Sync | 并发安全谓词 | 确保线程安全 |

**类型系统作为轻量级形式化验证**：

```rust
// 编译器验证的"霍尔三元组"
fn take_ownership(v: Vec<i32>) -> usize {
    v.len()  // v被消耗，返回后不能使用
}

fn borrow_shared(v: &Vec<i32>) -> usize {
    v.len()  // 借用v，返回后仍可用
}
```

> Rust的类型系统是一种**自动的、轻量级的**形式化验证，在编译时保证内存安全。
</details>

### 第4题 ⭐⭐ 实现一个简单的解释器

**题目背景**：
理解编程语言的实现需要综合运用形式语言和编译原理知识。

**问题**：

1. 为以下简单表达式语言设计AST（抽象语法树）

3. 实现一个递归下降解析器（parser）
4. 实现求值器（evaluator）

语法：

```
expr   ::= number | identifier | expr '+' expr | expr '*' expr | '(' expr ')'

```

<details>
<summary>💡 提示</summary>

1. 定义AST枚举类型
2. 使用正则或手工状态机实现lexer
3. 按照文法实现递归下降parser
4. 使用HashMap作为环境，递归求值

</details>

<details>
<summary>✅ 解答</summary>

**1. AST定义**（Rust）：

```rust
#[derive(Debug, Clone)]
enum Expr {
    Number(i32),
    Identifier(String),
    Add(Box<Expr>, Box<Expr>),
    Mul(Box<Expr>, Box<Expr>),
}
```

**2. 词法分析器**：

```rust
#[derive(Debug, Clone)]
enum Token {
    Number(i32),
    Identifier(String),
    Plus,
    Mul,
    LParen,
    RParen,
    EOF,
}

struct Lexer {
    input: Vec<char>,
    pos: usize,
}

impl Lexer {
    fn new(input: &str) -> Self {
        Self { input: input.chars().collect(), pos: 0 }
    }

    fn next_token(&mut self) -> Token {
        self.skip_whitespace();

        if self.pos >= self.input.len() {
            return Token::EOF;
        }

        match self.current() {
            '+' => { self.advance(); Token::Plus }
            '*' => { self.advance(); Token::Mul }
            '(' => { self.advance(); Token::LParen }
            ')' => { self.advance(); Token::RParen }
            c if c.is_digit(10) => self.read_number(),
            c if c.is_alphabetic() => self.read_identifier(),
            _ => panic!("Unknown character: {}", c),
        }
    }

    fn read_number(&mut self) -> Token {
        let mut num = 0;
        while self.pos < self.input.len() && self.current().is_digit(10) {
            num = num * 10 + (self.current() as i32 - '0' as i32);
            self.advance();
        }
        Token::Number(num)
    }

    fn read_identifier(&mut self) -> Token {
        let start = self.pos;
        while self.pos < self.input.len() && self.current().is_alphabetic() {
            self.advance();
        }
        Token::Identifier(self.input[start..self.pos].iter().collect())
    }

    fn skip_whitespace(&mut self) {
        while self.pos < self.input.len() && self.current().is_whitespace() {
            self.advance();
        }
    }

    fn current(&self) -> char { self.input[self.pos] }
    fn advance(&mut self) { self.pos += 1; }
}
```

**3. 解析器**：

```rust
struct Parser {
    lexer: Lexer,
    current: Token,
}

impl Parser {
    fn new(mut lexer: Lexer) -> Self {
        let current = lexer.next_token();
        Self { lexer, current }
    }

    fn eat(&mut self, expected: Token) {
        if std::mem::discriminant(&self.current) ==
           std::mem::discriminant(&expected) {
            self.current = self.lexer.next_token();
        } else {
            panic!("Expected {:?}, got {:?}", expected, self.current);
        }
    }

    // expr ::= term ((+|-) term)*
    fn parse_expr(&mut self) -> Expr {
        let mut left = self.parse_term();

        while let Token::Plus = self.current {
            self.eat(Token::Plus);
            let right = self.parse_term();
            left = Expr::Add(Box::new(left), Box::new(right));
        }

        left
    }

    // term ::= factor (* factor)*
    fn parse_term(&mut self) -> Expr {
        let mut left = self.parse_factor();

        while let Token::Mul = self.current {
            self.eat(Token::Mul);
            let right = self.parse_factor();
            left = Expr::Mul(Box::new(left), Box::new(right));
        }

        left
    }

    // factor ::= number | identifier | ( expr )
    fn parse_factor(&mut self) -> Expr {
        match self.current.clone() {
            Token::Number(n) => {
                self.eat(Token::Number(0));
                Expr::Number(n)
            }
            Token::Identifier(name) => {
                self.eat(Token::Identifier(String::new()));
                Expr::Identifier(name)
            }
            Token::LParen => {
                self.eat(Token::LParen);
                let expr = self.parse_expr();
                self.eat(Token::RParen);
                expr
            }
            _ => panic!("Unexpected token: {:?}", self.current),
        }
    }

    fn parse(&mut self) -> Expr {
        let expr = self.parse_expr();
        if let Token::EOF = self.current {
            expr
        } else {
            panic!("Unexpected token after expression")
        }
    }
}
```

**4. 求值器**：

```rust
use std::collections::HashMap;

fn eval(expr: &Expr, env: &HashMap<String, i32>) -> i32 {
    match expr {
        Expr::Number(n) => *n,
        Expr::Identifier(name) => *env.get(name)
            .expect(&format!("Undefined variable: {}", name)),
        Expr::Add(left, right) => eval(left, env) + eval(right, env),
        Expr::Mul(left, right) => eval(left, env) * eval(right, env),
    }
}

fn main() {
    let input = "2 + 3 * (x + 1)";
    let lexer = Lexer::new(input);
    let mut parser = Parser::new(lexer);
    let ast = parser.parse();

    let mut env = HashMap::new();
    env.insert("x".to_string(), 5);

    let result = eval(&ast, &env);
    println!("Result: {}", result);  // 2 + 3 * 6 = 20
}
```

> 这是迷你解释器的完整实现，展示了从源码到执行的完整流程。

### 第5题 ⭐⭐⭐ 异步运行时原理

**题目背景**：
现代高性能网络服务广泛使用异步编程（如Rust的tokio、Go的goroutine）。

**问题**：

1. 解释协作式多任务（Cooperative Multitasking）vs 抢占式多任务（Preemptive Multitasking）
2. 描述async/await的底层实现原理（状态机转换）
3. 实现一个简化的异步执行器（executor），支持poll机制

<details>
<summary>💡 提示</summary>

1. 协作式：任务主动让出；抢占式：调度器强制切换
2. async函数编译为状态机，await点为状态转换
3. 使用Waker通知机制，循环poll Ready的future

</details>

<details>
<summary>✅ 解答</summary>

**1. 协作式 vs 抢占式**

| 特性 | 协作式多任务 | 抢占式多任务 |
|------|-------------|-------------|
| 切换时机 | 任务主动让出（await/yield） | 时间片用完/中断 |
| 上下文切换 | 用户态，低开销 | 内核态，高开销 |
| 饿死风险 | 可能有（不yield的任务） | 无（强制切换） |
| 实现复杂度 | 简单 | 复杂（需要调度器） |
| 例子 | async/await, 协程 | 操作系统线程 |

**2. async/await的底层实现**

```rust
// 源代码
async fn example() -> i32 {
    let x = async_op1().await;
    let y = async_op2().await;
    x + y
}
```

**编译后状态机**（简化）：

```rust
enum ExampleFuture {
    Start,
    State1 { x: i32 },
    State2 { x: i32, y: i32 },
    Done,
}

impl Future for ExampleFuture {
    type Output = i32;

    fn poll(mut self: Pin<&mut Self>, cx: &mut Context) -> Poll<i32> {
        loop {
            match *self {
                ExampleFuture::Start => {
                    match async_op1().poll(cx) {
                        Poll::Ready(x) => {
                            *self = ExampleFuture::State1 { x };
                        }
                        Poll::Pending => return Poll::Pending,
                    }
                }
                ExampleFuture::State1 { x } => {
                    match async_op2().poll(cx) {
                        Poll::Ready(y) => {
                            *self = ExampleFuture::State2 { x, y };
                        }
                        Poll::Pending => return Poll::Pending,
                    }
                }
                ExampleFuture::State2 { x, y } => {
                    return Poll::Ready(x + y);
                }
                _ => panic!("Invalid state"),
            }
        }
    }
}
```

**3. 简化版Executor实现**：

```rust
use std::future::Future;
use std::pin::Pin;
use std::sync::mpsc::{channel, Sender, Receiver};
use std::sync::{Arc, Mutex};
use std::task::{Context, Poll, Wake, Waker};

// 任务队列
type Task = Pin<Box<dyn Future<Output = ()> + Send>>;

struct Executor {
    ready_queue: Receiver<Arc<TaskNode>>,
    sender: Sender<Arc<TaskNode>>,
}

struct TaskNode {
    future: Mutex<Option<Task>>,
    sender: Sender<Arc<TaskNode>>,
}

impl Wake for TaskNode {
    fn wake(self: Arc<Self>) {
        self.sender.send(self).unwrap();
    }
}

impl Executor {
    fn new() -> Self {
        let (sender, ready_queue) = channel();
        Executor { ready_queue, sender }
    }

    fn spawn<F>(&self, future: F)
    where
        F: Future<Output = ()> + Send + 'static,
    {
        let task = Arc::new(TaskNode {
            future: Mutex::new(Some(Box::pin(future))),
            sender: self.sender.clone(),
        });
        self.sender.send(task).unwrap();
    }

    fn run(&self) {
        while let Ok(task) = self.ready_queue.recv() {
            let mut future_slot = task.future.lock().unwrap();

            if let Some(mut future) = future_slot.take() {
                let waker = Waker::from(task.clone());
                let mut context = Context::from_waker(&waker);

                match future.as_mut().poll(&mut context) {
                    Poll::Ready(()) => {}
                    Poll::Pending => {
                        *future_slot = Some(future);
                    }
                }
            }
        }
    }
}

// 测试
async fn async_task(id: i32) {
    println!("Task {} started", id);
    // 模拟异步操作
    std::thread::sleep(std::time::Duration::from_millis(100));
    println!("Task {} completed", id);
}

fn main() {
    let executor = Executor::new();

    for i in 0..5 {
        executor.spawn(async_task(i));
    }

    executor.run();
}
```

> 这是tokio/async-std的极简版本，展示了异步执行器的核心机制。
</details>

### 第6题 ⭐⭐⭐ 函数式编程模式实践

**题目背景**：
函数式编程模式（如Monad、Functor）在现代编程中越来越重要（如Rust的Option/Result、JavaScript的Promise）。

**问题**：

1. 在Rust中实现Functor和Monad trait
2. 为 `Option<T>` 实现这些trait
3. 用实现的trait重写以下代码，展示组合优势：

```rust
fn process_data(data: Option<i32>) -> Option<String> {
    if let Some(x) = data {
        if x > 0 {
            let y = x * 2;
            if y < 100 {
                return Some(y.to_string());
            }
        }
    }
    None
}
```

<details>
<summary>💡 提示</summary>

1. Functor需要fmap，Monad需要pure和bind（flat_map）
2. Option天然支持这些操作
3. 使用and_then和map组合嵌套操作

</details>

<details>
<summary>✅ 解答</summary>

**1. Functor和Monad trait**：

```rust
trait Functor {
    type Wrapped<A>;

    fn fmap<A, B, F>(self, f: F) -> Self::Wrapped<B>
    where
        F: FnOnce(A) -> B;
}

trait Monad: Functor {
    fn pure<A>(value: A) -> Self::Wrapped<A>;

    fn bind<A, B, F>(self, f: F) -> Self::Wrapped<B>
    where
        F: FnOnce(A) -> Self::Wrapped<B>;
}
```

> 注：由于Rust类型系统限制，完整实现需要高阶类型，这里使用简化版本。

**2. 直接使用Option的方法**（更地道的Rust）：

```rust
fn process_data(data: Option<i32>) -> Option<String> {
    data.filter(|&x| x > 0)           // 保留正数
        .map(|x| x * 2)                // 乘以2
        .filter(|&y| y < 100)          // 保留小于100的
```

```rust
struct Identity<T>(T);

impl<T> Identity<T> {
    fn new(value: T) -> Self { Identity(value) }
    fn unwrap(self) -> T { self.0 }

    fn map<F, B>(self, f: F) -> Identity<B>
    where
        F: FnOnce(T) -> B,
    {
        Identity(f(self.0))
    }

    fn bind<F, B>(self, f: F) -> B
    where
        F: FnOnce(T) -> B,
    {
        f(self.0)
    }
}

// Option的Monad实现
fn pure<T>(value: T) -> Option<T> {
    Some(value)
}

fn bind<A, B, F>(opt: Option<A>, f: F) -> Option<B>
where
    F: FnOnce(A) -> Option<B>,
{
    opt.and_then(f)
}

fn map<A, B, F>(opt: Option<A>, f: F) -> Option<B>
where
    F: FnOnce(A) -> B,
{
    opt.map(f)
}

// Kleisli组合：f >=> g = \x -> f x >>= g
fn kleisli<A, B, C, F, G>(f: F, g: G) -> impl FnOnce(A) -> Option<C>
where
    F: FnOnce(A) -> Option<B>,
    G: FnOnce(B) -> Option<C>,
{
    move |x| f(x).and_then(g)
}

// 使用Kleisli组合重写
fn process_data_kleisli(data: Option<i32>) -> Option<String> {
    let validate_positive = |x: i32| (x > 0).then(|| x);
    let double = |x: i32| Some(x * 2);
    let validate_limit = |y: i32| (y < 100).then(|| y);
    let to_string = |y: i32| Some(y.to_string());

    let pipeline = |x| validate_positive(x)
        .and_then(double)
        .and_then(validate_limit)
        .and_then(to_string);

    data.and_then(pipeline)
}
```

|------|--------|--------|

| 可读性 | 深层嵌套 | 链式调用 |
| 可组合性 | 差 | 好（函数组合）|
| 测试性 | 需要模拟状态 | 纯函数易测试 |

### 第7题 ⭐⭐ 形式化规约与实现

**题目背景**：
在开发分布式系统时，需要先用形式化方法规约系统行为，再实现代码。

**问题**：

1. 用TLA+风格描述一个简单的分布式KV存储的一致性规约
2. 用状态机描述客户端与服务器的交互
3. 用Rust实现该系统的核心逻辑

<details>
<summary>💡 提示</summary>

1. 定义状态变量：store、inflight_requests
2. 定义操作：ClientRequest、ServerResponse
3. 不变式：一致性、原子性

</details>

<details>
<summary>✅ 解答</summary>

**1. TLA+风格规约**：

```
MODULE KVStore

VARIABLES store, requests, responses

Init ==
  /\\ store = [k \\in Keys |-> NoValue]
  /\\ requests = {}
  /\\ responses = {}

ClientRequest(c, op, key, value) ==
  /\\ requests' = requests \\cup {[client |-> c, op |-> op, key |-> key, value |-> value]}
  /\\ UNCHANGED <<store, responses>>

ServerProcess(req) ==
  /\\ req \\in requests
  /\\ IF req.op = "read"
     THEN responses' = responses \\cup {[client |-> req.client, result |-> store[req.key]]}
     ELSE /\\ store' = [store EXCEPT ![req.key] = req.value]
          /\\ responses' = responses \\cup {[client |-> req.client, result |-> "ok"]}
  /\\ requests' = requests \\ {req}

Consistency ==
  \\A r1, r2 \\in responses:
    (r1.key = r2.key /\\ r1.client = r2.client /\\ r1.seq < r2.seq)
    => r1.result precedes r2.result in linearization order
```

**2. 状态机**：

```
                    ClientRequest
Client(Idle) --------------------------> Server(Processing)
                                              |
                                              | Process
                                              v
Client(Waiting) <----------------------- Server(Ready)
                    ServerResponse
```

**3. Rust实现**：

```rust
use std::collections::HashMap;
use std::sync::{Arc, Mutex};

#[derive(Debug, Clone)]
enum Operation {
    Read { key: String },
    Write { key: String, value: String },
}

#[derive(Debug)]
enum Response {
    Value(Option<String>),
    Ok,
    Error(String),
}

struct KVStore {
    data: Mutex<HashMap<String, String>>,
}

impl KVStore {
    fn new() -> Self {
        Self {
            data: Mutex::new(HashMap::new()),
        }
    }

    fn execute(&self, op: Operation) -> Response {
        let mut data = self.data.lock().unwrap();

        match op {
            Operation::Read { key } => {
                Response::Value(data.get(&key).cloned())
            }
            Operation::Write { key, value } => {
                data.insert(key, value);
                Response::Ok
            }
        }
    }
}

// 模拟分布式客户端
struct Client {
    id: u64,
    store: Arc<KVStore>,
}

impl Client {
    fn request(&self, op: Operation) -> Response {
        println!("Client {} sending {:?}", self.id, op);
        let response = self.store.execute(op);
        println!("Client {} received {:?}", self.id, response);
        response
    }
}

fn main() {
    let store = Arc::new(KVStore::new());

    let mut clients = vec![];
    for i in 0..3 {
        clients.push(Client {
            id: i,
            store: Arc::clone(&store),
        });
    }

    // 客户端操作
    clients[0].request(Operation::Write {
        key: "foo".to_string(),
        value: "bar".to_string(),
    });

    clients[1].request(Operation::Read {
        key: "foo".to_string(),
    });

    clients[2].request(Operation::Write {
        key: "foo".to_string(),
        value: "baz".to_string(),
    });

    clients[0].request(Operation::Read {
        key: "foo".to_string(),
    });
}
```

> 形式化规约帮助我们在编码前澄清系统行为，Rust的类型系统帮助在实现时保持安全。

### 第8题 ⭐⭐⭐ 定理证明辅助工具

**题目背景**：
现代数学证明和程序验证越来越多地使用计算机辅助证明工具（如Lean、Coq、Isabelle）。

**问题**：

1. 解释Curry-Howard同构（命题即类型，证明即程序）
2. 用Rust类型系统表达简单的逻辑命题（如 $A \rightarrow B \rightarrow A$）
3. 实现一个简化版的类型检查器，验证证明项的类型正确性

<details>
<summary>💡 提示</summary>

1. Curry-Howard：逻辑命题对应类型，证明对应程序
2. 蕴含 $A \rightarrow B$ 对应函数类型 `Fn(A) -> B`
3. 类型检查器需要处理变量、函数类型、应用

</details>

<details>
<summary>✅ 解答</summary>

**1. Curry-Howard同构**：

| 逻辑 | 类型理论 | 编程 |
|------|---------|------|
| 命题 $P$ | 类型 $P$ | 类型定义 |
| 证明 $p: P$ | 项 $p: P$ | 程序/值 |
| $P \rightarrow Q$ | 函数类型 $P \to Q$ | `Fn(P) -> Q` |
| $P \wedge Q$ | 积类型 $P \times Q$ | 元组 `(P, Q)` |
| $P \vee Q$ | 和类型 $P + Q$ | 枚举 `Either<P, Q>` |
| $\forall x: P. Q(x)$ | 依赖函数 $\Pi x: P. Q(x)$ | 泛型 `forall<T>` |
| $\exists x: P. Q(x)$ | 依赖对 $\Sigma x: P. Q(x)$ | 存在类型 |
| 真 $\top$ | 单位类型 `()` | `()` |
| 假 $\bot$ | 空类型 `!` | 无值 |

**2. Rust中的命题表达**：

```rust
// A -> B -> A（肯定前件律）
fn affirmation_of_antecedent<A, B>(a: A, _b: B) -> A {
    a
}

// (A -> B) -> (B -> C) -> (A -> C)（蕴含传递）
fn implication_transitivity<A, B, C>(
    f: impl Fn(A) -> B,
    g: impl Fn(B) -> C,
) -> impl Fn(A) -> C {
    move |a| g(f(a))
}

// A -> B -> (A, B)（合取引入）
fn conjunction_intro<A, B>(a: A, b: B) -> (A, B) {
    (a, b)
}

// (A, B) -> A（合取消去）
fn conjunction_elim_left<A, B>(pair: (A, B)) -> A {
    pair.0
}

// A -> A ∨ B（析取引入）
fn disjunction_intro_left<A, B>(a: A) -> Result<A, B> {
    Ok(a)
}

// 排中律 A ∨ ¬A 在直觉主义逻辑中不成立
// 经典逻辑需要额外公理
```

**3. 简化类型检查器**：

```rust
#[derive(Debug, Clone)]
enum Term {
    Var(String),
    Lambda(String, Box<Term>),  // λx.e
    Apply(Box<Term>, Box<Term>), // f x
}

#[derive(Debug, Clone, PartialEq)]
enum Type {
    Var(String),
    Arrow(Box<Type>, Box<Type>), // A -> B
}

struct TypeChecker {
    env: Vec<(String, Type)>,
}

impl TypeChecker {
    fn new() -> Self {
        Self { env: vec![] }
    }

    fn extend(&mut self, var: String, ty: Type) {
        self.env.push((var, ty));
    }

    fn lookup(&self, var: &str) -> Option<&Type> {
        self.env.iter()
            .rev()
            .find(|(v, _)| v == var)
            .map(|(_, t)| t)
    }

    fn infer(&mut self, term: &Term) -> Option<Type> {
        match term {
            Term::Var(x) => self.lookup(x).cloned(),

            Term::Lambda(x, body) => {
                // 创建类型变量
                let arg_ty = Type::Var(format!("{}", x));
                self.extend(x.clone(), arg_ty.clone());
                let body_ty = self.infer(body)?;
                Some(Type::Arrow(Box::new(arg_ty), Box::new(body_ty)))
            }

            Term::Apply(f, arg) => {
                let f_ty = self.infer(f)?;
                let arg_ty = self.infer(arg)?;

                match f_ty {
                    Type::Arrow(domain, codomain) => {
                        if *domain == arg_ty {
                            Some(*codomain)
                        } else {
                            println!("Type mismatch: expected {:?}, got {:?}", domain, arg_ty);
                            None
                        }
                    }
                    _ => {
                        println!("Expected arrow type");
                        None
                    }
                }
            }
        }
    }
}

fn main() {
    let mut checker = TypeChecker::new();

    // λx.λy.x : A -> B -> A
    let k_combinator = Term::Lambda(
        "x".to_string(),
        Box::new(Term::Lambda(
            "y".to_string(),
            Box::new(Term::Var("x".to_string())),
        )),
    );

    println!("K combinator type: {:?}", checker.infer(&k_combinator));
    // 输出: K combinator type: Some(Arrow(Var("x"), Arrow(Var("y"), Var("x"))))
    // 即 x -> y -> x，对应 A -> B -> A ✓
}
```

> 这是Lean/Coq等证明助手的基础，展示了编程和证明的深层统一。

**3. 类型检查器**（简化版）：

```rust
use std::collections::HashMap;

struct TypeChecker {
    var_counter: usize,
    subst: HashMap<String, Type>,
}

impl TypeChecker {
    fn new() -> Self {
        Self { var_counter: 0, subst: HashMap::new() }
    }

    fn fresh_var(&mut self) -> Type {
        self.var_counter += 1;
        Type::Var(format!("t{}", self.var_counter))
    }

    fn apply_subst(&self, ty: &Type) -> Type {
        match ty {
            Type::Var(name) => {
                if let Some(t) = self.subst.get(name) {
                    self.apply_subst(t)
                } else {
                    ty.clone()
                }
            }
            Type::Arrow(t1, t2) => {
                Type::Arrow(
                    Box::new(self.apply_subst(t1)),
                    Box::new(self.apply_subst(t2)),
                )
            }
            _ => ty.clone(),
        }
    }

    fn infer(&mut self, env: &HashMap<String, Type>, expr: &Expr) -> Result<Type, String> {
        match expr {
            Expr::Int(_) => Ok(Type::Int),
            Expr::Bool(_) => Ok(Type::Bool),

            Expr::Var(name) => {
                env.get(name)
                    .cloned()
                    .ok_or_else(|| format!("Unbound variable: {}", name))
            }

            Expr::Lambda(param, body) => {
                let param_ty = self.fresh_var();
                let mut new_env = env.clone();
                new_env.insert(param.clone(), param_ty.clone());
                let body_ty = self.infer(&new_env, body)?;
                Ok(Type::Arrow(Box::new(param_ty), Box::new(body_ty)))
            }

            Expr::App(func, arg) => {
                let func_ty = self.infer(env, func)?;
                let arg_ty = self.infer(env, arg)?;
                let result_ty = self.fresh_var();

                self.unify(&func_ty, &Type::Arrow(Box::new(arg_ty), Box::new(result_ty.clone())))?;
                Ok(result_ty)
            }

            Expr::Let(name, value, body) => {
                let value_ty = self.infer(env, value)?;
                let mut new_env = env.clone();
                new_env.insert(name.clone(), value_ty);
                self.infer(&new_env, body)
            }

            Expr::If(cond, then_branch, else_branch) => {
                let cond_ty = self.infer(env, cond)?;
                self.unify(&cond_ty, &Type::Bool)?;

                let then_ty = self.infer(env, then_branch)?;
                let else_ty = self.infer(env, else_branch)?;
                self.unify(&then_ty, &else_ty)?;

                Ok(then_ty)
            }

            Expr::Add(left, right) => {
                let left_ty = self.infer(env, left)?;
                let right_ty = self.infer(env, right)?;
                self.unify(&left_ty, &Type::Int)?;
                self.unify(&right_ty, &Type::Int)?;
                Ok(Type::Int)
            }

            Expr::Gt(left, right) => {
                let left_ty = self.infer(env, left)?;
                let right_ty = self.infer(env, right)?;
                self.unify(&left_ty, &Type::Int)?;
                self.unify(&right_ty, &Type::Int)?;
                Ok(Type::Bool)
            }
        }
    }

    fn unify(&mut self, t1: &Type, t2: &Type) -> Result<(), String> {
        let t1 = self.apply_subst(t1);
        let t2 = self.apply_subst(t2);

        match (&t1, &t2) {
            (Type::Var(v), t) | (t, Type::Var(v)) => {
                if t1 != t2 {
                    self.subst.insert(v.clone(), t.clone());
                }
                Ok(())
            }
            (Type::Arrow(a1, r1), Type::Arrow(a2, r2)) => {
                self.unify(a1, a2)?;
                self.unify(r1, r2)
            }
            _ if t1 == t2 => Ok(()),
            _ => Err(format!("Cannot unify {:?} with {:?}", t1, t2)),
        }
    }
}

fn main() {
    let mut checker = TypeChecker::new();

    // let add = fn x => fn y => x + y in add 3 4
    let expr = Expr::Let(
        "add".to_string(),
        Box::new(Expr::Lambda(
            "x".to_string(),
            Box::new(Expr::Lambda(
                "y".to_string(),
                Box::new(Expr::Add(
                    Box::new(Expr::Var("x".to_string())),
                    Box::new(Expr::Var("y".to_string())),
                )),
            )),
        )),
        Box::new(Expr::App(
            Box::new(Expr::App(
                Box::new(Expr::Var("add".to_string())),
                Box::new(Expr::Int(3)),
            )),
            Box::new(Expr::Int(4)),
        )),
    );

    let env: HashMap<String, Type> = HashMap::new();
    match checker.infer(&env, &expr) {
        Ok(ty) => println!("Type: {:?}", ty),  // Type: Int
        Err(e) => println!("Error: {}", e),
    }
}
```

**4. 分布式锁服务验证案例**：

**系统设计**：

```rust
// 简化版分布式锁服务
trait DistributedLock {
    // 获取锁，返回lock token
    fn acquire(&self, resource: &str, client: &str) -> Result<String, LockError>;

    // 释放锁，需要token验证
    fn release(&self, token: &str) -> Result<(), LockError>;

    // 续约（防止持有锁的客户端崩溃）
    fn renew(&self, token: &str) -> Result<(), LockError>;
}
```

**安全属性（需要形式化验证）**：

```tla
(* 互斥性：同一时刻只有一个客户端持有锁 *)
Mutex ==
  \A c1, c2 \in Clients, r \in Resources:
    (holdsLock(c1, r) /\ holdsLock(c2, r)) => (c1 = c2)

(* 安全性：只有获得锁的客户端能释放 *)
Safety ==
  \A c \in Clients, r \in Resources:
    release(c, r) => holdsLock(c, r)

(* 活性：请求锁的客户端最终获得（无饥饿） *)
Liveness ==
  \A c \in Clients, r \in Resources:
    requestLock(c, r) ~> holdsLock(c, r)

(* 容错性：客户端崩溃后锁最终释放 *)
FaultTolerance ==
  \A c \in Clients, r \in Resources:
    (crashed(c) /\ holdsLock(c, r)) ~> free(r)
```

**实现验证**：

```rust
// 使用Rust类型系统保证部分安全属性
struct LockToken {
    resource: String,
    client: String,
    expiry: Instant,
    _private: (),  // 阻止外部构造
}

impl LockToken {
    // 只有锁服务能创建token
    fn new(resource: String, client: String, expiry: Instant) -> Self {
        Self { resource, client, expiry, _private: () }
    }

    fn verify(&self, client: &str) -> bool {
        self.client == client && self.expiry > Instant::now()
    }
}

struct LockService {
    locks: Mutex<HashMap<String, LockToken>>,
}

impl DistributedLock for LockService {
    fn acquire(&self, resource: &str, client: &str) -> Result<String, LockError> {
        let mut locks = self.locks.lock().unwrap();

        // 检查是否已有锁
        if let Some(token) = locks.get(resource) {
            if token.expiry > Instant::now() {
                return Err(LockError::AlreadyHeld);
            }
            // 过期，可以抢占
        }

        let token = LockToken::new(
            resource.to_string(),
            client.to_string(),
            Instant::now() + Duration::from_secs(30),
        );

        let token_str = format!("{}:{}", resource, client);
        locks.insert(resource.to_string(), token);

        Ok(token_str)
    }

    fn release(&self, token_str: &str) -> Result<(), LockError> {
        let parts: Vec<&str> = token_str.split(':').collect();
        if parts.len() != 2 {
            return Err(LockError::InvalidToken);
        }

        let mut locks = self.locks.lock().unwrap();

        if let Some(token) = locks.get(parts[0]) {
            if token.verify(parts[1]) {
                locks.remove(parts[0]);
                return Ok(());
            }
        }

        Err(LockError::NotHeld)
    }
    // ...
}
```

**验证策略**：

1. **类型系统**：保证token不可伪造
2. **单元测试**：覆盖正常和异常路径
3. **模型检查**：用TLA+验证并发安全性
4. **模糊测试**：随机输入发现边界情况
5. **形式化证明**：对关键路径进行Coq证明

> 形式化方法不是银弹，但在关键系统中是降低风险的有力工具。随着工具成熟和自动化程度提高，其应用范围将不断扩大。
