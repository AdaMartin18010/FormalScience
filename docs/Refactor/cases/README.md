## 推荐阅读顺序

### 路径一：循序渐进（推荐）

```
案例3（类型安全数据库）
       ↓
案例1（操作系统验证）
       ↓
案例4（异步运行时）
       ↓
案例2（分布式共识）
       ↓
案例5（AI调度器）
```

**理由：** 从类型系统入手较为直观，逐步过渡到并发、分布式，最后挑战AI混合系统。

### 路径二：按应用领域

**系统软件方向：**

```
案例1（OS）→ 案例4（运行时）→ 案例3（数据库）
```

**分布式系统方向：**

```
案例2（共识）→ 案例4（异步）→ 案例5（AI调度）
```

**形式化验证方向：**

```
案例3（类型）→ 案例1（OS）→ 案例2（共识）→ 案例5（AI）
```

### 路径三：按技术深度

**快速概览（了解应用）：**

```
案例3 → 案例1 → 案例4
```

**深入研究（掌握方法）：**

```
案例1 → 案例2 → 案例4 → 案例5
```

**全栈掌握（建议全部）：**

```
按循序渐进路径完成全部5个案例
```
