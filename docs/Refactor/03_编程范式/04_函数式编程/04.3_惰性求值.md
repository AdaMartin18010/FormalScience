## 1.1 求值策略

### 1.1.1 严格求值与非严格求值

**定义 1.1.1**：

- **严格求值（Strict）**：先求值参数，再应用函数
- **非严格求值（Non-strict）**：按需求值参数

形式化：
$$
\text{Strict}: f(e) \Rightarrow \text{eval}(e) \text{ then } apply(f, value)
$$

$$
\text{Non-strict}: f(e) \Rightarrow \text{store}(e) \text{ and } \text{eval}(e) \text{ when needed}
$$

```rust
// Rust：默认严格求值
fn strict_evaluation() {
    let x = expensive_computation();  // 立即求值
    println!("Done");
    // 可能不使用 x，但已经计算了
}

fn expensive_computation() -> i32 {
    println!("Computing...");
    42
}

// 模拟惰性求值
fn lazy_evaluation() {
    let x = || expensive_computation();  // Thunk：延迟计算
    println!("Done");
    // 只有调用 x() 时才计算
    println!("Value: {}", x());
}
```

### 1.1.2 按值调用与按名调用

| 策略 | 描述 | 示例 |
|------|------|------|
| Call-by-Value | 先求值参数 | Rust, C, Java |
| Call-by-Name | 按需求值，每次重新求值 | Algol 60 |
| Call-by-Need | 按需求值，结果缓存 | Haskell |

```rust
// 按值调用（Rust 默认）
fn call_by_value(x: i32) -> i32 {
    x + x  // x 已经求值
}

// 模拟按名调用（每次都重新计算）
fn call_by_name<F>(f: F) -> i32
where
    F: Fn() -> i32,
{
    f() + f()  // 调用两次，计算两次
}

// 模拟按需调用（缓存结果）
use std::cell::RefCell;

struct Lazy<T, F> {
    thunk: RefCell<Option<T>>,
    computation: RefCell<Option<F>>,
}

impl<T: Clone, F: FnOnce() -> T> Lazy<T, F> {
    fn new(f: F) -> Self {
        Lazy {
            thunk: RefCell::new(None),
            computation: RefCell::new(Some(f)),
        }
    }

    fn force(&self) -> T {
        if self.thunk.borrow().is_none() {
            let f = self.computation.borrow_mut().take().unwrap();
            *self.thunk.borrow_mut() = Some(f());
        }
        self.thunk.borrow().clone().unwrap()
    }
}
```

## 1.2 Thunk 机制

### 1.2.1 Thunk 定义

**定义 1.2.1**：Thunk 是延迟计算的单位，封装了尚未求值的表达式。

形式化：
$$
\text{Thunk}\langle T \rangle = () \rightarrow T
$$

```rust
// Thunk 类型
type Thunk<T> = Box<dyn Fn() -> T>;

fn make_thunk<T, F>(f: F) -> Thunk<T>
where
    F: Fn() -> T + 'static,
    T: 'static,
{
    Box::new(f)
}

fn force<T>(thunk: Thunk<T>) -> T {
    thunk()
}

fn thunk_examples() {
    // 创建 Thunk
    let lazy_value = make_thunk(|| {
        println!("Computing...");
        42
    });

    println!("Thunk created");

    // 强制求值
    let value = force(lazy_value);
    println!("Value: {}", value);
}
```

### 1.2.2 记忆化 Thunk

**定义 1.2.2**：记忆化 Thunk 在首次求值后缓存结果。

```rust
use std::cell::RefCell;
use std::rc::Rc;

struct MemoThunk<T> {
    value: RefCell<Option<T>>,
    computation: RefCell<Box<dyn FnOnce() -> T>>,
}

impl<T: Clone> MemoThunk<T> {
    fn new<F>(f: F) -> Self
    where
        F: FnOnce() -> T + 'static,
    {
        MemoThunk {
            value: RefCell::new(None),
            computation: RefCell::new(Box::new(f)),
        }
    }

    fn force(&self) -> T {
        // 检查是否已求值
        if let Some(ref v) = *self.value.borrow() {
            return v.clone();
        }

        // 首次求值
        let f = self.computation.borrow_mut().take().unwrap();
        let result = f();
        *self.value.borrow_mut() = Some(result.clone());
        result
    }
}

fn memo_thunk_example() {
    let thunk = MemoThunk::new(|| {
        println!("Computing expensive value...");
        expensive_calculation()
    });

    println!("First force: {}", thunk.force());  // 计算
    println!("Second force: {}", thunk.force()); // 使用缓存
    println!("Third force: {}", thunk.force());  // 使用缓存
}

fn expensive_calculation() -> i32 {
    std::thread::sleep(std::time::Duration::from_secs(1));
    42
}
```

### 1.2.3 递归 Thunk

```rust
// 递归定义的惰性值
use std::cell::RefCell;
use std::rc::Rc;

struct LazyList<T> {
    head: T,
    tail: RefCell<Option<Rc<LazyList<T>>>>,
}

fn lazy_list_example() {
    // 创建无限的 1, 1, 1, ... 列表
    fn ones() -> Rc<LazyList<i32>> {
        Rc::new(LazyList {
            head: 1,
            tail: RefCell::new(None),
        })
    }

    let list = ones();
    println!("Head: {}", list.head);

    // 延迟创建 tail
    if list.tail.borrow().is_none() {
        *list.tail.borrow_mut() = Some(ones());
    }

    println!("Tail head: {}", list.tail.borrow().as_ref().unwrap().head);
}
```

## 1.3 惰性流

### 1.3.1 流的基本概念

**定义 1.3.1**：流（Stream）是惰性的序列，元素按需计算。

形式化：
$$
\text{Stream}\langle T \rangle = \text{Nil} \mid \text{Cons}(T, \text{Thunk}\langle \text{Stream}\langle T \rangle \rangle)
$$

```rust
// 惰性流定义
enum Stream<T> {
    Empty,
    Cons(T, Box<dyn FnOnce() -> Stream<T>>),
}

impl<T: Clone> Stream<T> {
    fn head(&self) -> Option<T> {
        match self {
            Stream::Empty => None,
            Stream::Cons(h, _) => Some(h.clone()),
        }
    }

    fn tail(self) -> Stream<T> {
        match self {
            Stream::Empty => Stream::Empty,
            Stream::Cons(_, t) => t(),
        }
    }

    fn take(self, n: usize) -> Vec<T> {
        if n == 0 {
            return vec![];
        }

        match self {
            Stream::Empty => vec![],
            Stream::Cons(h, t) => {
                let mut rest = t().take(n - 1);
                rest.insert(0, h);
                rest
            }
        }
    }
}

fn stream_examples() {
    // 创建有限流
    fn range(start: i32, end: i32) -> Stream<i32> {
        if start >= end {
            Stream::Empty
        } else {
            Stream::Cons(start, Box::new(move || range(start + 1, end)))
        }
    }

    let s = range(1, 10);
    let first_five: Vec<_> = s.take(5);
    println!("{:?}", first_five);  // [1, 2, 3, 4, 5]
}
```

### 1.3.2 流操作

```rust
impl<T: Clone> Stream<T> {
    fn map<F, B>(self, f: F) -> Stream<B>
    where
        F: Fn(T) -> B + Clone + 'static,
        B: Clone + 'static,
    {
        match self {
            Stream::Empty => Stream::Empty,
            Stream::Cons(h, t) => {
                let fh = f(h);
                let ft = t;
                let f_clone = f.clone();
                Stream::Cons(fh, Box::new(move || ft().map(f_clone)))
            }
        }
    }

    fn filter<F>(self, pred: F) -> Stream<T>
    where
        F: Fn(&T) -> bool + Clone + 'static,
        T: 'static,
    {
        match self {
            Stream::Empty => Stream::Empty,
            Stream::Cons(h, t) => {
                let pred_clone = pred.clone();
                if pred(&h) {
                    Stream::Cons(h, Box::new(move || t().filter(pred_clone)))
                } else {
                    t().filter(pred)
                }
            }
        }
    }

    fn fold<B, F>(self, init: B, f: F) -> B
    where
        F: Fn(B, T) -> B + Clone + 'static,
        T: 'static,
    {
        match self {
            Stream::Empty => init,
            Stream::Cons(h, t) => {
                let new_acc = f(init, h);
                t().fold(new_acc, f)
            }
        }
    }
}

fn stream_operations() {
    fn range(start: i32, end: i32) -> Stream<i32> {
        if start >= end {
            Stream::Empty
        } else {
            Stream::Cons(start, Box::new(move || range(start + 1, end)))
        }
    }

    let s = range(1, 100);

    // 惰性 map 和 filter
    let result: Vec<_> = s
        .map(|x| x * 2)
        .filter(|x| x % 3 == 0)
        .take(5);

    println!("{:?}", result);  // [6, 12, 18, 24, 30]

    // 只计算了必要的元素，而不是全部 99 个
}
```

## 1.4 无限数据结构

### 1.4.1 无限列表

**定义 1.4.1**：无限列表（或流）可以通过递归定义创建。

```rust
// 自然数流
fn naturals(start: i32) -> Stream<i32> {
    Stream::Cons(start, Box::new(move || naturals(start + 1)))
}

// 斐波那契数列
fn fibonacci() -> Stream<i32> {
    fn fib(a: i32, b: i32) -> Stream<i32> {
        Stream::Cons(a, Box::new(move || fib(b, a + b)))
    }
    fib(0, 1)
}

// 素数流（埃拉托斯特尼筛）
fn sieve(s: Stream<i32>) -> Stream<i32> {
    match s {
        Stream::Empty => Stream::Empty,
        Stream::Cons(p, t) => {
            let p_val = p;
            Stream::Cons(
                p,
                Box::new(move || {
                    sieve(t().filter(move |x| x % p_val != 0))
                })
            )
        }
    }
}

fn primes() -> Stream<i32> {
    sieve(naturals(2))
}

fn infinite_examples() {
    // 获取前 10 个自然数
    let first_10: Vec<_> = naturals(1).take(10);
    println!("Naturals: {:?}", first_10);

    // 获取前 20 个斐波那契数
    let fib_20: Vec<_> = fibonacci().take(20);
    println!("Fibonacci: {:?}", fib_20);

    // 获取前 10 个素数
    let prime_10: Vec<_> = primes().take(10);
    println!("Primes: {:?}", prime_10);
}
```

### 1.4.2 互递归定义

```rust
// 互递归的无限列表
fn even_numbers() -> Stream<i32> {
    Stream::Cons(0, Box::new(|| odd_numbers().map(|x| x + 1)))
}

fn odd_numbers() -> Stream<i32> {
    Stream::Cons(1, Box::new(|| even_numbers().map(|x| x + 1)))
}

fn mutually_recursive_example() {
    let evens: Vec<_> = even_numbers().take(5);
    println!("Evens: {:?}", evens);  // [0, 2, 4, 6, 8]

    let odds: Vec<_> = odd_numbers().take(5);
    println!("Odds: {:?}", odds);    // [1, 3, 5, 7, 9]
}
```

## 1.5 严格性分析

### 1.5.1 严格性分析基础

**定义 1.5.1**：严格性分析（Strictness Analysis）确定函数参数是否总是被求值。

形式化：
$$f \text{ is strict in } x \iff f(\bot) = \bot$$

```rust
// 严格函数
fn strict_add(x: i32, y: i32) -> i32 {
    x + y  // 严格于 x 和 y
}

// 非严格函数
fn const_first<T>(x: T, _y: T) -> T {
    x  // 非严格于 y
}

// 条件严格性
fn conditional_strict(x: i32, y: i32, flag: bool) -> i32 {
    if flag {
        x  // 非严格于 y
    } else {
        y  // 非严格于 x
    }
}
```

### 1.5.2 Rust 中的严格性注解

```rust
// Rust 编译器使用严格性分析优化

// #[inline(always)] 提示严格性
#[inline(always)]
fn strict_function(x: i32) -> i32 {
    x * 2
}

// 使用 drop 明确严格性
fn explicit_strict<T>(x: T, y: T) -> T {
    drop(x);  // 明确表示需要 x
    y
}

// 编译器优化：识别未使用的值
fn optimizer_example() {
    let _ = expensive();  // 编译器可能优化掉

    let x = expensive();
    let _ = black_box(x);  // 强制保留
}

fn expensive() -> i32 {
    42
}

fn black_box<T>(x: T) -> T {
    unsafe {
        std::ptr::read_volatile(&x)
    }
}
```

## 1.6 性能考量

### 1.6.1 空间-时间权衡

```rust
// 惰性求值的空间开销
fn space_overhead() {
    // Thunk 占用额外空间
    let thunk = || expensive_computation();

    // 如果最终要计算，可能不如立即计算
    // 但如果可能不计算，惰性更有利
}

// 记忆化的空间开销
fn memoization_space() {
    use std::collections::HashMap;
    use std::cell::RefCell;

    let cache: RefCell<HashMap<i32, i32>> = RefCell::new(HashMap::new());

    let fib = |n: i32| -> i32 {
        if let Some(&result) = cache.borrow().get(&n) {
            return result;
        }

        let result = if n <= 1 {
            n
        } else {
            // 递归计算
            n  // 简化
        };

        cache.borrow_mut().insert(n, result);
        result
    };

    // 缓存占用内存，但避免了重复计算
}
```

### 1.6.2 节流与稀疏求值

```rust
// 节流：避免过度惰性
fn throttle_example() {
    // 批量处理而不是逐个处理
    let chunks: Vec<Vec<i32>> = (0..1000)
        .collect::<Vec<_>>()
        .chunks(100)
        .map(|c| c.to_vec())
        .collect();

    for chunk in chunks {
        process_chunk(chunk);
    }
}

fn process_chunk(chunk: Vec<i32>) {
    println!("Processing {} items", chunk.len());
}

// 稀疏求值：只在需要时计算
fn sparse_evaluation() {
    let data: Vec<_> = (0..1000000).collect();

    // 只计算需要的部分
    let needed: Vec<_> = data
        .iter()
        .enumerate()
        .filter(|(i, _)| i % 1000 == 0)
        .map(|(_, &x)| expensive_transform(x))
        .collect();

    println!("Computed {} values", needed.len());
}

fn expensive_transform(x: i32) -> i32 {
    x * x
}
```
