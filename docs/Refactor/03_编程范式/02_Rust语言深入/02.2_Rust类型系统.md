## 1.1 类型系统基础

### 1.1.1 类型系统的目标

**定义 1.1.1**：类型系统（Type System）是编程语言中用于分类表达式并规定类型间转换规则的规则集合。

形式化定义：
$$
\text{TypeSystem}: \text{Expression} \rightarrow \text{Type} \cup \{\text{Error}\}
$$

Rust 类型系统的核心目标：

1. **内存安全**：编译时保证无悬空指针和数据竞争
2. **零成本抽象**：高级抽象不引入运行时开销
3. **表达力**：支持多态和泛型编程

```rust
// 类型系统保证内存安全
fn type_safety() {
    let x: i32 = 5;           // 整数类型
    let y: &i32 = &x;         // 引用类型
    let s: String = "hello".to_string();  // 所有权类型

    // 类型不匹配会导致编译错误
    // let z: i32 = s;  // 错误：expected `i32`, found `String`
}
```

### 1.1.2 基本类型分类

| 类别 | 类型 | 描述 |
|------|------|------|
| 标量 | `i32`, `u64`, `f32` | 整数和浮点数 |
| 布尔 | `bool` | 真/假 |
| 字符 | `char` | Unicode 标量值 |
| 复合 | 元组、数组 | 固定大小集合 |
| 引用 | `&T`, `&mut T` | 借用 |
| 指针 | `Box<T>`, `*const T` | 堆分配/原始指针 |
| 函数 | `fn(i32) -> i32` | 函数指针 |

## 1.2 Trait 系统

### 1.2.1 Trait 基础

**定义 1.2.1**：Trait 定义类型的共享行为，类似于其他语言的接口。

形式化定义：
$$
\text{Trait} = \{(method: Signature)\}
$$

$$
T: Trait \iff \forall m \in Trait, \exists \text{impl } m \text{ for } T
$$

```rust
// 定义 Trait
trait Drawable {
    fn draw(&self);
    fn description(&self) -> String {
        String::from("A drawable object")
    }
}

// 为类型实现 Trait
struct Circle {
    radius: f64,
}

impl Drawable for Circle {
    fn draw(&self) {
        println!("Drawing a circle with radius {}", self.radius);
    }
}

struct Rectangle {
    width: f64,
    height: f64,
}

impl Drawable for Rectangle {
    fn draw(&self) {
        println!("Drawing a rectangle {}x{}", self.width, self.height);
    }

    fn description(&self) -> String {
        String::from("A rectangle")
    }
}
```

### 1.2.2 Trait 作为参数

```rust
// Trait 边界
fn draw_item<T: Drawable>(item: &T) {
    item.draw();
}

// impl Trait 语法
fn draw_item_impl(item: &impl Drawable) {
    item.draw();
}

// 多重 Trait 边界
fn draw_and_describe<T: Drawable + Debug>(item: &T) {
    item.draw();
    println!("{:?}", item.description());
}

// where 子句
fn some_function<T, U>(t: &T, u: &U) -> i32
where
    T: Drawable + Clone,
    U: Clone + Debug,
{
    0
}
```

### 1.2.3 关联类型

**定义 1.2.2**：关联类型（Associated Types）在 Trait 定义中指定类型占位符。

```rust
trait Iterator {
    type Item;  // 关联类型

    fn next(&mut self) -> Option<Self::Item>;
}

struct Counter {
    count: u32,
}

impl Iterator for Counter {
    type Item = u32;

    fn next(&mut self) -> Option<Self::Item> {
        if self.count < 5 {
            self.count += 1;
            Some(self.count)
        } else {
            None
        }
    }
}
```

### 1.2.4 默认实现

```rust
trait Summary {
    fn summarize_author(&self) -> String;

    // 默认实现
    fn summarize(&self) -> String {
        format!("(Read more from {}...)", self.summarize_author())
    }
}

struct Article {
    author: String,
}

impl Summary for Article {
    fn summarize_author(&self) -> String {
        self.author.clone()
    }
    // 使用 summarize 的默认实现
}
```

## 1.3 泛型编程

### 1.3.1 泛型基础

**定义 1.3.1**：泛型（Generics）允许编写适用于多种类型的代码。

形式化定义：
$$
\forall T \in \text{TypeSet}, \text{Generic}(T) \text{ is valid}
$$

```rust
// 泛型函数
fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];

    for item in list {
        if item > largest {
            largest = item;
        }
    }

    largest
}

// 泛型结构体
struct Point<T> {
    x: T,
    y: T,
}

// 多泛型参数
struct Point2<T, U> {
    x: T,
    y: U,
}

// 泛型枚举
enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

### 1.3.2 泛型实现

```rust
impl<T> Point<T> {
    fn x(&self) -> &T {
        &self.x
    }
}

// 特定类型的实现
impl Point<f64> {
    fn distance_from_origin(&self) -> f64 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}
```

### 1.3.3 单态化

**定义 1.3.2**：单态化（Monomorphization）将泛型代码转换为具体类型的代码。

```rust
// 编译前的泛型代码
fn generic<T>(x: T) -> T {
    x
}

// 编译后的单态化代码（概念上）
fn generic_i32(x: i32) -> i32 { x }
fn generic_f64(x: f64) -> f64 { x }
fn generic_string(x: String) -> String { x }

// 使用
let a = generic(5i32);      // 调用 generic_i32
let b = generic(3.14f64);   // 调用 generic_f64
```

## 1.4 关联类型

### 1.4.1 关联类型 vs 泛型

**定理 1.4.1**：关联类型与泛型 Trait 的区别：

- 关联类型：每个实现只有一个具体类型
- 泛型 Trait：可以有多个实现，每个类型不同

```rust
// 关联类型：一个类型只能实现一次
trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}

// 泛型 Trait：可以多次实现
trait Add<Rhs = Self> {
    type Output;
    fn add(self, rhs: Rhs) -> Self::Output;
}

// 可以为同一类型实现多次
struct Point {
    x: i32,
    y: i32,
}

impl Add for Point {
    type Output = Point;
    fn add(self, rhs: Point) -> Point {
        Point {
            x: self.x + rhs.x,
            y: self.y + rhs.y,
        }
    }
}

impl Add<i32> for Point {
    type Output = Point;
    fn add(self, rhs: i32) -> Point {
        Point {
            x: self.x + rhs,
            y: self.y + rhs,
        }
    }
}
```

### 1.4.2 完全限定语法

```rust
trait Pilot {
    fn fly(&self);
}

trait Wizard {
    fn fly(&self);
}

struct Human;

impl Pilot for Human {
    fn fly(&self) {
        println!("This is your captain speaking.");
    }
}

impl Wizard for Human {
    fn fly(&self) {
        println!("Up!");
    }
}

impl Human {
    fn fly(&self) {
        println!("*waving arms furiously*");
    }
}

fn fully_qualified_syntax() {
    let person = Human;

    person.fly();  // 调用 Human 的方法
    Pilot::fly(&person);  // 调用 Pilot trait 的方法
    Wizard::fly(&person);  // 调用 Wizard trait 的方法

    // 完全限定语法
    <Human as Pilot>::fly(&person);
}
```

## 1.5 类型推断

### 1.5.1 Hindley-Milner 类型推断

**定义 1.5.1**：Rust 使用基于 Hindley-Milner 算法的类型推断。

形式化：
$$
\Gamma \vdash e: \tau \quad \text{（在环境 } \Gamma \text{ 中，表达式 } e \text{ 的类型为 } \tau \text{）}
$$

```rust
fn type_inference() {
    // 编译器推断类型
    let x = 5;        // i32
    let y = 3.14;     // f64
    let s = "hello";  // &str

    // 复杂推断
    let v = vec![1, 2, 3];  // Vec<i32>
    let iter = v.iter();     // std::slice::Iter<i32>

    // 有时需要显式标注
    let data: Vec<u8> = vec![];
}
```

### 1.5.2 泛型类型推断

```rust
fn generic_inference() {
    // 编译器推断泛型参数
    let sum: i32 = [1, 2, 3].iter().sum();  // Sum<i32>

    // turbofish 语法显式指定
    let sum = [1, 2, 3].iter().sum::<i32>();

    // collect 需要显式类型
    let v: Vec<i32> = [1, 2, 3].iter().map(|x| x * 2).collect();
}
```

## 1.6 高级类型特性

### 1.6.1 类型别名

```rust
// 类型别名
type Kilometers = i32;
type Thunk = Box<dyn Fn() + Send + 'static>;

type Result<T> = std::result::Result<T, std::io::Error>;

fn type_aliases() {
    let x: i32 = 5;
    let y: Kilometers = 5;

    println!("x + y = {}", x + y);  // 类型别名不改变底层类型
}
```

### 1.6.2 Never 类型

**定义 1.6.1**：Never 类型 `!` 表示永不返回的函数。

```rust
fn diverging() -> ! {
    panic!("This function never returns!");
}

fn never_type_usage() {
    let x: i32 = match Some(5) {
        Some(n) => n,
        None => diverging(),  // ! 可以强制转换为任何类型
    };
}
```

### 1.6.3 动态大小类型

**定义 1.6.2**：动态大小类型（DST）的大小在编译时未知。

```rust
// DST 必须通过指针使用
fn dst_examples() {
    // str 是 DST
    let s1: &str = "hello";
    let s2: Box<str> = "world".into();

    // [T] 是 DST
    let arr: &[i32] = &[1, 2, 3];

    // Trait 对象也是 DST
    let drawable: &dyn Drawable = &Circle { radius: 5.0 };
}
```

### 1.6.4 类型级编程

```rust
// PhantomData 用于类型标记
use std::marker::PhantomData;

struct Id<T>(u64, PhantomData<T>);

struct User;
struct Post;

fn type_level_programming() {
    let user_id: Id<User> = Id(1, PhantomData);
    let post_id: Id<Post> = Id(1, PhantomData);

    // 编译错误：不能混用不同类型的 Id
    // let wrong = user_id == post_id;
}

// 类型状态模式
struct Draft;
struct PendingReview;
struct Published;

struct Post<State> {
    content: String,
    state: PhantomData<State>,
}

impl Post<Draft> {
    fn new() -> Post<Draft> {
        Post {
            content: String::new(),
            state: PhantomData,
        }
    }

    fn add_text(&mut self, text: &str) {
        self.content.push_str(text);
    }

    fn request_review(self) -> Post<PendingReview> {
        Post {
            content: self.content,
            state: PhantomData,
        }
    }
}

impl Post<PendingReview> {
    fn approve(self) -> Post<Published> {
        Post {
            content: self.content,
            state: PhantomData,
        }
    }
}

impl Post<Published> {
    fn content(&self) -> &str {
        &self.content
    }
}
```

### 1.6.5 形式化类型规则

**定理 1.6.3**：Rust 类型系统的核心规则：

```lean
-- Lean 风格的形式化
-- 类型规则

inductive Ty
  | int
  | bool
  | ref (t : Ty) (mut : Bool)
  | fn (args : List Ty) (ret : Ty)

-- 子类型关系
inductive SubTy : Ty → Ty → Prop
  | refl {t} : SubTy t t
  | ref_immutable {t} : SubTy (&mut t) (&t)

-- 类型良好性
inductive HasTy : Expr → Ty → Prop
  | lit_int {n} : HasTy (Expr.lit n) Ty.int
  | lit_bool {b} : HasTy (Expr.bool b) Ty.bool
  | var {x t} : HasTy (Expr.var x) t
  | app {f arg t1 t2} :
      HasTy f (Ty.fn [t1] t2) →
      HasTy arg t1 →
      HasTy (Expr.app f arg) t2
```
