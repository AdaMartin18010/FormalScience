# 02.3 内存安全形式化

## 02.3.1 概述

Rust的内存安全保证基于形式化模型。**RustBelt**和**Stacked Borrows**等项目为Rust的类型系统提供了数学基础。

| 安全属性 | 威胁 | Rust保证 |
|----------|------|----------|
| 空间安全 | 越界访问 | 长度检查 |
| 时间安全 | use-after-free | 所有权系统 |
| 线程安全 | 数据竞争 | Send/Sync |
| 类型安全 | 类型混淆 | 类型系统 |

## 02.3.2 内存模型基础

### 02.3.2.1 内存结构

**定义 02.3.1 (内存)**

内存 $M$ 是位置到值的偏函数：

$$M : \text{Loc} \rightharpoonup \text{Val}$$

其中位置 $\ell \in \text{Loc}$ 包含：

- 分配标识符
- 偏移量
- 大小

```rust
// 概念模型
struct Memory {
    allocations: HashMap<AllocId, Allocation>,
}

struct Allocation {
    bytes: Vec<Option<u8>>,
    provenance: Vec<Option<Provenance>>,
    align: usize,
    mutability: Mutability,
}
```

### 02.3.2.2 指针表示

**定义 02.3.2 (指针)**

指针 $p = (\ell, o, \pi)$ 包含：

- $\ell$：分配标识符
- $o$：偏移量
- $\pi$：来源信息 (Provenance)

```rust
// 概念表示
struct Pointer {
    alloc_id: AllocId,
    offset: Size,
    provenance: Provenance,
}

enum Provenance {
    Root(AllocId),           // 原始分配
    Derived(Box<Pointer>),   // 派生指针
    Wild,                    // 不确定来源（不安全）
}
```

### 02.3.2.3 有效性

**定义 02.3.3 (有效性谓词)**

值 $v$ 在类型 $T$ 上有效，记为 $v :_\text{valid} T$：

$$
\begin{aligned}
n :_\text{valid} \texttt{u32} &\iff n \in [0, 2^{32}-1] \\
p :_\text{valid} \&T &\iff p \text{ 是有效的非空指针} \\
&\quad \land \text{ptr 指向有效的 } T \\
() :_\text{valid} () &\iff \text{always true} \\
(v_1, v_2) :_\text{valid} (T_1, T_2) &\iff v_1 :_\text{valid} T_1 \land v_2 :_\text{valid} T_2
\end{aligned}
$$

## 02.3.3 Stacked Borrows模型

### 02.3.3.1 借用栈

**定义 02.3.4 (借用栈)**

每个内存位置有一个**借用栈**，追踪活跃的借用：

```
栈: [ Unique(0) | SharedReadOnly | Unique(1) | ... ]
      ↑原始指针     ↑共享引用       ↑可变引用
```

**栈操作**

- **Push**: 创建新借用时压栈
- **Pop**: 使用父指针时弹出子借用
- **Deallocate**: 释放内存时清空栈

### 02.3.3.2 标签系统

```rust
enum Tag {
    Untagged,                    // 原始指针
    Tagged(PtrId),              // 带标签指针
    AliasableUntagged,          // 内部可变性
}

enum Permission {
    Unique,                      // 独占访问
    SharedReadWrite,            // 共享读写
    SharedReadOnly,             // 共享只读
    Disabled,                   // 已禁用
}
```

### 02.3.3.3 操作规则

**规则 02.3.1 (读取)**

从位置 $\ell$ 读取时：

1. 找到栈中包含该标签的最高位置
2. 确保该位置权限允许读取
3. 若为 `Unique`，上方所有条目标记为 `Disabled`

**规则 02.3.2 (写入)**

向位置 $\ell$ 写入时：

1. 标签必须是栈顶
2. 权限必须是 `Unique` 或 `SharedReadWrite`
3. 上方所有条目弹出

```rust
// 示例代码
let mut x = 5;
let r1 = &mut x;    // 压栈: [..., Unique(r1)]
let r2 = &mut x;    // 错误！r1 仍在使用

let mut y = 5;
let r1 = &mut y;    // [Unique(r1)]
*r1 = 6;            // OK，栈顶
let r2 = &mut y;    // r1 被弹出，[Unique(r2)]
*r2 = 7;            // OK
```

## 02.3.4 RustBelt形式化

### 02.3.4.1 Iris分离逻辑

RustBelt使用**Iris**框架，一种高阶并发分离逻辑：

**定义 02.3.5 (资源)**

资源 $r \in \text{iProp}$ 是程序状态的部分描述：

```
ℓ ↦ v           -- 独占所有权：位置ℓ存储值v
Shared(ℓ, P)    -- 共享读取权限
Mutable(ℓ, P)   -- 可变借用权限
Own(T, v)       -- 类型T的所有权
```

### 02.3.4.2 类型解释

**定义 02.3.6 (语义类型)**

语义类型 $\text{SemTy}$ 是值到资源的函数：

$$\llbracket T \rrbracket : \text{Val} \to \text{iProp}$$

```rust
// 基本类型的解释
[[i32]] = λv. ∃n. v = n ∧ n ∈ [-2^31, 2^31-1]

[[&'a T]] = λv. ∃ℓ. v = ptr(ℓ) ∧ ℓ ↦_readonly [[T]]

[[&'a mut T]] = λv. ∃ℓ. v = ptr(ℓ) ∧ ℓ ↦ [[T]]

[[Box<T>]] = λv. ∃ℓ. v = ptr(ℓ) ∧ ℓ ↦ [[T]] ∧ Dealloc(ℓ)
```

### 02.3.4.3 不变式

```rust
// 内部可变性解释
[[Cell<T>]] = λv. ∃ℓ. v = ptr(ℓ) ∧ Inv(ℓ, [[T]])

// RefCell使用运行时检查
[[RefCell<T>]] = λv. ∃ℓ, state.
    v = ptr(ℓ) ∧
    ℓ.state ↦ state ∧
    if state = Unshared then ℓ.data ↦ [[T]]
    else if state = Shared(n) then Shared(ℓ.data, [[T]])
    else state = Mutable ∧ ℓ.data ↦ [[T]]
```

## 02.3.5 类型系统与内存安全

### 02.3.5.1 Send与Sync

**定义 02.3.7 (线程安全Trait)**

```rust
// 可跨线程转移所有权
trait Send {}

// 可跨线程共享引用
trait Sync: Send {}

// 自动推导规则
impl<T> Send for T where T: 'static + ?Sized {}
impl<T> !Send for Rc<T> {}  // 显式禁用

// Sync的关系
T: Sync ⇔ &T: Send
```

### 02.3.5.2 变型规则

**定义 02.3.8 (类型构造器变型)**

```rust
// 协变：子类型关系保持
&'a T         // 'a和T都是协变
Box<T>        // T协变
Vec<T>        // T协变

// 逆变：子类型关系反转
fn(T) -> U    // T逆变，U协变

// 不变：子类型关系不传递
&mut T        // T不变
Cell<T>       // T不变
UnsafeCell<T> // T不变

fn(T) -> U    // 整体对T逆变
// 因为若T1 <: T2，则接受T2的函数可以接受T1
```

### 02.3.5.3 不安全代码契约

```rust
/// # Safety
/// - `ptr` 必须是对齐的非空指针
/// - `ptr` 必须指向已初始化的T值
/// - 该位置的值在调用后被视为"已移动"
unsafe fn read<T>(ptr: *const T) -> T { ... }

/// # Safety
/// - `dst` 必须对齐且非空
/// - `dst` 指向的内存必须已分配
unsafe fn write<T>(dst: *mut T, src: T) { ... }
```

## 02.3.6 Lean4形式化示例

### 02.3.6.1 内存模型基础

```lean4
-- 分配标识符
def AllocId := ℕ

-- 内存位置
def Loc := AllocId × ℕ  -- (分配ID, 偏移)

-- 值
def Val : Type := Option (Array ℕ)
  -- None 表示未初始化

-- 内存
def Memory := AllocId → Option {
  data : Array (Option ℕ),
  align : ℕ
}

-- 指针
def Ptr := Option (Loc × Provenance)
```

### 02.3.6.2 有效性谓词

```lean4
inductive Valid : Val → Ty → Prop
  | valid_int {n sz} :
      n < 2^sz → Valid (some [n]) (Ty.int sz)
  | valid_unit :
      Valid (some []) Ty.unit
  | valid_pair {v1 v2 t1 t2} :
      Valid (some v1) t1 → Valid (some v2) t2 →
      Valid (some (v1 ++ v2)) (Ty.pair t1 t2)
  | valid_ref {ℓ t} :
      points_to ℓ t → Valid (some [ℓ]) (Ty.ref t)
```

### 02.3.6.3 安全定理

```lean4
theorem type_safety {e : Expr} {τ : Ty} {σ σ' : State} :
    (⊢ e : τ) →            -- e 有类型 τ
    (e, σ) ⟹* (v, σ') →   -- e 求值为 v
    Valid v τ := by        -- 则 v 对 τ 有效
  intro h_type h_eval
  induction h_eval generalizing τ
  case refl =>
    -- 立即求值的情况
    apply validity_preservation h_type
  case trans e1 e2 e3 h12 h23 ih =>
    -- 多步求值：保持性保证中间状态有效
    have h_mid : ValidStep e1 e2 := progress h_type
    exact ih (preservation h_type h_mid)
```

## 02.3.7 验证工具

### 02.3.7.1 Miri

**MIR解释器**用于检测未定义行为：

```bash
# 运行Miri
rustup component add miri
miri run

# 检测示例
fn main() {
    let x = 5;
    let r = &x as *const i32;
    unsafe {
        println!("{}", *r);  // OK
    }
    drop(x);
    unsafe {
        println!("{}", *r);  // Miri: 报错！悬垂指针
    }
}
```

### 02.3.7.2 Kani

**模型检查器**用于验证属性：

```rust
#[kani::proof]
fn check_abs() {
    let x: i32 = kani::any();
    kani::assume(x != i32::MIN);
    let result = x.abs();
    assert!(result >= 0);
    assert!(result == x || result == -x);
}
```

### 02.3.7.3 Prusti

**Viper框架**的Rust前端：

```rust
use prusti_contracts::*;

#[requires(n >= 0)]
#[ensures(result == n * (n + 1) / 2)]
fn sum(n: i32) -> i32 {
    let mut i = 0;
    let mut s = 0;

    #[invariant(i <= n)]
    #[invariant(s == i * (i - 1) / 2)]
    while i < n {
        i += 1;
        s += i;
    }

    s
}
```
