## 1.1 所有权基础

### 1.1.1 所有权规则

**定义 1.1.1**：所有权（Ownership）是 Rust 管理内存的核心机制，遵循三条基本规则：

1. **唯一所有权**：每个值在任意时刻有且只有一个所有者
2. **作用域约束**：当所有者离开作用域时，值被自动丢弃
3. **移动语义**：赋值和传参会转移所有权

形式化定义：
$$
\text{Ownership}: \text{Value} \rightarrow \text{Owner}
$$

$$
\forall v \in \text{Value}, |\{o \mid \text{Ownership}(v) = o\}| = 1 \lor |\{o\}| = 0
$$

```rust
fn ownership_basics() {
    // s1 是 String 的所有者
    let s1 = String::from("hello");

    // 所有权移动到 s2，s1 不再有效
    let s2 = s1;

    // println!("{}", s1);  // 编译错误：s1 已被移动
    println!("{}", s2);     // OK

    // s2 离开作用域，内存自动释放
}
```

### 1.1.2 移动语义

**定义 1.1.2**：移动语义（Move Semantics）指值的所有权在赋值或传参时转移给新变量。

形式化模型：
$$
\text{Move}(v, o_1, o_2): \text{Ownership}(v) = o_1 \Rightarrow \text{Ownership}(v) = o_2 \land o_1 \text{ invalid}
$$

```rust
fn move_semantics() {
    let s = String::from("data");

    // 传递给函数，所有权移动
    takes_ownership(s);
    // s 不再有效

    let x = 5;
    // Copy 类型，不移动而是复制
    makes_copy(x);
    // x 仍然有效
}

fn takes_ownership(s: String) {
    println!("{}", s);
    // s 在这里被释放
}

fn makes_copy(i: i32) {
    println!("{}", i);
}
```

### 1.1.3 Copy trait

**定义 1.1.3**：Copy trait 标记可以在赋值时自动按位复制的类型。

```rust
// 实现 Copy 的类型
#[derive(Copy, Clone)]
struct Point {
    x: i32,
    y: i32,
}

fn copy_trait() {
    let p1 = Point { x: 1, y: 2 };
    let p2 = p1;  // p1 被复制，不是移动

    println!("p1: ({}, {})", p1.x, p1.y);  // OK
    println!("p2: ({}, {})", p2.x, p2.y);  // OK
}

// 以下类型实现 Copy：
// - 所有整数类型
// - 所有浮点数类型
// - 布尔类型
// - 字符类型
// - 只包含 Copy 类型的元组
```

### 1.1.4 Drop trait

**定义 1.1.4**：Drop trait 定义值离开作用域时的清理行为。

```rust
struct CustomResource {
    name: String,
}

impl Drop for CustomResource {
    fn drop(&mut self) {
        println!("Dropping CustomResource: {}", self.name);
        // 执行清理操作
    }
}

fn drop_example() {
    let res = CustomResource {
        name: String::from("resource1"),
    };
    // res 离开作用域时自动调用 drop
}
```

## 1.2 借用与引用

### 1.2.1 引用规则

**定义 1.2.1**：借用（Borrowing）允许在不转移所有权的情况下使用值。

形式化规则：
$$
\text{Borrow}(v) \Rightarrow \begin{cases}
\&v & \text{不可变借用} \\
\&mut \ v & \text{可变借用}
\end{cases}
$$

**定理 1.2.2**：借用检查器保证以下规则：

- 任一时刻，只能有一个可变引用，或多个不可变引用
- 引用必须始终有效（不能悬空）

$$
\text{Valid}(\&mut \ v) \iff \neg \exists \&v \land \neg \exists \&'mut \ v
$$

$$
\forall r, \text{Valid}(r) \Rightarrow \text{Lifetime}(r) \subseteq \text{Lifetime}(\text{data}(r))
$$

```rust
fn borrowing_rules() {
    let mut s = String::from("hello");

    // 不可变借用
    let r1 = &s;
    let r2 = &s;
    println!("{} {}", r1, r2);  // 可以同时存在多个

    // 借用结束后才能可变借用
    let r3 = &mut s;
    r3.push_str(" world");

    // 不能再使用 r1 或 r2
    // println!("{}", r1);  // 编译错误

    println!("{}", r3);
}
```

### 1.2.2 切片引用

```rust
fn slice_examples() {
    let s = String::from("hello world");

    // 字符串切片
    let hello = &s[0..5];
    let world = &s[6..11];

    println!("{} {}", hello, world);

    // 数组切片
    let arr = [1, 2, 3, 4, 5];
    let slice = &arr[1..3];

    assert_eq!(slice, &[2, 3]);
}
```

### 1.2.3 解引用

**定义 1.2.3**：Deref trait 允许自定义解引用行为。

```rust
use std::ops::Deref;

struct MyBox<T>(T);

impl<T> MyBox<T> {
    fn new(x: T) -> MyBox<T> {
        MyBox(x)
    }
}

impl<T> Deref for MyBox<T> {
    type Target = T;

    fn deref(&self) -> &T {
        &self.0
    }
}

fn deref_example() {
    let x = 5;
    let y = MyBox::new(x);

    assert_eq!(5, x);
    assert_eq!(5, *y);  // 使用 Deref
}

// 自动解引用转换
fn hello(name: &str) {
    println!("Hello, {}!", name);
}

fn deref_coercion() {
    let m = MyBox::new(String::from("Rust"));
    hello(&m);  // &MyBox<String> -> &String -> &str
}
```

## 1.3 生命周期

### 1.3.1 生命周期标注

**定义 1.3.1**：生命周期（Lifetime）是引用的有效范围。

形式化定义：
$$
\text{Lifetime}: Reference \rightarrow Scope
$$

$$
\text{Valid}(r: \&'a T) \iff 'a \subseteq \text{Lifetime}(T)
$$

```rust
// 显式生命周期标注
fn longest<'a>(s1: &'a str, s2: &'a str) -> &'a str {
    if s1.len() > s2.len() {
        s1
    } else {
        s2
    }
}

// 结构体中的生命周期
struct ImportantExcerpt<'a> {
    part: &'a str,
}

impl<'a> ImportantExcerpt<'a> {
    fn level(&self) -> i32 {
        3
    }

    // 返回的生命周期与 self 相同
    fn announce_and_return_part(&self, announcement: &str) -> &str {
        println!("Attention please: {}", announcement);
        self.part
    }
}
```

### 1.3.2 生命周期省略规则

**定理 1.3.2**：编译器自动应用的生命周期省略规则：

1. 每个引用参数都有独立的生命周期
2. 如果只有一个输入生命周期，它赋给所有输出生命周期
3. 如果有多个输入生命周期，但一个是 `&self` 或 `&mut self`，则输出生命周期与 `self` 相同

```rust
// 规则应用示例

// 规则 1：自动添加生命周期
fn first_word(s: &str) -> &str {  // 等价于 fn first_word<'a>(s: &'a str) -> &'a str
    &s[0..1]
}

// 规则 2：单个输入
fn foo(x: &i32) -> &i32 {  // 等价于 fn foo<'a>(x: &'a i32) -> &'a i32
    x
}

// 规则 3：&self
impl<'a> ImportantExcerpt<'a> {
    fn get_part(&self) -> &str {  // 等价于 fn get_part(&'a self) -> &'a str
        self.part
    }
}
```

### 1.3.3 静态生命周期

**定义 1.3.3**：`'static` 表示整个程序运行期间都有效的生命周期。

```rust
fn static_lifetime() {
    // 字符串字面量拥有 'static 生命周期
    let s: &'static str = "I have a static lifetime.";

    // 可以存储在全局变量中
    static GLOBAL: &str = "Global static";

    // Box::leak 创建 'static 引用
    let leaked: &'static str = Box::leak(String::from("leaked").into_boxed_str());
}
```

## 1.4 智能指针

### 1.4.1 Box

**定义 1.4.1**：Box 是在堆上存储数据的智能指针。

```rust
fn box_example() {
    // 堆分配
    let b = Box::new(5);
    println!("b = {}", b);

    // 用于递归类型
    enum List {
        Cons(i32, Box<List>),
        Nil,
    }

    use List::{Cons, Nil};

    let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
}
```

### 1.4.2 Rc 和 Arc

**定义 1.4.2**：Rc（引用计数）和 Arc（原子引用计数）允许多所有权。

```rust
use std::rc::Rc;

fn rc_example() {
    let data = Rc::new(String::from("shared data"));

    println!("Reference count: {}", Rc::strong_count(&data));

    {
        let data2 = Rc::clone(&data);
        println!("Reference count: {}", Rc::strong_count(&data));

        // data2 离开作用域
    }

    println!("Reference count: {}", Rc::strong_count(&data));
}

// Arc 用于多线程
use std::sync::Arc;
use std::thread;

fn arc_example() {
    let data = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let data = Arc::clone(&data);
        let handle = thread::spawn(move || {
            let mut num = data.lock().unwrap();
            *num += 1;
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("Result: {}", *data.lock().unwrap());
}
```

### 1.4.3 Weak 引用

**定义 1.4.3**：Weak 引用不增加引用计数，用于打破循环引用。

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct Node {
    value: i32,
    parent: RefCell<Weak<Node>>,
    children: RefCell<Vec<Rc<Node>>>,
}

fn weak_reference() {
    let leaf = Rc::new(Node {
        value: 3,
        parent: RefCell::new(Weak::new()),
        children: RefCell::new(vec![]),
    });

    println!("leaf parent = {:?}", leaf.parent.borrow().upgrade());

    let branch = Rc::new(Node {
        value: 5,
        parent: RefCell::new(Weak::new()),
        children: RefCell::new(vec![Rc::clone(&leaf)]),
    });

    *leaf.parent.borrow_mut() = Rc::downgrade(&branch);

    println!("leaf parent = {:?}", leaf.parent.borrow().upgrade());
}
```

## 1.5 内部可变性

### 1.5.1 Cell

**定义 1.5.1**：Cell 允许在不可变引用时修改数据，只适用于 Copy 类型。

```rust
use std::cell::Cell;

fn cell_example() {
    let cell = Cell::new(5);

    // 即使 cell 是不可变绑定，也可以修改内部值
    cell.set(10);

    println!("{}", cell.get());  // 10
}
```

### 1.5.2 RefCell

**定义 1.5.2**：RefCell 在运行时而非编译时检查借用规则。

```rust
use std::cell::RefCell;

fn refcell_example() {
    let data = RefCell::new(vec![1, 2, 3]);

    {
        let mut vec = data.borrow_mut();
        vec.push(4);
    }

    {
        let vec = data.borrow();
        println!("{:?}", *vec);
    }

    // 运行时检查借用规则
    // let mut b1 = data.borrow_mut();  // OK
    // let b2 = data.borrow();          // 运行时 panic！
}
```

### 1.5.3 组合使用

```rust
// Rc<RefCell<T>> 模式
use std::rc::Rc;
use std::cell::RefCell;

fn rc_refcell_combination() {
    let shared = Rc::new(RefCell::new(5));

    let shared1 = Rc::clone(&shared);
    let shared2 = Rc::clone(&shared);

    *shared1.borrow_mut() += 1;
    *shared2.borrow_mut() += 1;

    println!("shared: {:?}", shared.borrow());  // 7
}

// 实现图结构
#[derive(Debug)]
struct Graph {
    nodes: Vec<Rc<RefCell<Node>>>,
}

#[derive(Debug)]
struct Node {
    value: i32,
    edges: Vec<Weak<RefCell<Node>>>,
}
```

## 1.6 形式化分析

### 1.6.1 所有权系统的形式化模型

**定义 1.6.1**：所有权系统的状态可以形式化为：

$$
\Sigma = (O, B, L)
$$

其中：

- $O$：所有权映射 $Value \rightarrow Owner$
- $B$：借用集合 $\{(Reference, Mutable, Lifetime)\}$
- $L$：生命周期关系

**定理 1.6.2**（内存安全）：
$$
\forall r \in Reference, \text{Valid}(r) \Rightarrow \neg \text{Dangling}(r) \land \neg \text{UseAfterFree}(r)
$$

### 1.6.2 借用规则的形式化

**定理 1.6.3**（借用互斥）：
$$
\forall r_1, r_2 \in B, (r_1.mutable \lor r_2.mutable) \Rightarrow r_1 \cap r_2 = \emptyset
$$

```lean
-- Lean 形式化（概念示例）
-- 所有权系统的类型定义

def Owner := String
def Value := String
def Lifetime := Nat

inductive BorrowType
  | mutable
  | immutable

structure Borrow where
  ref : Value
  owner : Owner
  ty : BorrowType
  lifetime : Lifetime

structure OwnershipState where
  owner_map : Value → Option Owner
  borrows : List Borrow

-- 借用有效性检查
def valid_borrow (state : OwnershipState) (b : Borrow) : Prop :=
  -- 不能同时存在可变和不可变借用
  (b.ty = BorrowType.mutable →
    ∀ b' ∈ state.borrows, b'.ref = b.ref → b'.ty = BorrowType.mutable)
  ∧
  -- 生命周期有效
  (b.lifetime ≤ (state.owner_map b.ref).length)
```

### 1.6.3 生命周期推导

**算法 1.6.4**：生命周期推导

```
对于每个引用 r：
  1. 找到其指向值 v 的定义点
  2. 计算 v 的作用域范围 S
  3. r 的生命周期 'r 必须满足：'r ⊆ S

对于函数调用：
  1. 收集所有输入引用的生命周期
  2. 根据省略规则确定输出引用的生命周期
  3. 验证所有约束条件
```
