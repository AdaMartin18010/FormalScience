# 10.4.1 Kolmogorov复杂度

> 基于 Kolmogorov (1965), Chaitin (1966), Solomonoff (1964) 和 Li & Vitányi (2008)

## 10.4.1.1 引言

**算法信息论**（Algorithmic Information Theory）将信息论与可计算性理论相结合，提供了信息复杂度的根本度量。
**Kolmogorov复杂度**是这一领域的核心概念，由柯尔莫哥洛夫（Andrey Kolmogorov）于1965年独立提出（同时Chaitin和Solomonoff也有类似贡献）。
它度量描述一个对象所需的最短程序长度，是对象信息内容的绝对度量。

## 10.4.1.2 Kolmogorov复杂度的定义

### 定义 10.4.1.1（Kolmogorov复杂度）

设 $U$ 是通用图灵机，二元字符串 $x$ 的**Kolmogorov复杂度** $K(x)$ 定义为：

$$K(x) = \min_{p: U(p) = x} |p|$$

即：输出 $x$ 并停止的最短程序 $p$ 的长度。

**直观理解**：

- $K(x)$ 是 $x$ 的"信息量"或"描述复杂度"
- 随机串具有高复杂度
- 规律串具有低复杂度

### 条件Kolmogorov复杂度

$$K(x|y) = \min_{p: U(p, y) = x} |p|$$

在已知 $y$ 的条件下，描述 $x$ 所需的最短程序长度。

## 10.4.1.3 Kolmogorov复杂度的基本性质

### 定理 10.4.1.1（上界）

$$K(x) \leq |x| + O(1)$$

**证明**：存在程序直接输出 $x$（如 `print("x")`）。

### 定理 10.4.1.2（不可计算性）

$K(x)$ 是不可计算的（不存在通用算法计算所有 $x$ 的 $K(x)$）。

**证明**（反证法）：

假设存在程序 $P$ 可计算 $K(x)$。构造程序 $Q$：

1. 枚举所有字符串 $s$（按长度递增）
2. 对每个 $s$，计算 $K(s)$
3. 输出第一个满足 $K(s) > |Q|$ 的 $s$

矛盾：$Q$ 输出了 $s$，所以 $K(s) \leq |Q|$，但 $Q$ 选择 $s$ 使得 $K(s) > |Q|$。

### 定理 10.4.1.3（通用性）

对于任意图灵机 $M$，存在常数 $c_M$ 使得：
$$K(x) \leq K_M(x) + c_M$$

其中 $K_M$ 是基于机器 $M$ 的复杂度。

**意义**：复杂度在常数范围内与通用机的选择无关。

## 10.4.1.4 不可压缩性

### 定义 10.4.1.2（不可压缩串）

若 $K(x) \geq |x| - c$（$c$ 为常数），则称 $x$ 是**不可压缩的**或**随机的**。

### 定理 10.4.1.4（不可压缩串的存在性）

对于任意 $n$，存在长度为 $n$ 的不可压缩串。

**证明**：

长度为 $n$ 的程序最多有 $2^n$ 个，只能产生 $2^n$ 个不同输出。

但长度为 $n$ 的串有 $2^n$ 个，长度为 $n-1$ 的程序只能产生 $2^n - 1$ 个输出。

因此至少有一个 $n$ 长串不能被 $n-1$ 长程序产生，即 $K(x) \geq n$。

### 定理 10.4.1.5（大多数串都是不可压缩的）

长度为 $n$ 的串中，不可压缩串的比例：
$$\frac{|\{x \in \{0,1\}^n : K(x) \geq n - c\}|}{2^n} \geq 1 - 2^{-c}$$

## 10.4.1.5 柯尔莫哥洛夫复杂度与香农熵的关系

### 定理 10.4.1.6

设 $X_1, X_2, \ldots, X_n$ 是i.i.d.随机变量，熵为 $H(X)$，则：
$$\frac{K(X_1 \cdots X_n)}{n} \to H(X) \quad \text{(依概率)}$$

**意义**：

- 对于典型序列，Kolmogorov复杂度 ≈ $n \cdot H(X)$
- 连接了算法信息论与香农信息论

## 10.4.1.6 算法随机性

### 定义 10.4.1.3（算法随机性）

无限序列 $x \in \{0,1\}^\infty$ 是**算法随机的**，如果存在常数 $c$ 使得对所有前缀 $x^n$：
$$K(x^n) \geq n - c$$

**定理 10.4.1.7（Martin-Löf）**：算法随机序列关于均匀测度测度为1。

```python
import zlib
import gzip
import lzma
from typing import List
import math

def compression_ratio(s: bytes) -> float:
    """
    使用压缩算法估计复杂度
    注意：这是K(x)的上界，不是精确值
    """
    compressed = zlib.compress(s)
    return len(compressed) / len(s)

def kolmogorov_estimate(s: str) -> int:
    """
    估计Kolmogorov复杂度（上界）
    使用压缩长度作为估计
    """
    encoded = s.encode('utf-8')
    compressed = lzma.compress(encoded)
    return len(compressed)

import random

def is_prime(n: int) -> bool:
    if n < 2:
        return False
    for i in range(2, int(math.sqrt(n)) + 1):
        if n % i == 0:
            return False
    return True

for name, s in test_strings:

    ratio = compressed_len / original_len if original_len > 0 else 0
    print(f"{name:<10} {original_len:<10} {compressed_len:<10} {compressed_len:<12} {ratio:<10.4f}")

def analyze_compressibility(n: int, num_samples: int = 100):
    """分析n长串的压缩性分布"""
    ratios = []
    for _ in range(num_samples):
        s = ''.join(str(random.randint(0, 1)) for _ in range(n))
        encoded = s.encode()
        compressed = lzma.compress(encoded)
        ratios.append(len(compressed) / n)

    avg_ratio = sum(ratios) / len(ratios)
    min_ratio = min(ratios)
    high_complexity = sum(1 for r in ratios if r > 0.95) / len(ratios)

    return avg_ratio, min_ratio, high_complexity

for n in [100, 500, 1000]:

    print(f"n={n}: 平均压缩比={avg:.4f}, 最小={min_r:.4f}, 高复杂度比例≈{high:.2%}")

# 复杂度与熵的关系

print("\n复杂度与熵的关系:")

def generate_iid(p: float, n: int) -> str:
    """生成熵为H(p)的i.i.d序列"""
    return ''.join('1' if random.random() < p else '0' for _ in range(n))

for p in [0.1, 0.3, 0.5, 0.7, 0.9]:
    entropy = -(p * math.log2(p) + (1-p) * math.log2(1-p)) if 0 < p < 1 else 0

    # 生成多个样本
    complexities = []
    for _ in range(20):
        s = generate_iid(p, 1000)
        k_est = kolmogorov_estimate(s)
        complexities.append(k_est / 1000)

    avg_k = sum(complexities) / len(complexities)
    print(f"p={p:.1f}: H(X)={entropy:.4f}, K/n≈{avg_k:.4f}")

# Chaitin的Ω数（停机概率）概念介绍
print("\n" + "="*50)
print("\nChaitin的Ω数（停机概率）:")
print("Ω = Σ 2^{-|p|}，对所有停机程序p求和")

print("  2. Ω是算法随机的（不可压缩）")
```

## 10.4.1.8 总结

```mermaid

    A[对象x] --> B[Kolmogorov复杂度 K(x)]
```

**核心结论**：

1. **定义**：$K(x)$ = 输出 $x$ 的最短程序长度
2. **不可计算性**：不存在通用算法计算 $K(x)$
3. **通用性**：复杂度在常数范围内与机器无关
4. **不可压缩串**：大多数串都是近似不可压缩的
5. **随机性**：算法随机序列测度为1
