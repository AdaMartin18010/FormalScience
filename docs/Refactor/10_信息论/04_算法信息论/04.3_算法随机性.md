# 10.4.3 算法随机性

> 基于 Martin-Löf (1966), Schnorr (1971) 和 Downey & Hirschfeldt (2010)

## 10.4.3.1 引言

**算法随机性**（Algorithmic Randomness）提供了"随机序列"的严格数学定义。
与概率论中"以概率1成立"的表述不同，算法随机性针对单个无限序列，使用可计算性理论来刻画随机性的本质。
这一领域由Martin-Löf、Kolmogorov、Chaitin、Schnorr等人奠基，已成为数理逻辑和理论计算机科学的重要分支。

## 10.4.3.2 随机性的悖论

### 直观困难

考虑两个无限二元序列：

- $x = 000000\ldots$（全零）
- $y = $ 抛硬币结果的记录

从概率论角度，两者出现的概率都是0（在均匀测度下）。但从直观上，$x$ 明显不是随机的，而 $y$ 应该是随机的。

**问题**：如何精确定义"随机序列"？

## 10.4.3.3 Martin-Löf随机性

### 有效零测集

**定义 10.4.3.1（Martin-Löf测试）**：

**Martin-Löf测试**是一列可计算枚举的开集 $\{U_n\}_{n \in \mathbb{N}}$，满足：

1. $U_{n+1} \subseteq U_n$（嵌套性）
2. $\mu(U_n) \leq 2^{-n}$（测度衰减）

其中 $\mu$ 是均匀测度（硬币投掷测度）。

### Martin-Löf随机性的定义

**定义 10.4.3.2（Martin-Löf随机性）**：

序列 $x \in \{0,1\}^\infty$ 是**Martin-Löf随机的**，如果对于所有Martin-Löf测试 $\{U_n\}$：
$$x \notin \bigcap_{n} U_n$$

**直观**：随机序列不能通过任何有效测试被"检测"为非随机。

### 存在性定理

**定理 10.4.3.1（Martin-Löf, 1966）**：

Martin-Löf随机序列的集合测度为1。

**证明概要**：

存在**通用Martin-Löf测试** $\{\hat{U}_n\}$，使得：
$$\bigcap_n \hat{U}_n = \bigcup_{\{U_n\} \text{测试}} \bigcap_n U_n$$

由于 $\mu(\hat{U}_n) \leq 2^{-n}$，有 $\mu(\bigcap_n \hat{U}_n) = 0$。

因此，随机序列的补集测度为0，即随机序列测度为1。

## 10.4.3.4 算法随机性的等价刻画

### Kolmogorov复杂度刻画

**定理 10.4.3.2**：

$x \in \{0,1\}^\infty$ 是Martin-Löf随机的，当且仅当存在常数 $c$ 使得对所有前缀 $x^n$：
$$K(x^n) \geq n - c$$

**意义**：随机序列的所有前缀都是不可压缩的。

### 鞅刻画（Schnorr）

**定义 10.4.3.3（鞅）**：

函数 $d: \{0,1\}^* \to \mathbb{R}_{\geq 0}$ 是**鞅**，如果：
$$d(w) = \frac{d(w0) + d(w1)}{2}$$

**定理 10.4.3.3（Schnorr）**：

$x$ 是Martin-Löf随机的，当且仅当不存在可计算鞅在 $x$ 上成功（即 $d(x^n)$ 无界）。

**意义**：随机序列不能被任何有效的赌博策略预测。

### 不可预测性刻画

**定理 10.4.3.4**：

$x$ 是Martin-Löf随机的，当且仅当不存在可计算函数 $f$ 能够基于前缀预测下一位的正确率显著高于1/2。

## 10.4.3.5 其他随机性概念

### Schnorr随机性

**定义 10.4.3.4（Schnorr测试）**：

Schnorr测试是Martin-Löf测试加上额外条件：$\mu(U_n)$ 是**精确可计算**的（不仅是上界）。

**关系**：Schnorr随机 $\subset$ Martin-Löf随机

### 可计算随机性

**定义 10.4.3.5**：

$x$ 是**可计算随机的**，如果没有**可计算**鞅在 $x$ 上成功。

**关系链**：
$$\text{可计算随机} \subset \text{Schnorr随机} \subset \text{Martin-Löf随机}$$

### Kurtz随机性（弱随机性）

**定义 10.4.3.6**：

$x$ 是**Kurtz随机的**，如果不在任何**闭**的有效零测集中。

这是 weakest 的随机性概念。

## 10.4.3.6 算法随机序列的性质

### 定理 10.4.3.5（大数定律）

若 $x$ 是Martin-Löf随机的，则：
$$\lim_{n \to \infty} \frac{x_1 + \cdots + x_n}{n} = \frac{1}{2}$$

### 定理 10.4.3.6（重对数律）

Martin-Löf随机序列满足重对数律：
$$\limsup_{n \to \infty} \frac{\sum_{i=1}^n (x_i - 1/2)}{\sqrt{n \log \log n}} = \frac{1}{\sqrt{2}}$$

### 定理 10.4.3.7（Von Mises-Church-Wald）

Martin-Löf随机序列在任何**可计算选择函数**下满足大数定律。

## 10.4.3.7 随机性与信息论

### 熵与随机性

对于Martin-Löf随机序列 $x$：
$$\lim_{n \to \infty} \frac{K(x^n)}{n} = 1$$

这对应于均匀分布的熵率。

### 非均匀分布

对于偏斜分布 $p \neq 1/2$，可以定义相对于该分布的随机性：
$$\lim_{n \to \infty} \frac{K(x^n)}{n} = H(p)$$

```python
import random
import math
from typing import List, Callable

def kolmogorov_estimate(s: str) -> int:
    """
    使用压缩估计Kolmogorov复杂度
    """
    import lzma
    return len(lzma.compress(s.encode()))

def is_prefix_incompressible(s: str, c: int = 10) -> bool:
    """
    检查字符串是否前缀不可压缩
    K(s) >= |s| - c
    """
    k_est = kolmogorov_estimate(s)
    return k_est >= len(s) - c

def prefix_complexity_ratio(s: str) -> float:
    """
    返回 K(s)/|s|
    """
    if len(s) == 0:
        return 0
    return kolmogorov_estimate(s) / len(s)

def martin_lof_test_pass(sequences: List[str], n: int) -> List[str]:
    """
    简单的Martin-Löf风格测试：筛选出高复杂度的序列

    实际Martin-Löf测试涉及可计算枚举的零测集，这里用复杂度作为代理
    """
    threshold = 1.0 - 2**(-n)  # 复杂度比例阈值
    return [s for s in sequences
            if prefix_complexity_ratio(s) >= threshold]

def von_mises_test(s: str, max_subseq_length: int = 10) -> bool:
    """
    Von Mises风格测试：检查子序列的1的频率
    """
    for length in range(1, min(len(s), max_subseq_length) + 1):
        for start in range(0, len(s) - length + 1, length):
            subseq = s[start:start+length]
            if length > 0:
                freq = subseq.count('1') / length
                # 允许一定的偏差
                if abs(freq - 0.5) > 0.3 and length >= 5:
                    return False
    return True

def law_of_large_numbers_test(s: str) -> bool:
    """
    大数定律测试
    """
    if len(s) == 0:
        return True
    freq = s.count('1') / len(s)
    # 对于长序列，频率应接近0.5
    expected_deviation = 1.0 / math.sqrt(len(s))
    return abs(freq - 0.5) < 3 * expected_deviation

def generate_random_sequence(length: int) -> str:
    """生成随机序列"""
    return ''.join(str(random.randint(0, 1)) for _ in range(length))

def generate_pattern_sequence(length: int, pattern: str = "01") -> str:
    """生成模式序列"""
    repeats = length // len(pattern) + 1
    return (pattern * repeats)[:length]

def is_prime(n: int) -> bool:
    if n < 2:
        return False
    for i in range(2, int(math.sqrt(n)) + 1):
        if n % i == 0:
            return False
    return True

def simple_martingale_test(s: str, initial_capital: float = 1.0) -> float:
    """
    简单的鞅测试：基于简单策略的赌博
    如果序列随机，资本不应无界增长
    """
    capital = initial_capital

    # 策略：如果连续出现两个相同的，押注下一个不同
    for i in range(2, len(s)):
        bet = capital * 0.1  # 下注10%

        if s[i-2] == s[i-1]:  # 连续两个相同
            prediction = '1' if s[i-1] == '0' else '0'
        else:
            prediction = s[i-1]  # 跟随趋势

        if prediction == s[i]:
            capital += bet  # 赢
        else:
            capital -= bet  # 输

    return capital

print("\n随机序列的资本应保持相对稳定，非随机序列可能出现极端值")
```

## 10.4.3.9 总结

```mermaid

    A[算法随机性] --> B[Martin-Löf随机]
    A --> C[Schnorr随机]
    A --> D[可计算随机]
    A --> E[Kurtz随机]

    B --> F[通过所有有效检验]
    C --> G[精确可计算检验]
    D --> H[无可计算鞅成功]

    B -.->|蕴含| C
    C -.->|蕴含| D
    D -.->|蕴含| E
```

**核心结论**：

1. **Martin-Löf随机性**：通过所有有效统计检验
2. **等价刻画**：前缀不可压缩性、鞅不可成功性、不可预测性
3. **测度为1**：随机序列构成满测集
4. **统计定律**：随机序列满足大数定律、重对数律等
5. **层次结构**：ML-随机 $\subset$ Schnorr随机 $\subset$ 可计算随机 $\subset$ Kurtz随机
