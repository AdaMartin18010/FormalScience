## 1. 引言

### 1.1 背景与动机

线性时序逻辑（LTL）是形式化验证领域的核心理论工具，由 Amir Pnueli 于 1977 年提出。LTL 扩展了命题逻辑，引入时间算子以描述系统行为的时序特性。

**核心问题**：如何形式化描述并发系统的时序性质，并自动验证这些性质？

### 1.2 与其他逻辑的关系

| 逻辑系统 | 时间模型 | 表达能力 | 模型检测复杂度 |
|---------|---------|---------|--------------|
| PL (命题逻辑) | 无 | 静态性质 | P-完全 |
| LTL | 线性 | 路径性质 | PSPACE-完全 |
| CTL | 分支 | 状态性质 | P-完全 |
| CTL* | 分支+线性 | 完整 | PSPACE-完全 |

> **交叉引用**：参见 [01.2_计算树逻辑_CTL.md](./01.2_计算树逻辑_CTL.md) 了解分支时序逻辑。

## 2. 语法定义

### 2.1 形式化语法

**定义 2.1** (LTL 语法)。设 $AP$ 为原子命题集合，LTL 公式 $\varphi$ 的语法由以下文法归纳定义：

$$\varphi ::= \top \mid \bot \mid p \mid \neg \varphi \mid \varphi \land \varphi \mid \varphi \lor \varphi \mid \mathbf{X}\varphi \mid \mathbf{F}\varphi \mid \mathbf{G}\varphi \mid \varphi \mathbf{U} \varphi$$

其中 $p \in AP$，各时序算子的含义如下：

| 算子 | 名称 | 直观含义 |
|-----|-----|---------|
| $\mathbf{X}\varphi$ | Next (下一时刻) | 下一状态满足 $\varphi$ |
| $\mathbf{F}\varphi$ | Future/Eventually (最终) | 未来某时刻满足 $\varphi$ |
| $\mathbf{G}\varphi$ | Globally/Always (总是) | 所有未来时刻满足 $\varphi$ |
| $\varphi \mathbf{U} \psi$ | Until (直到) | $\varphi$ 持续成立直到 $\psi$ 成立 |

### 2.2 导出算子

**定义 2.2** (导出时序算子)。以下算子可由基本算子定义：

- **Release**: $\varphi \mathbf{R} \psi \equiv \neg(\neg\varphi \mathbf{U} \neg\psi)$
- **Weak Until**: $\varphi \mathbf{W} \psi \equiv (\varphi \mathbf{U} \psi) \lor \mathbf{G}\varphi$
- **Implication**: $\varphi \to \psi \equiv \neg\varphi \lor \psi$

**定理 2.1** (算子完备性)。$\{\mathbf{X}, \mathbf{U}\}$ 是 LTL 的函数完备算子集。

_证明_：需证所有其他算子均可由此表示。

- $\mathbf{F}\varphi \equiv \top \mathbf{U} \varphi$
- $\mathbf{G}\varphi \equiv \neg\mathbf{F}\neg\varphi \equiv \neg(\top \mathbf{U} \neg\varphi)$
- $\mathbf{R}$ 由定义 2.2 可得。 ∎

### 2.3 语法优先级

运算符优先级（从高到低）：

```
¬, X  (一元，右结合)
∧    (二元，左结合)
∨    (二元，左结合)
U, R (二元，左结合)
→    (二元，右结合)
```

## 3. 语义理论

### 3.1 Kripke 结构

**定义 3.1** (Kripke 结构)。一个 Kripke 结构是一个四元组 $M = (S, S_0, R, L)$，其中：

- $S$: 有限状态集合
- $S_0 \subseteq S$: 初始状态集合
- $R \subseteq S \times S$: 全转移关系（即 $\forall s \in S, \exists s' \in S: (s, s') \in R$）
- $L: S \to 2^{AP}$: 标记函数，将每个状态映射到在该状态为真的原子命题集合

```mermaid

    S0[Initial State] --> S1[s₁: {p}]
    S1 --> S2[s₂: {q}]
    S1 --> S3[s₃: {p,q}]
    S2 --> S2
    S3 --> S1

    style S2 fill:#FFE4B5
    style S3 fill:#FFE4B5
```

### 3.2 计算路径

**定义 3.2** (路径)。在 Kripke 结构 $M$ 中，从状态 $s$ 出发的**路径** $\pi$ 是状态序列：

$$\pi = s_0 s_1 s_2 \ldots \quad \text{满足} \quad s_0 = s \land \forall i \geq 0: (s_i, s_{i+1}) \in R$$

**定义 3.3** (路径操作)。设 $\pi = s_0 s_1 s_2 \ldots$：

- $\pi[i] = s_i$: 第 $i$ 个状态
- $\pi^i = s_i s_{i+1} s_{i+2} \ldots$: 从第 $i$ 个状态开始的后缀

### 3.3 形式语义

**定义 3.4** (LTL 满足关系)。设 $\pi$ 为路径，$\models$ 关系归纳定义如下：

| 公式 | 满足条件 | 形式化定义 |
|-----|---------|-----------|
| $\pi \models \top$ | 恒真 | - |
| $\pi \models p$ | 原子命题 | $p \in L(\pi[0])$ |
| $\pi \models \neg\varphi$ | 否定 | $\pi \not\models \varphi$ |
| $\pi \models \varphi \land \psi$ | 合取 | $\pi \models \varphi$ 且 $\pi \models \psi$ |
| $\pi \models \mathbf{X}\varphi$ | 下一时刻 | $\pi^1 \models \varphi$ |
| $\pi \models \mathbf{F}\varphi$ | 最终 | $\exists k \geq 0: \pi^k \models \varphi$ |
| $\pi \models \mathbf{G}\varphi$ | 总是 | $\forall k \geq 0: \pi^k \models \varphi$ |
| $\pi \models \varphi \mathbf{U} \psi$ | 直到 | $\exists k \geq 0: (\pi^k \models \psi) \land (\forall 0 \leq j < k: \pi^j \models \varphi)$ |

### 3.4 语义等价与对偶

**定理 3.1** (对偶律)。对于任意 LTL 公式：

1. $\neg\mathbf{X}\varphi \equiv \mathbf{X}\neg\varphi$ （Next 的自对偶）
2. $\neg\mathbf{F}\varphi \equiv \mathbf{G}\neg\varphi$ （F/G 对偶）
3. $\neg\mathbf{G}\varphi \equiv \mathbf{F}\neg\varphi$ （G/F 对偶）
4. $\neg(\varphi \mathbf{U} \psi) \equiv (\neg\varphi) \mathbf{R} (\neg\psi)$ （U/R 对偶）

_证明_（以性质2为例）：

$\pi \models \neg\mathbf{F}\varphi$
$\Leftrightarrow \neg(\exists k \geq 0: \pi^k \models \varphi)$
$\Leftrightarrow \forall k \geq 0: \pi^k \not\models \varphi$
$\Leftrightarrow \forall k \geq 0: \pi^k \models \neg\varphi$
$\Leftrightarrow \pi \models \mathbf{G}\neg\varphi$ ∎

## 4. 常用规约模式

### 4.1 安全性与活性

**定义 4.1** (安全性)。安全性性质断言"坏事永不会发生"：

$$\mathbf{G}\neg bad$$

**定义 4.2** (活性)。活性性质断言"好事终将发生"：

$$\mathbf{G}(req \to \mathbf{F}ack)$$

**定理 4.1** (Alpern-Schneider)。任意 LTL 公式可表示为安全性与活性的合取。

### 4.2 响应性质

| 模式 | LTL 公式 | 含义 |
|-----|---------|-----|
| 请求-响应 | $\mathbf{G}(req \to \mathbf{F}res)$ | 每个请求终将被响应 |
| 强响应 | $\mathbf{G}(req \to (\neg cancel \mathbf{U} res))$ | 请求持续直到响应，除非取消 |
| 无饥饿 | $\mathbf{G}(req \to \mathbf{F}^{\leq k} res)$ | 响应在 $k$ 步内完成（有界） |

### 4.3 公平性约束

**定义 4.3** (公平性)。

- **弱公平** (justice): $\mathbf{GF}enabled \to \mathbf{GF}executed$
- **强公平** (compassion): $\mathbf{FG}enabled \to \mathbf{GF}executed$

> **交叉引用**：公平性约束在模型检测中至关重要，参见 [01.3_时序逻辑与规约.md](./01.3_时序逻辑与规约.md) 第3节。

## 5. 模型检测算法

### 5.1 问题定义

**定义 5.1** (LTL 模型检测)。给定 Kripke 结构 $M$ 和 LTL 公式 $\varphi$，判定是否 $M \models \varphi$。

**定理 5.1** (PSPACE-完全性)。LTL 模型检测问题是 PSPACE-完全的。

### 5.2 基于自动机的检测

#### 5.2.1 B"uchi 自动机

**定义 5.2** (B"uchi 自动机)。一个 B"uchi 自动机是一个五元组 $A = (\Sigma, Q, Q_0, \delta, F)$：

- $\Sigma$: 输入字母表
- $Q$: 状态集合
- $Q_0 \subseteq Q$: 初始状态
- $\delta: Q \times \Sigma \to 2^Q$: 转移函数
- $F \subseteq Q$: 接受状态集合

**定义 5.3** (B"uchi 接受)。运行 $r = q_0 q_1 q_2 \ldots$ 被接受当且仅当 $\inf(r) \cap F \neq \emptyset$。

#### 5.2.2 LTL 到 B"uchi 自动机的转换

**定理 5.2** (Vardi-Wolper)。对于任意 LTL 公式 $\varphi$，存在 B"uchi 自动机 $A_\varphi$ 使得 $\mathcal{L}(A_\varphi) = \{\pi \mid \pi \models \varphi\}$，且 $|A_\varphi| = O(2^{|\varphi|})$。

**构造算法** (tableau 构造)：

```python
def ltl_to_buchi(phi):
    """
    将LTL公式转换为Buchi自动机
    时间复杂度: O(2^|phi|)
    """
    # 1. 子公式闭包计算
    subformulas = closure(phi)

    # 2. 状态为极大一致子集
    states = maximal_consistent_sets(subformulas)

    # 3. 定义转移关系
    transitions = {}
    for s in states:
        for next_s in states:
            if consistent_transition(s, next_s):
                label = atomic_propositions(s)
                transitions[(s, label)] = transitions.get((s, label), []) + [next_s]

    # 4. 接受条件: 满足所有until子公式的公平性
    accepting = compute_fairness_constraints(phi, states)

    return BuchiAutomaton(states, transitions, initial_states, accepting)
```

### 5.3 检测过程

**算法 5.1** (LTL 模型检测)。

```python
def check_ltl(M, phi):
    """
    LTL模型检测算法
    输入: Kripke结构 M, LTL公式 phi
    输出: M |= phi ?
    """
    # 1. 构建公式自动机 A_phi
    A_phi = ltl_to_buchi(phi)

    # 2. 构建系统自动机 A_M (从M构造)
    A_M = kripke_to_buchi(M)

    # 3. 构造乘积自动机 A = A_M × A_¬phi
    A_neg_phi = ltl_to_buchi(Negation(phi))
    product = product_automaton(A_M, A_neg_phi)

    # 4. 检查乘积自动机的语言是否为空
    # 即检查是否存在被接受的运行
    if has_accepting_run(product):
        # 找到反例
        counterexample = extract_counterexample(product)
        return False, counterexample
    else:
        return True, None
```

**定理 5.3** (复杂度)。算法 5.1 的时间和空间复杂度为 $O(|M| \cdot 2^{|\varphi|})$。

## 6. Lean 形式化

### 6.1 LTL 语法定义

```lean4
import Mathlib

-- 原子命题类型
inductive Atom
  | p | q | r | req | ack | grant
  deriving DecidableEq, Repr

-- LTL 公式归纳定义
inductive LTL (A : Type) where
  | true  : LTL A                    -- ⊤
  | false : LTL A                    -- ⊥
  | atom  : A → LTL A                -- 原子命题
  | neg   : LTL A → LTL A            -- ¬φ
  | and   : LTL A → LTL A → LTL A    -- φ ∧ ψ
  | or    : LTL A → LTL A → LTL A    -- φ ∨ ψ
  | next  : LTL A → LTL A            -- X φ
  | until : LTL A → LTL A → LTL A    -- φ U ψ
  deriving DecidableEq, Repr

-- 派生算子
def LTL.finally {A} (φ : LTL A) : LTL A := LTL.until LTL.true φ  -- F φ

def LTL.globally {A} (φ : LTL A) : LTL A := LTL.neg (LTL.finally (LTL.neg φ))  -- G φ

def LTL.implies {A} (φ ψ : LTL A) : LTL A := LTL.or (LTL.neg φ) ψ  -- φ → ψ
```

### 6.2 路径语义定义

```lean4
-- 路径是无限状态序列
def Path (S : Type) := ℕ → S

-- 路径后缀
def Path.suffix {S} (π : Path S) (k : ℕ) : Path S := λ n => π (k + n)

-- 标记函数
def Labeling (S A : Type) := S → Set A

-- 满足关系 (归纳定义)
inductive Satisfies {S A : Type}
    (L : Labeling S A) : Path S → LTL A → Prop where
  | tt {π} : Satisfies L π LTL.true
  | atom {π a} : a ∈ L (π 0) → Satisfies L π (LTL.atom a)
  | neg {π φ} : ¬(Satisfies L π φ) → Satisfies L π (LTL.neg φ)
  | and {π φ ψ} : Satisfies L π φ → Satisfies L π ψ →
                  Satisfies L π (LTL.and φ ψ)
  | next {π φ} : Satisfies L (π.suffix 1) φ → Satisfies L π (LTL.next φ)
  | until {π φ ψ} :
      (∃ k, Satisfies L (π.suffix k) ψ ∧
        ∀ j < k, Satisfies L (π.suffix j) φ) →
      Satisfies L π (LTL.until φ ψ)

-- 记号: π ⊨ φ
infix:50 " ⊨ " => Satisfies
```

### 6.3 定理证明示例

```lean4
-- 定理: G φ ≡ ¬F ¬φ
example {S A L π φ} :
    (π ⊨ LTL.globally φ) ↔ ¬(π ⊨ LTL.finally (LTL.neg φ)) := by
  constructor
  · -- 正向: G φ → ¬F ¬φ
    intro h
    simp [LTL.globally, LTL.finally, LTL.true] at h ⊢
    assumption
  · -- 反向: ¬F ¬φ → G φ
    intro h
    simp [LTL.globally, LTL.finally, LTL.true] at h ⊢
    assumption
```

```lean4
-- LTL公式的归纳定义
inductive LTL (AP : Type) where
  | true  : LTL AP
  | false : LTL AP
  | atom  : AP → LTL AP
  | neg   : LTL AP → LTL AP
  | and   : LTL AP → LTL AP → LTL AP
  | or    : LTL AP → LTL AP → LTL AP
  | next  : LTL AP → LTL AP
  | until : LTL AP → LTL AP → LTL AP

-- 满足关系（归纳定义）
inductive Satisfies {S AP : Type} (L : Labeling S AP) :
    Path S → LTL AP → Prop where
  | tt {π} : Satisfies L π ⊤
  | atom {π a} : a ∈ L (π 0) → Satisfies L π (LTL.atom a)
  | neg {π φ} : ¬(Satisfies L π φ) → Satisfies L π (LTL.neg φ)
  | and {π φ ψ} : Satisfies L π φ → Satisfies L π ψ →
                  Satisfies L π (LTL.and φ ψ)
  | next {π φ} : Satisfies L (π.suffix 1) φ → Satisfies L π (LTL.next φ)
  | until {π φ ψ} :
      (∃ k, Satisfies L (π.suffix k) ψ ∧
        ∀ j < k, Satisfies L (π.suffix j) φ) →
      Satisfies L π (LTL.until φ ψ)

-- F/G 对偶性: ¬Fφ ↔ G¬φ
theorem finally_duality {S AP L π φ} :
    (π ⊨ (¬(F φ)) ↔ π ⊨ (G (¬φ))) := by
  constructor
  · intro h; simp [LTL.globally, LTL.finally] at h ⊢; assumption
  · intro h; simp [LTL.globally, LTL.finally] at h ⊢; assumption
```
