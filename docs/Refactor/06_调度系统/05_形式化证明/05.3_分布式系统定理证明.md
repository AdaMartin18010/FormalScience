## 1. CAP定理形式化证明

### 1.1 形式化定义

**定义 1.1（分布式系统）**: 分布式系统是一个三元组 $\mathcal{D} = (\mathcal{N}, \mathcal{M}, \mathcal{O})$：

- $\mathcal{N} = \{n_1, \ldots, n_k\}$：节点集合
- $\mathcal{M}$：消息传递系统
- $\mathcal{O}$：操作集合（读/写）

**定义 1.2（一致性 - Consistency）**:
系统满足一致性，若所有读操作返回最近一次写操作的结果：
\begin{equation}
\forall r \in \text{Reads}: r \text{ returns } v \iff \text{last write before } r \text{ wrote } v
\end{equation}

**定义 1.3（可用性 - Availability）**:
系统满足可用性，若每个非故障节点对请求都能在有限时间内响应：
\begin{equation}
\forall n \in \mathcal{N} \setminus \mathcal{F}: \forall \text{req}: \text{response}(n, \text{req}) \text{ within } T < \infty
\end{equation}
其中 $\mathcal{F}$ 是故障节点集合。

**定义 1.4（分区容忍性 - Partition Tolerance）**:
系统满足分区容忍性，若在网络分区情况下仍能继续运行：
\begin{equation}
\forall P \subseteq \mathcal{N}: \text{system operates correctly even when } \mathcal{N} \setminus P \text{ is partitioned from } P
\end{equation}

---

### 1.2 CAP定理

\begin{theorem}[CAP定理]
对于分布式数据存储，不可能同时满足一致性（Consistency）、可用性（Availability）和分区容忍性（Partition Tolerance）。最多只能同时满足其中两个。
\end{theorem}

\begin{proof}

**证明策略**：假设系统同时满足C、A、P，导出矛盾。

**构造场景**：

考虑两个节点 $n_1, n_2$ 组成的最小系统，数据项 $x$ 的初始值为 $v_0$。

**步骤1：网络分区**

网络发生分区，$n_1$ 和 $n_2$ 之间无法通信。

由分区容忍性（P），系统必须继续运行。

**步骤2：客户端请求**

- 客户端 $c_1$ 连接到 $n_1$，写 $x = v_1$
- 客户端 $c_2$ 连接到 $n_2$，读 $x$

**步骤3：可用性要求**

由可用性（A）：

- $n_1$ 必须响应写请求
- $n_2$ 必须响应读请求

**步骤4：一致性冲突**

$n_1$ 接受了写请求，本地 $x = v_1$。
$n_2$ 未收到更新（网络分区），本地 $x = v_0$。

$n_2$ 响应读请求：

- 若返回 $v_0$：违反一致性（C），因为最新值是 $v_1$
- 若返回 $v_1$：$n_2$ 不可能知道 $v_1$（未通信），故不可能

**步骤5：矛盾**

无法同时满足所有三个属性。
\end{proof}

---

### 1.3 CAP权衡的形式化

\begin{corollary}[CAP系统分类]
分布式系统必须选择以下三类之一：
\begin{enumerate}
    \item CP系统：牺牲可用性，保证一致性和分区容忍性
    \item AP系统：牺牲一致性，保证可用性和分区容忍性
    \item CA系统：牺牲分区容忍性，仅在无分区时运行（非分布式）
\end{enumerate}
\end{corollary}

## 2. FLP不可能性定理完整证明

### 2.1 异步系统模型

**定义 2.1（异步消息传递系统）**:

- 进程集合 $\mathcal{P} = \{p_1, \ldots, p_n\}$
- 消息传递延迟：有限但无界，不可预测
- 进程故障：崩溃停止（crash-stop）

**定义 2.2（配置）**: 配置 $C = (S, M)$ 包含：

- $S$：所有进程的本地状态
- $M$：在途消息集合

**定义 2.3（事件）**: 事件 $e = (p, m)$ 表示进程 $p$ 接收消息 $m$。

**定义 2.4（执行）**: 执行是配置和事件交替的无限序列：
\begin{equation}
C_0 \xrightarrow{e_1} C_1 \xrightarrow{e_2} C_2 \xrightarrow{e_3} \cdots
\end{equation}

---

### 2.2 价态理论

**定义 2.5（决策状态）**: 配置 $C$ 是**决策状态**，若某进程已输出决策值。

**定义 2.6（价态）**:

- $C$ 是**0-价**的：所有从 $C$ 可达的配置决策0
- $C$ 是**1-价**的：所有从 $C$ 可达的配置决策1
- $C$ 是**双价**的：存在可达的0-决策和1-决策配置

**引理 2.1（初始双价性）**: 存在双价的初始配置。

\begin{proof}
考虑两个初始配置：

- $I_0$：所有进程提议0。由有效性，必须决策0，故是0-价的。
- $I_1$：所有进程提议1。由有效性，必须决策1，故是1-价的。

考虑从 $I_0$ 逐步改变进程提议值为1的序列：
\begin{equation}
I_0 = J_0, J_1, J_2, \ldots, J_n = I_1
\end{equation}
其中 $J_k$ 是前 $k$ 个进程提议1，其余提议0。

$J_0$ 是0-价的，$J_n$ 是1-价的。
必存在相邻配置 $J_k$（0-价或双价）和 $J_{k+1}$（1-价或双价）。

若 $J_k$ 0-价且 $J_{k+1}$ 1-价，考虑 $p_{k+1}$ 崩溃的情况。
剩余进程无法区分 $J_k$ 和 $J_{k+1}$（$p_{k+1}$ 无影响）。
若它们决策不同值，违反一致性。

因此，必存在双价初始配置。
\end{proof}

**引理 2.2（可迁延性）**: 设 $C$ 是双价配置，$e = (p, m)$ 是可应用于 $C$ 的事件。则存在从 $C$ 可达的配置 $C'$，使得：
\begin{enumerate}
    \item $e$ 可应用于 $C'$
    \item $C'$ 是双价的
\end{enumerate}

\begin{proof}
设 $\mathcal{C}$ 是从 $C$ 可达且 $e$ 可应用的配置集合。

假设所有 $C' \in \mathcal{C}$ 都是单价（非双价）。

由于 $C$ 是双价的，存在从 $C$ 到0-决策配置的执行 $E_0$，和到1-决策配置的 $E_1$。

考虑 $E_0$ 和 $E_1$ 中第一个应用 $e$ 的位置：

- 若都不应用 $e$：设 $D_0$ 是 $E_0$ 的最终配置（0-决策），$e$ 仍可应用，$D_0$ 是0-价的
- 类似 $D_1$ 是1-价的

但 $D_0$ 和 $D_1$ 可通过不应用 $e$ 的执行到达，且都允许 $e$，这与价态定义矛盾。

详细构造：设 $F_0$ 是 $E_0$ 中最后一个 $e$ 可应用前的配置，$F_1$ 类似。

由假设，$F_0$ 和 $F_1$ 都是单价的。
由于 $E_0$ 决策0，$F_0$ 必为0-价。
由于 $E_1$ 决策1，$F_1$ 必为1-价。

现在考虑交换性：

- 从 $C$ 执行 $E_0$ 的事件（跳过 $e$）到 $F_0$
- 从 $C$ 执行 $E_1$ 的事件（跳过 $e$）到 $F_1$

若这些执行涉及不同于 $p$ 的进程，则它们可以交换。
通过精心构造，可以导出 $F_0$ 和 $F_1$ 可从同一配置到达的矛盾。

因此，必存在双价的 $C' \in \mathcal{C}$。
\end{proof}

### 2.3 FLP主定理

\begin{theorem}[FLP不可能性]
在异步消息传递系统中，即使只有一个故障进程，也不存在确定性共识算法同时满足：
\begin{enumerate}
    \item 终止性
    \item 一致性
    \item 有效性
\end{enumerate}
\end{theorem}

\begin{proof}

构造无限执行：

**步骤1**：由引理2.1，存在双价初始配置 $C_0$。

**步骤2**：对 $i = 0, 1, 2, \ldots$：

- 由引理2.2，存在双价配置 $C_{i+1}$ 从 $C_i$ 可达，且某事件 $e_{i+1}$ 可应用
- 构造 $C_i \xrightarrow{\sigma_i} C_{i+1}$，其中 $\sigma_i$ 是不包含 $e_{i+1}$ 的执行片段

**步骤3**：组合无限执行：
\begin{equation}
C_0 \xrightarrow{\sigma_0} C_1 \xrightarrow{\sigma_1} C_2 \xrightarrow{\sigma_2} \cdots
\end{equation}

**关键观察**：

- 每个 $C_i$ 都是双价的
- 双价意味着尚未决策（决策意味着单价）
- 因此，没有任何进程在此执行中决策

**为什么这是有效的异步执行？**

通过延迟消息传递：

- 对于事件 $e_{i+1} = (p, m)$，简单地延迟消息 $m$ 的传递
- 异步系统允许任意有限延迟
- 因此可以永远延迟某些消息

**结论**：

我们构造了一个公平执行（所有正确进程无限次获得机会），其中没有进程决策。

这违反了终止性。

因此，不存在满足所有三个属性的确定性共识算法。
\end{proof}

## 3. 一致性模型等价性

### 3.1 一致性层次结构

**定义 3.1（线性一致性 - Linearizability）**:
操作历史 $H$ 是线性一致的，若存在扩展 $H'$ 和顺序 $<$ 使得：

1. $<$ 是 $H'$ 所有操作的线性序
2. $<$ 尊重实时序：若操作 $op_1$ 完成于 $op_2$ 开始前，则 $op_1 < op_2$
3. 每个读操作返回最近写操作的值

**定义 3.2（顺序一致性 - Sequential Consistency）**:
操作历史 $H$ 是顺序一致的，若存在扩展 $H'$ 和顺序 $<$ 使得：

1. $<$ 是 $H'$ 所有操作的线性序
2. 每个进程的操作按程序序排列
3. 每个读操作返回最近写操作的值

**定义 3.3（因果一致性 - Causal Consistency）**:
操作历史 $H$ 是因果一致的，若存在每个进程的视图使得：

1. 每个进程看到的操作顺序是因果一致的
2. 因果序：若 $op_1$ 发生在 $op_2$ 之前（程序序或读依赖于写），则所有进程看到 $op_1$ 在 $op_2$ 之前

---

### 3.2 等价关系定理

\begin{theorem}[一致性层次]
\begin{equation}
\text{Linearizability} \subsetneq \text{Sequential Consistency} \subsetneq \text{Causal Consistency}
\end{equation}
即：线性一致性 $\Rightarrow$ 顺序一致性 $\Rightarrow$ 因果一致性，但反之不成立。
\end{theorem}

\begin{proof}

**线性一致性 $\Rightarrow$ 顺序一致性**：

线性一致性要求：

- 全局顺序尊重实时序

顺序一致性要求：

- 全局顺序尊重程序序（每个进程内的操作顺序）

由于程序序蕴含实时序（同一进程的操作按程序序执行，完成时间也按程序序），线性一致的顺序必然满足顺序一致性的要求。

严格性：存在顺序一致但非线性一致的历史。

**构造反例**：

```
进程P1: W(x,1) ------> W(x,2)
时间:    t1         t2

进程P2:          R(x)=2 ------> R(x)=1
时间:            t3           t4
```

其中 $t1 < t2 < t3 < t4$。

此历史是顺序一致的（顺序：W(x,1), W(x,2), R(x)=2, R(x)=1 尊重程序序）。

但不是线性一致的：R(x)=2 在 t3，W(x,1) 在 t1，但 R(x)=2 后 R(x)=1 意味着 W(x,2) 在 W(x,1) 之前（矛盾于实时序 $t1 < t2$）。

**顺序一致性 $\Rightarrow$ 因果一致性**：

因果一致性只要求：

- 因果相关的操作全局有序

顺序一致性提供：

- 所有操作全局有序
- 程序序（因果序的一部分）被尊重

因此，顺序一致的历史必然因果一致。

严格性：存在因果一致但非顺序一致的历史。

**构造反例**：

```
进程P1: W(x,1)
进程P2:          W(x,2)
进程P3:                   R(x)=1  R(x)=2
进程P4:                   R(x)=2  R(x)=1
```

P3 看到顺序：W(x,1), W(x,2)
P4 看到顺序：W(x,2), W(x,1)

没有全局顺序同时满足两者，故非顺序一致。

但因果一致：P1 和 P2 无因果关系，不同进程可以不同顺序看到它们的写。
\end{proof}

### 3.3 实现等价性

\begin{theorem}[寄存器层次]
对于共享存储器：
\begin{equation}
\text{Atomic Register} \subsetneq \text{Regular Register} \subsetneq \text{Safe Register}
\end{equation}
\end{theorem}

\begin{proof}

**安全寄存器（Safe）**：

- 无并发写时，读返回最后写的值
- 并发写时，读可返回任意值

**正则寄存器（Regular）**：

- 无并发写时，读返回最后写的值
- 并发写时，读返回被写的某个值或旧值

**原子寄存器（Atomic）**：

- 等价于线性一致性
- 读操作看起来在单个时间点瞬间完成

**包含关系**：

- 原子寄存器满足正则性（更强的条件）
- 正则寄存器满足安全性

**严格性**：

正则但非原子：考虑并发读写。

```
写W(x,1)开始 ---- 写W(x,2)开始 ---- 读R(x) ---- 写W(x,2)完成 ---- 写W(x,1)完成
```

正则允许读返回1或2（因为两个写都并发）。
但原子性要求：若读在写完成后开始，则必须返回该写的值。这里的读在两个写完成前开始，原子性也允许1或2。

更复杂的例子可以区分。
\end{proof}

## 4. 拜占庭容错界限

### 4.1 拜占庭将军问题

**定义 4.1（拜占庭故障）**: 故障节点可能：

- 发送矛盾消息给不同接收者
- 停止响应
- 任意行为（包括恶意）

**定义 4.2（拜占庭共识）**:

- 终止性：所有正确进程决策
- 一致性：所有正确进程决策相同值
- 有效性：若所有正确进程提议相同值 $v$，则决策 $v$

---

### 4.2 拜占庭容错下界

\begin{theorem}[拜占庭容错界限]
在 $n$ 个进程的系统中，若存在 $f$ 个拜占庭故障进程，则拜占庭共识可达当且仅当：
\begin{equation}
n \geq 3f + 1
\end{equation}
\end{theorem}

\begin{proof}

**必要性 ($\Rightarrow$)：若 $n \leq 3f$，则不可能**

假设 $n = 3$，$f = 1$（一般情况类似）。

设三个将军：$G_1$（指挥官），$G_2, G_3$（副官）。

**场景A**：$G_1$ 是拜占庭的

- $G_1$ 发送 $v$ 给 $G_2$，$v'$ 给 $G_3$
- 从 $G_2$ 的视角：可能是 $G_1$ 拜占庭，或 $G_3$ 拜占庭
- 无法区分

**场景B**：$G_2$ 是拜占庭的

- $G_1$ 发送 $v$ 给两者
- $G_2$ 告诉 $G_3$ 说收到了 $v'$
- $G_3$ 看到矛盾：$G_1$ 说 $v$，$G_2$ 说 $v'$

$G_3$ 无法区分场景A和场景B，故无法正确决策。

一般地，若 $n \leq 3f$，可将网络划分为三个组，每组最多 $f$ 个节点。
拜占庭节点可以模拟不同组之间的任意行为，使正确节点无法区分故障源。

**充分性 ($\Leftarrow$)：若 $n \geq 3f + 1$，则可能**

构造Lamport-Shostak-Pease算法：

**算法概述**：

1. 指挥官发送值给所有副官
2. 每个副官收集值（包括转发）
3. 使用多数表决

**OM(m)算法**（ oral message，假设认证）：

```
OM(0):
  指挥官发送值v给所有副官
  每个副官使用接收到的值

OM(m), m > 0:
  指挥官发送v给所有副官
  对每个副官i:
    设v_i是从指挥官收到的值
    i运行OM(m-1)作为指挥官，向其他副官广播v_i
  每个副官j:
    收集n-1个值（从其他副官）
    使用majority(v_1, ..., v_{n-1})
```

**正确性证明（归纳）**：

**基础**：$m = 0$，无故障时直接通信正确。

**归纳假设**：OM(m-1)在 $n-1 \geq 3f + 1$ 节点和 $f$ 故障时正确。

**归纳步骤**：

**情况1**：指挥官正确

- 发送相同值 $v$ 给所有副官
- 正确副官收到 $v$，在OM(m-1)中广播 $v$
- 每个副官从至少 $(n-1) - f \geq 2f$ 个正确副官收到 $v$
- 最多 $f$ 个拜占庭副官发送其他值
- 多数表决正确选择 $v$

**情况2**：指挥官拜占庭

- 最多 $f-1$ 个其他拜占庭副官
- 对任意两个正确副官 $i, j$：
  - $i$ 运行OM(m-1)，$n-1 \geq 3(f-1) + 1$ 节点
  - 由归纳假设，$j$ 从 $i$ 的广播中收到正确值
- 正确副官之间交换的值是一致的
- 多数表决在正确副官中产生相同结果

因此，当 $n \geq 3f + 1$ 时，拜占庭共识可达。
\end{proof}

### 4.3 带认证的拜占庭容错

\begin{theorem}[带签名时的容错]
若消息可数字签名（不可伪造），则拜占庭共识可在 $n \geq 2f + 1$ 时达成。
\end{theorem}

\begin{proof}

**签名机制**：

- 每条消息由发送者签名
- 签名可验证发送者身份
- 不可伪造：拜占庭节点无法伪造正确节点的签名

**算法**：

1. 指挥官签名并发送值给所有副官
2. 副官转发带签名的消息
3. 每个副官验证签名链

**正确性**：

若指挥官正确，发送带签名的 $v$。
所有正确副官收到并验证签名，知道指挥官发送了 $v$。
即使拜占庭节点转发矛盾信息，签名验证会失败。

若指挥官拜占庭，最多 $f-1$ 个其他拜占庭节点。
由于 $n \geq 2f + 1$，正确副官占多数。
通过带签名的消息交换，正确副官可以识别并忽略拜占庭消息。

签名提供可追责性，使得 $2f + 1$ 足够（而非 $3f + 1$）。
\end{proof}

## 5. 时钟同步界限

### 5.1 时钟同步问题

**定义 5.1（时钟偏移）**: 物理时钟 $C_i(t)$ 与参考时钟的差：
\begin{equation}
\delta_i(t) = C_i(t) - t
\end{equation}

**定义 5.2（时钟同步）**:
系统达到时钟同步当：
\begin{equation}
\forall i, j: |C_i(t) - C_j(t)| \leq \epsilon
\end{equation}
对于某个小界限 $\epsilon$。

---

### 5.2 同步界限定理

\begin{theorem}[时钟同步下界]
在存在 $f$ 个拜占庭时钟的系统中，若 $n \leq 3f$，则无法实现可靠的时钟同步。
\end{theorem}

\begin{proof}

与拜占庭共识类似。

时钟同步要求所有正确进程就时间达成一致（共识的一种形式）。

若 $n \leq 3f$，拜占庭节点可以：

- 向不同正确节点发送不同的时间估计
- 模拟网络延迟差异
- 使正确节点无法区分真实时钟偏移和拜占庭干扰

因此，拜占庭容错时钟同步也需要 $n \geq 3f + 1$。
\end{proof}
