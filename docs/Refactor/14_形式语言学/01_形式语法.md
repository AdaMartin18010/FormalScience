## 1. 引言

### 1.1 形式语法的研究对象

形式语法（Formal Grammar）是运用数学方法精确描述自然语言和人工语言结构的理论体系。
它将语言视为形式系统，通过严格的规则生成或识别合法的语言表达式。

### 1.2 形式化的必要性

**自然语言描述的局限性**:

- 模糊性："修饰语修饰什么？"
- 不完备：遗漏边缘情况
- 不一致：规则之间的冲突

**形式化的优势**:

```
形式化描述                自然语言描述
─────────────────────────────────────────────────
S → NP VP                 "句子由名词短语和动词短语组成"
NP → Det N                "名词短语由限定词和名词组成"
VP → V NP                 "动词短语由动词和名词短语组成"
─────────────────────────────────────────────────
精确、无歧义              直观但可能模糊
可计算、可验证            依赖解释者理解
```

### 1.3 历史发展脉络

```
1956        1960s       1970s       1980s       1990s       2000s
  │           │           │           │           │           │
  ▼           ▼           ▼           ▼           ▼           ▼
Chomsky    转换语法    标准理论   管辖与约束   最简方案   范畴语法
```

## 2. Chomsky 层次结构

### 2.1 形式文法的定义

**定义 2.1.1** (形式文法)
一个形式文法是一个四元组 $G = (N, \Sigma, P, S)$，其中：

- $N$：非终结符的有限集合
- $\Sigma$：终结符的有限集合，$N \cap \Sigma = \emptyset$
- $P$：产生式规则的有限集合，每条规则形如 $\alpha \rightarrow \beta$
- $S \in N$：开始符号

**定义 2.1.2** (直接推导)
对于文法 $G$，若存在规则 $\alpha \rightarrow \beta \in P$，以及字符串 $\gamma, \delta \in (N \cup \Sigma)^*$，使得：
$$\gamma\alpha\delta \Rightarrow_G \gamma\beta\delta$$
则称 $\gamma\alpha\delta$ **直接推导**为 $\gamma\beta\delta$。

**定义 2.1.3** (推导与生成语言)
$\Rightarrow_G^*$ 是 $\Rightarrow_G$ 的自反传递闭包。文法 $G$ 生成的语言为：
$$L(G) = \{w \in \Sigma^* : S \Rightarrow_G^* w\}$$

### 2.2 0型文法（无限制文法）

**定义 2.2.1** (0型文法/短语结构文法)
产生式规则无任何限制：$\alpha \rightarrow \beta$，其中 $\alpha \in (N \cup \Sigma)^+$, $\beta \in (N \cup \Sigma)^*$，且 $|\alpha| > 0$。

**性质**:

- 生成能力：递归可枚举语言（Recursively Enumerable）
- 识别装置：图灵机
- 停机问题：不可判定

**示例 2.2.1** (复制语言)
文法生成 $L = \{ww : w \in \{a,b\}^*\}$（需要非上下文无关的能力）：

```
S → ABCS | ABC
AB → BA, BA → AB, BC → CB, CB → BC, AC → CA, CA → AC
A → a, B → b, C → c
ca → aXc, Xc → cX, Xa → aaX
Xb → bbX, Xc → ε
```

### 2.3 1型文法（上下文有关文法）

**定义 2.3.1** (1型文法/上下文有关文法 CSG)
产生式规则形式：$\alpha A \beta \rightarrow \alpha \gamma \beta$，其中 $A \in N$, $\gamma \in (N \cup \Sigma)^+$, $\alpha, \beta \in (N \cup \Sigma)^*$。

等价形式：$|\alpha| \leq |\beta|$（单调文法），不包含 $\varepsilon$-产生式。

**定理 2.3.1** (Kuroda范式)
任何上下文有关语言都可由满足以下形式的文法生成：

- $AB \rightarrow CD$
- $A \rightarrow BC$
- $A \rightarrow a$

**应用**: 自然语言中的某些交叉依赖现象（如瑞士德语交叉序列）

### 2.4 2型文法（上下文无关文法）

**定义 2.4.1** (上下文无关文法 CFG)
产生式规则形式：$A \rightarrow \alpha$，其中 $A \in N$, $\alpha \in (N \cup \Sigma)^*$。

**示例 2.4.1** (简单英语子集)

```
S → NP VP
NP → Det N | ProperNoun
VP → V NP | V NP PP | V
PP → P NP
Det → 'the' | 'a'
N → 'cat' | 'dog' | 'mat'
V → 'chased' | 'sat'
P → 'on' | 'in'
ProperNoun → 'John' | 'Mary'
```

**定理 2.4.1** (Chomsky范式 CNF)
任何CFG都可转化为等价的形式，其中所有产生式为：

- $A \rightarrow BC$（$B, C$ 为非终结符）
- $A \rightarrow a$（$a$ 为终结符）

**定理 2.4.2** (Greibach范式 GNF)
任何CFG都可转化为等价的形式，所有产生式为：

- $A \rightarrow a\alpha$，其中 $a \in \Sigma$, $\alpha \in N^*$

### 2.5 3型文法（正则文法）

**定义 2.5.1** (正则文法 RG)

- **右线性文法**: $A \rightarrow aB$ 或 $A \rightarrow a$
- **左线性文法**: $A \rightarrow Ba$ 或 $A \rightarrow a$

**定理 2.5.1** (正则语言的封闭性)
正则语言在以下运算下封闭：

- 并：$L_1 \cup L_2$
- 连接：$L_1 \cdot L_2$
- Kleene星：$L^*$
- 补：$\overline{L}$
- 交：$L_1 \cap L_2$

**定理 2.5.2** (泵引理 - 正则语言)
若 $L$ 是正则语言，则存在常数 $p \geq 1$，使得对任意 $|w| \geq p$ 的 $w \in L$，可分解为 $w = xyz$，满足：

1. $|y| > 0$
2. $|xy| \leq p$
3. $\forall i \geq 0: xy^iz \in L$

### 2.6 层次包含关系与泵引理

**Chomsky层次包含**:
$$\text{正则语言} \subset \text{上下文无关语言} \subset \text{上下文有关语言} \subset \text{递归可枚举语言}$$

**泵引理对比**:

| 语言类 | 泵引理核心 | 典型非成员 |
|--------|-----------|-----------|
| 正则 | 中间部分可重复 | $L = \{a^nb^n : n \geq 0\}$ |
| CFL | 一对部分可同步重复 | $L = \{a^nb^nc^n : n \geq 0\}$ |
| CSL | 更复杂的交换性质 | - |

## 3. 上下文无关文法的深度分析

### 3.1 CFG的形式定义

**定义 3.1.1** (派生树)
对于CFG $G = (N, \Sigma, P, S)$，字符串 $w \in L(G)$ 的派生树是满足以下条件的树：

- 根节点标记为 $S$
- 内部节点标记为非终结符
- 叶子节点标记为终结符或 $\varepsilon$
- 若节点 $A$ 有子节点 $X_1, X_2, ..., X_k$，则 $A \rightarrow X_1X_2...X_k \in P$
- 叶子的从左到右连接形成 $w$

### 3.2 歧义性与固有歧义

**定义 3.2.1** (歧义性)
文法 $G$ 是**歧义的**，若存在 $w \in L(G)$ 有多于一个最左派生（或等价地，多于一棵派生树）。

**定义 3.2.2** (固有歧义)
语言 $L$ 是**固有歧义的**，若所有生成 $L$ 的CFG都是歧义的。

**示例 3.2.1** (算术表达式歧义)

```
E → E + E | E * E | (E) | id
```

字符串 "id + id * id" 有两种解析：

```
    E              E
   /|\            /|\
  E + E          E * E
 /|\   \        /   /|\
E+E   id      id  E + E
|             |   |   |
id            id  id  id

(左结合加法)   (右结合乘法)
```

**消歧方法**: 引入优先级和结合性

```
E → E + T | T
T → T * F | F
F → (E) | id
```

### 3.3 范式转换

**算法**: CFG到CNF的转换

```
输入: CFG G = (N, Σ, P, S)
输出: 等价的CNF文法 G'

步骤:
1. 消除ε-产生式（保留S→ε若ε∈L(G)）
2. 消除单位产生式（A→B）
3. 消除无用符号
4. 分解长产生式: A→X₁X₂...Xₖ (k>2) 替换为:
   A → X₁A₁
   A₁ → X₂A₂
   ...
   A_{k-2} → X_{k-1}X_k
5. 将终结符移到单独产生式
```

### 3.4 下推自动机（PDA）

**定义 3.4.1** (下推自动机)
PDA是一个七元组 $M = (Q, \Sigma, \Gamma, \delta, q_0, Z_0, F)$，其中：

- $Q$：状态的有限集合
- $\Sigma$：输入字母表
- $\Gamma$：栈字母表
- $\delta: Q \times (\Sigma \cup \{\varepsilon\}) \times \Gamma \rightarrow \mathcal{P}_{\text{finite}}(Q \times \Gamma^*)$：转移函数
- $q_0 \in Q$：初始状态
- $Z_0 \in \Gamma$：栈底符号
- $F \subseteq Q$：接受状态集

**定理 3.4.1** (CFG与PDA的等价性)
语言 $L$ 是上下文无关的当且仅当存在PDA $M$ 使得 $L = L(M)$。

### 3.5 确定性与非确定性CFL

**定义 3.5.1** (确定性CFL)
语言 $L$ 是**确定性的**（DCFL），若存在确定性PDA（DPDA）接受 $L$。

**性质**:

- DCFL在补运算下封闭
- DCFL不是CFL的真子集
- DCFL $\subset$ 无歧义CFL $\subset$ CFL

**示例 3.5.1**:

- DCFL: $L = \{a^nb^n : n \geq 0\}$
- 非DCFL但CFL: $L = \{ww^R : w \in \{a,b\}^*\}$（回文）

## 4. 转换语法

### 4.1 经典转换语法（TG）

**核心思想**: 通过转换规则将深层结构映射到表层结构。

**基本操作**:

- 移位（Movement）：将成分移至新位置
- 删除（Deletion）：删除某些成分
- 插入（Insertion）：插入新成分
- 替换（Replacement）：替换成分

**示例 4.1.1** (疑问句转换)

```
深层结构: [S you [VP will [VP read what]]]
转换: Wh-移位
表层结构: [S whatᵢ [S you [VP will [VP read tᵢ]]]]
```

### 4.2 标准理论与扩展标准理论

**标准理论（ST）** (1965):

- 深层结构 → 转换 → 表层结构 → 语音形式
-                        ↓
-                   语义解释

**扩展标准理论（EST）**:

- 语义部分由深层结构解释
- 引入迹（trace）理论处理移位

### 4.3 管辖与约束理论（GB）

**理论架构**:

```
D-结构（深层结构）
    ↓ 移位α
S-结构（表层结构）
    ├──► 语音形式（PF）
    └──► 逻辑形式（LF）
```

**核心子理论**:

| 子理论 | 内容 | 形式化目标 |
|--------|------|-----------|
| X-bar理论 | 短语结构的普遍模式 | 层级结构的约束 |
| Θ理论 | 题元角色分配 | 语义关系的映射 |
| 格理论 | 抽象格的分配 | 句法-形态接口 |
| 约束理论 | 指称依赖关系 | 回指的局域性 |
| 控制理论 | PRO的解释 | 非限定子句 |
| 界限理论 | 移位的局域性 | 句法计算的约束 |

**定义 4.3.1** (界限理论)

- **管辖（Government）**: 节点 $A$ 管辖节点 $B$ 当...
- **界限节点（Bounding Node）**: IP, DP, CP等
- **子语类化（Subjacency）**: 移位不能跨越超过一个界限节点

### 4.4 最简方案（MP）

**核心思想**: 语法是满足界面条件的最优计算系统。

**操作**:

- **合并（Merge）**: $\{α, β\}$，创建新句法对象
- **移位（Move/Internal Merge）**: 复制 + 合并
- **一致（Agree）**: 特征匹配与删除

**经济原则**:

- 晚合并（Merge over Move）
- 最短移位（Shortest Move/Minimal Link Condition）
- 全力推导（Full Interpretation）

**形式化表示**:

```
运算:
  Merge(X, Y) = {X, Y}  或  {X, {X, Y}}

特征系统:
  [±N], [±V], [±D], [±T], [±C]
  [±interpretable], [±valued]

一致操作:
  Agree(Probe, Goal) 当：
    a. Probe 有未赋值特征 [uF]
    b. Goal 有可解释特征 [iF]
    c. 在最简语域内
```

## 5. 范畴语法

### 5.1 抽象范畴语法（AB Grammar）

**定义 5.1.1** (范畴)
基本范畴：$Prim = \{S, N, NP, ...\}$
复合范畴：若 $A, B$ 是范畴，则 $A/B$ 和 $A\backslash B$ 也是范畴。

**直观解释**:

- $A/B$：向右寻找 $B$ 形成 $A$
- $A\backslash B$：向左寻找 $B$ 形成 $A$

**示例 5.1.1**:

```
John: NP
runs: S\NP  （向左找NP，形成S）
likes: (S\NP)/NP  （向右找NP，再向左找NP，形成S）
```

**演算规则**:
$$\frac{x : A/B \quad y : B}{xy : A} (>)
\quad
\frac{y : B \quad x : B\backslash A}{yx : A} (<)$$

### 5.2 Lambek演算

**公理**: $A \rightarrow A$

**规则**:
```
右引入（/R）:     右引入（\R）:
  Γ, A ⊢ B          A, Γ ⊢ B
  ─────────         ─────────
  Γ ⊢ B/A          Γ ⊢ A\B

左引入（/L）:     左引入（\L）:
  Γ ⊢ A   Δ, B, Λ ⊢ C    Γ ⊢ A   Δ, B, Λ ⊢ C
  ───────────────────    ───────────────────
     Δ, Γ, B/A, Λ ⊢ C       Δ, A\B, Γ, Λ ⊢ C

切断（Cut）:
  Γ ⊢ A   Δ, A, Λ ⊢ B
  ───────────────────
     Δ, Γ, Λ ⊢ B
```

**定理 5.2.1** (切割消除)
Lambek演算满足切割消除，因此是内涵逻辑的自然演绎系统。

### 5.3 组合范畴语法（CCG）

**扩展范畴**:
- 函子范畴：$A/B$, $A\backslash B$
- 次级范畴：$A|B$（|可以是/或\）
- 复杂函子：$(A/B)/C$

**组合规则**:
```
前向应用（>）:      后向应用（<）:
  X/Y  Y              Y  X\Y
  ─────               ─────
   X                    X

前向组合（>B）:     后向组合（<B）:
  X/Y  Y/Z           Y\Z  X\Y
  ────────           ────────
   X/Z                X\Z

前向类型提升（>T）:  后向类型提升（<T）:
    X                 X
  ───────            ───────
  T/(T\X)            T\(T/X)
```

**示例 5.3.1** (长距离依存)
```
what: S/(S\NP)  （类型提升）
      S/(S/NP)  （替代提升路径）

John likes: (S\NP)/NP  S\NP  →  (S\NP)  （通过 < 应用）
→ 这里需要类型提升处理非成分协调
```

### 5.4 类型提升与组合规则

**定义 5.4.1** (类型提升)
$$\uparrow : A \Rightarrow T/(T\backslash A)$$

**谐和标记**:
CCG引入谐和标记（*h* 和 *x*）约束组合：
- $A/_{*h} B$：可与 $B$ 谐和组合
- $A/_{*x} B$：不可谐和组合（仅允许应用）

**示例 5.4.1** (主语-动词一致)
```
runs: S\NP[3sg]        run: S\NP[pl]
John: NP[3sg]          they: NP[pl]
─────────────────      ─────────────────
  John runs : S          they run : S
```

## 6. 扩展文法形式体系

### 6.1 树邻接语法（TAG）

**定义 6.1.1** (基本树)
基本树是满足以下条件的初始树或辅助树：
- 初始树：根标记为 $S$，内部节点为非终结符，叶子为终结符或非终结符
- 辅助树：有一个标记为某非终结符的足节点（foot node），与根标记相同

**操作**:
- **替换（Substitution）**: 将初始树插入到叶子非终结符处
- **附加（Adjunction）**: 将辅助树插入到内部节点处

**适度上下文相关性**:
TAG生成能力介于CFG和CSG之间，可处理：
- 交叉依赖（如瑞士德语）
- 特定类型的复制语言

### 6.2 头部驱动的短语结构语法（HPSG）

**核心特征**:
- 基于特征合一的语法
- 词汇主义（词汇承载大部分语法信息）
- 类型层级与继承

**特征结构**:
```
phrase
├── HEAD: 词类特征
├── VALENCE: 价特征（SUBJ, COMPS, SPR）
├── MOD: 修饰语信息
└── ...
```

**原则**:
- 头部特征原则（Head Feature Principle）
- 语义组合原则（Semantic Compositionality Principle）
- 价原则（Valence Principle）

### 6.3 词汇功能语法（LFG）

**双层次表示**:
- c-结构（constituent structure）：表层句法树
- f-结构（functional structure）：属性-值矩阵（AVM）

**对应原则**:
```
c-结构                 f-结构
─────────────────────────────────
IP ─┬─ DP[subj]        [ PRED 'pro'
    │      │             TENSE past
    │      'John'        SUBJ [ PRED 'John'
    │                      CASE nom ]
    └─ I' ─┬─ I          PRED 'chase<SUBJ,OBJ>'
           │   'chased'   OBJ [ PRED 'Mary'
           └─ VP ─┬─ V              CASE acc ]
                   │  'chased'
                   └─ DP[obj]
                        'Mary'
```

### 6.4 依存语法

**核心思想**: 句子结构由词与词之间的依存关系构成。

**定义 6.4.1** (依存树)
- 单根：有且仅有一个根节点（通常是主要动词）
- 连通：所有节点通过依存弧连接
- 无环：不存在依存环
- 投射性（可选）：依存弧不交叉

**形式化**:
句子是图 $G = (V, A)$，其中：
- $V$：词汇节点集合
- $A \subseteq V \times V \times L$：带标签的依存弧

## 7. 方法对比矩阵

### 7.1 文法形式体系对比

| 特性 | CFG | TAG | HPSG | LFG | CCG |
|------|-----|-----|------|-----|-----|
| **形式基础** | 产生式规则 | 树操作 | 特征合一 | 双层次结构 | 范畴演算 |
| **生成能力** | 上下文无关 | 适度上下文相关 | 上下文无关 | 上下文无关 | 上下文无关 |
| **歧义处理** | 派生树 | 派生树 | 特征合一 | f-结构合一 | 证明结构 |
| **词汇主义** | 弱 | 中 | 强 | 强 | 强 |
| **组合性** | 树组合 | 树替换/附加 | 特征组合 | 结构映射 | 范畴应用 |
| **计算复杂度** | $O(n^3)$ | $O(n^6)$ | NP完全 | NP完全 | 多项式 |
| **自然语言现象** | 基本结构 | 长距离依存 | 一致、移位 | 一致、话题化 | 协调、提升 |
| **实现工具** | CYK, Earley | XTAG | LKB | XLE | C&C, EasyCCG |

### 7.2 转换语法 vs 范畴语法

| 维度 | GB/MP | CCG |
|------|-------|-----|
| **基本操作** | 移位、合并 | 函数应用、组合 |
| **表征** | 派生树 + 移位痕迹 | 证明/派生 |
| **局域性** | 子语类化条件 | 邻接约束 |
| **词汇信息** | 特征集合 | 范畴类型 |
| **语义组合** | LF映射 |  Curry-Howard对应 |
| **心理现实性** | 声称 | 计算效率 |

## 9. 应用案例

### 9.1 案例1：编程语言语法设计

**背景**: 设计一门类Python语言的语法分析器。

**解决方案**: 使用ANTLR生成基于LL(*)的解析器。

**文法片段**:
```antlr
grammar SimplePy;

program : statement+ EOF;

statement
    : assignment
    | if_stmt
    | while_stmt
    | expr_stmt
    ;

assignment : NAME '=' expression NEWLINE;

if_stmt : 'if' expression ':' block
          ('elif' expression ':' block)*
          ('else' ':' block)?
        ;

while_stmt : 'while' expression ':' block;

block : NEWLINE INDENT statement+ DEDENT
      | statement
      ;

expression
    : expression ('+' | '-') expression
    | expression ('*' | '/') expression
    | expression ('==' | '!=' | '<' | '>') expression
    | NUMBER
    | NAME
    | '(' expression ')'
    ;
```

**复杂度分析**: 表达式文法是歧义的，通过优先级和结合性规则解决。

### 9.2 案例2：自然语言句法分析器

**背景**: 构建英文问答系统的句法分析模块。

**方案**: 使用CCG进行句法分析，直接映射到语义表示。

**示例分析**:
```
输入: "Which student read every book?"

CCG派生:
  which student    read          every book
  NP/N    N      (S\NP)/NP   (NP/N)/N   N
  ─────────────────────────   ─────────────
      NP          (S\NP)/NP         NP
      ───────────────────────────────────
                S\NP
      ───────────────────────────────────
              S/(S\NP)  S\NP
              ───────────────────────────
                         S

语义组合:
  which: λP.λQ.∃x[P(x)∧Q(x)]
  student: λx.student(x)
  read: λy.λx.read(x,y)
  every: λP.λQ.∀x[P(x)→Q(x)]
  book: λx.book(x)

最终结果: λQ.∃x[student(x)∧∀y[book(y)→read(x,y)]∧Q(x)]
```

### 9.3 案例3：编译器优化中的文法应用

**背景**: 在编译器中进行公共子表达式消除。

**方法**: 使用属性文法计算表达式的可用性。

**属性文法定义**:
```
E → E₁ + T    { E.val = E₁.val + T.val;
                E.gen = E₁.gen ∪ T.gen ∪ {E.val};
                E.kill = ... }
E → T         { E.val = T.val; ... }
T → T₁ * F    { T.val = T₁.val * F.val; ... }
T → F         { T.val = F.val; ... }
F → (E)       { F.val = E.val; ... }
F → id        { F.val = lookup(id.lexeme); ... }
```
