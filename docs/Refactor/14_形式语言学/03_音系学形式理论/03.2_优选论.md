# 14.3.2 优选论（Optimality Theory）

## 14.3.2.1 引言

优选论（Optimality Theory, OT）由Alan Prince和Paul Smolensky于1993年提出，是生成音系学的重要理论革新。
OT摒弃了基于规则推导的音系分析，转而采用**制约条件交互**（constraint interaction）的框架。

> **核心思想**：音系表达通过评估一组普遍制约条件的层级排序来确定，最优输出是违反制约条件最少的候选者。

## 14.3.2.2 理论框架

### 14.3.2.2.1 三个核心组件

> **定义 14.3.2.1**（OT语法）
> OT语法由三个组件构成：
>
> 1. **GEN（生成器）**：从输入生成所有可能的候选输出集合
> 2. **CON（制约条件集）**：普遍存在的音系制约条件集合
> 3. **EVAL（评估器）**：根据制约条件层级排序选择最优输出

```
OT推导流程：

输入 /input/ ──GEN──> {候选1, 候选2, 候选3, ...}
                            │
                            ▼
                      制约条件层级：
                      C₁ >> C₂ >> C₃ >> ... >> Cₙ
                            │
                            ▼
                      EVAL评估
                            │
                            ▼
                      最优输出 [output]
```

```haskell
-- 优选论的Haskell实现

data OTGrammar = OTGrammar {
    otGen :: Input -> [Candidate],
    otCon :: [Constraint],
    otEval :: [Constraint] -> [Candidate] -> Candidate
} deriving (Show)

data Input = Input {
    inputUR :: String,       -- 底层形式
    inputMorphs :: [String]  -- 形态信息
} deriving (Show)

data Candidate = Candidate {
    candSR :: String,        -- 表层形式
    candViolations :: Map String Int,  -- 制约违反次数
    candParse :: [ParseStep] -- 推导步骤
} deriving (Show)

data ParseStep = ParseStep String String deriving (Show)  -- (操作, 结果)

data Constraint = Constraint {
    conName :: String,
    conType :: ConType,
    conDescription :: String,
    conEvaluator :: Candidate -> Int  -- 返回违反次数
} deriving (Show)

data ConType = Faithfulness | Markedness deriving (Show)

-- 层级排序：C₁ >> C₂ 表示 C₁ 比 C₂ 更重要
data Ranking = Ranking [String] deriving (Show)

-- 评估函数
selectOptimal :: Ranking -> [Constraint] -> [Candidate] -> Candidate
selectOptimal (Ranking rankedNames) constraints candidates =
    let rankedCons = map (findConstraint constraints) rankedNames
        survivors = foldl (\cands con -> filterByConstraint con cands) candidates rankedCons
    in head survivors  -- 假设总有最优解

findConstraint :: [Constraint] -> String -> Constraint
findConstraint cons name = head (filter (\c -> conName c == name) cons)

-- 按制约条件过滤：保留违反最少的候选
filterByConstraint :: Constraint -> [Candidate] -> [Candidate]
filterByConstraint con cands =
    let violations = map (conEvaluator con) cands
        minViol = minimum violations
    in [c | (c, v) <- zip cands violations, v == minViol]
```

### 14.3.2.2.2 制约条件类型

**忠实性制约条件**（Faithfulness）：要求输出忠实于输入

| 制约条件 | 描述 |
|----------|------|
| MAX-IO | 输入中的音段必须在输出中出现（无删除） |
| DEP-IO | 输出中的音段必须在输入中出现（无插入） |
| IDENT(F) | 输入和输出的特征F值必须相同（无变更） |
| UNIFORMITY | 不允许融合（coalescence） |
| INTEGRITY | 不允许分裂（fission） |

**标记性制约条件**（Markedness）：限制输出的结构

| 制约条件 | 描述 |
|----------|------|
| *COMPLEX | 禁止复杂音节首/尾 |
| NOCODA | 禁止音节尾 |
| ONSET | 音节必须有首音 |
| *VOICED-CODA | 禁止浊音在音节尾 |
| SSP | 响度顺序原则 |
| *KLUSTER | 禁止辅音丛 |

```haskell
-- 忠实性制约条件
maxIO :: Constraint
maxIO = Constraint "MAX-IO" Faithfulness "No deletion"
    (\c -> countDeleted (inputUR undefined) (candSR c))

depIO :: Constraint
depIO = Constraint "DEP-IO" Faithfulness "No epenthesis"
    (\c -> countInserted (inputUR undefined) (candSR c))

identVoice :: Constraint
identVoice = Constraint "IDENT(voice)" Faithfulness "Preserve voicing"
    (\c -> countFeatureChanges (inputUR undefined) (candSR c) Voiced)

-- 标记性制约条件
noCoda :: Constraint
noCoda = Constraint "NOCODA" Markedness "No syllable coda"
    (\c -> countCodas (candSR c))

onset :: Constraint
onset = Constraint "ONSET" Markedness "Syllables must have onsets"
    (\c -> countOnsetlessSyllables (candSR c))

complexOnset :: Constraint
complexOnset = Constraint "*COMPLEX-ONSET" Markedness "No complex onsets"
    (\c -> countComplexOnsets (candSR c))

-- 辅助计数函数（简化实现）
countDeleted :: String -> String -> Int
countDeleted input output = length input - length output + countInserted input output

countInserted :: String -> String -> Int
countInserted input output = max 0 (length output - length input)

countFeatureChanges :: String -> String -> Feature -> Int
countFeatureChanges = undefined  -- 需要特征表示

countCodas :: String -> Int
countCodas = undefined

countOnsetlessSyllables :: String -> Int
countOnsetlessSyllables = undefined

countComplexOnsets :: String -> Int
countComplexOnsets = undefined
```

## 14.3.2.3 表格式分析（Tableau）

### 14.3.2.3.1 评估表

> **定义 14.3.2.2**（OT表）
> OT表展示候选者在各制约条件下的违反情况：
>
> - "*"：违反一次
> - "*!"：致命违反（导致淘汰）
> - "☞"：标记最优候选

```
示例表：/pat+red/ "patted"（英语过去时）

                  *COMPLEX   NOCODA   MAX    DEP
/pat+red/
→ [pæt.təd]             *        *      **     *
→ [pæt.təd] ☞                         *      **     *
→ [pæt]                        ***!    ***
→ [pæ.təd]             *        **     *      *

层级：*COMPLEX >> NOCODA >> MAX >> DEP

注：英语通过插入元音避免复杂音节尾
```

```haskell
-- OT表生成
data Tableau = Tableau {
    tabInput :: Input,
    tabConstraints :: [Constraint],
    tabCandidates :: [Candidate],
    tabOptimal :: Candidate,
    tabViolations :: [[Violation]]
} deriving (Show)

data Violation = VNone | VFatal | VNormal Int deriving (Show)

-- 生成OT表
generateTableau :: OTGrammar -> Input -> Tableau
generateTableau grammar input =
    let candidates = otGen grammar input
        optimal = otEval grammar (otCon grammar) candidates
        violations = map (\c -> map (evaluateConstraint c) candidates) (otCon grammar)
    in Tableau input (otCon grammar) candidates optimal violations

evaluateConstraint :: Constraint -> Candidate -> Violation
evaluateConstraint con cand =
    let v = conEvaluator con cand
    in if v == 0 then VNone else VNormal v

-- 表格输出
printTableau :: Tableau -> String
printTableau tab =
    let header = "Input: " ++ inputUR (tabInput tab) ++ "\n"
        conRow = "Candidate\t" ++ concatMap (\c -> conName c ++ "\t") (tabConstraints tab) ++ "\n"
        candRows = map (printCandidateRow (tabConstraints tab) (tabOptimal tab)) (tabCandidates tab)
    in header ++ conRow ++ concat candRows

printCandidateRow :: [Constraint] -> Candidate -> Candidate -> String
printCandidateRow cons optimal cand =
    let marker = if candSR cand == candSR optimal then "☞ " else "  "
        violations = map (\con -> showViolation (conEvaluator con cand)) cons
    in marker ++ candSR cand ++ "\t" ++ concatMap (++ "\t") violations ++ "\n"

showViolation :: Int -> String
showViolation 0 = ""
showViolation 1 = "*"
showViolation n = "*" ++ show n
```

```haskell
englishPluralOT :: Input -> Tableau

generatePluralCandidates (Input ur _) =

      Candidate (ur ++ "z") Map.empty [],  -- 加/z/
      Candidate (ur ++ "əz") Map.empty [], -- 加/əz/
      Candidate ur Map.empty []             -- 无变化
    ]

noVoicedCoda :: Constraint
noVoicedCoda = Constraint "*VOICED-CODA" Markedness "No voiced coda"
    (\c -> countViolations (candSR c) isVoicedCoda)

isVoicedCoda :: String -> Bool
isVoicedCoda = undefined

countViolations :: String -> (String -> Bool) -> Int
countViolations s pred = length (filter pred (substrings s))

substrings :: String -> [String]
substrings s = [take n (drop i s) | i <- [0..length s-1], n <- [1..length s - i]]
```

## 14.3.2.4 音节优化

### 14.3.2.4.1 音节结构制约条件

```
常见音节结构制约条件层级：

ONSET >> NOCODA >> *COMPLEX >> MAX/DEP

示例：/bli/ → [bli]（英语允许复杂音节首）
       /bli/ → [bə.li]（某些语言禁止复杂音节首）

不同语言的差异体现在制约条件层级上：
- 英语：MAX >> *COMPLEX >> NOCODA
- 日语：*COMPLEX >> MAX >> NOCODA
```

```haskell

    ssCoda :: String

    (\c -> countOnsetless (candSR c))

noCodaConstraint :: Constraint
noCodaConstraint = Constraint "NOCODA" Markedness "Syllables must not have codas"
    (\c -> countCodasOT (candSR c))

    (\c -> countComplexOnsetsOT (candSR c))

    (\c -> countComplexCodasOT (candSR c))

-- 计数函数
countOnsetless :: String -> Int
```

## 14.3.2.5 和谐推导过程

### 14.3.2.5.1 和谐序列推导

> **定义 14.3.2.3**（和谐推导）
> 某些OT分析使用**和谐推导**（Harmonic Derivation），即逐步优化候选者。

```
和谐推导示例：

步骤1：输入 /pat+red/
步骤2：音节划分 pa.t.red
步骤3：处理复杂音节尾 pa.t.rəd（违反DEP）
步骤4：处理无音节首 *无法解决
步骤5：重新划分 pæ.təd

最终结果：[pæ.təd]
```

```haskell
-- 和谐推导
data HarmonicStep = HarmonicStep {
    hsInput :: String,
    hsOutput :: String,
    hsOperation :: String,
    hsConstraint :: Constraint
} deriving (Show)

data HarmonicDerivation = HarmonicDerivation {
    hdSteps :: [HarmonicStep],
    hdFinal :: String
} deriving (Show)

-- 和谐推导函数
harmonicDerive :: OTGrammar -> String -> HarmonicDerivation
harmonicDerive grammar input =
    let steps = deriveSteps grammar input []
        final = case steps of
            [] -> input
            _ -> hsOutput (last steps)
    in HarmonicDerivation steps final

deriveSteps :: OTGrammar -> String -> [HarmonicStep] -> [HarmonicStep]
deriveSteps grammar current steps =
    let candidates = otGen grammar (Input current [])
        optimal = otEval grammar (otCon grammar) candidates
        newOutput = candSR optimal
    in if newOutput == current
       then steps
       else let step = HarmonicStep current newOutput "optimization" (head (otCon grammar))
            in deriveSteps grammar newOutput (steps ++ [step])
```

## 14.3.2.6 OT的高级扩展

### 14.3.2.6.1 对应理论（Correspondence Theory）

> **定义 14.3.2.4**（对应关系）
> McCarthy & Prince（1995）提出输入-输出、输出-输出、基式-重复之间的对应关系。

**对应制约条件**：

- **MAX-BR**：基式-重复对应，重复不能删除基式音段
- **DEP-BR**：重复不能添加基式中没有的音段
- **IDENT-BR(F)**：重复必须保持基式的特征F

## 14.3.2.7 OT与语言习得

### 14.3.2.7.1 约束重排算法

> **算法 14.3.2.1**（约束降级）
> Tesar & Smolensky（1998）提出的学习算法：
>
> 1. 观察目标语言的表层形式
> 2. 如果当前层级产生错误输出，将违反的制约条件降级

```haskell
-- 约束重排学习
data LearningData = LearningData {
    ldInput :: Input,
    ldTarget :: String
} deriving (Show)

data ConstraintRanking = CR [(Constraint, Int)] deriving (Show)  -- (制约, 层级)

-- 错误驱动学习
errorDrivenLearning :: [LearningData] -> [Constraint] -> ConstraintRanking
errorDrivenLearning dataSet constraints =
    let initialRanking = CR (zip constraints [1..])
    in foldl processExample initialRanking dataSet

processExample :: ConstraintRanking -> LearningData -> ConstraintRanking
processExample (CR ranking) (LearningData input target) =
    let grammar = OTGrammar undefined (map fst ranking) undefined
        candidates = otGen grammar input
        currentRanking = map fst (sortBy (\(_,a) (_,b) -> compare a b) ranking)
        predicted = otEval grammar currentRanking candidates
    in if candSR predicted == target
       then CR ranking
       else demoteViolatedConstraints CR ranking input target predicted

demoteViolatedConstraints :: (ConstraintRanking -> LearningData -> Candidate -> ConstraintRanking)
demoteViolatedConstraints (CR ranking) input target predicted =
    -- 找到目标输出违反但预测输出未违反的最高制约条件
    -- 将该制约降级
    CR ranking  -- 简化
```

## 14.3.2.8 OT与句法

虽然OT最初用于音系学，但其框架也被应用于句法和形态学。

```
OT句法示例：

输入：语义表示
制约条件：
- STAY：避免移位
- EPP：Spec-TP必须被填充
- *LEX-V：避免词汇动词在T位置

"What did John eat?"

候选1: [CP What did [TP John [T' eat]]]  -- *LEX-V, EPP
候选2: [CP What [TP John [T' did eat]]]  -- EPP, *LEX-V
候选3: [CP What did [TP John [T' did eat]]]  -- EPP, STAY

层级：EPP >> *LEX-V >> STAY
最优：候选3（do-support）
```
