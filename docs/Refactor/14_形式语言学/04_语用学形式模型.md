## 1. 引言

### 1.1 语用学的形式化挑战

语用学研究语言在语境中的使用，其核心挑战在于：**如何将"语境"和"使用"这两个本质上是动态的、主观的概念形式化？**

**语义与语用的分界**:

```
真值条件语义 ─────────────────────────► 语境依赖语义
     │                                      │
     │ 组合性、真值条件                      │ 使用条件、语境更新
     │                                      │
     ▼                                      ▼
"What is said"                        "What is meant/communicated"
(语义内容)                              (交际内容)
```

### 1.2 语境的形式表示

**定义 1.2.1** (语境)
语境 $C$ 是一个多元组，包含以下成分：

- $C_G$：共同背景知识（common ground）
- $C_S$：说话者信息
- $C_H$：听话者信息
- $C_T$：时间、地点等指示信息
- $C_D$：话语上下文（discourse context）

**形式化表示**:
$$C = \langle A, S, H, T, L, D \rangle$$
其中：

- $A$：话语参与者集合
- $S$：说话者
- $H$：听话者
- $T$：话语时间
- $L$：话语地点
- $D$：前文信息（discourse representation）

### 1.3 从语义到语用的过渡

**三层次意义模型**:

```
句子 "It's cold in here"
      │
      ├──► 语义内容: "室内温度低" (真值条件)
      │
      ├──► 语用内容: "说话者觉得冷" (指示表达)
      │
      └──► 交际意图: "请关窗/调高温度" (言语行为)
```

## 2. 言语行为理论的形式化

### 2.1 Austin与Searle的经典理论

**Austin的言语行为三分法**:

| 层次 | 定义 | 例子 |
|------|------|------|
| 以言表意行为 (Locutionary) | 说出有意义的词句 | "我承诺明天来" |
| 以言行事行为 (Illocutionary) | 实施某种行为 | 做出承诺 |
| 以言取效行为 (Perlocutionary) | 对听话者产生影响 | 使听话者放心 |

**Searle的言语行为分类**:

| 类别 | 目的 | 适从方向 | 心理状态 | 例子 |
|------|------|---------|---------|------|
| 断言类 (Assertive) | 陈述事实 | 词→世 | 相信 | "下雨了" |
| 指令类 (Directive) | 使听话者做某事 | 世→词 | 愿望 | "请关门" |
| 承诺类 (Commissive) | 承诺做某事 | 世→词 | 意图 | "我保证..." |
| 表达类 (Expressive) | 表达心理状态 | 无 | 可变 | "谢谢！" |
| 宣告类 (Declarative) | 改变世界状态 | 双 | 可变 | "我宣布..." |

### 2.2 言语行为的逻辑结构

**形式化表示**:
$$F(P)$$
其中：

- $F$：以言行事力量（illocutionary force）
- $P$：命题内容

**以言行事标记函数** (Searle & Vanderveken):
$$Force(P) = \langle F, P, M, S, C \rangle$$

- $F$：力量类型
- $P$：命题内容
- $M$：命题内容模式
- $S$：力量成分（准备条件、真诚条件等）
- $C$：语境要求

### 2.3 适切条件的形式化

**定义 2.3.1** (适切条件 Felicity Conditions)

对于断言类言语行为 $Assert(A, P)$：

| 条件类型 | 内容 | 形式化 |
|---------|------|--------|
| 命题内容条件 | $P$ 必须为命题 | $Proposition(P)$ |
| 预备条件 | $S$ 有证据相信 $P$ | $Bel(S, P) \wedge Evidence(S, P)$ |
| 真诚条件 | $S$ 相信 $P$ | $Bel(S, P)$ |
| 本质条件 | 承担 $P$ 为真的承诺 | $Obligation(S, True(P))$ |

**形式化规则**:
$$Appropriate(Assert(S, H, P), C) \leftrightarrow $$
$$Proposition(P) \wedge Bel(S, P) \wedge Evidence(S, P) \wedge$$
$$\neg Bel(S, H, Bel(S, P)) \wedge$$
$$\neg Obvious(C, P)$$

### 2.4 间接言语行为

**定义 2.4.1** (间接言语行为)
说话者通过实施一个言语行为来间接实施另一个言语行为。

**形式化分析**:

```
字面行为: Assert("我能打开窗户吗？", "我有能力打开窗户")
间接行为: Request(S, H, "打开窗户")

推理链 (Searle):
1. 问能力通常不是为获取信息
2. 在语境中，开窗是相关行为
3. 询问能力暗示希望执行该行为
4. 因此是请求
```

**形式化推导**:

```
规则: Question(S, H, Can(S, A)) ∧ Relevant(A, C) ⊨ Request(S, H, A)

适用条件:
  - 询问能力不是为获取信息
  - 执行A对S有利
  - H有能力执行A
```

## 3. 会话含义理论

### 3.1 Grice的合作原则

**合作原则 (Cooperative Principle)**:
> 使你的话语在交谈的当前阶段符合所参与的交谈的公认目的或方向。

**四大会话准则**:

| 准则 | 内容 | 违反产生的含义 |
|------|------|---------------|
| 量的准则 | 信息量适度 | 少说 → 暗示更多 |
| 质的准则 | 说真话 | 反讽、隐喻 |
| 关系准则 | 相关 | 不相关暗示另有含义 |
| 方式准则 | 清晰明了 | 隐晦表达复杂态度 |

### 3.2 会话准则的形式化

**定义 3.2.1** (会话准则的形式化)

设 $U$ 为话语，$C$ 为语境：

**量的准则**:
$$Maxim_{Quantity}(U, C) :=$$
$$Info(U) \geq InfoRequired(C) \wedge Info(U) \leq InfoWarranted(C)$$

**质的准则**:
$$Maxim_{Quality}(U, C) :=$$
$$\neg Believe(S, \neg True(U)) \wedge Evidence(S, True(U))$$

**关系准则**:
$$Maxim_{Relation}(U, C) := Relevant(U, C)$$

**方式准则**:
$$Maxim_{Manner}(U, C) :=$$
$$\neg Obscure(U) \wedge \neg Ambiguous(U) \wedge Brief(U) \wedge Orderly(U)$$

### 3.3 含义的推导机制

**Grice的推导模式**:

```
1. 说话者说 "U"
2. 没有违反合作原则的明显理由
3. 若要维持合作假设，假设说话者相信 "P"
4. 说话者未表明相信 "P"
5. 说话者意图让听话者推断 "P"
6. 因此，说话者会话含义 "P"
```

**形式化推导**:
$$U \wedge Cooperative(S) \wedge (Maxim_{Q}(U) \rightarrow P) \wedge \neg Assert(S, P)$$
$$\vDash_{implicate} P$$

### 3.4 等级含义（Scalar Implicature）

**定义 3.4.1** (等级含义)
从使用较弱表达推导出的否定较强表达的会话含义。

**Horn等级**:

```
⟨all, most, many, some⟩          量词等级
⟨and, or⟩                        逻辑连接词
⟨n, ..., 3, 2, 1⟩                数字
⟨must, should, may⟩              情态动词
⟨know, believe⟩                  认知动词
⟨always, often, sometimes⟩       频率副词
```

**推导机制**:

```
等级: ⟨all, some⟩ 其中 all > some (信息强度)

话语: "Some students passed"
推导:
  - 说话者说 "some" 而非更强的 "all"
  - 若说 "all" 更信息丰富且为真，应说 "all"
  - 因此 "not all"
含义: "Some but not all students passed"
```

**形式化**:
$$Scalar(S, \langle w, s \rangle) \wedge Assert(S, w) \vDash \neg s$$

**类型对比**:

| 类型 | 例子 | 推导依据 |
|------|------|---------|
| 上限含义 | "some" → "not all" | 量的准则 |
| 下限含义 | "almost all" → "not all" | 质的准则 |
| 双重否定含义 | "not impossible" → "possible but not certain" | 方式准则 |

## 4. 形式语用学框架

### 4.1 语境更新语义学

**核心思想**: 话语的意义在于改变语境（信息状态）。

**定义 4.1.1** (语境更新)
$$C[U] = C'$$
其中 $C$ 是输入语境，$U$ 是话语，$C'$ 是更新后的语境。

**断言更新**:
$$C[Assert(\phi)] = \{\langle g, h \rangle \in C : h \in [[\phi]]\}$$

**问题更新**:
$$C[Ask(\phi?)] = C \cap \{(g, h) : h \in [[\phi]] \vee h \in [[\neg\phi]]\}$$

**更新语义与DRT的联系**:

```
DRT: 合并DRS表示更新话语内容
语境更新: 信息状态集合的交集/更新

统一: 两者都是动态语义的形式化
```

### 4.2 信息系统语义学

**定义 4.2.1** (信息系统)
信息系统 $I = \langle S, \vdash \rangle$，其中：

- $S$：句子集合
- $\vdash \subseteq \mathcal{P}(S) \times S$：推导关系

**支持关系** (Support):
$$s \Vdash \phi \text{ (状态 } s \text{ 支持 } \phi \text{)}$$

**语义定义**:

```
原子: s ⊩ p  iff  p ∈ s
否定: s ⊩ ¬φ  iff  ∀t ⊆ s: t ⋠ φ
合取: s ⊩ φ∧ψ  iff  s ⊩ φ and s ⊩ ψ
析取: s ⊩ φ∨ψ  iff  s ⊩ φ or s ⊩ ψ
蕴涵: s ⊩ φ→ψ  iff  ∀t ⊆ s: if t ⊩ φ then t ⊩ ψ
```

### 4.3 博弈论语用学

**核心思想**: 交际是说话者与听话者之间的博弈。

**信号博弈 (Signaling Game)**:

```
参与者:
  - 发送者 (Sender): 知道世界状态 t ∈ T
  - 接收者 (Receiver): 选择行动 a ∈ A

博弈过程:
  1. 自然选择类型 t (按先验分布)
  2. 发送者观察 t，选择信号 m ∈ M
  3. 接收者观察 m，选择行动 a
  4. 收益: u_S(t,a), u_R(t,a)

均衡: 完美贝叶斯均衡
  - 发送者策略最优给定接收者策略
  - 接收者信念与策略一致
```

**语用学应用**:

```
场景: 说话者选择表达描述状态

类型: t₁="all", t₂="some"
信号: m₁="all", m₂="some"

均衡分离策略:
  - t₁类型选 m₁
  - t₂类型选 m₂

含义涌现: 说 "some" 暗示 t₂ (即 "not all")
```

### 4.4 决策理论语用学

**RSA框架** (Rational Speech Act):

```
字面听者 (Literal Listener):
  L₀(t | m) ∝ P(m | t) · P(t)

实用说话者 (Pragmatic Speaker):
  S₁(m | t) ∝ exp(α · U(m; t))
  U(m; t) = log L₀(t | m)

实用听者 (Pragmatic Listener):
  L₁(t | m) ∝ S₁(m | t) · P(t)
```

**迭代推理**:

```
L₀: 字面解释
S₁: 考虑L₀，选择最大化信息性的表达
L₂: 考虑S₁，推断说话者意图
...

收敛: 话语的语用解释
```

## 5. 指示与照应

### 5.1 指示词的形式分析

**定义 5.1.1** (指示词)
指示词是其指称依赖于话语语境的表达。

**指示词类型**:

| 类型 | 例子 | 语境依赖 |
|------|------|---------|
| 人称指示 | 我、你、他 | 话语参与者 |
| 时间指示 | 今天、现在、昨天 | 话语时间 |
| 地点指示 | 这里、那里、这个 | 话语地点 |
| 话语指示 | 上文、前一句 | 话语顺序 |
| 社交指示 | 您、阁下 | 社会关系 |

**形式化解释**:

```
[["我"]]^C = C.S  (说话者)
[["你"]]^C = C.H  (听话者)
[["这里"]]^C = C.L  (地点)
[["现在"]]^C = C.T  (时间)
[["这个"]]^C = ιx[Demonstrated(x, C)]  (演示对象)
```

### 5.2 话语表示与回指

**回指解析规则** (DRT框架):

```
话语: "A man came in. He smiled."

DRS构建:
  DRS₁: [x | man(x), came-in(x)]
  DRS₂: [ | smiled(x)]  (He 解析为 x)

可及性条件:
  x 在 DRS₁ 中引入，在后续话语中可及
```

**约束理论 (Binding Theory) 形式化**:

| 原则 | 内容 | 形式化 |
|------|------|--------|
| 原则A | 反身代词必须在最小管辖语域内被约束 | $Reflexive \rightarrow Bound\_in\_GC$ |
| 原则B | 代词必须在最小管辖语域内自由 | $Pronoun \rightarrow Free\_in\_GC$ |
| 原则C | R-表达必须自由 | $R\_expression \rightarrow Free$ |

### 5.3 预设的形式处理

**预设触发语** (Karttunen & Peters):

| 类型 | 例子 | 预设内容 |
|------|------|---------|
| 定指描述 | the N | ∃!x N(x) |
| 事实动词 | regret S | S |
| 变化动词 | stop V-ing | 之前V-ing |
| 迭代词 | too, again | 类似情况曾发生 |
| 比较句 | X is as Y as Z | Z is Y |

**预设投射问题**:

```
嵌入语境中的预设行为:

简单句: "John regrets leaving"
  → 预设: John left

条件句: "If John regrets leaving, he should apologize"
  → 预设不确定 (条件悬挂)

合取: "John left and he regrets it"
  → 预设: John left (在第一合取支满足)
```

## 6. 信息结构

### 6.1 话题-述题结构

**定义 6.1.1** (话题与述题)

- **话题 (Topic)**: 话语所关于的对象（旧信息）
- **述题 (Comment/Predicate)**: 关于话题的新信息

**形式化**:

```
句子: "As for John, Mary likes him"
结构: Topic(John, Mary likes him)

DRT表示:
  ┌─────────────────────────────┐
  │ x y                         │
  │ John(x)                     │
  │ Mary(y)                     │
  │ ┌───────────────────────┐   │
  │ │ 关于x:                │   │
  │ │ like(y, x)            │   │
  │ └───────────────────────┘   │
  └─────────────────────────────┘
```

**话题标记策略**:

| 策略 | 语言 | 形式 |
|------|------|------|
| 句法 | 日语 | NP-wa ... |
| 语调 | 英语 | 话题升调 |
| 语序 | 汉语 | 话题前置 |
| 词汇 | 英语 | As for X, ... |

### 6.2 焦点与预设

**定义 6.2.1** (焦点-预设结构)
焦点突出句子中信息最显著的部分，预设是焦点所依赖的背景。

**形式化**:

```
句子: "JOHN ate the apple" (John为焦点)
预设: 某人吃了苹果
断言: 这人是John

结构: Focus(John, λx.eat(x, the apple))
```

**焦点敏感算子**:

```
"Only JOHN ate the apple"
语义: ∀x[eat(x, apple) → x = john]

焦点关联:
  Only(λx.eat(x, apple))(john)
  = john ∈ {x : eat(x, apple)} ∧ |{x : eat(x, apple)}| = 1
```

### 6.3 信息状态的动态更新

**信息状态模型**:
$$IS = \langle W, \leq, s \rangle$$

- $W$：可能世界集合
- $\leq$：偏序关系（信息增多）
- $s$：当前信息状态

**更新操作**:

```
断言更新: s[φ] = s ∩ [[φ]]

问题更新: s[?φ] = {w ∈ s : w ∈ [[φ]] 或 w ∈ [[¬φ]]}

焦点更新: s[Focus(x, φ)] = s ∩ {w : ιx.φ(x) = c}
```

### 8.2 语用推理流程

```
输入: 话语 U + 语境 C
        │
        ├──► 字面语义解释 [[U]]^C
        │
        ├──► 言语行为识别 (断言/疑问/请求/...)
        │
        ├──► 预设处理
        │       ├──► 识别预设触发语
        │       └──► 投射或满足预设
        │
        ├──► 会话含义推导
        │       ├──► 检查准则违反
        │       └──► 推导含义
        │
        ├──► 指示词解析
        │
        └──► 信息结构处理
                ├──► 话题识别
                └──► 焦点解析

输出: 完整语用解释
```

### 9.2 案例2：等级含义的计算推导

**应用场景**: 自然语言推理系统。

**推导实现**:

```python
def scalar_implicature(utterance, alternatives, context):
    """
    推导等级含义
    utterance: 实际话语
    alternatives: 可选的更强表达
    context: 语境信息
    """
    # 检查是否有更强的可选表达
    stronger = [alt for alt in alternatives
                if is_stronger(alt, utterance)
                and is_relevant(alt, context)]

    if stronger:
        # 推导: 说话者不相信更强的表达为真
        implicature = Neg(Belief(Speaker, stronger[0]))
        return implicature
    return None

# 示例
utterance = "some students passed"
alternatives = ["all students passed", "most students passed"]

# 推导: "not all students passed"
```
