# 14.2.4 博弈论语义学

## 14.2.4.1 引言

博弈论语义学（Game-Theoretical Semantics, GTS）由Jaakko Hintikka（1973）提出，将逻辑公式的真值条件解释为两个玩家（验证者Verifier和证伪者Falsifier）之间的博弈。
Paul Lorenzen更早提出了对话逻辑（dialogical logic）的类似思想。

> **核心思想**：公式的真值通过博弈确定——验证者试图证明公式为真，证伪者试图证明为假。如果验证者有必胜策略，则公式为真。

## 14.2.4.2 逻辑博弈基础

### 14.2.4.2.1 博弈定义

> **定义 14.2.4.1**（语义博弈）
> 给定模型 M = ⟨D, I⟩ 和公式 φ，语义博弈 G(φ, M) 定义如下：
>
> - **玩家**：验证者（V，Myself）和证伪者（F，Nature）
> - **位置**：⟨ψ, g⟩，其中 ψ 是子公式，g 是赋值函数
> - **初始位置**：⟨φ, g₀⟩
> - **终止位置**：原子公式 ⟨P(t₁,...,tₙ), g⟩
> - **规则**：根据主连接词确定

### 14.2.4.2.2 博弈规则

| 公式 | 玩家 | 动作 | 结果位置 |
|------|------|------|----------|
| φ ∧ ψ | F | 选择 | ⟨φ, g⟩ 或 ⟨ψ, g⟩ |
| φ ∨ ψ | V | 选择 | ⟨φ, g⟩ 或 ⟨ψ, g⟩ |
| ∀xφ | F | 选择 a ∈ D | ⟨φ, g[x/a]⟩ |
| ∃xφ | V | 选择 a ∈ D | ⟨φ, g[x/a]⟩ |
| ¬φ | 交换 | 角色互换 | ⟨φ, g⟩（玩家交换） |
| φ → ψ | F | 选择 | ⟨¬φ, g⟩ 或 ⟨ψ, g⟩ |

```
博弈树示例：∀x∃y L(x,y)

                    Myself
                   /      \
                  /        \
             (F选a)      (F选b)
                |            |
             Nature       Nature
            /     \       /     \
           /       \     /       \
      (V选c)   (V选d) (V选e)  (V选f)
          |       |       |       |
       L(a,c)  L(a,d)  L(b,e)  L(b,f)

Myself有必胜策略当且仅当 ∀a∃b L(a,b)
```

```haskell
-- 博弈论语义学的Haskell实现

data Player = Myself | Nature deriving (Eq, Show)

-- 博弈位置
data GamePosition = GP {
    gpFormula :: Formula,
    gpAssignment :: Assignment
} deriving (Show)

data Formula = FAtom String [Term]
             | FConj Formula Formula    -- φ ∧ ψ
             | FDisj Formula Formula    -- φ ∨ ψ
             | FImp Formula Formula     -- φ → ψ
             | FNeg Formula             -- ¬φ
             | FForall String Formula   -- ∀xφ
             | FExists String Formula   -- ∃xφ
             deriving (Show)

data Term = TVar String | TConst String deriving (Show)

type Assignment = Map String Entity
type Entity = String

-- 模型
data Model = Model {
    modelDomain :: [Entity],
    modelInterp :: Map String [Entity] -> Bool  -- 谓词解释
} deriving (Show)

-- 博弈状态
data GameState = GameState {
    gsPosition :: GamePosition,
    gsCurrentPlayer :: Player,
    gsModel :: Model
} deriving (Show)

-- 获胜判定
winner :: GameState -> Maybe Player
winner (GameState (GP (FAtom p terms) assign) player model) =
    let entities = map (evalTerm assign) terms
        isTrue = modelInterp model entities
    in case (isTrue, player) of
        (True, Myself) -> Just Myself
        (False, Nature) -> Just Nature
        _ -> Nothing
winner _ = Nothing

evalTerm :: Assignment -> Term -> Entity
evalTerm assign (TVar x) = case Map.lookup x assign of
    Just e -> e
    Nothing -> error "Unbound variable"
evalTerm _ (TConst c) = c
```

### 14.2.4.2.3 策略与真值

> **定义 14.2.4.2**（策略与真值）
>
> - **策略**：从玩家选择的位置到合法移动的函数
> - **必胜策略**：无论对手如何选择，都保证获胜的策略
> - **真值**：φ 在 M 中为真当且仅当 Myself 在 G(φ, M) 中有必胜策略

```haskell
-- 策略定义
data Strategy = Strategy {
    stratMoves :: Map GamePosition Move,
    stratPlayer :: Player
} deriving (Show)

data Move = MLeft | MRight | MChoose Entity deriving (Show)

-- 应用策略获得移动
applyStrategy :: Strategy -> GamePosition -> Maybe Move
applyStrategy strat pos = Map.lookup pos (stratMoves strat)

-- 判断是否为必胜策略
isWinning :: Model -> Formula -> Strategy -> Bool
isWinning model formula strategy =
    let initialState = GameState (GP formula Map.empty) Myself model
    in allWinningPlays initialState strategy

allWinningPlays :: GameState -> Strategy -> Bool
allWinningPlays state strategy =
    case winner state of
        Just p -> p == stratPlayer strategy
        Nothing ->
            let moves = legalMoves state
                responses = map (\m -> makeMove state m) moves
            in all (\s -> allWinningPlays s strategy) responses

legalMoves :: GameState -> [Move]
legalMoves (GameState (GP (FDisj _ _) _) Myself _) = [MLeft, MRight]
legalMoves (GameState (GP (FConj _ _) _) Nature _) = [MLeft, MRight]
legalMoves (GameState (GP (FForall _ _) _) Nature m) =
    map MChoose (modelDomain m)
legalMoves (GameState (GP (FExists _ _) _) Myself m) =
    map MChoose (modelDomain m)
legalMoves _ = []

makeMove :: GameState -> Move -> GameState
makeMove gs@(GameState (GP (FDisj l r) assign) Myself m) MLeft =
    GameState (GP l assign) Nature m
makeMove gs@(GameState (GP (FDisj l r) assign) Myself m) MRight =
    GameState (GP r assign) Nature m
makeMove gs@(GameState (GP (FForall x phi) assign) Nature m) (MChoose e) =
    GameState (GP phi (Map.insert x e assign)) Myself m
makeMove gs@(GameState (GP (FExists x phi) assign) Myself m) (MChoose e) =
    GameState (GP phi (Map.insert x e assign)) Nature m
makeMove gs _ = gs
```

## 14.2.4.3 IF逻辑

### 14.2.4.3.1 独立性友好逻辑

> **定义 14.2.4.3**（IF逻辑）
> IF逻辑（Independence-Friendly Logic）允许量词之间的依赖性/独立性显式标记。
>
> (∀x/∀y)φ 表示 x 的选择独立于 y 的选择

```
IF逻辑示例：

∀x∃y/∀x L(x,y)
含义：对于每个x，存在y，且y的选择不依赖于特定的x
这等价于：∃y∀x L(x,y)

博弈解释：
- Nature选择x
- Myself选择y，但不知道Nature选择了哪个x
- Myself必须有统一的策略应对所有可能的x
```

```haskell
-- IF逻辑公式
data IFFormula = IFAtom String [Term]
               | IFConj IFFormula IFFormula
               | IFDisj IFFormula IFFormula
               | IFNeg IFFormula
               | IFForall String [String] IFFormula  -- ∀x/(y₁,...,yₙ)φ
               | IFExists String [String] IFFormula  -- ∃x/(y₁,...,yₙ)φ
               deriving (Show)

-- 信息集：Myself不知道的选择
newtype InformationSet = IS {
    isVisible :: [String],      -- 可见变量
    isHidden :: [String]        -- 隐藏变量（独立性）
} deriving (Show)

-- IF博弈状态
data IFGameState = IFGS {
    ifgsPosition :: IFFormula,
    ifgsAssignment :: Assignment,
    ifgsVisible :: Set String,   -- 当前可见变量
    ifgsPlayer :: Player
} deriving (Show)

-- IF博弈的移动
ifLegalMoves :: IFGameState -> [Move]
ifLegalMoves (IFGS (IFForall x hidden phi) assign visible Nature) =
    -- Nature选择x，但Myself可能看不到
    map MChoose (modelDomain undefined)  -- 简化
ifLegalMoves (IFGS (IFExists x hidden phi) assign visible Myself) =
    -- Myself选择x，但受限于可见变量
    let allowedChoices = filter (\e -> isConsistent x e visible assign) (modelDomain undefined)
    in map MChoose allowedChoices
ifLegalMoves _ = []

isConsistent :: String -> Entity -> Set String -> Assignment -> Bool
isConsistent var entity visible assign = True  -- 简化
```

### 14.2.4.3.2 混合策略均衡

> **定义 14.2.4.4**（混合策略）
> 在IF逻辑中，Myself可能没有纯策略获胜，但可能有混合策略（概率分布）获胜。

```haskell
-- 混合策略
data MixedStrategy = MixedStrategy {
    msDistribution :: Map Move Double  -- 移动的概率分布
} deriving (Show)

-- 计算获胜概率
winningProbability :: IFGameState -> MixedStrategy -> Double
winningProbability state strategy =
    sum [prob * winProb state move | (move, prob) <- Map.toList (msDistribution strategy)]
  where
    winProb state move =
        let nextState = ifMakeMove state move
        in if isTerminal nextState
           then if winnerIF nextState == Just Myself then 1.0 else 0.0
           else optimalWinProb nextState

    optimalWinProb s = winningProbability s (optimalResponse s)

isTerminal :: IFGameState -> Bool
isTerminal (IFGS (IFAtom _ _) _ _ _) = True
isTerminal _ = False

winnerIF :: IFGameState -> Maybe Player
winnerIF = undefined  -- 依赖于模型

ifMakeMove :: IFGameState -> Move -> IFGameState
ifMakeMove = undefined

optimalResponse :: IFGameState -> MixedStrategy
optimalResponse = undefined  -- 博弈论均衡计算
```

## 14.2.4.4 对话逻辑

### 14.2.4.4.1 对话游戏

Paul Lorenzen的对话逻辑将证明视为玩家之间的辩论。

> **定义 14.2.4.5**（对话游戏）
>
> - **主张者**（Proponent，P）维护公式为真
> - **反对者**（Opponent，O）质疑公式
> - 通过攻击和辩护规则进行对话

```
攻击规则：

公式      攻击       辩护
φ ∧ ψ    ?L 或 ?R   φ 或 ψ
φ ∨ ψ    ?          φ 或 ψ
φ → ψ    φ          ψ
¬φ       φ          （无，游戏结束）
∀xφ      ?a（选a）   φ[a/x]
∃xφ      ?          φ[a/x]（选a）

胜利条件：
- P成功辩护所有攻击
- O无法提出合法攻击
```

```haskell
-- 对话逻辑
data DialoguePlayer = Proponent | Opponent deriving (Eq, Show)

data DialogueMove = Attack AttackType
                  | Defense Formula
                  | Surrender
                  deriving (Show)

data AttackType = AL | AR | AQ String | AAtomic deriving (Show)

-- 对话状态
data DialogueState = DS {
    dsFormula :: Formula,
    dsAttacker :: DialoguePlayer,
    dsHistory :: [(DialoguePlayer, DialogueMove)],
    dsConstraints :: [Formula]  -- O的承诺
} deriving (Show)

-- 合法攻击
attacks :: Formula -> DialoguePlayer -> [AttackType]
attacks (FConj _ _) Proponent = [AL, AR]
attacks (FDisj _ _) Opponent = [AQ "which?"]
attacks (FImp phi _) Opponent = [AQ "prove antecedent"]
attacks (FNeg phi) Opponent = [AQ "prove"]
attacks (FForall x phi) Opponent = map (AQ . ("choose " ++)) ["a", "b", "c"]
attacks (FExists x phi) Proponent = [AQ "witness"]
attacks (FAtom _ _) _ = [AAtomic]
attacks _ _ = []

-- 合法辩护
defenses :: Formula -> AttackType -> Maybe Formula
defenses (FConj l _) AL = Just l
defenses (FConj _ r) AR = Just r
defenses (FDisj l r) (AQ _) = Just l  -- 或 r，P选择
defenses (FImp _ psi) _ = Just psi
defenses (FExists x phi) (AQ witness) = Just (substitute phi x witness)
defenses _ _ = Nothing

substitute :: Formula -> String -> String -> Formula
substitute = undefined
```

### 14.2.4.4.2 形式辩证法

```haskell
-- 对话协议
data DialogueProtocol = DP {
    dpAttackRule :: Formula -> [AttackType],
    dpDefenseRule :: Formula -> AttackType -> Maybe Formula,
    dpWinningCond :: DialogueState -> Maybe DialoguePlayer
} deriving (Show)

-- 经典协议
classicalProtocol :: DialogueProtocol
classicalProtocol = DP {
    dpAttackRule = attacks,
    dpDefenseRule = defenses,
    dpWinningCond = classicalWinning
}

classicalWinning :: DialogueState -> Maybe DialoguePlayer
classicalWinning (DS (FAtom p _) Proponent _ _) = Just Proponent
classicalWinning (DS (FAtom p _) Opponent _ _) = Just Opponent
classicalWinning (DS _ _ history _) =
    if length history > 100 then Just Proponent else Nothing

-- 运行对话
runDialogue :: DialogueProtocol -> Formula -> DialoguePlayer -> DialogueState
runDialogue protocol formula starter =
    let initial = DS formula starter [] []
    in playDialogue protocol initial

playDialogue :: DialogueProtocol -> DialogueState -> DialogueState
playDialogue protocol state =
    case dpWinningCond protocol state of
        Just winner -> state
        Nothing ->
            let move = selectMove protocol state
                nextState = updateState state move
            in playDialogue protocol nextState

selectMove = undefined

updateState :: DialogueState -> (DialoguePlayer, DialogueMove) -> DialogueState
```

## 14.2.4.5 博弈论语义与自然语言

### 14.2.4.5.1 量词范围歧义

```
"Every man loves some woman"

解释1（宽域存在）：∃y∀x[man(x) → (woman(y) ∧ love(x,y))]
博弈：Myself先选择y，然后Nature选择x

解释2（窄域存在）：∀x∃y[man(x) → (woman(y) ∧ love(x,y))]
博弈：Nature先选择x，然后Myself选择y

GTS通过玩家移动顺序消解歧义
```

```haskell
-- 量词范围的博弈解释
quantifierScopeGame :: [Quantifier] -> Formula -> GameState
quantifierScopeGame quants phi =
    let initialPos = GP phi Map.empty
        initialPlayer = if quantPlayer (head quants) == Myself then Myself else Nature
    in GameState initialPos initialPlayer undefined

data Quantifier = ForallQ String | ExistsQ String deriving (Show)

quantPlayer :: Quantifier -> Player
quantPlayer (ForallQ _) = Nature
quantPlayer (ExistsQ _) = Myself

quantOrder :: [Quantifier] -> [(Player, String)]
quantOrder = map (\q -> (quantPlayer q, quantVar q))
  where
    quantVar (ForallQ x) = x
    quantVar (ExistsQ x) = x
```

### 14.2.4.5.2 无定名词

```
"A man walks. He whistles."

博弈解释：
1. "A man walks"：Myself选择x，使得man(x) ∧ walk(x)
2. "He whistles"：x绑定到前文选择的实体
3. 验证：whistle(x)

DRS与GTS的关系：
DRS的话语指称 = GTS的Myself选择
DRS条件 = 博弈的获胜条件
```

## 14.2.4.6 实现与算法

```prolog
% Prolog: 博弈论语义引擎

% 博弈树生成
game_tree(Formula, Model, Player, Tree) :-
    build_tree(Formula, Model, Player, [], Tree).

% 原子公式：终止节点
build_tree(Atom, Model, Player, Assign, leaf(Atom, Assign, Winner)) :-
    atomic_formula(Atom),
    evaluate(Atom, Model, Assign, Truth),
    winner(Truth, Player, Winner).

% 合取：证伪者选择
build_tree(and(Phi, Psi), Model, Player, Assign,
           node(Player, [LeftTree, RightTree])) :-
    switch_player(Player, Opponent),
    build_tree(Phi, Model, Opponent, Assign, LeftTree),
    build_tree(Psi, Model, Opponent, Assign, RightTree).

% 析取：验证者选择
build_tree(or(Phi, Psi), Model, Player,
           node(Player, [LeftTree, RightTree])) :-
    build_tree(Phi, Model, Player, LeftTree),
    build_tree(Psi, Model, Player, RightTree).

% 全称量词：证伪者选择实例
build_tree(forall(X, Phi), Model, Player, Assign,
           node(Player, SubTrees)) :-
    domain(Model, Domain),
    switch_player(Player, Opponent),
    findall(Tree, (
        member(E, Domain),
        update_assign(Assign, X, E, NewAssign),
        build_tree(Phi, Model, Opponent, NewAssign, Tree)
    ), SubTrees).

% 存在量词：验证者选择实例
build_tree(exists(X, Phi), Model, Player, Assign,
           node(Player, SubTrees)) :-
    domain(Model, Domain),
    findall(Tree, (
        member(E, Domain),
        update_assign(Assign, X, E, NewAssign),
        build_tree(Phi, Model, Player, NewAssign, Tree)
    ), SubTrees).

% 判断必胜策略
has_winning_strategy(Tree, Player) :-
    winning_tree(Tree, Player).

winning_tree(leaf(_, _, Player), Player).
winning_tree(node(Player, Children), Player) :-
    % 玩家选择：至少有一个子树获胜
    member(Child, Children),
    winning_tree(Child, Player).
winning_tree(node(Opponent, Children), Player) :-
    % 对手选择：所有子树都必须获胜
    Opponent \= Player,
    forall(member(Child, Children), winning_tree(Child, Player)).
```

## 14.2.4.7 总结

| 特性 | Tarski语义 | 博弈论语义 |
|------|------------|------------|
| 真值定义 | 递归定义 | 策略存在性 |
| 量词 | 变元赋值 | 玩家选择 |
| 逻辑常项 | 真值函数 | 游戏规则 |
| 直觉主义 | 额外机制 | 自然对应 |
| IF逻辑 | 困难 | 直接处理 |
| 计算复杂度 | 低 | 高（博弈求解） |
