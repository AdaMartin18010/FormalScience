## 14.4.1.2 有限自动机与正则语言

### 14.4.1.2.1 确定性有限自动机

> **定义 14.4.1.1**（DFA）
> 确定性有限自动机是一个五元组 M = (Q, Σ, δ, q₀, F)，其中：
>
> - Q：状态有限集
> - Σ：输入字母表
> - δ: Q × Σ → Q：转移函数
> - q₀ ∈ Q：初始状态
> - F ⊆ Q：接受状态集

```
DFA示例：识别以"ab"结尾的字符串

      a           b
   ┌─────┐     ┌─────┐
   ▼     │     ▼     │
→ (q₀) ──┘    (q₁) ──┐
   │     b      │     a
   ▼            ▼
  [q₂] ←───────┘

接受状态: q₂
转移表:
    δ   a   b
    q₀  q₁  q₀
    q₁  q₁  q₂
    q₂  q₁  q₀
```

```haskell
-- DFA的Haskell实现
data DFA state symbol = DFA {
    dfaStates :: [state],
    dfaAlphabet :: [symbol],
    dfaTransition :: state -> symbol -> state,
    dfaStart :: state,
    dfaAccept :: [state]
} deriving (Show)

-- 运行DFA
runDFA :: (Eq state, Eq symbol) => DFA state symbol -> [symbol] -> Bool
runDFA dfa input =
    let finalState = foldl (dfaTransition dfa) (dfaStart dfa) input
    in finalState `elem` dfaAccept dfa

-- DFA配置
data DFAConfig state = DFAConfig {
    dfaConfigState :: state,
    dfaConfigInput :: [symbol]
} deriving (Show)

-- 单步转移
dfaStep :: DFA state symbol -> DFAConfig state -> Maybe (DFAConfig state)
dfaStep dfa (DFAConfig q []) = Nothing
dfaStep dfa (DFAConfig q (s:ss)) =
    Just (DFAConfig (dfaTransition dfa q s) ss)

-- 示例：识别偶数个a的DFA
evenAsDFA :: DFA Int Char
evenAsDFA = DFA {
    dfaStates = [0, 1],
    dfaAlphabet = ['a', 'b'],
    dfaTransition = \q s -> case (q, s) of
        (0, 'a') -> 1
        (0, 'b') -> 0
        (1, 'a') -> 0
        (1, 'b') -> 1,
    dfaStart = 0,
    dfaAccept = [0]
}
```

### 14.4.1.2.2 非确定性有限自动机

> **定义 14.4.1.2**（NFA）
> NFA允许：
>
> - 同一状态-符号对转移到多个状态
> - ε-转移（不消耗输入符号）

**定理 14.4.1.1**（DFA与NFA等价性）
对于每个NFA，存在等价的DFA接受相同语言。

**证明**：子集构造法

- 新状态 = NFA状态的子集
- 转移 = 所有可能转移的并集
- 接受状态 = 包含NFA接受状态的子集

```haskell
-- NFA实现
data NFA state symbol = NFA {
    nfaStates :: [state],
    nfaAlphabet :: [symbol],
    nfaTransition :: state -> Maybe symbol -> [state],
    nfaStart :: state,
    nfaAccept :: [state]
} deriving (Show)

-- ε-闭包
epsilonClosure :: Eq state => NFA state symbol -> [state] -> [state]
epsilonClosure nfa states =
    let closure = concatMap (\s -> nfaTransition nfa s Nothing) states
        newStates = nub (states ++ closure)
    in if newStates == states then states else epsilonClosure nfa newStates

-- NFA转DFA（子集构造）
nfaToDFA :: (Eq state, Eq symbol) => NFA state symbol -> DFA [state] symbol
nfaToDFA nfa =
    let start = epsilonClosure nfa [nfaStart nfa]
        allSubsets = powerset (nfaStates nfa)
        validSubsets = filter (\subset -> not (null subset)) allSubsets
    in DFA {
        dfaStates = validSubsets,
        dfaAlphabet = nfaAlphabet nfa,
        dfaTransition = \subset s ->
            epsilonClosure nfa (concatMap (\q -> nfaTransition nfa q (Just s)) subset),
        dfaStart = start,
        dfaAccept = filter (\subset -> any (\s -> s `elem` nfaAccept nfa) subset) validSubsets
    }

powerset :: [a] -> [[a]]
powerset [] = [[]]
powerset (x:xs) = let ps = powerset xs in ps ++ map (x:) ps

nub :: Eq a => [a] -> [a]
nub = map head . group . sort
```

### 14.4.1.2.3 正则表达式

> **定义 14.4.1.3**（正则表达式）
> 正则表达式语法：
>
> - ∅：空语言
> - ε：空字符串
> - a ∈ Σ：单个符号
> - R₁ + R₂：并
> - R₁ · R₂：连接
> - R*：Kleene闭包

**Kleene定理**：正则表达式、正则文法、DFA、NFA识别相同语言类。

```haskell
-- 正则表达式
data Regex = Empty
           | Epsilon
           | Symbol Char
           | Union Regex Regex
           | Concat Regex Regex
           | Star Regex
           deriving (Eq, Show)

-- 正则表达式转NFA（Thompson构造）
regexToNFA :: Regex -> NFA Int Char
regexToNFA Empty = NFA [0,1] [] (\_ _ -> []) 0 []
regexToNFA Epsilon = NFA [0,1] [] (\q ms -> if q == 0 && ms == Nothing then [1] else []) 0 [1]
regexToNFA (Symbol c) = NFA [0,1] [c] (\q ms -> if q == 0 && ms == Just c then [1] else []) 0 [1]
regexToNFA (Union r1 r2) = unionNFA (regexToNFA r1) (regexToNFA r2)
regexToNFA (Concat r1 r2) = concatNFA (regexToNFA r1) (regexToNFA r2)
regexToNFA (Star r) = starNFA (regexToNFA r)

-- NFA组合操作
unionNFA :: NFA Int Char -> NFA Int Char -> NFA Int Char
unionNFA n1 n2 = undefined  -- 需要状态重编号

concatNFA :: NFA Int Char -> NFA Int Char -> NFA Int Char
concatNFA n1 n2 = undefined

starNFA :: NFA Int Char -> NFA Int Char
starNFA n = undefined
```

## 14.4.1.3 上下文无关文法与下推自动机

### 14.4.1.3.1 下推自动机

> **定义 14.4.1.4**（PDA）
> PDA = (Q, Σ, Γ, δ, q₀, Z₀, F)，其中：
>
> - Γ：栈字母表
> - Z₀ ∈ Γ：初始栈符号
> - δ: Q × (Σ ∪ {ε}) × Γ → P(Q × Γ*)

```
PDA示例：识别 { aⁿbⁿ | n ≥ 0 }

状态: q₀ (初始), q₁, qf (接受)
栈: Z₀ (初始), A

转移:
1. δ(q₀, a, Z₀) = {(q₀, AZ₀)}
2. δ(q₀, a, A) = {(q₀, AA)}
3. δ(q₀, b, A) = {(q₁, ε)}
4. δ(q₁, b, A) = {(q₁, ε)}
5. δ(q₁, ε, Z₀) = {(qf, Z₀)}

推导 "aaabbb":
(q₀, aaabbb, Z₀) ⊢ (q₀, aabbb, AZ₀)
                ⊢ (q₀, abbb, AAZ₀)
                ⊢ (q₀, bbb, AAAZ₀)
                ⊢ (q₁, bb, AAZ₀)
                ⊢ (q₁, b, AZ₀)
                ⊢ (q₁, ε, Z₀)
                ⊢ (qf, ε, Z₀) ✓
```

```haskell
-- PDA实现
data PDA state input stack = PDA {
    pdaStates :: [state],
    pdaAlphabet :: [input],
    pdaStackAlphabet :: [stack],
    pdaTransitions :: state -> Maybe input -> stack -> [(state, [stack])],
    pdaStart :: state,
    pdaInitialStack :: stack,
    pdaAccept :: [state]
} deriving (Show)

-- PDA配置
data PDAConfig state stack = PDAConfig {
    pdaState :: state,
    pdaInput :: [input],
    pdaStack :: [stack]
} deriving (Show)

-- PDA转移
pdaTransitionsAll :: PDA state input stack -> PDAConfig state stack -> [PDAConfig state stack]
pdaTransitionsAll pda config@(PDAConfig q (i:is) (s:ss)) =
    let epsilonMoves = pdaTransitions pda q Nothing s
        inputMoves = pdaTransitions pda q (Just i) s
    in [PDAConfig q' is (s' ++ ss) | (q', s') <- inputMoves] ++
       [PDAConfig q' (i:is) (s' ++ ss) | (q', s') <- epsilonMoves]
pdaTransitionsAll pda (PDAConfig q [] (s:ss)) =
    let epsilonMoves = pdaTransitions pda q Nothing s
    in [PDAConfig q' [] (s' ++ ss) | (q', s') <- epsilonMoves]
pdaTransitionsAll _ _ = []

-- 接受性判断（空栈接受）
acceptsEmpty :: (Eq state, Eq stack) => PDA state input stack -> [input] -> Bool
acceptsEmpty pda input = any (\c -> pdaStack c == [])
    (reachableConfigs pda (PDAConfig (pdaStart pda) input [pdaInitialStack pda]))

reachableConfigs :: Eq state => PDA state input stack -> PDAConfig state stack -> [PDAConfig state stack]
reachableConfigs pda initial =
    let go seen [] = seen
        go seen (c:cs) =
            let next = filter (\n -> not (n `elem` seen)) (pdaTransitionsAll pda c)
            in go (c:seen) (cs ++ next)
    in go [] [initial]

instance (Eq state, Eq stack) => Eq (PDAConfig state stack) where
    (PDAConfig q1 i1 s1) == (PDAConfig q2 i2 s2) = q1 == q2 && i1 == i1 && s1 == s2
```

### 14.4.1.3.2 CFG与PDA等价性

**定理 14.4.1.2**（CFG与PDA等价）
语言L是上下文无关的当且仅当存在PDA M使得L = L(M)。

**构造方法**：

- **CFG → PDA**：栈模拟最左推导
- **PDA → CFG**：将PDA计算转化为产生式

```haskell
-- CFG数据类型
data CFG = CFG {
    cfgNonTerminals :: [String],
    cfgTerminals :: [String],
    cfgRules :: [(String, [String])],
    cfgStart :: String
} deriving (Show)

-- CFG转PDA（自顶向下分析）
cfgToPDA :: CFG -> PDA String String String
cfgToPDA cfg = PDA {
    pdaStates = ["q", "f"],
    pdaAlphabet = cfgTerminals cfg,
    pdaStackAlphabet = cfgNonTerminals cfg ++ cfgTerminals cfg,
    pdaTransitions = \state input top -> case (state, input, top) of
        ("q", Just a, a') | a == a' -> [("q", [])]  -- 匹配终结符
        ("q", Nothing, nt) | nt `elem` cfgNonTerminals cfg ->
            [("q", rhs) | (lhs, rhs) <- cfgRules cfg, lhs == nt]  -- 扩展
        ("q", Nothing, "$") -> [("f", [])]  -- 接受
        _ -> [],
    pdaStart = "q",
    pdaInitialStack = cfgStart cfg,
    pdaAccept = ["f"]
}
```

## 14.4.1.4 图灵机与递归可枚举语言

### 14.4.1.4.1 确定性图灵机

> **定义 14.4.1.5**（DTM）
> DTM = (Q, Σ, Γ, δ, q₀, B, F)，其中：
>
> - Γ ⊇ Σ：带字母表
> - B ∈ Γ - Σ：空白符号
> - δ: Q × Γ → Q × Γ × {L, R}：部分函数

```haskell
-- 图灵机
data TM state symbol = TM {
    tmStates :: [state],
    tmAlphabet :: [symbol],
    tmTapeAlphabet :: [symbol],
    tmTransition :: state -> symbol -> Maybe (state, symbol, Direction),
    tmStart :: state,
    tmBlank :: symbol,
    tmAccept :: [state]
} deriving (Show)

data Direction = L | R deriving (Show)

-- 磁带表示
data Tape symbol = Tape {
    tapeLeft :: [symbol],   -- 反转的左半部分
    tapeHead :: symbol,
    tapeRight :: [symbol]
} deriving (Show)

-- TM配置
data TMConfig state symbol = TMConfig {
    tmState :: state,
    tmTape :: Tape symbol
} deriving (Show)

-- TM单步
tmStep :: TM state symbol -> TMConfig state symbol -> Maybe (TMConfig state symbol)
tmStep tm (TMConfig q tape) = do
    (q', sym', dir) <- tmTransition tm q (tapeHead tape)
    let newTape = case dir of
            L -> case tapeLeft tape of
                [] -> Tape [] (tmBlank tm) (sym' : tapeRight tape)
                (l:ls) -> Tape ls l (sym' : tapeRight tape)
            R -> case tapeRight tape of
                [] -> Tape (sym' : tapeLeft tape) (tmBlank tm) []
                (r:rs) -> Tape (sym' : tapeLeft tape) r rs
    return (TMConfig q' newTape)

-- 运行直到停机
tmRun :: TM state symbol -> TMConfig state symbol -> TMConfig state symbol
tmRun tm config = case tmStep tm config of
    Nothing -> config
    Just config' -> tmRun tm config'
```

### 14.4.1.4.2 图灵机的语言学意义

| 文法类型 | 自动机 | 复杂度 | 自然语言应用 |
|----------|--------|--------|--------------|
| 正则（Type-3） | DFA/NFA | O(n) | 形态分析（有限） |
| 上下文无关（Type-2） | PDA | O(n³) | 句法分析 |
| 上下文有关（Type-1） | LBA | 不确定 | 某些句法现象 |
| 递归可枚举（Type-0） | TM | 不可判定 | 语义/语用 |

## 14.4.1.5 有限状态转录机

### 14.4.1.5.1 FST与形态分析

> **定义 14.4.1.6**（FST）
> 有限状态转录机是一个五元组 T = (Q, Σ, Γ, δ, q₀, F)，其中：
>
> - Σ：输入字母表
> - Γ：输出字母表
> - δ: Q × (Σ ∪ {ε}) → P(Q × (Γ ∪ {ε}))

```haskell
-- 有限状态转录机
data FST state input output = FST {
    fstStates :: [state],
    fstInputAlphabet :: [input],
    fstOutputAlphabet :: [output],
    fstTransitions :: state -> Maybe input -> [(state, Maybe output)],
    fstStart :: state,
    fstAccept :: [state]
} deriving (Show)

-- FST运行（生成所有可能的输出）
runFST :: (Eq state) => FST state input output -> [input] -> [[output]]
runFST fst input =
    let configs = [(fstStart fst, input, [])]
    in map reverse (runFSTConfigs fst configs)

runFSTConfigs :: (Eq state) => FST state input output -> [(state, [input], [output])] -> [[output]]
runFSTConfigs _ [] = []
runFSTConfigs fst ((q, [], out):cs)
    | q `elem` fstAccept fst = out : runFSTConfigs fst cs
    | otherwise = runFSTConfigs fst cs
runFSTConfigs fst ((q, (i:is), out):cs) =
    let trans = fstTransitions fst q (Just i)
        epsilonTrans = fstTransitions fst q Nothing
        newConfigs = [(q', is, maybe out (\o -> o:out) mo) | (q', mo) <- trans] ++
                     [(q', i:is, maybe out (\o -> o:out) mo) | (q', mo) <- epsilonTrans]
    in runFSTConfigs fst (cs ++ newConfigs)

-- 形态分析FST示例
twoLevelMorphology :: FST Int Char (Char, Char)
twoLevelMorphology = FST {
    fstStates = [0, 1, 2],
    fstInputAlphabet = ['a', 'b', 'N', 'G'],
    fstOutputAlphabet = zip ['a', 'b', 'N', 'G'] ['a', 'b', 'n', 'g'],
    fstTransitions = \q mi -> case (q, mi) of
        (0, Just 'a') -> [(0, Just ('a', 'a'))]
        (0, Just 'b') -> [(0, Just ('b', 'b'))]
        (0, Just 'N') -> [(1, Just ('N', 'n'))]  -- 准备删除
        (1, Just 'G') -> [(2, Just ('G', 'ŋ'))]  -- 鼻音同化
        (2, Nothing) -> [(0, Nothing)]  -- 删除标记
        _ -> [],
    fstStart = 0,
    fstAccept = [0]
}
```

## 14.4.1.6 总结

```
自动机层次：

正则语言                    上下文无关语言               上下文有关语言
    |                            |                            |
    ▼                            ▼                            ▼
┌─────────┐                ┌─────────┐                  ┌─────────┐
│  DFA/   │                │   PDA   │                  │   LBA   │
│  NFA    │                │         │                  │         │
└────┬────┘                └────┬────┘                  └────┬────┘
     │                          │                            │
     └────────────┬─────────────┘                            │
                  │                                          │
                  ▼                                          ▼
            ┌──────────┐                              ┌──────────┐
            │    FST   │                              │    TM    │
            │ (转录机) │                              │          │
            └──────────┘                              └──────────┘

应用范围：
- FST: 形态分析、音系规则
- PDA: 句法分析、括号匹配
- LBA: 某些复杂句法现象
- TM: 完全计算能力
```
