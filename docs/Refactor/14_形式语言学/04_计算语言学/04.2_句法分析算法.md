## 14.4.2.1 引言

句法分析（Parsing）是将输入字符串映射到句法树结构的过程。本章介绍两种经典的上下文无关文法分析算法：CKY算法（Cocke-Kasami-Younger）和Earley算法，它们在自然语言处理中有广泛应用。

## 14.4.2.2 CKY算法

### 14.4.2.2.1 算法概述

CKY算法是一种**动态规划**算法，用于识别和解析上下文无关语言。它要求文法为**乔姆斯基范式**（Chomsky Normal Form, CNF）。

> **定义 14.4.2.1**（CNF）
> 上下文无关文法处于乔姆斯基范式，如果所有产生式为：
>
> - A → B C（两个非终结符）
> - A → a（一个终结符）
> - S → ε（仅开始符号可产生空串）

**CKY算法复杂度**：O(n³|G|)，其中n为输入长度，|G|为文法大小。

### 14.4.2.2.2 算法原理

```
CKY解析表：

        j=0     j=1     j=2     j=3     j=4
      ───────────────────────────────────────
i=4  │       │       │       │       │ T[4,4]│
      ───────────────────────────────────────
i=3  │       │       │       │ T[3,3]│ T[3,4]│
      ───────────────────────────────────────
i=2  │       │       │ T[2,2]│ T[2,3]│ T[2,4]│
      ───────────────────────────────────────
i=1  │       │ T[1,1]│ T[1,2]│ T[1,3]│ T[1,4]│
      ───────────────────────────────────────
i=0  │ T[0,0]│ T[0,1]│ T[0,2]│ T[0,3]│ T[0,4]│
      ───────────────────────────────────────
      w₀      w₁      w₂      w₃      w₄

T[i,j] 包含能够生成子串 w[i..j] 的所有非终结符
```

```haskell
-- CKY算法的Haskell实现

data CNFRule = Terminal String String      -- A -> a
             | Binary String String String  -- A -> B C
             deriving (Show)

data CNFGrammar = CNFGrammar {
    cnfStart :: String,
    cnfRules :: [CNFRule]
} deriving (Show)

-- CKY表项
data CKYCell = CKYCell {
    ckyNonTerms :: [(String, ParseNode)],  -- (非终结符, 解析树)
    ckySpan :: (Int, Int)
} deriving (Show)

data ParseNode = PNode String [ParseNode]  -- 非终结符节点
               | PLeaf String              -- 终结符叶子
               deriving (Show)

-- CKY解析表
type CKYTable = [[CKYCell]]

-- CKY算法
ckyParse :: CNFGrammar -> [String] -> Maybe ParseNode
ckyParse grammar words =
    let n = length words
        table = initializeCKY grammar words n
        filledTable = fillTable grammar words table n
        finalCell = (filledTable !! 0) !! (n - 1)
    in lookup (cnfStart grammar) (ckyNonTerms finalCell)

-- 初始化（处理终结符产生式）
initializeCKY :: CNFGrammar -> [String] -> Int -> CKYTable
initializeCKY grammar words n =
    [[if i == j then initCell grammar words i else emptyCell (i,j)
      | j <- [0..n-1]] | i <- [0..n-1]]

initCell :: CNFGrammar -> [String] -> Int -> CKYCell
initCell grammar words pos =
    let word = words !! pos
        matching = [(lhs, PLeaf word) | Terminal lhs rhs <- cnfRules grammar, rhs == word]
    in CKYCell matching (pos, pos)

emptyCell :: (Int, Int) -> CKYCell
emptyCell span = CKYCell [] span

-- 填充解析表
fillTable :: CNFGrammar -> [String] -> CKYTable -> Int -> CKYTable
tillTable grammar words table n =
    foldl (\t span -> fillSpan grammar t span n) table
          [(i, j) | span <- [1..n-1], i <- [0..n-span-1], let j = i + span]

fillSpan :: CNFGrammar -> CKYTable -> (Int, Int) -> Int -> CKYTable
fillSpan grammar table (i, j) n =
    let newEntries = concatMap (\k -> combine grammar table i k j) [i..j-1]
        updatedCell = CKYCell (nubBy (\(a,_) (b,_) -> a == b) newEntries) (i, j)
    in updateTable table i j updatedCell

combine :: CNFGrammar -> CKYTable -> Int -> Int -> Int -> [(String, ParseNode)]
combine grammar table i k j =
    let leftCell = (table !! i) !! k
        rightCell = (table !! (k+1)) !! j
        leftNTs = ckyNonTerms leftCell
        rightNTs = ckyNonTerms rightCell
    in [(lhs, PNode lhs [leftTree, rightTree])
        | Binary lhs b c <- cnfRules grammar,
          (b', leftTree) <- leftNTs, b' == b,
          (c', rightTree) <- rightNTs, c' == c]

updateTable :: CKYTable -> Int -> Int -> CKYCell -> CKYTable
updateTable table i j cell =
    [if row == i then updateRow row j cell else row | (rowIdx, row) <- zip [0..] table]
  where
    updateRow r j cell = [if col == j then cell else c | (col, c) <- zip [0..] r]

-- 辅助函数
nubBy :: (a -> a -> Bool) -> [a] -> [a]
nubBy eq [] = []
nubBy eq (x:xs) = x : nubBy eq (filter (\y -> not (eq x y)) xs)

lookup :: Eq a => a -> [(a, b)] -> Maybe b
lookup key [] = Nothing
lookup key ((k,v):rest) = if key == k then Just v else lookup key rest
```

### 14.4.2.2.3 文法转换

```haskell
-- CNF转换
data CFGRule = CRule String [String] deriving (Show)

data CFG = CFG {
    cfgStart :: String,
    cfgRules :: [CFGRule]
} deriving (Show)

-- 转换CFG到CNF
toCNF :: CFG -> CNFGrammar
toCNF cfg =
    let step1 = eliminateStart cfg
        step2 = eliminateEpsilon step1
        step3 = eliminateUnit step2
        step4 = eliminateTerminals step3
        step5 = binarize step4
    in CNFGrammar (cfgStart step5) (convertRules (cfgRules step5))

eliminateStart :: CFG -> CFG
eliminateStart cfg = cfg  -- 简化

eliminateEpsilon :: CFG -> CFG
eliminateEpsilon cfg = cfg  -- 简化

eliminateUnit :: CFG -> CFG
eliminateUnit cfg = cfg  -- 简化

eliminateTerminals :: CFG -> CFG
eliminateTerminals cfg = cfg  -- 简化

binarize :: CFG -> CFG
binarize cfg =
    let newRules = concatMap binarizeRule (cfgRules cfg)
    in CFG (cfgStart cfg) newRules

binarizeRule :: CFGRule -> [CFGRule]
binarizeRule (CRule lhs [rhs1, rhs2]) = [CRule lhs [rhs1, rhs2]]
binarizeRule (CRule lhs rhs) | length rhs > 2 =
    let newNT = lhs ++ "_" ++ head rhs
        rest = tail rhs
    in CRule lhs [head rhs, newNT] : binarizeRule (CRule newNT rest)
binarizeRule r = [r]

convertRules :: [CFGRule] -> [CNFRule]
convertRules = concatMap convertRule
  where
    convertRule (CRule lhs [rhs]) | isTerminal rhs = [Terminal lhs rhs]
    convertRule (CRule lhs [rhs1, rhs2]) = [Binary lhs rhs1 rhs2]
    convertRule _ = []  -- 不应该发生

isTerminal :: String -> Bool
isTerminal s = not (null s) && head s >= 'a' && head s <= 'z'
```

## 14.4.2.3 Earley算法

### 14.4.2.3.1 算法概述

Earley算法（1970）是一种**图表解析**（chart parsing）算法，不需要文法处于CNF，可以处理任意CFG。

> **定义 14.4.2.2**（Earley项）
> Earley项形如 [A → α • β, i, j]，表示：
>
> - 产生式 A → αβ
> - 点（•）位置表示已分析/待分析边界
> - i：开始位置
> - j：当前位置

**Earley算法复杂度**：

- 一般情况：O(n³)
- 无歧义文法：O(n²)
- 确定性文法：O(n)

### 14.4.2.3.2 算法规则

```
Earley三种操作：

1. 预测（Predictor）：
   若 [A → α • B β, i, j] 在图表中，
   对所有 B → γ，添加 [B → • γ, j, j]

2. 扫描（Scanner）：
   若 [A → α • a β, i, j] 在图表中，且 w[j] = a，
   添加 [A → α a • β, i, j+1]

3. 完成（Completer）：
   若 [B → γ •, j, k] 在图表中，
   且 [A → α • B β, i, j] 在图表中，
   添加 [A → α B • β, i, k]
```

```haskell
-- Earley算法
data EarleyItem = EarleyItem {
    eiRule :: (String, [String], [String]),  -- (A, α, β) 其中 A → α•β
    eiStart :: Int,
    eiDot :: Int
} deriving (Show)

-- eiRule: (lhs, 已分析部分, 待分析部分)
-- A → α • β 表示为 (A, α, β)

data EarleyGrammar = EarleyGrammar {
    egStart :: String,
    egRules :: [(String, [String])]  -- (LHS, RHS)
} deriving (Show)

-- Earley图表：按位置索引
type EarleyChart = [[EarleyItem]]

-- Earley解析
earleyParse :: EarleyGrammar -> [String] -> Bool
earleyParse grammar words =
    let n = length words
        initialChart = replicate (n + 1) []
        initialItem = EarleyItem (egStart grammar, [], [rhs | (lhs, rhs) <- egRules grammar, lhs == egStart grammar]) 0 0
        chart0 = updateChart initialChart 0 [initialItem]
        finalChart = processAll grammar words chart0 n
    in any (\item -> isComplete item && eiStart item == 0 && eiDot item == n) (finalChart !! n)

isComplete :: EarleyItem -> Bool
isComplete (EarleyItem (_, _, beta) _ _) = null beta

-- 处理所有位置
processAll :: EarleyGrammar -> [String] -> EarleyChart -> Int -> EarleyChart
processAll grammar words chart n =
    foldl (\c k -> processPosition grammar words c k n) chart [0..n]

-- 处理位置k
processPosition :: EarleyGrammar -> [String] -> EarleyChart -> Int -> Int -> EarleyChart
processPosition grammar words chart k n =
    let items = chart !! k
        processed = processItems grammar words chart k items n
    in updateChart chart k processed

processItems :: EarleyGrammar -> [String] -> EarleyChart -> Int -> [EarleyItem] -> Int -> [EarleyItem]
processItems grammar words chart k items n =
    let newByPredict = concatMap (predictor grammar chart k) items
        newByScan = concatMap (scanner words chart k) items
        newByComplete = concatMap (completer chart k) items
        allNew = nub (items ++ newByPredict ++ newByScan ++ newByComplete)
    in if length allNew == length items
       then items
       else processItems grammar words chart k allNew n

-- 预测操作
predictor :: EarleyGrammar -> EarleyChart -> Int -> EarleyItem -> [EarleyItem]
predictor grammar chart k (EarleyItem (_, _, []) _ _) = []  -- 已完成
predictor grammar chart k (EarleyItem (_, alpha, b:beta) i j) =
    if isNonTerminal b
    then [EarleyItem (b, [], rhs) j j | (lhs, rhs) <- egRules grammar, lhs == b]
    else []

-- 扫描操作
scanner :: [String] -> EarleyChart -> Int -> EarleyItem -> [EarleyItem]
scanner words chart k (EarleyItem (_, _, []) _ _) = []
scanner words chart k (EarleyItem rule@(a, alpha, x:beta) i j) =
    if k < length words && x == words !! k
    then [EarleyItem (a, alpha ++ [x], beta) i (j + 1)]
    else []

-- 完成操作
completer :: EarleyChart -> Int -> EarleyItem -> [EarleyItem]
completer chart k item@(EarleyItem (b, gamma, []) j k') =
    [EarleyItem (a, alpha ++ [b], beta') i k
     | EarleyItem (a, alpha, c:beta') i j' <- chart !! j,
       c == b, j' == j]
completer _ _ _ = []

-- 辅助函数
isNonTerminal :: String -> Bool
isNonTerminal s = not (null s) && head s >= 'A' && head s <= 'Z'

updateChart :: EarleyChart -> Int -> [EarleyItem] -> EarleyChart
updateChart chart k items = take k chart ++ [items] ++ drop (k + 1) chart

nub :: Eq a => [a] -> [a]
nub [] = []
nub (x:xs) = x : nub (filter (/= x) xs)
```

### 14.4.2.3.3 解析树重构

```haskell
-- 从Earley图表重构解析树
data EarleyTree = ETNode String [EarleyTree]
                | ETLeaf String
                deriving (Show)

reconstructTree :: EarleyChart -> EarleyGrammar -> Int -> [EarleyTree]
reconstructTree chart grammar n =
    [buildTree item chart
     | item <- chart !! n,
       isComplete item,
       let (lhs, _, _) = eiRule item,
       lhs == egStart grammar,
       eiStart item == 0]

buildTree :: EarleyItem -> EarleyChart -> EarleyTree
buildTree (EarleyItem (lhs, alpha, []) start end) chart =
    ETNode lhs (buildChildren alpha start chart)

buildChildren :: [String] -> Int -> EarleyChart -> [EarleyTree]
buildChildren [] _ _ = []
buildChildren (sym:syms) pos chart =
    if isNonTerminal sym
    then let matching = [item | item <- chart !! pos,
                                let (lhs, _, _) = eiRule item,
                                lhs == sym,
                                isComplete item]
         in case matching of
              (item:_) -> buildTree item chart : buildChildren syms (eiDot item) chart
              [] -> [ETLeaf "error"]
    else ETLeaf sym : buildChildren syms (pos + 1) chart
```

## 14.4.2.5 概率上下文无关文法

### 14.4.2.5.1 PCFG

> **定义 14.4.2.3**（PCFG）
> PCFG = (N, Σ, S, R, P)，其中 P 为产生式概率，满足：
> ∀A ∈ N: Σ P(A → α) = 1

```haskell
-- PCFG
data PCFGRule = PCFGRule {
    pcfgLHS :: String,
    pcfgRHS :: [String],
    pcfgProb :: Double
} deriving (Show)

data PCFG = PCFG {
    pcfgStart :: String,
    pcfgRules :: [PCFGRule]
} deriving (Show)

-- Viterbi解析（CKY扩展）
data ViterbiCell = ViterbiCell {
    vcProbs :: [(String, Double, ParseNode)]  -- (非终结符, 概率, 树)
} deriving (Show)

viterbiParse :: PCFG -> [String] -> (Double, Maybe ParseNode)
viterbiParse pcfg words =
    let n = length words
        table = initializeViterbi pcfg words n
        filledTable = fillViterbi pcfg words table n
        finalCell = (filledTable !! 0) !! (n - 1)
    in case lookupMax (pcfgStart pcfg) (vcProbs finalCell) of
         Just (prob, tree) -> (prob, Just tree)
         Nothing -> (0, Nothing)

initializeViterbi :: PCFG -> [String] -> Int -> [[ViterbiCell]]
initializeViterbi pcfg words n =
    [[if i == j then initViterbiCell pcfg words i else ViterbiCell []
      | j <- [0..n-1]] | i <- [0..n-1]]

initViterbiCell :: PCFG -> [String] -> Int -> ViterbiCell
initViterbiCell pcfg words pos =
    let word = words !! pos
        matching = [(lhs, pcfgProb r, PLeaf word)
                   | r@(PCFGRule lhs [rhs] _) <- pcfgRules pcfg, rhs == word]
    in ViterbiCell matching

fillViterbi :: PCFG -> [String] -> [[ViterbiCell]] -> Int -> [[ViterbiCell]]
fillViterbi pcfg words table n = undefined  -- 类似CKY，但保留最大概率

lookupMax :: Eq a => a -> [(a, Double, b)] -> Maybe (Double, b)
lookupMax key entries =
    let matches = [(p, t) | (k, p, t) <- entries, k == key]
    in if null matches then Nothing else Just (maximumBy (comparing fst) matches)

comparing :: Ord b => (a -> b) -> a -> a -> Ordering
comparing f x y = compare (f x) (f y)

maximumBy :: (a -> a -> Ordering) -> [a] -> a
maximumBy cmp (x:xs) = foldl (\acc y -> if cmp acc y == LT then y else acc) x xs
```
