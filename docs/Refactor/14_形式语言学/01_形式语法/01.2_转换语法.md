# 14.1.2 转换语法

## 14.1.2.1 引言

转换语法（Transformational Grammar）是Chomsky在其经典著作《句法结构》（_Syntactic Structures_, 1957）中提出的理论框架。
该理论认为自然语言的句法生成涉及两个层面：深层结构（Deep Structure）通过转换规则映射到表层结构（Surface Structure）。

## 14.1.2.2 标准理论（Standard Theory）

### 14.1.2.2.1 基础组件

> **定义 14.1.2.1**（标准理论的句法组件）
> 标准理论包含以下组件：
>
> 1. **短语结构规则**（Phrase Structure Rules）：生成深层结构
> 2. **词汇插入**（Lexical Insertion）：将词汇项插入深层结构
> 3. **转换规则**（Transformations）：将深层结构映射到表层结构

### 14.1.2.2.2 短语结构规则

```
S → NP Aux VP
NP → (Det) N (PP) | NP S'
VP → V (NP) (PP) (S')
PP → P NP
Aux → Tense (M) (have+en) (be+ing)
Det → the | a | every | ...
N → man | woman | book | idea | ...
V → love | read | believe | seem | ...
P → in | on | by | with | ...
Tense → past | present
M → will | can | may | must | ...
```

**示例推导**：

```
S
→ NP Aux VP
→ Det N Tense M VP
→ the N Tense M VP
→ the man Tense M VP
→ the man present will VP
→ the man present will V NP
→ the man present will read NP
→ the man present will read Det N
→ the man present will read a N
→ the man present will read a book
```

```haskell
-- 短语结构规则的Haskell表示
data Category = S | NP | VP | PP | Aux | Det | N | V | P | Tense | M
              | AP | AdvP | CP | C' | TP | T' | vP | v' | DP
              deriving (Eq, Show)

data Feature = Past | Present | Third | Sing | Plur | Nom | Acc
             | Finite | Inf | Part | Ger deriving (Eq, Show)

data LexicalItem = LexicalItem {
    liForm :: String,
    liCategory :: Category,
    liFeatures :: [Feature]
} deriving (Show)

-- 上下文无关规则
data PSRRule = PSRRule {
    psrLHS :: Category,
    psrRHS :: [Either Category LexicalItem]
} deriving (Show)

-- 示例规则库
standardRules :: [PSRRule]
standardRules = [
    PSRRule S [Left NP, Left Aux, Left VP],
    PSRRule NP [Left Det, Left N],
    PSRRule NP [Left Det, Left N, Left PP],
    PSRRule VP [Left V, Left NP],
    PSRRule VP [Left V, Left NP, Left PP],
    PSRRule PP [Left P, Left NP]
    ]
```

### 14.1.2.2.3 转换规则的形式化

> **定义 14.1.2.2**（转换规则）
> 转换规则 T 是一个结构映射：T: (Structᵢ, Cond) → Structₒ
> 其中：
>
> - Structᵢ：输入结构（分析树中的子树）
> - Cond：应用条件
> - Structₒ：输出结构

**主要转换类型**：

| 转换 | 描述 | 示例 |
|------|------|------|
| 被动化 | NP₁ V NP₂ → NP₂ be V-en (by NP₁) | John loves Mary → Mary is loved by John |
| 疑问形成 | NP Aux VP → Aux NP VP | John can swim → Can John swim? |
| 关系化 | NP₁ VP → NP₁ [who VP]' | the man sleeps → the man who sleeps |
| 提升 | it seems [S' NP VP] → NP seems [S' t VP] | it seems that John is tall → John seems to be tall |

```
被动化转换：
输入结构：    [NP₁ Aux V NP₂]VP
输出结构：    [NP₂ Aux be V+en (by NP₁)]VP

示例：
      VP                    VP
     /  \                  /  |  \
    V    NP              be   V'  PP
    |    |               |    |    \
  love  Mary    →       is  loved  by John
   /
 John
```

## 14.1.2.3 扩展标准理论（EST）

### 14.1.2.3.1 X-bar理论

X-bar理论提供了一种跨范畴的句法结构统一描述。

> **定义 14.1.2.3**（X-bar 模式）
> X-bar 理论规定以下层级结构：
>
> - X⁰（词汇项/词头）
> - X'（X-bar，中间投射）
> - XP（X-phrase，最大投射）

```
      XP                   X'
     /  \                 /  \
    Spec X'      或      X'   YP (Adjunct)
         / \            /  \
        X   ZP         X    WP (Complement)
        |              |
       X⁰             X⁰ (Head)
```

**X-bar 模式**：

```
XP → (YP) X'      [Specifier]
X' → X' (ZP)      [Adjunct递归]
X' → X⁰ (WP)      [Complement]
```

```haskell
-- X-bar结构的Haskell表示
data XBar a = X0 a                          -- 词头
            | XBar (XBar a) (Maybe (XBar a))  -- X' = X' + Adjunct 或 X' = X⁰ + Comp
            | XP (Maybe (XBar a)) (XBar a)    -- XP = Spec + X'
            deriving (Show)

data HeadType = N | V | A | P | T | C | v deriving (Eq, Show)

data XBarNode = XBarNode {
    xbHeadType :: HeadType,
    xbFeatures :: [Feature],
    xbLabel    :: String
} deriving (Show)

-- 构建X-bar树
buildXBar :: HeadType -> String -> Maybe (XBar XBarNode) -> XBar XBarNode
buildXBar ht label comp =
    let head = X0 (XBarNode ht [] label)
    in case comp of
        Nothing -> XP Nothing (XBar head Nothing)
        Just c  -> XP Nothing (XBar head (Just c))

-- 添加标定语
addSpecifier :: XBar XBarNode -> XBar XBarNode -> XBar XBarNode
addSpecifier (XP _ x') spec = XP (Just spec) x'
addSpecifier xbar _ = xbar
```

## 14.1.2.4 管辖与约束理论（GB Theory）

### 14.1.2.4.1 理论框架

GB理论（Chomsky, 1981）将语法分为四个子理论：

```
┌─────────────────────────────────────────┐
│           管辖与约束理论                │
├─────────────────────────────────────────┤
│  1. X-bar理论  - 短语结构               │
│  2. θ-理论    - 题元角色分配            │
│  3. 格理论    - 格位分配                │
│  4. 约束理论  - 照应语约束              │
│  5. 控制理论  - PRO的控制               │
│  6. 界限理论  - 移位界限                │
└─────────────────────────────────────────┘
```

### 14.1.2.4.2 D-结构与S-结构

> **定义 14.1.2.4**（D-结构与S-结构）
>
> - **D-结构**（深层结构）：词汇项插入后的初始结构，所有题元角色在此分配
> - **S-结构**（表层结构）：应用移位转换后的结构

```
D-结构:                          S-结构:
    TP                              TP
   /  \                            /  \
  DP   T'                         DP   T'
  |    |                          |    |
 John  T   vP                    who  T'   vP
       |   / \                        |    / \
      will DP  v'                    will  DP  v'
           |    \                          |    \
         who   v   VP                    t₁   v   VP
               |    \                         |    \
              v⁰    V   DP                   v⁰    V  DP
               |    |    |                    |    |   |
              read read  t₁                  read read  the book
                     |
                  the book

"Who will read the book?"
```

### 14.1.2.4.3 θ-理论

> **定义 14.1.2.5**（θ-准则 θ-Criterion）
> 每个论元角色必须且只能分配给一个论元，每个论元必须且只能接收一个论元角色。
>
> ∀x[Argument(x) → ∃!θ(θ(x))] ∧ ∀θ∃!x(θ(x))

**题元角色类型**：

- **施事（Agent）**：动作发出者
- **受事（Patient/Theme）**：动作承受者
- **经历者（Experiencer）**：心理状态主体
- **目标（Goal）**：动作的终点
- **来源（Source）**：动作的起点
- **工具（Instrument）**：动作使用的工具

```haskell
-- θ-理论的形式化
data ThetaRole = Agent | Patient | Theme | Experiencer | Goal | Source |
                 Instrument | Location | Benefactive
               deriving (Eq, Show)

data Argument = Argument {
    argDP :: String,
    argTheta :: Maybe ThetaRole,
    argPosition :: Position
} deriving (Show)

data Position = Internal  -- 动词内部位置（如VP内部）
              | External  -- 动词外部位置（如Spec-vP）
              | Adjunct   -- 附加语位置
              deriving (Eq, Show)

-- 动词题元结构
data ThetaGrid = ThetaGrid {
    externalArg :: Maybe ThetaRole,  -- 外部论元
    internalArgs :: [ThetaRole]      -- 内部论元（按顺序）
} deriving (Show)

-- 动词词典条目
transitiveVerb :: ThetaGrid
transitiveVerb = ThetaGrid (Just Agent) [Patient]  -- kill: <Agent, Patient>

intransitiveVerb :: ThetaGrid
intransitiveVerb = ThetaGrid (Just Agent) []       -- run: <Agent>

unaccusativeVerb :: ThetaGrid
unaccusativeVerb = ThetaGrid Nothing [Theme]       -- arrive: <Theme>
```

### 14.1.2.4.4 格理论

> **定义 14.1.2.6**（格过滤条件 Case Filter）
> 所有显式DP必须被赋予格位（抽象格）。
> *DP → DP 当且仅当 DP 被赋予了格位

**格位分配规则**：

- **主格**：由finite T（时态）赋予Spec-TP位置的DP
- **宾格**：由及物动词v赋予Compl-V位置的DP
- **属格**：由D（限定词）赋予Spec-DP位置的DP
- **与格**：由介词P赋予Compl-P位置的DP

```
        TP
       /  \
   [Nom]   T'
      DP₁  |
      |   T⁰   vP
     John will   |
              [Acc] v'
                 DP₂  v⁰
                 |    |
                the  v⁰   VP
                book      |
                        [Acc] V'
                           DP₃
                           |
                          Mary

格位分配：
- John: 主格（由will赋予）
- the book: 宾格（由v⁰赋予）
- Mary: 宾格（由kill赋予）
```

### 14.1.2.4.5 约束理论

> **定义 14.1.2.7**（约束理论）
> 根据照应语的性质定义约束域：
>
> - **A类照应语**（reflexives如himself）：必须在最小程序域内被约束
> - **B类照应语**（pronouns如him）：必须在最小程序域内自由
> - **C类照应语**（R-expressions如John）：必须完全自由

```
约束域定义：
α 是 β 的管辖范畴（governing category）当且仅当：
1. α 是包含 β, β 的管辖语和 β 的可及主语的最小XP

约束原则：
A: 照应语必须在管辖范畴内被A-约束
B: 代词必须在管辖范畴内自由
C: R-表达式必须完全自由
```

```prolog
% Prolog: 约束理论检查

% 照应语类别
anaphora_type(himself, a).
anaphora_type(herself, a).
anaphora_type(themselves, a).
anaphora_type(him, b).
anaphora_type(her, b).
anaphora_type(them, b).
anaphora_type(he, b).
anaphora_type(she, b).
anaphora_type(they, b).

% 检查是否A-约束（被c-command的先行语约束）
c_commands(X, Y, Tree) :-
    node_dominates(X, Z, Tree),
    node_sisters(Z, Y, Tree).

% 约束原则A：照应语必须在管辖范畴内有约束语
principle_a(Expression, Tree) :-
    anaphora_type(Expression, a),
    governing_category(Expression, GC, Tree),
    binder(Expression, B, GC),
    c_commands(B, Expression, GC).

% 约束原则B：代词在管辖范畴内自由
principle_b(Expression, Tree) :-
    anaphora_type(Expression, b),
    governing_category(Expression, GC, Tree),
    not((binder(Expression, B, GC), c_commands(B, Expression, GC))).

% 约束原则C：R-表达式完全自由
principle_c(Expression, Tree) :-
    not(anaphora_type(Expression, _)),
    not((binder(Expression, B, Tree), c_commands(B, Expression, Tree))).
```

## 14.1.2.5 最简方案（Minimalist Program）

### 14.1.2.5.1 核心概念

最简方案（Chomsky, 1995）是对GB理论的精简和重构，追求理论的最小性。

> **定义 14.1.2.8**（合并操作 Merge）
> 合并是句法的基本操作：
>
> - **外部合并**：将两个句法对象 α 和 β 合并为新对象 γ = {α, β}
> - **内部合并**（移位）：将已存在于句法树中的成分 α 重新合并到新位置

### 14.1.2.5.2 特征核查理论

> **定义 14.1.2.9**（特征核查）
> 句法操作由形式特征驱动：
>
> - **可解释特征**（如语义特征）：在接口层面有意义
> - **不可解释特征**（如EPP特征、φ特征）：必须在句法推导中被核查删除

```haskell
-- 最简方案的Haskell实现
data FeatureVal = Plus | Minus | Val String deriving (Eq, Show)

data FormalFeature = FF {
    ffName :: String,
    ffValue :: FeatureVal,
    ffInterpretable :: Bool,
    ffStrong :: Bool  -- 强特征触发显性移位
} deriving (Show)

data SO = SO {
    soLabel :: String,
    soFeatures :: [FormalFeature],
    soPhonetic :: String
} deriving (Show)

-- 句法对象可以是词汇项或合并结果
data SyntaxObject = Lexical SO
                  | Merged SyntaxObject SyntaxObject  -- {α, β}
                  | Moved SyntaxObject Int           -- 移位痕迹（原始位置深度）
                  deriving (Show)

-- 合并操作
merge :: SyntaxObject -> SyntaxObject -> SyntaxObject
merge alpha beta = Merged alpha beta

-- 内部合并（移位）
internalMerge :: SyntaxObject -> Int -> SyntaxObject
internalMerge obj depth = Moved obj depth

-- 特征核查
agree :: SyntaxObject -> SyntaxObject -> Maybe SyntaxObject
agree probe goal =
    let pFeatures = getUninterpretable probe
        gFeatures = getInterpretable goal
    in if canAgree pFeatures gFeatures
       then Just (deleteFeatures probe pFeatures)
       else Nothing

getUninterpretable :: SyntaxObject -> [FormalFeature]
getUninterpretable (Lexical so) = filter (not . ffInterpretable) (soFeatures so)
getUninterpretable (Merged a b) = getUninterpretable a ++ getUninterpretable b
getUninterpretable (Moved obj _) = getUninterpretable obj

getInterpretable :: SyntaxObject -> [FormalFeature]
getInterpretable (Lexical so) = filter ffInterpretable (soFeatures so)
getInterpretable _ = []

canAgree :: [FormalFeature] -> [FormalFeature] -> Bool
canAgree [] _ = True
canAgree (p:ps) goals = any (matchFeature p) goals && canAgree ps goals

matchFeature :: FormalFeature -> FormalFeature -> Bool
matchFeature (FF n1 v1 _ _) (FF n2 v2 _ _) = n1 == n2 && v1 == v2

deleteFeatures :: SyntaxObject -> [FormalFeature] -> SyntaxObject
deleteFeatures (Lexical so) toDelete =
    Lexical (so { soFeatures = soFeatures so \\ toDelete })
deleteFeatures obj _ = obj
```

### 14.1.2.5.3 相位理论（Phase Theory）

> **定义 14.1.2.10**（相位）
> 相位是句法推导中的完整命题性结构：
>
> - CP（标句短语）是命题性相位
> - vP（轻动词短语）是及物性相位

**相位不可穿透性条件（PIC）**：
一旦相位完成，相位边缘之外的成分无法访问相位内部的补语位置。

```
相位结构：

        CP
       /  \
   (Force) C'
           /  \
          C⁰   TP  ← 相位边缘（可访问）
               /  \
              DP   T'
                   |
                  T⁰   vP  ← v*P相位
                       /  \
                   (Exp)  v'
                          /  \
                         v⁰   VP
                              |
                            (Th)

移位必须经由相位边缘（Spec-CP, Spec-vP）
```

### 14.1.2.5.4 推导示例

```
"What will John read?"

步骤1：构造VP
  V⁰ = read, 选择 DP[wh] 作为内论元
  VP = {read, what}

步骤2：构造v*P
  v* 选择 VP 作为补语，携带 [uφ] [uCase] [EPP]
  v*P = {v*, {read, what}}

步骤3：合并主语
  John 合并到Spec-v*P
  v*P = {John, {v*, {read, what}}}
  特征核查: v*[uφ] ↔ John[iφ], v*[uCase] → John[Acc]

步骤4：构造TP
  T[will] 选择 v*P，携带 [uφ] [EPP]
  特征核查: T[uφ] ↔ John[iφ]
  John 移位到Spec-TP（EPP驱动）
  TP = {John, {will, {t, {v*, {read, what}}}}}

步骤5：what移位到Spec-CP（wh特征驱动）
  CP = {what, {C, {will, {John, {t, {v*, {read, t}}}}}}}

最终结构:
        CP
       /  \
   what   C'
          /  \
         C⁰   TP
              /  \
           John   T'
                  /  \
                will  vP
                      /  \
                     t₁   v'
                          /  \
                         v⁰   VP
                              /  \
                           read  t₂
```
