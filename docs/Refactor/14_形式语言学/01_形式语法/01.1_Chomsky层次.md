## 14.1.1.1 引言

Chomsky层次（Chomsky Hierarchy）是Noam Chomsky在1956年提出的形式语言分类体系，将形式语言按照生成能力分为四个层次。
该理论奠定了形式语言学的基础，对计算机科学、语言学和认知科学产生了深远影响。

> **定义 14.1.1.1**（形式文法）
> 一个形式文法 G 是一个四元组 G = (V, Σ, R, S)，其中：
>
> - V：非终结符的有限集合
> - Σ：终结符的有限集合，V ∩ Σ = ∅
> - R：重写规则的有限集合，形式为 α → β，其中 α ∈ (V ∪ Σ)_V(V ∪ Σ)_，β ∈ (V ∪ Σ)*
> - S ∈ V：起始符号

## 14.1.1.2 正则语言（Type-3）

### 14.1.1.2.1 定义

> **定义 14.1.1.2**（正则文法）
> 正则文法（右线性文法）是满足以下条件的文法 G = (V, Σ, R, S)：
> 所有规则形如 A → aB 或 A → a，其中 A, B ∈ V，a ∈ Σ ∪ {ε}

**定理 14.1.1.1**（正则语言的封闭性）
正则语言类在并、交、补、连接、Kleene闭包运算下封闭。

**证明**：
设 L₁, L₂ 为正则语言，分别由DFA M₁ = (Q₁, Σ, δ₁, q₀₁, F₁) 和 M₂ = (Q₂, Σ, δ₂, q₀₂, F₂) 识别。

**并运算**：构造 M_∪ = (Q₁ × Q₂, Σ, δ, (q₀₁, q₀₂), F₁ × Q₂ ∪ Q₁ × F₂)
其中 δ((p, q), a) = (δ₁(p, a), δ₂(q, a))

**补运算**：构造 M_¬ = (Q₁, Σ, δ₁, q₀₁, Q₁ - F₁)

### 14.1.1.2.2 有限自动机

> **定义 14.1.1.3**（确定性有限自动机 DFA）
> DFA 是一个五元组 M = (Q, Σ, δ, q₀, F)，其中：
>
> - Q：状态的有限集合
> - Σ：输入字母表
> - δ: Q × Σ → Q：转移函数
> - q₀ ∈ Q：初始状态
> - F ⊆ Q：接受状态集合

```haskell
-- DFA 的 Haskell 实现
data DFA state symbol = DFA {
    states     :: [state],
    alphabet   :: [symbol],
    transition :: state -> symbol -> state,
    startState :: state,
    acceptStates :: [state]
}

-- 运行DFA接受字符串
runDFA :: Eq state => DFA state symbol -> [symbol] -> Bool
runDFA dfa input = finalState `elem` acceptStates dfa
  where
    finalState = foldl (transition dfa) (startState dfa) input

-- 示例：识别以"ab"结尾的DFA
data ABState = Q0 | Q1 | Q2 deriving (Eq, Show)

transitionAB :: ABState -> Char -> ABState
transitionAB Q0 'a' = Q1
transitionAB Q0 _   = Q0
transitionAB Q1 'b' = Q2
transitionAB Q1 'a' = Q1
transitionAB Q1 _   = Q0
transitionAB Q2 'a' = Q1
transitionAB Q2 _   = Q0

dfaAB :: DFA ABState Char
dfaAB = DFA [Q0, Q1, Q2] ['a', 'b'] transitionAB Q0 [Q2]
```

### 14.1.1.2.3 Pumping Lemma

**定理 14.1.1.2**（正则语言的Pumping Lemma）
对于任何正则语言 L，存在常数 p（泵长度），使得对于任何字符串 w ∈ L 且 |w| ≥ p，w 可分解为 w = xyz，满足：

1. |xy| ≤ p
2. |y| > 0
3. ∀i ≥ 0: xyⁱz ∈ L

**证明**：设 L 被具有 n 个状态的DFA M 接受。对 |w| ≥ n，根据鸽巢原理，M 在读取 w 时必重复访问某状态。令 y 为两次访问之间的子串，则泵作用保持接受状态。

```prolog
% Prolog: 检查正则表达式匹配
% 简单正则表达式匹配器

% 基本正则表达式类型
% re_char(C) - 匹配字符C
% re_concat(RE1, RE2) - 连接
% re_union(RE1, RE2) - 并
% re_star(RE) - Kleene星号

matches(re_char(C), [C]).
matches(re_concat(RE1, RE2), S) :-
    append(S1, S2, S),
    matches(RE1, S1),
    matches(RE2, S2).
matches(re_union(RE1, _), S) :- matches(RE1, S).
matches(re_union(_, RE2), S) :- matches(RE2, S).
matches(re_star(_), []).
matches(re_star(RE), S) :-
    append(S1, S2, S),
    S1 \= [],
    matches(RE, S1),
    matches(re_star(RE), S2).
```

## 14.1.1.3 上下文无关语言（Type-2）

### 14.1.1.3.1 定义

> **定义 14.1.1.4**（上下文无关文法 CFG）
> CFG 是满足以下条件的文法 G = (V, Σ, R, S)：
> 所有规则形如 A → α，其中 A ∈ V，α ∈ (V ∪ Σ)*

**示例**：英语句子结构的CFG片段

```
S → NP VP
NP → Det N | NP PP | ProperN
VP → V NP | V NP PP | V PP
PP → P NP
Det → 'the' | 'a'
N → 'man' | 'woman' | 'book'
V → 'reads' | 'loves'
P → 'in' | 'on'
ProperN → 'John' | 'Mary'
```

### 14.1.1.3.2 下推自动机

> **定义 14.1.1.5**（下推自动机 PDA）
> PDA 是一个七元组 M = (Q, Σ, Γ, δ, q₀, Z₀, F)，其中：
>
> - Q：状态集合
> - Σ：输入字母表
> - Γ：栈字母表
> - δ: Q × (Σ ∪ {ε}) × Γ → P(Q × Γ*)：转移函数
> - q₀ ∈ Q：初始状态
> - Z₀ ∈ Γ：初始栈符号
> - F ⊆ Q：接受状态集合

**定理 14.1.1.3**（CFG与PDA的等价性）
语言 L 是上下文无关的当且仅当存在PDA M 使得 L = L(M)。

```haskell
-- PDA的Haskell实现
data PDA state input stack = PDA {
    pdaStates      :: [state],
    pdaInputAlphabet :: [input],
    pdaStackAlphabet :: [stack],
    pdaTransition  :: state -> Maybe input -> stack -> [(state, [stack])],
    pdaStartState  :: state,
    pdaStartStack  :: stack,
    pdaAcceptStates :: [state]
}

-- PDA配置
data Config state input stack = Config {
    configState  :: state,
    configInput  :: [input],
    configStack  :: [stack]
} deriving (Show)

-- 单步转移
stepPDA :: Eq input => PDA state input stack -> Config state input stack
        -> [Config state input stack]
stepPDA pda (Config q (i:is) (s:ss)) =
    [(Config q' is (s' ++ ss)) | (q', s') <- pdaTransition pda (Just i) s]
    ++ [(Config q' (i:is) (s' ++ ss)) | (q', s') <- pdaTransition pda Nothing s]
stepPDA pda (Config q [] (s:ss)) =
    [(Config q' [] (s' ++ ss)) | (q', s') <- pdaTransition pda Nothing s]
stepPDA _ (Config _ _ []) = []
```

### 14.1.1.3.3 Pumping Lemma for CFL

**定理 14.1.1.4**（上下文无关语言的Pumping Lemma）
对于任何CFL L，存在常数 p，使得对于任何字符串 w ∈ L 且 |w| ≥ p，w 可分解为 w = uvxyz，满足：

1. |vxy| ≤ p
2. |vy| > 0
3. ∀i ≥ 0: uvⁱxyⁱz ∈ L

### 14.1.1.3.4 语法树与歧义性

```
        S                      S
       / \                    / \
      NP  VP                 NP  VP
      |   / \                |   |
     John V  NP              John V'
         |   / \                 |
       reads Det N            reads
             |   |              |
             a  book            NP
                                 / \
                               Det  N
                                |   |
                                a  book
```

> **定义 14.1.1.6**（固有歧义性）
> 语言 L 是固有歧义的，如果所有生成 L 的CFG都是歧义的。

## 14.1.1.4 上下文有关语言（Type-1）

### 14.1.1.4.1 定义

> **定义 14.1.1.7**（上下文有关文法 CSG）
> CSG 是满足以下条件的文法 G = (V, Σ, R, S)：
> 所有规则形如 αAβ → αγβ，其中 A ∈ V，α, β ∈ (V ∪ Σ)*，γ ∈ (V ∪ Σ)+
> 且 |αAβ| ≤ |αγβ|（单调性条件）

**定理 14.1.1.5**（Kuroda范式）
任何CSL都可由仅含以下形式规则的文法生成：

- AB → CD
- A → BC
- A → a

### 14.1.1.4.2 线性有界自动机

> **定义 14.1.1.8**（线性有界自动机 LBA）
> LBA 是一个受限的图灵机，其读写头只能在输入字符串占据的磁带区域内移动。

**定理 14.1.1.6**（CSL与LBA的等价性）
语言 L 是上下文有关的当且仅当存在LBA M 使得 L = L(M)。

```haskell
-- LBA的抽象表示
data LBA state symbol = LBA {
    lbaStates      :: [state],
    lbaAlphabet    :: [symbol],
    lbaTapeAlphabet :: [symbol],
    lbaTransition  :: state -> symbol -> [(state, symbol, Direction)],
    lbaStartState  :: state,
    lbaAcceptStates :: [state]
} deriving (Show)

data Direction = L | R | S deriving (Show)

-- LBA配置：磁带内容 + 读写头位置 + 当前状态
data LBAConfig state symbol = LBAConfig {
    leftTape   :: [symbol],    -- 反转的左半部分
    headSymbol :: symbol,      -- 当前符号
    rightTape  :: [symbol],    -- 右半部分
    lbaState   :: state
} deriving (Show)
```

## 14.1.1.5 递归可枚举语言（Type-0）

### 14.1.1.5.1 无限制文法

> **定义 14.1.1.9**（无限制文法 / 短语结构文法）
> 无限制文法 G = (V, Σ, R, S) 对规则 α → β 无限制，仅要求 α 至少包含一个非终结符。

**定理 14.1.1.7**（Chomsky, 1956）
语言 L 是递归可枚举的当且仅当存在无限制文法 G 使得 L = L(G)。

### 14.1.1.5.2 图灵机

> **定义 14.1.1.10**（图灵机）
> 图灵机 M = (Q, Σ, Γ, δ, q₀, B, F)，其中：
>
> - Q：状态的有限集合
> - Σ：输入字母表
> - Γ ⊇ Σ：磁带字母表
> - δ: Q × Γ → Q × Γ × {L, R}：转移函数（部分函数）
> - q₀ ∈ Q：初始状态
> - B ∈ Γ - Σ：空白符号
> - F ⊆ Q：接受状态集合

```haskell
-- 图灵机的Haskell实现
data TM state symbol = TM {
    tmStates       :: [state],
    tmInputAlphabet :: [symbol],
    tmTapeAlphabet  :: [symbol],
    tmTransition   :: state -> symbol -> Maybe (state, symbol, Direction),
    tmStartState   :: state,
    tmBlank        :: symbol,
    tmAcceptStates :: [state]
}

-- 图灵机配置
data TMConfig state symbol = TMConfig {
    tmLeft  :: [symbol],   -- 左半部分（反转）
    tmHead  :: symbol,     -- 当前符号
    tmRight :: [symbol],   -- 右半部分
    tmState :: state
} deriving (Show)

-- 单步执行
tmStep :: TM state symbol -> TMConfig state symbol -> Maybe (TMConfig state symbol)
tmStep tm config = do
    (newState, newSym, dir) <- tmTransition tm (tmState config) (tmHead config)
    let (newLeft, newHead, newRight) = case dir of
            L -> case tmLeft config of
                []     -> ([], tmBlank tm, newSym : tmRight config)
                (l:ls) -> (ls, l, newSym : tmRight config)
            R -> case tmRight config of
                []     -> (newSym : tmLeft config, tmBlank tm, [])
                (r:rs) -> (newSym : tmLeft config, r, rs)
            S -> (tmLeft config, newSym, tmRight config)
    return $ TMConfig newLeft newHead newRight newState

-- 执行直到停机
tmRun :: TM state symbol -> TMConfig state symbol -> Maybe (TMConfig state symbol)
tmRun tm config = case tmStep tm config of
    Nothing  -> Just config
    Just cfg -> tmRun tm cfg
```

## 14.1.1.6 Chomsky层次的包含关系

```
┌─────────────────────────────────────────┐
│      递归可枚举 (Type-0)                │
│   ┌─────────────────────────────────┐   │
│   │     上下文有关 (Type-1)          │   │
│   │   ┌─────────────────────────┐   │   │
│   │   │   上下文无关 (Type-2)    │   │   │
│   │   │   ┌─────────────────┐   │   │   │
│   │   │   │   正则 (Type-3)  │   │   │   │
│   │   │   │   {aⁿbⁿ | n≥0}  │   │   │   │
│   │   │   │   {ww | w∈Σ*}   │   │   │   │
│   │   │   │   {aⁿbⁿcⁿ}      │   │   │   │
│   │   │   └─────────────────┘   │   │   │
│   │   └─────────────────────────┘   │   │
│   └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

**定理 14.1.1.8**（真包含层次）

1. 正则语言 ⊂ 上下文无关语言 ⊂ 上下文有关语言 ⊂ 递归可枚举语言
2. 上述所有包含关系都是真包含

**证明要点**：

- L = {aⁿbⁿ | n ≥ 0} 是CFL但不是正则的
- L = {aⁿbⁿcⁿ | n ≥ 0} 是CSL但不是CFL
- 停机问题是递归可枚举但非递归的

## 14.1.1.7 语言学意义

Chomsky层次对自然语言分析有深远影响：

1. **句法分析**：自然语言大多可用CFG近似描述
2. **语言能力**：人类语言的生成能力对应哪个层次是持续研究的问题
3. **计算复杂性**：语言层次与识别复杂度直接相关

| 层次 | 文法类型 | 自动机 | 时间复杂度 |
|------|----------|--------|------------|
| Type-3 | 正则 | DFA/NFA | O(n) |
| Type-2 | 上下文无关 | PDA | O(n³) |
| Type-1 | 上下文有关 | LBA | 不确定 |
| Type-0 | 无限制 | 图灵机 | 不可判定 |
