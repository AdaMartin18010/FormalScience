## 14.1.3.1 引言

范畴语法（Categorial Grammar）是一种词汇驱动的形式语法理论，起源于Ajdulkiewicz（1935）和Bar-Hillel（1953），后经Lambek（1958）发展为**Lambek演算**，并进一步扩展为**组合范畴语法**（Combinatory Categorial Grammar, CCG）。

> **核心思想**：句法范畴通过函数式组合来构建，每个词汇项被赋予一个范畴类型，句子通过函数应用规则组合而成。

## 14.1.3.2 基本范畴语法（AB Grammar）

### 14.1.3.2.1 范畴类型定义

> **定义 14.1.3.1**（基本范畴类型）
> 基本范畴集合包含：
>
> - 原子范畴：S（句子）、N（名词）、NP（名词短语）等
> - 函数范畴：X/Y（向右寻找Y以形成X）、X\\Y（向左寻找Y以形成X）

**语法**：

```
A, B ::= S | N | NP | PP | ...    ; 原子范畴
       | A/B | A\B                ; 函数范畴
```

### 14.1.3.2.2 函数应用规则

> **定义 14.1.3.2**（应用规则）
>
> - **前向应用**（>）：X/Y  Y → X
> - **后向应用**（<）：Y  X\\Y → X

```
前向应用 (>):
    X/Y    Y
      \    /
         X

后向应用 (<):
    Y    X\Y
      \    /
         X
```

**示例分析**："John sleeps"

```
John: NP
sleeps: S\\NP

      John       sleeps
       |           |
      NP        S\NP
        \       /
           S

应用规则：<规则（后向应用）
结果: S（完整句子）
```

```haskell
-- 范畴语法的Haskell实现
data Cat = S | NP | N | PP | AP | AdvP
         | Slash Cat Cat       -- X/Y
         | Backslash Cat Cat   -- X\Y
         deriving (Eq, Show)

-- 词汇项
data LexicalItem = LI {
    form :: String,
    category :: Cat
} deriving (Show)

-- 语法表达式（带有范畴标注的短语）
data Expr = Word LexicalItem
          | App Expr Expr Cat  -- 函数应用结果
          deriving (Show)

-- 前向应用：X/Y + Y → X
forwardApp :: Expr -> Expr -> Maybe Expr
forwardApp f a = case getCat f of
    Slash x y | getCat a == y -> Just (App f a x)
    _ -> Nothing

-- 后向应用：Y + X\Y → X
backwardApp :: Expr -> Expr -> Maybe Expr
backwardApp a f = case getCat f of
    Backslash x y | getCat a == y -> Just (App a f x)
    _ -> Nothing

getCat :: Expr -> Cat
getCat (Word li) = category li
getCat (App _ _ c) = c

-- 示例词汇
john :: LexicalItem
john = LI "John" NP

sleeps :: LexicalItem
sleeps = LI "sleeps" (Backslash S NP)

loves :: LexicalItem
loves = LI "loves" (Backslash (Slash S NP) NP)

mary :: LexicalItem
mary = LI "Mary" NP

-- "John loves Mary"
exampleSentence :: Maybe Expr
exampleSentence = do
    lovesMary <- forwardApp (Word loves) (Word mary)  -- loves Mary : S\NP
    backwardApp (Word john) lovesMary                 -- John (loves Mary) : S
```

## 14.1.3.3 Lambek演算

### 14.1.3.3.1 演算系统

Lambek演算（1958）提供了一个逻辑基础，将范畴语法与**直觉主义命题逻辑**相关联。

> **定义 14.1.3.3**（Lambek演算 L）
> 一个相继式演算，包含：
>
> - **恒等公理**：A → A
> - **右蕴涵引入**（/R）：若 Γ, A → B，则 Γ → B/A
> - **左蕴涵引入**（/L）：若 Γ → A 且 Δ, B, Δ' → C，则 Δ, B/A, Γ, Δ' → C
> - **左后向蕴涵**（\L）：若 Γ → A 且 Δ, B, Δ' → C，则 Δ, Γ, A\\B, Δ' → C
> - **右后向蕴涵**（\\R）：若 A, Γ → B，则 Γ → A\\B
> - **结合律**：省略（Lambek演算非结合）或显式处理

### 14.1.3.3.2 相继式演算

```
公理:
    ---- (Id)
    A → A

右规则 (/R):
    Γ, A → B
    ---------
    Γ → B/A

右规则 (\R):
    A, Γ → B
    ---------
    Γ → A\B

左规则 (/L):
    Γ → A    Δ, B, Δ' → C
    ----------------------
    Δ, B/A, Γ, Δ' → C

左规则 (\L):
    Γ → A    Δ, B, Δ' → C
    ----------------------
    Δ, Γ, A\B, Δ' → C

切规则 (Cut):
    Γ → A    Δ, A, Δ' → B
    ----------------------
    Δ, Γ, Δ' → B
```

**示例证明**：推导 "John loves Mary"

```
目标：证明 NP, (S\NP)/NP, NP → S

1.  NP → NP          (Id)
2.  S → S            (Id)
3.  NP → NP          (Id)
4.  NP, S\NP → S     (<规则，需要证明)

子证明：验证 loves 的范畴
    NP → NP    S → S
    ----------------- (/L with empty Δ, Δ')
    (S\NP)/NP, NP → S\NP

完整推导：
    NP → NP    S → S    NP → NP
    ---------------------------
    (S\NP)/NP, NP → S\NP    (右应用规则变体)

    NP → NP    (S\NP)/NP, NP → S\NP
    -------------------------------
    NP, (S\NP)/NP, NP → S          (Cut + <规则)
```

```prolog
% Prolog: Lambek演算定理证明器

% 相继式：sequent(Gamma, A) 表示 Gamma |- A

% 公理
prove_sequent([A], A) :- atomic(A), !.

% 右规则 /R: Gamma, A |- B  =>  Gamma |- B/A
prove_sequent(Gamma, slash(B, A)) :-
    append(GammaPrefix, [A], GammaSuffix),
    prove_sequent(GammaSuffix, B).

% 右规则 \R: A, Gamma |- B  =>  Gamma |- A\B
prove_sequent(Gamma, backslash(A, B)) :-
    append([A], GammaPrefix, GammaSuffix),
    prove_sequent(GammaPrefix, B).

% 左规则 /L: Gamma |- A, Delta, B, Delta' |- C
%           => Delta, B/A, Gamma, Delta' |- C
prove_sequent(GammaResult, C) :-
    select(slash(B, A), GammaResult, GammaRest),
    append(Delta, [slash(B,A)|GammaDeltaPrime], GammaResult),
    append(Gamma, DeltaPrime, GammaDeltaPrime),
    prove_sequent(Gamma, A),
    append(Delta, [B|DeltaPrime], GammaMid),
    prove_sequent(GammaMid, C).

% 左规则 \L: Gamma |- A, Delta, B, Delta' |- C
%           => Delta, Gamma, A\B, Delta' |- C
prove_sequent(GammaResult, C) :-
    select(backslash(A, B), GammaResult, GammaRest),
    append(Delta, GammaABDeltaPrime, GammaResult),
    append(Gamma, [backslash(A,B)|DeltaPrime], GammaABDeltaPrime),
    prove_sequent(Gamma, A),
    append(Delta, [B|DeltaPrime], GammaMid),
    prove_sequent(GammaMid, C).
```

### 14.1.3.3.3 无割性质

**定理 14.1.3.1**（Lambek演算的无割性质）
Lambek演算具有无割性质：任何可证明的相继式都可以不使用切规则证明。

**证明概要**：通过Gentzen式的割消算法，展示所有使用切规则的证明都可以转化为无切证明。

### 14.1.3.3.4 Curry-Howard对应

> **定理 14.1.3.2**（Curry-Howard对应）
> Lambek演算与简单类型λ演算存在对应关系：
>
> - 命题 ↔ 类型
> - 证明 ↔ λ项
> - 证明归约 ↔ β归约

```
范畴      λ演算类型
--------------------------
A/B      B → A    (函数类型)
A\B      B → A    (函数类型)
```

```haskell
-- Curry-Howard对应的Haskell实现
-- 将Lambek演算证明翻译为λ项

data LType = Atom String
           | LImp LType LType  -- B → A 对应 A/B 或 A\B
           deriving (Eq, Show)

data LTerm = Var String
           | Lam String LType LTerm
           | AppT LTerm LTerm
           deriving (Show)

-- 翻译函数：从范畴到类型
toLambdaType :: Cat -> LType
toLambdaType S = Atom "S"
toLambdaType NP = Atom "NP"
toLambdaType N = Atom "N"
toLambdaType (Slash a b) = LImp (toLambdaType b) (toLambdaType a)
toLambdaType (Backslash a b) = LImp (toLambdaType b) (toLambdaType a)

-- 语义解释：每个词汇项对应一个λ项
semantics :: LexicalItem -> LTerm
semantics (LI "John" NP) = Var "john"
semantics (LI "Mary" NP) = Var "mary"
semantics (LI "sleeps" (Backslash S NP)) =
    Lam "x" (Atom "NP") (AppT (Var "sleep") (Var "x"))
semantics (LI "loves" (Backslash (Slash S NP) NP)) =
    Lam "y" (Atom "NP") (Lam "x" (Atom "NP")
        (AppT (AppT (Var "love") (Var "x")) (Var "y")))
semantics _ = Var "unknown"
```

## 14.1.3.4 组合范畴语法（CCG）

### 14.1.3.4.1 扩展规则

组合范畴语法（Steedman, 1985, 2000）扩展了Lambek演算，增加了**组合规则**。

> **定义 14.1.3.4**（CCG基本规则）
>
> - **函数应用**（>，<）：X/Y Y → X；Y X\\Y → X
> - **前向组合**（>B）：X/Y Y/Z → X/Z
> - **后向组合**（<B）：Y\\Z X\\Y → X\\Z
> - **前向置换**（>S）：(X/Y)/Z Y/Z → X/Z
> - **后向置换**（<S）：Y\\Z (X\\Y)\\Z → X\\Z
> - **类型提升**（>T，<T）：X → T/(T\\X)；X → T\\(T/X)

### 14.1.3.4.2 组合规则的形式化

```
前向组合 (>B):
    X/Y    Y/Z
      \    /
       X/Z

后向组合 (<B):
    Y\Z    X\Y
      \    /
       X\Z

前向置换 (>S):
    (X/Y)/Z    Y/Z
         \      /
            X/Z

名词提升:
    NP → S/(S\NP)   (>T)
    NP → S\(S/NP)   (<T)
```

```haskell
-- CCG的Haskell实现
data Direction = Forward | Backward deriving (Eq, Show)

data CCGCat = AtomCat String
            | FunCat CCGCat Direction CCGCat  -- X/Y 或 X\Y
            deriving (Eq, Show)

-- 前向函数范畴构造
(/>) :: CCGCat -> CCGCat -> CCGCat
a /> b = FunCat a Forward b

-- 后向函数范畴构造
(\>) :: CCGCat -> CCGCat -> CCGCat
a \> b = FunCat a Backward b

-- CCG表达式
data CCGExpr = CCGWord String CCGCat
             | CCGApply CCGExpr CCGExpr CCGCat
             | CCGCompose CCGExpr CCGExpr CCGCat
             | CCGRaise CCGExpr CCGCat
             deriving (Show)

-- 函数应用
apply :: CCGExpr -> CCGExpr -> Maybe CCGExpr
apply f a = case (getCCGCat f, getCCGCat a) of
    (FunCat x Forward y, z) | y == z -> Just (CCGApply f a x)
    (z, FunCat x Backward y) | y == z -> Just (CCGApply a f x)
    _ -> Nothing

-- 前向组合 (>B): X/Y + Y/Z → X/Z
composeForward :: CCGExpr -> CCGExpr -> Maybe CCGExpr
composeForward f g = case (getCCGCat f, getCCGCat g) of
    (FunCat x Forward y, FunCat y' Forward z) | y == y' ->
        Just (CCGCompose f g (FunCat x Forward z))
    _ -> Nothing

-- 后向组合 (<B): Y\Z + X\Y → X\Z
composeBackward :: CCGExpr -> CCGExpr -> Maybe CCGExpr
composeBackward g f = case (getCCGCat g, getCCGCat f) of
    (FunCat y Backward z, FunCat x Backward y') | y == y' ->
        Just (CCGCompose g f (FunCat x Backward z))
    _ -> Nothing

-- 名词提升 (>T): NP → S/(S\NP)
raiseForward :: CCGExpr -> Maybe CCGExpr
raiseForward x = case getCCGCat x of
    AtomCat "NP" -> let raised = FunCat (AtomCat "S") Forward
                                   (FunCat (AtomCat "S") Backward (AtomCat "NP"))
                   in Just (CCGRaise x raised)
    _ -> Nothing

getCCGCat :: CCGExpr -> CCGCat
getCCGCat (CCGWord _ c) = c
getCCGCat (CCGApply _ _ c) = c
getCCGCat (CCGCompose _ _ c) = c
getCCGCat (CCGRaise _ c) = c
```

### 14.1.3.4.3 长距离依存分析

CCG通过组合规则优雅地处理长距离依存：

```
"John thinks Mary loves"

John: NP
thinks: (S\\NP)/S
Mary: NP
loves: (S\\NP)/NP

分析过程：
1. Mary loves: (S\\NP)/NP 的 NP → S\\NP  (应用)
2. John thinks (Mary loves):
   - NP 提升: NP → S/(S\\NP)
   - S/(S\\NP) + (S\\NP)/S → S/S  (>B组合)
   - S/S + S\\NP → 无法组合...

更简洁分析：
Mary loves: 等待主语的VP
thinks: 选择S补语

CCG组合允许：
  (S\\NP)/S    S\\NP    (通过>应用，但loves缺少宾语)

长距离依存：
"Who does John think Mary loves?"

Who: S/(S\\NP)  或  (S\\NP)\\((S\\NP)/NP)
通过类型提升和组合，宾语空缺可以被Who远程填充
```

### 14.1.3.4.4 协调性分析

CCG对协调的处理特别优雅：

```
"John sings and dances"

sings: S\\NP
dances: S\\NP
and: (X\\X)/X  （协调连词，X可为任意范畴）

分析：
  sings       and       dances
  S\NP     (X\X)/X      S\NP
             |           |
      (S\NP\S\NP)/S\NP  S\NP
             \         /
            (S\NP)\(S\NP)
                 |
              S\NP    S\NP  (<组合)
                 \    /
                   S\NP

John (sings and dances): S
```

## 14.1.3.5 证明搜索与句法分析

```haskell
-- CCG句法分析器
type ParseState = [CCGExpr]

-- 尝试所有可能的分析
parse :: [CCGExpr] -> [CCGExpr]
parse [single] = [single]
parse stack =
    let applications = [result | (f:a:rest) <- tails stack,
                                 Just result <- [apply f a]]
        compositions = [result | (f:g:rest) <- tails stack,
                                 Just result <- [composeForward f g
                                               <|> composeBackward f g]]
        nextStates = applications ++ compositions
    in concatMap (parse . (: drop 2 stack)) nextStates

-- 辅助函数
tails :: [a] -> [[a]]
tails [] = []
tails xs@(_:rest) = xs : tails rest

-- 完整解析函数
fullParse :: [String] -> [(CCGExpr, CCGCat)]
fullParse words =
    let initial = map lookupLexicon words
        results = parse initial
    in [(r, getCCGCat r) | r <- results, isComplete (getCCGCat r)]

lookupLexicon :: String -> CCGExpr
lookupLexicon w = case w of
    "John" -> CCGWord "John" (AtomCat "NP")
    "Mary" -> CCGWord "Mary" (AtomCat "NP")
    "sleeps" -> CCGWord "sleeps" (FunCat (AtomCat "S") Backward (AtomCat "NP"))
    "loves" -> CCGWord "loves" (FunCat (FunCat (AtomCat "S") Backward (AtomCat "NP"))
                                          Forward (AtomCat "NP"))
    "thinks" -> CCGWord "thinks" (FunCat (FunCat (AtomCat "S") Backward (AtomCat "NP"))
                                            Forward (AtomCat "S"))
    _ -> CCGWord w (AtomCat "UNKNOWN")

isComplete :: CCGCat -> Bool
isComplete (AtomCat "S") = True
isComplete _ = False
```

## 14.1.3.6 总结

| 特征 | AB语法 | Lambek演算 | CCG |
|------|--------|------------|-----|
| 规则复杂度 | 简单 | 中等 | 丰富 |
| 逻辑基础 | 无 | 命题逻辑 | 组合逻辑 |
| 结合性 | 非结合 | 非结合 | 自由结合 |
| 长距离依存 | 有限 | 中等 | 优秀 |
| 计算复杂度 | 低 | 中等 | 多项式 |
| 与λ演算关系 | 间接 | Curry-Howard | 直接 |
