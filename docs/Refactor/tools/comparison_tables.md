## 调度算法对比

### CPU调度算法

| 算法 | 类型 | 平均等待时间 | 响应时间 | 饥饿 | 适用场景 |
|------|------|-------------|----------|------|---------|
| **FCFS** | 非抢占 | 高 | 差 | 无 | 批处理系统 |
| **SJF** | 可抢占 | 最优* | 一般 | 可能 | 批处理，已知服务时间 |
| **RR** | 抢占 | 中 | 好 | 无 | 分时系统 |
| **优先级** | 可抢占 | 中 | 好 | 可能 | 实时系统 |
| **MLFQ** | 抢占 | 中 | 好 | 可能 | 通用交互系统 |
| **CFS** | 抢占 | 中 | 好 | 无 | Linux默认调度器 |

\* SJF在已知服务时间下平均等待时间最优

### 关键指标定义

```
周转时间 = 完成时间 - 到达时间
等待时间 = 周转时间 - 服务时间
响应时间 = 首次运行时间 - 到达时间
```

### 实时调度

| 算法 | 类别 | 可调度性条件 | 复杂度 | 特点 |
|------|------|-------------|--------|------|
| **RMS** | 静态优先级 | $U \leq n(2^{1/n}-1)$ | $O(n)$ | 周期任务，速率单调 |
| **EDF** | 动态优先级 | $U \leq 1$ | $O(\log n)$ | 最优单处理器调度 |
| **LLF** | 动态优先级 | $U \leq 1$ | $O(n)$ | 最小松弛度优先 |

**利用率界限公式**:

- RMS: $U_{lub} = n(2^{1/n} - 1) \to \ln 2 \approx 0.693$ (当 $n \to \infty$)
- EDF: $U_{lub} = 1$

## 综合决策矩阵

### 形式化方法选择指南

| 需求 | 推荐工具/方法 | 理由 |
|------|--------------|------|
| 验证复杂算法 | Lean/Coq + 提取代码 | 高保证，可提取可执行代码 |
| 验证并发协议 | TLA+ / SPIN | 擅长时序性质和状态空间探索 |
| C代码安全 | CBMC + Frama-C | 直接验证现有代码 |
| 快速类型检查 | Rust/ML | 强类型系统，良好推断 |
| 数学形式化 | Lean 4 / Isabelle | 强大的数学库，良好自动化 |
| 实时系统验证 | UPPAAL | 时序自动机支持 |
| 轻量级属性检查 | SMT (Z3/CVC5) | 快速原型，良好集成 |

### 语言特性权衡

```
安全性 ←————————————————————————→ 灵活性
Rust    Java    Go    C++    C    Assembly
        ↑ 生产率

验证成本 ←————————————————————————→ 运行效率
Agda    Haskell  Rust   C++    C
        ↑ 抽象层次
```

## 快速参考卡片

### 调度算法速选

```
交互式系统 → RR / MLFQ / CFS
批处理系统 → SJF / FCFS
实时系统   → EDF / RMS (周期性)
公平性要求 → CFS / 公平队列
```

### 并发模型速选

```
共享内存多核 → 共享内存 + 锁/RwLock
高可用分布式 → Actor模型
简洁通信   → CSP (Go风格)
高并发I/O  → 异步/事件驱动
数据并行   → GPU/向量化
```

### 证明工具速选

```
数学定理   → Lean 4 / Isabelle
软件验证   → Coq / Lean
协议验证   → TLA+ / SPIN
快速检查   → SMT求解器
C代码      → CBMC / Frama-C
```
