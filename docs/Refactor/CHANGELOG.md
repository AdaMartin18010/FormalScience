## [4.0.0] - 2025-11-14 - 重构完成

### 🎉 重大里程碑

FormalScience 项目 v4.0.0 正式发布！这是项目历史上的一个重要里程碑，标志着从内容积累阶段全面转向系统化、标准化的知识工程阶段。

### ✨ 新增内容

#### 核心框架
- **全新项目结构**：采用八目录分层架构
  - `01_数学基础` - 数学理论根基
  - `02_形式语言` - 形式化语法系统
  - `03_编程范式` - 编程方法论
  - `04_软件工程` - 工程实践体系
  - `05_形式化理论` - 理论验证框架
  - `06_调度系统` - 核心调度实现
  - `07_交叉视角` - 跨领域分析
  - `08_附录` - 参考资料集合

#### 调度系统 v4.0
- **NetworkScheduler V4** - 企业级网络调度系统
  - 多层级架构设计（核心层/管理层/执行层/策略层）
  - 支持多协议栈（HTTP/2、TCP、UDP、QUIC）
  - 自适应负载均衡算法（加权轮询、最少连接、一致性哈希）
  - 智能故障转移机制（健康检查、熔断降级、自动恢复）
  - 完整性能监控体系（延迟追踪、吞吐量统计、资源利用率）

- **SchedulerFactory V4** - 统一调度工厂
  - 插件化架构支持
  - 运行时策略热加载
  - 多租户资源隔离
  - 配额管理与限流控制

#### 工具链
- **智能文件分析器** (`smart_file_analyzer.py`)
  - 多维度内容分析
  - 智能分类建议
  - 重复内容检测
  - 知识图谱构建

- **自动化重构工具**
  - 批量文件迁移
  - 自动链接修复
  - 元数据提取

#### 文档体系
- **98个核心文档** - 覆盖所有知识领域
- **README 矩阵** - 每层目录独立README
- **交叉引用系统** - 1200+ 内部链接
- **术语词典** - 500+ 专业术语定义

### ⚠️ 破坏性变更 (Breaking Changes)

#### 目录结构调整
| 旧路径 | 新路径 | 说明 |
|--------|--------|------|
| `/数学/` | `/01_数学基础/` | 统一编号前缀 |
| `/形式语言/` | `/02_形式语言/` | 统一编号前缀 |
| `/编程/` | `/03_编程范式/` | 重命名并加前缀 |
| `/工程/` | `/04_软件工程/` | 重命名并加前缀 |
| `/理论/` | `/05_形式化理论/` | 重命名并加前缀 |
| `/调度/` | `/06_调度系统/` | 重命名并加前缀 |
| `/交叉/` | `/07_交叉视角/` | 重命名并加前缀 |
| `/附录/` | `/08_附录/` | 统一编号前缀 |

#### API 变更
```python
# 旧版本 (v3.x)
from scheduler import NetworkScheduler
scheduler = NetworkScheduler(config)

# 新版本 (v4.0)
from scheduler.v4 import NetworkScheduler, SchedulerFactory
factory = SchedulerFactory()
scheduler = factory.create_network_scheduler(config)
```

#### 配置格式变更
- 配置文件从 `.yaml` 迁移至 `.toml`
- 旧配置需使用迁移工具转换：`python tools/migrate_config.py`

## 迁移指南

### 从 v3.x 迁移至 v4.0

#### 1. 目录结构适配
```bash
# 使用自动迁移工具
python tools/migrate_v3_to_v4.py --source ./old_project --target ./new_project
```

#### 2. 配置转换
```bash
# 转换配置文件格式
python tools/migrate_config.py --input config.yaml --output config.toml
```

#### 3. 代码适配
- 更新所有 `scheduler` 导入语句
- 替换已废弃的 API 调用
- 更新单元测试用例

#### 4. 数据迁移
- 备份现有数据
- 执行数据库 schema 更新（如有）
