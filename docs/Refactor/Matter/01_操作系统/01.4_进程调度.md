# 01.4 进程调度

## 1. 概述

进程调度是操作系统内核的核心功能之一，负责决定哪个进程在何时使用CPU。它是实现多任务、提高系统吞吐量和响应性的关键机制。

### 1.1 调度目标

- **公平性**：每个进程都能获得合理的CPU时间
- **效率**：最大化CPU利用率，最小化调度开销
- **响应时间**：快速响应交互式用户的请求
- **吞吐量**：单位时间内完成尽可能多的作业
- **截止时间**：满足实时任务的时限要求

## 2. 进程状态模型

### 2.1 基本状态转换

```
        创建
          ↓
    ┌─────┴─────┐
    ↓           ↓
 就绪态 ←─────→ 运行态 ←────┐
    ↑      调度    ↓ 时间片到│
    └──────────── 阻塞态 ←───┘
                       ↑
                       I/O完成
```

### 2.2 状态定义

| 状态 | 描述 | 触发条件 |
|------|------|----------|
| 创建态 | 进程正在被创建 | 系统调用fork/exec |
| 就绪态 | 等待CPU分配 | 进程创建完成或I/O完成 |
| 运行态 | 正在CPU上执行 | 被调度器选中 |
| 阻塞态 | 等待某事件发生 | 发起I/O请求或等待信号 |
| 终止态 | 进程执行完毕 | 调用exit或被杀死 |

## 3. 调度层级

### 3.1 长程调度（作业调度）

决定哪些作业被允许进入系统，控制多道程序的道数。

**主要任务**：

- 从后备队列中选择作业
- 创建进程并分配资源
- 控制内存中的进程数量

### 3.2 中程调度（内存调度）

决定哪些进程被换入/换出内存，参与交换（swapping）机制。

**交换条件**：

- 内存不足时换出低优先级进程
- 需要时换入就绪进程

### 3.3 短程调度（CPU调度）

最频繁的调度，决定哪个就绪进程获得CPU。

**触发时机**：

- 时钟中断（时间片用完）
- I/O中断
- 系统调用
- 信号处理

## 4. 调度算法分类

### 4.1 先来先服务（FCFS）

**原理**：按照进程到达的先后顺序执行。

**特点**：

- 实现简单，使用FIFO队列
- 非抢占式
- 护航效应（Convoy Effect）

**示例**：

```
进程  到达时间  执行时间
P1     0        24
P2     0        3
P3     0        3

FCFS执行顺序：P1 → P2 → P3
等待时间：P1=0, P2=24, P3=27
平均等待时间：(0+24+27)/3 = 17
```

### 4.2 最短作业优先（SJF）

**原理**：优先执行预计执行时间最短的进程。

**形式化定义**：
$$\text{选择 } P_i \text{ 使得 } T_{burst}(P_i) = \min_{P_j \in ReadyQueue} T_{burst}(P_j)$$

**变体**：

- **非抢占式SJF**：一旦开始执行，直到完成
- **抢占式SJF（SRTF）**：新进程到达时重新评估

**示例**：

```
进程  到达时间  执行时间
P1     0        7
P2     2        4
P3     4        1
P4     5        4

SRTF执行顺序：P1(0-2)→P2(2-4)→P3(4-5)→P2(5-7)→P4(7-11)→P1(11-16)
平均等待时间：((11-2)+(5-4)+(4-4)+(7-5))/4 = 3
```

### 4.3 优先级调度

**原理**：每个进程分配优先级，高优先级先执行。

**优先级计算因素**：

- 用户指定优先级
- 进程类型（系统/用户）
- 资源需求
- 等待时间（动态调整）

**解决饥饿**：

- **老化（Aging）**：随等待时间增加优先级

$$Priority_{new} = Priority_{old} + \alpha \times WaitingTime$$

### 4.4 时间片轮转（RR）

**原理**：每个进程分配固定时间片，轮流执行。

**关键参数**：

- **时间片长度（q）**：通常为10-100ms

**性能分析**：

- 若q太大 → 退化为FCFS
- 若q太小 → 上下文切换开销过大

**上下文切换开销占比**：
$$\text{开销比} = \frac{T_{context\_switch}}{T_{context\_switch} + q}$$

**示例**：

```
进程  执行时间
P1     10
P2     3
P3     5

时间片q=4:
时间线：P1(0-4)→P2(4-7)→P3(7-11)→P1(11-15)→P3(15-16)→P1(16-18)
```

### 4.5 多级队列调度

**原理**：将就绪队列划分为多个独立队列，每个队列有自己的调度算法。

**队列划分依据**：

- 进程类型（前台/后台）
- 优先级
- 内存需求

```
┌─────────────────────────────────────┐
│  最高优先级队列（系统进程）  ←  RR   │
├─────────────────────────────────────┤
│  高优先级队列（交互式进程）  ←  RR   │
├─────────────────────────────────────┤
│  中优先级队列（批处理进程）  ←  FCFS │
├─────────────────────────────────────┤
│  低优先级队列（学生进程）    ←  FCFS │
└─────────────────────────────────────┘
```

### 4.6 多级反馈队列（MLFQ）

**原理**：允许进程在队列间移动，根据行为动态调整优先级。

**调度规则**：

1. 新进程进入最高优先级队列（Q₀）
2. 若时间片用完未结束，降级到下一队列
3. 低优先级队列的时间片更长
4. 等待一定时间后，进程升级

```
Q0 (q=8ms):  P1 ──→ 超时降级
Q1 (q=16ms):      P1 ──→ 超时降级
Q2 (FCFS):              P1 执行完成
```

**参数设置**：

- 队列数量
- 每队列时间片长度
- 升级/降级策略

## 5. 实时调度

### 5.1 硬实时 vs 软实时

| 特性 | 硬实时 | 软实时 |
|------|--------|--------|
| 截止时间 | 必须满足 | 尽量满足 |
| 错过后果 | 系统失败 | 性能下降 |
| 示例 | 汽车ABS、医疗设备 | 视频流、游戏 |

### 5.2 速率单调调度（RMS）

**原理**：周期越短，优先级越高。

**可调度性条件**：
$$\sum_{i=1}^{n} \frac{C_i}{T_i} \leq n(2^{1/n} - 1)$$

其中：$C_i$为执行时间，$T_i$为周期

### 5.3 最早截止时间优先（EDF）

**原理**：截止时间最早的进程优先执行。

**最优性**：如果存在可行调度，EDF一定能找到。

**可调度性条件**：
$$\sum_{i=1}^{n} \frac{C_i}{T_i} \leq 1$$

## 6. 多处理器调度

### 6.1 调度方式

**亲和性调度**：

- 尽量让进程在同一CPU上执行
- 减少缓存失效

**负载均衡**：

- push迁移：空闲处理器主动获取任务
- pull迁移：忙碌处理器推送任务

### 6.2 处理器亲和性

$$\text{亲和性强度} = \frac{\text{本地缓存命中次数}}{\text{总缓存访问次数}}$$

## 7. 调度性能指标

### 7.1 常用指标

| 指标 | 公式 | 说明 |
|------|------|------|
| 周转时间 | $T_{turnaround} = T_{completion} - T_{arrival}$ | 从提交到完成 |
| 等待时间 | $T_{waiting} = T_{turnaround} - T_{burst}$ | 在就绪队列等待 |
| 响应时间 | $T_{response} = T_{first\_run} - T_{arrival}$ | 首次获得CPU |
| 吞吐量 | $\frac{\text{完成作业数}}{\text{时间}}$ | 单位时间完成量 |
| CPU利用率 | $\frac{\text{忙碌时间}}{\text{总时间}} \times 100\%$ | CPU有效工作时间 |

### 7.2 公平性指标

**Jain公平指数**：
$$J(x_1, x_2, ..., x_n) = \frac{(\sum_{i=1}^{n} x_i)^2}{n \cdot \sum_{i=1}^{n} x_i^2}$$

其中$x_i$为第i个进程获得的CPU时间份额。

## 8. Linux调度器演进

### 8.1 O(1)调度器（Linux 2.5-2.6.22）

- 活跃数组和过期数组
- 常数时间选择下一个进程
- 140个优先级

### 8.2 CFS完全公平调度器（Linux 2.6.23+）

**核心思想**：

- 使用红黑树管理可运行进程
- 虚拟运行时间（vruntime）作为调度依据

**vruntime计算**：
$$vruntime = vruntime_{old} + \frac{actual\_runtime \times 1024}{weight}$$

**权重计算**：
$$weight = \frac{1024}{1.25^{nice}}$$

### 8.3 调度策略

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| SCHED_FIFO | 实时先进先出 | 硬实时任务 |
| SCHED_RR | 实时轮转 | 实时任务 |
| SCHED_OTHER | 普通分时 | 普通进程 |
| SCHED_BATCH | 批处理 | CPU密集型 |
| SCHED_IDLE | 空闲 | 低优先级任务 |

## 9. 代码示例

### 9.1 简单优先级队列实现

```c
#include <stdio.h>
#include <stdlib.h>

#define MAX_PROCESSES 100

typedef struct {
    int pid;
    int priority;
    int burst_time;
    int remaining_time;
} Process;

typedef struct {
    Process *processes[MAX_PROCESSES];
    int size;
} PriorityQueue;

void init_queue(PriorityQueue *q) {
    q->size = 0;
}

void enqueue(PriorityQueue *q, Process *p) {
    int i = q->size++;
    while (i > 0 && q->processes[(i-1)/2]->priority < p->priority) {
        q->processes[i] = q->processes[(i-1)/2];
        i = (i-1)/2;
    }
    q->processes[i] = p;
}

Process* dequeue(PriorityQueue *q) {
    if (q->size == 0) return NULL;
    Process *max = q->processes[0];
    Process *last = q->processes[--q->size];

    int i = 0, child;
    while (2*i + 1 < q->size) {
        child = 2*i + 1;
        if (child + 1 < q->size &&
            q->processes[child+1]->priority > q->processes[child]->priority)
            child++;
        if (last->priority >= q->processes[child]->priority)
            break;
        q->processes[i] = q->processes[child];
        i = child;
    }
    q->processes[i] = last;
    return max;
}

int main() {
    PriorityQueue q;
    init_queue(&q);

    // 添加进程
    Process p1 = {1, 3, 10, 10};
    Process p2 = {2, 1, 5, 5};
    Process p3 = {3, 5, 8, 8};

    enqueue(&q, &p1);
    enqueue(&q, &p2);
    enqueue(&q, &p3);

    // 按优先级执行
    while (q.size > 0) {
        Process *p = dequeue(&q);
        printf("执行进程 %d (优先级:%d, 执行时间:%d)\n",
               p->pid, p->priority, p->burst_time);
    }

    return 0;
}
```

### 9.2 轮转调度模拟

```python
from collections import deque

def round_robin(processes, time_quantum):
    """
    轮转调度算法模拟
    processes: [(pid, burst_time), ...]
    """
    queue = deque(processes)
    time = 0
    completion_times = {}

    while queue:
        pid, burst = queue.popleft()

        if burst <= time_quantum:
            time += burst
            completion_times[pid] = time
            print(f"时间 {time-time_quantum if burst == time_quantum else time-burst}-{time}: 进程 {pid} 完成")
        else:
            time += time_quantum
            print(f"时间 {time-time_quantum}-{time}: 进程 {pid} 执行 {time_quantum}ms")
            queue.append((pid, burst - time_quantum))

    return completion_times

# 测试
processes = [(1, 10), (2, 3), (3, 5)]
round_robin(processes, time_quantum=4)
```

### 10.1 算法复杂度对比

| 算法 | 选择复杂度 | 插入复杂度 | 特点 |
|------|-----------|-----------|------|
| FCFS | O(1) | O(1) | FIFO队列 |
