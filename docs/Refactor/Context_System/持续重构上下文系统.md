# 持续重构上下文系统

## 重构任务概述

### 任务目标
1. 分析 `/docs` 目录下（除 Refactor）的所有递归子目录中的所有文件内容
2. 梳理各个主题的相关内容知识，分析论证思路
3. 进行哲学和科学的批判性分析，梳理分类所有内容的相关性
4. 重构到 `/docs/Refactor` 目录下，建立各个梳理后的主题子目录
5. 输出符合数学规范的形式化规范的 markdown 文件
6. 构建能持续性不间断的上下文提醒体系

### 重构原则
- **形式化规范**：所有内容必须符合数学和学术规范
- **批判性分析**：避免辩证法，保持严谨的批判性思维
- **多表征方式**：包含图表、数学符号、形式化证明等
- **一致性保证**：内容一致性、证明一致性、相关性一致性、语义一致性
- **避免重复**：消除冗余内容，建立清晰的层次结构

## 重构策略

### 1. 内容分析策略
- 深度阅读所有文件内容
- 识别核心概念和定理
- 分析论证逻辑和证明过程
- 评估内容的完整性和准确性

### 2. 批判性分析策略
- 识别逻辑漏洞和论证缺陷
- 评估形式化程度
- 检查一致性和完备性
- 提出改进建议

### 3. 重构策略
- 建立清晰的层次结构
- 消除重复和冗余
- 补充缺失的证明和论证
- 统一术语和符号

### 4. 形式化策略
- 使用严格的数学符号
- 提供完整的证明过程
- 建立形式化定义
- 确保逻辑严密性

## 质量控制

### 1. 内容质量
- 准确性：所有陈述必须准确
- 完整性：关键概念和定理必须完整
- 一致性：术语和符号使用一致
- 相关性：内容与主题高度相关

### 2. 形式质量
- 数学规范：符合数学写作规范
- 逻辑严密：论证逻辑严密
- 结构清晰：文档结构清晰
- 可读性：内容易于理解

### 3. 学术规范
- 引用规范：正确引用参考文献
- 术语规范：使用标准术语
- 符号规范：使用标准数学符号
- 格式规范：符合学术写作格式
