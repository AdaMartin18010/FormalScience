# 弃用计划

> 本文档记录即将弃用的功能、迁移路径和时间线，帮助用户平稳过渡到新版本。
> 
> 最后更新: 2026-04-11

---

## ⚠️ 弃用政策

### 版本支持策略

| 版本类型 | 支持周期 | 说明 |
|----------|----------|------|
| 主要版本 (x.0.0) | 18个月 | 包括安全更新和关键补丁 |
| 次要版本 (x.y.0) | 12个月 | 包括功能更新和修复 |
| 补丁版本 (x.y.z) | 直到下个补丁 | 仅关键修复 |

### 弃用流程

```
功能弃用流程:

当前版本        下个次要版本      下个主要版本       再下个主要版本
    |                |                |                 |
    v                v                v                 v
+--------+       +--------+       +--------+       +--------+
| 标记弃用 |  -->  | 发出警告 |  -->  | 默认禁用 |  -->  | 完全移除 |
| (软弃用) |       | (硬弃用) |       | (过渡)   |       | (清理)   |
+--------+       +--------+       +--------+       +--------+
    |                |                |                 |
  文档更新        迁移工具         兼容性层          代码清理
  警告日志        升级指南         配置选项          文档更新
```

### 弃用等级

| 等级 | 名称 | 说明 | 影响 |
|------|------|------|------|
| 🟡 等级 1 | 软弃用 | 标记为弃用，但仍可用 | 提示警告 |
| 🟠 等级 2 | 硬弃用 | 发出明确弃用警告 | 日志警告 |
| 🔴 等级 3 | 禁用 | 默认禁用，需显式启用 | 功能受限 |
| ⚫ 等级 4 | 移除 | 完全移除功能 | 破坏性变更 |

## 🛣️ 迁移路径

### 从 v4.0 到 v4.1

#### API 迁移

**步骤 1: 识别已弃用的 API 调用**
```bash
# 使用提供的检查工具
./tools/check-deprecated-apis.sh

# 输出示例:
# [WARNING] Found deprecated API usage:
#   File: src/client.py:45
#   Deprecated: /api/v1/tasks/create
#   Use instead: /api/v2/tasks
```

**步骤 2: 更新 API 调用**
```python
# 迁移前
response = requests.post('/api/v1/tasks/create', json=data)

# 迁移后
response = requests.post('/api/v2/tasks', json=data)
```

**步骤 3: 验证迁移**
```bash
# 运行测试确保功能正常
./scripts/verify-api-migration.sh
```

#### 配置迁移

**自动迁移工具:**
```bash
# 运行配置迁移脚本
./tools/migrate-config.py --from v4.0 --to v4.1 \
  --input config.old.yaml \
  --output config.new.yaml
```

**手动迁移指南:**
1. 备份现有配置: `cp config.yaml config.yaml.backup`
2. 参考 [配置迁移文档](./migration/config-v4.0-to-v4.1.md)
3. 逐项更新配置项
4. 验证新配置: `./bin/validate-config config.new.yaml`

### 从 v4.x 到 v5.0

#### 重大变更预览

v5.0 将包含以下破坏性变更：

| 变更项 | 影响 | 迁移难度 | 提前准备 |
|--------|------|----------|----------|
| 协议升级 | 通信协议变更 | 中 | 启用协议 v2 |
| 配置格式 | 配置结构重组 | 中 | 使用新配置格式 |
| API 版本 | v1 API 移除 | 高 | 迁移到 v2 API |
| 存储格式 | 数据格式升级 | 低 | 自动迁移 |

#### 兼容性层

v4.2 将提供兼容性层，帮助平滑过渡:

```yaml
# 启用兼容性模式
compatibility:
  mode: "v4-v5-transition"
  enable_legacy_apis: true
  enable_protocol_v1: true
```

**注意:** 兼容性模式会有性能开销，建议在过渡期后关闭。
