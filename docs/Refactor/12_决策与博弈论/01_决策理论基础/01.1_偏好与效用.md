## 12.1.1.1 引言

决策理论的核心问题在于：如何刻画理性决策者的选择行为？
von Neumann 和 Morgenstern (1944) 在《博弈论与经济行为》中奠定了现代决策理论的数学基础。
本章从偏好关系的公理化出发，建立效用理论的形式化框架。

## 12.1.1.2 选择结构与偏好关系

### 12.1.1.2.1 定义：选择结构

**定义 12.1.1.1**（选择结构）

设 $X$ 为备选方案集（alternative set）。**选择结构**（choice structure）是一个二元组 $(\mathcal{B}, C(\cdot))$，其中：

- $\mathcal{B} \subseteq 2^X \setminus \{\emptyset\}$ 是**预算集族**（collection of budget sets），表示决策者可能面临的所有选择情境
- $C: \mathcal{B} \to 2^X$ 是**选择规则**（choice rule），满足：对所有 $B \in \mathcal{B}$，$C(B) \subseteq B$ 且 $C(B) \neq \emptyset$

**定义 12.1.1.2**（显示偏好）

对于选择结构 $(\mathcal{B}, C(\cdot))$，方案 $x$ 在预算集 $B$ 中**显示偏好**于 $y$（记作 $x \succeq^* y$），当且仅当存在 $B \in \mathcal{B}$ 使得 $x, y \in B$ 且 $x \in C(B)$。

### 12.1.1.2.2 弱公理与强公理

**定义 12.1.1.3**（弱显示偏好公理 WARP）

选择结构 $(\mathcal{B}, C(\cdot))$ 满足**弱显示偏好公理**（Weak Axiom of Revealed Preference），如果对任意 $B, B' \in \mathcal{B}$，当 $x, y \in B \cap B'$ 且 $x \in C(B)$、$y \in C(B')$ 时，必有 $x \in C(B')$。

**命题 12.1.1.1**（WARP 的等价表述）

选择结构满足 WARP 当且仅当：若 $x$ 显示偏好于 $y$，则 $y$ 不严格显示偏好于 $x$。

**证明**：

($\Rightarrow$) 设 $x \succeq^* y$，则存在 $B$ 使 $x \in C(B)$ 且 $y \in B$。假设 $y \succ^* x$（严格显示偏好），则存在 $B'$ 使 $y \in C(B')$ 且 $x \in B' \setminus C(B')$。但 $x, y \in B \cap B'$，由 WARP，$y \in C(B')$ 且 $x \in C(B)$ 蕴含 $x \in C(B')$，矛盾。

($\Leftarrow$) 设 $x, y \in B \cap B'$，$x \in C(B)$，$y \in C(B')$。则 $x \succeq^* y$ 且 $y \succeq^* x$。若 $x \notin C(B')$，则 $y \succ^* x$，与 $x \succeq^* y$ 矛盾（若假设偏好关系的非对称性）。∎

## 12.1.1.3 理性偏好关系

### 12.1.1.3.1 偏好关系的公理

**定义 12.1.1.4**（理性偏好）

备选方案集 $X$ 上的二元关系 $\succeq$ 称为**理性偏好关系**（rational preference relation），若满足：

1. **完备性**（Completeness）：对所有 $x, y \in X$，$x \succeq y$ 或 $y \succeq x$（或两者皆成立）
2. **传递性**（Transitivity）：对所有 $x, y, z \in X$，若 $x \succeq y$ 且 $y \succeq z$，则 $x \succeq z$

**定义 12.1.1.5**（导出关系）

由 $\succeq$ 导出：

- **严格偏好** $x \succ y$：$x \succeq y$ 且非 $y \succeq x$
- **无差异** $x \sim y$：$x \succeq y$ 且 $y \succeq x$

### 12.1.1.3.2 效用表示定理

**定理 12.1.1.1**（Debreu 表示定理，1954）

设 $X$ 是连通的、可分的拓扑空间，$\succeq$ 是 $X$ 上的连续完备传递偏好关系，则存在连续效用函数 $u: X \to \mathbb{R}$ 表示 $\succeq$，即：

$$x \succeq y \iff u(x) \geq u(y)$$

**证明概要**：

1. 构造 $X$ 的可数稠密子集 $D = \{x_1, x_2, ...\}$
2. 对每个 $x_n \in D$，定义 $u(x_n)$ 为与标准偏好序列的比较结果
3. 利用连续性将 $u$ 扩展到整个 $X$
4. 验证 $u$ 保持偏好顺序

**定义 12.1.1.6**（偏好连续性）

偏好关系 $\succeq$ 是**连续的**，如果对任意 $x \in X$，集合 $\{y \in X: y \succeq x\}$ 和 $\{y \in X: x \succeq y\}$ 都是闭集。

## 12.1.1.4 序数效用与基数效用

### 12.1.1.4.1 序数效用

**定义 12.1.1.7**（序数效用）

效用函数 $u$ 是**序数**的，如果它只保留偏好顺序信息。对任意严格递增函数 $\phi: \mathbb{R} \to \mathbb{R}$，$\phi \circ u$ 表示同一偏好关系。

### 12.1.1.4.2 基数效用与线性变换

**定理 12.1.1.2**（基数效用的唯一性）

在期望效用理论框架下，效用函数在正仿射变换下唯一：若 $u$ 和 $v$ 都表示同一偏好，则存在 $a > 0$ 和 $b$ 使得 $v = a \cdot u + b$。

**证明**：

设 $u$ 和 $v$ 都表示同一偏好关系。取 $x^* \succ x_*$（最偏好和最不偏好）。对任意 $x$，存在唯一的 $\alpha \in [0,1]$ 使得 $x \sim \alpha x^* + (1-\alpha) x_*$。

则 $u(x) = \alpha u(x^*) + (1-\alpha) u(x_*)$，$v(x) = \alpha v(x^*) + (1-\alpha) v(x_*)$。

令 $a = \frac{v(x^*) - v(x_*)}{u(x^*) - u(x_*)}$，$b = v(x_*) - a \cdot u(x_*)$，可验证 $v = a \cdot u + b$。∎

## 12.1.1.5 偏好关系的基本性质

### 12.1.1.5.1 凸性

**定义 12.1.1.8**（凸偏好）

偏好关系 $\succeq$ 是**凸的**，如果对任意 $x, y, z \in X$，若 $y \succeq x$ 且 $z \succeq x$，则对所有 $\alpha \in [0,1]$，$\alpha y + (1-\alpha) z \succeq x$。

**命题 12.1.1.2**

效用函数 $u$ 表示凸偏好当且仅当 $u$ 是拟凹的，即：

$$u(\alpha x + (1-\alpha) y) \geq \min\{u(x), u(y)\}$$

### 12.1.1.5.2 单调性

**定义 12.1.1.9**（单调偏好）

设 $X = \mathbb{R}^n_+$（商品空间），偏好关系 $\succeq$ 是**单调的**，若 $x \geq y$（分量-wise）且 $x \neq y$ 蕴含 $x \succ y$。

## 12.1.1.6 Python 实现：偏好与效用

```python
"""
偏好与效用理论的 Python 实现
基于 von Neumann & Morgenstern (1944) 框架
"""

from typing import List, Tuple, Callable, Set
from dataclasses import dataclass
from functools import total_ordering
import numpy as np


@dataclass
class Alternative:
    """备选方案"""
    name: str
    attributes: dict

    def __hash__(self):
        return hash(self.name)

    def __eq__(self, other):
        return isinstance(other, Alternative) and self.name == other.name


class PreferenceRelation:
    """
    理性偏好关系
    满足完备性和传递性
    """

    def __init__(self, alternatives: Set[Alternative]):
        self.alternatives = alternatives
        self._preference_pairs: Set[Tuple[Alternative, Alternative]] = set()
        self._indifference_pairs: Set[Tuple[Alternative, Alternative]] = set()

    def add_preference(self, x: Alternative, y: Alternative):
        """添加偏好 x ≽ y"""
        self._preference_pairs.add((x, y))
        self._check_transitivity()

    def add_indifference(self, x: Alternative, y: Alternative):
        """添加无差异 x ~ y"""
        self._indifference_pairs.add((x, y))
        self._indifference_pairs.add((y, x))

    def _check_transitivity(self):
        """验证传递性"""
        for x in self.alternatives:
            for y in self.alternatives:
                for z in self.alternatives:
                    if self.is_preferred(x, y) and self.is_preferred(y, z):
                        if not self.is_preferred(x, z):
                            raise ValueError(f"传递性违反: {x.name} ≽ {y.name} ≽ {z.name}")

    def is_preferred(self, x: Alternative, y: Alternative) -> bool:
        """检查 x ≽ y"""
        return (x, y) in self._preference_pairs or x == y

    def is_strictly_preferred(self, x: Alternative, y: Alternative) -> bool:
        """检查 x ≻ y"""
        return self.is_preferred(x, y) and not self.is_preferred(y, x)

    def is_indifferent(self, x: Alternative, y: Alternative) -> bool:
        """检查 x ~ y"""
        return self.is_preferred(x, y) and self.is_preferred(y, x)

    def check_completeness(self) -> bool:
        """验证完备性"""
        for x in self.alternatives:
            for y in self.alternatives:
                if not (self.is_preferred(x, y) or self.is_preferred(y, x)):
                    return False
        return True

    def get_choice(self, budget_set: Set[Alternative]) -> Set[Alternative]:
        """
        在预算集中选择最优方案
        C(B) = {x ∈ B : ∀y ∈ B, x ≽ y}
        """
        best = set()
        for x in budget_set:
            is_best = True
            for y in budget_set:
                if not self.is_preferred(x, y):
                    is_best = False
                    break
            if is_best:
                best.add(x)
        return best


class UtilityFunction:
    """
    效用函数表示
    """

    def __init__(self, utility_map: dict):
        """
        utility_map: {Alternative: float}
        """
        self.utility_map = utility_map

    def u(self, x: Alternative) -> float:
        """计算效用值"""
        return self.utility_map.get(x, 0.0)

    def get_preference_relation(self) -> PreferenceRelation:
        """从效用函数导出偏好关系"""
        alternatives = set(self.utility_map.keys())
        pref = PreferenceRelation(alternatives)

        for x in alternatives:
            for y in alternatives:
                if self.u(x) >= self.u(y):
                    pref.add_preference(x, y)

        return pref

    def affine_transform(self, a: float, b: float) -> 'UtilityFunction':
        """
        正仿射变换：v = a*u + b
        基数效用下表示同一偏好
        """
        new_map = {x: a * u + b for x, u in self.utility_map.items()}
        return UtilityFunction(new_map)

    def is_ordinal_equivalent(self, other: 'UtilityFunction') -> bool:
        """
        检查两个效用函数是否序数等价
        （保持相同的偏好排序）
        """
        alts = set(self.utility_map.keys())
        if alts != set(other.utility_map.keys()):
            return False

        for x in alts:
            for y in alts:
                if (self.u(x) >= self.u(y)) != (other.u(x) >= other.u(y)):
                    return False
        return True


# 示例：验证基数效用的唯一性
def demonstrate_cardinal_uniqueness():
    """演示基数效用的正仿射变换唯一性"""

    # 定义备选方案
    A = Alternative("A", {"value": 10})
    B = Alternative("B", {"value": 5})
    C = Alternative("C", {"value": 2})

    # 原始效用函数
    u = UtilityFunction({A: 10, B: 5, C: 2})

    # 正仿射变换：v = 2*u + 3
    v = u.affine_transform(2, 3)

    print("原始效用 u:", {x.name: val for x, val in u.utility_map.items()})
    print("变换后效用 v = 2u + 3:", {x.name: val for x, val in v.utility_map.items()})

    # 验证序数等价性
    print(f"\n序数等价: {u.is_ordinal_equivalent(v)}")

    # 导出偏好关系
    pref_u = u.get_preference_relation()
    pref_v = v.get_preference_relation()

    print("\n偏好排序 (两者相同):")
    for x in [A, B, C]:
        for y in [A, B, C]:
            if pref_u.is_strictly_preferred(x, y):
                print(f"  {x.name} ≻ {y.name}")


if __name__ == "__main__":
    demonstrate_cardinal_uniqueness()
```
