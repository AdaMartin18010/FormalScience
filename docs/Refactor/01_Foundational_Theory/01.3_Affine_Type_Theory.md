## 1. 仿射逻辑基础

### 1.1 仿射逻辑公理系统

**定义 1.1 (仿射上下文)**
仿射上下文 $\Gamma$ 是变量到类型的映射，其中每个变量最多使用一次：
$$\Gamma : \text{Var} \rightarrow \text{Type}$$

**定义 1.2 (仿射类型)**
仿射类型系统包含以下类型构造子：
$$\tau ::= \text{Base} \mid \tau_1 \rightarrow \tau_2 \mid \tau_1 \& \tau_2 \mid \tau_1 \oplus \tau_2$$

其中：

- $\rightarrow$ 表示仿射函数类型
- $\&$ 表示仿射积类型（with类型）
- $\oplus$ 表示仿射和类型（plus类型）

**公理 1.1 (仿射变量规则)**
$$\frac{x : \tau \in \Gamma}{\Gamma \vdash x : \tau} \quad \text{(AVar)}$$

**公理 1.2 (仿射抽象)**
$$\frac{\Gamma, x : \tau_1 \vdash e : \tau_2}{\Gamma \vdash \lambda x.e : \tau_1 \rightarrow \tau_2} \quad \text{(AAbs)}$$

**公理 1.3 (仿射应用)**
$$\frac{\Gamma \vdash e_1 : \tau_1 \rightarrow \tau_2 \quad \Gamma \vdash e_2 : \tau_1}{\Gamma \vdash e_1 e_2 : \tau_2} \quad \text{(AApp)}$$

**公理 1.4 (仿射弱化)**
$$\frac{\Gamma \vdash e : \tau}{\Gamma, x : \tau' \vdash e : \tau} \quad \text{(AWeakening)}$$

### 1.2 仿射性约束

**定理 1.1 (仿射性保持)**
在仿射类型系统中，如果 $\Gamma \vdash e : \tau$，则 $\Gamma$ 中的每个变量在 $e$ 中最多出现一次。

**证明：** 通过结构归纳法证明。对于每个语法构造：

1. **变量**：直接满足仿射性
2. **抽象**：通过归纳假设，变量在体中最多出现一次
3. **应用**：通过仿射性约束，确保变量不重复使用
4. **弱化**：允许忽略未使用的变量

**定理 1.2 (仿射上下文性质)**
仿射上下文支持弱化操作，但不支持收缩操作。

**证明：**

- 弱化：$\Gamma \vdash e : \tau$ 蕴含 $\Gamma, x : \tau' \vdash e : \tau$
- 收缩：$\Gamma, x : \tau, y : \tau \vdash e : \sigma$ 不蕴含 $\Gamma, z : \tau \vdash e[z/x, z/y] : \sigma$

### 1.3 仿射上下文管理

**定义 1.3 (仿射上下文操作)**
仿射上下文支持以下操作：

1. **扩展**：$\Gamma, x : \tau$
2. **弱化**：$\Gamma, x : \tau' \vdash e : \tau$（如果 $x$ 未在 $e$ 中使用）
3. **投影**：$\Gamma \setminus \{x\}$（移除变量 $x$）

**引理 1.1 (仿射上下文投影)**
如果 $\Gamma, x : \tau \vdash e : \sigma$ 且 $x$ 未在 $e$ 中使用，则 $\Gamma \vdash e : \sigma$。

## 2. 资源管理理论

### 2.1 仿射资源类型

**定义 2.1 (仿射资源类型)**
仿射资源类型表示最多使用一次的系统资源：
$$\text{AffineResource} ::= \text{FileHandle} \mid \text{MemoryRef} \mid \text{NetworkConn} \mid \text{DatabaseConn}$$

**定义 2.2 (仿射资源操作)**
仿射资源操作包括创建、使用和销毁：

```haskell
data AffineResourceOp a where
  Create :: ResourceType -> AffineResourceOp AffineResource
  Use    :: AffineResource -> (a -> b) -> AffineResourceOp b
  Destroy :: AffineResource -> AffineResourceOp ()
  Ignore :: AffineResource -> AffineResourceOp ()  -- 仿射特有
```

**定理 2.1 (仿射资源安全)**
在仿射类型系统中，资源不会被重复使用，但可以被忽略。

**证明：** 通过仿射性约束：

1. 每个资源变量最多使用一次
2. 资源可以被忽略（弱化）
3. 无法重复访问已使用的资源

### 2.2 内存管理

**定义 2.3 (仿射引用)**
仿射引用确保内存安全：

```haskell
data AffineRef a where
  NewRef :: a -> AffineRef a
  ReadRef :: AffineRef a -> (a, AffineRef a)
  WriteRef :: AffineRef a -> a -> AffineRef a
  FreeRef :: AffineRef a -> ()
  IgnoreRef :: AffineRef a -> ()  -- 仿射特有
```

**定理 2.2 (仿射内存安全)**
仿射引用系统保证：

1. 不会出现悬空指针
2. 不会重复释放内存
3. 支持忽略未使用的引用
4. 不会出现数据竞争

### 2.3 资源安全保证

**定义 2.4 (仿射资源安全类型)**
仿射资源安全类型系统：
$$\tau ::= \text{Base} \mid \tau_1 \rightarrow \tau_2 \mid \text{AffineResource} \tau$$

**公理 2.1 (仿射资源创建)**
$$\frac{\Gamma \vdash e : \tau}{\Gamma \vdash \text{new}(e) : \text{AffineResource} \tau} \quad \text{(AffineResourceNew)}$$

**公理 2.2 (仿射资源使用)**
$$\frac{\Gamma \vdash e_1 : \text{AffineResource} \tau_1 \quad \Gamma, x : \tau_1 \vdash e_2 : \tau_2}{\Gamma \vdash \text{use}(e_1, \lambda x.e_2) : \tau_2} \quad \text{(AffineResourceUse)}$$

**公理 2.3 (仿射资源忽略)**
$$\frac{\Gamma \vdash e : \text{AffineResource} \tau}{\Gamma \vdash \text{ignore}(e) : \text{Unit}} \quad \text{(AffineResourceIgnore)}$$

## 3. 仿射逻辑语义

### 3.1 指称语义

**定义 3.1 (仿射函数空间)**
仿射函数空间 $A \rightarrow B$ 的语义：
$$\llbracket A \rightarrow B \rrbracket = \llbracket A \rrbracket \rightarrow \llbracket B \rrbracket$$

**定义 3.2 (仿射积语义)**
仿射积 $A \& B$ 的语义：
$$\llbracket A \& B \rrbracket = \llbracket A \rrbracket \times \llbracket B \rrbracket$$

**定义 3.3 (仿射和语义)**
仿射和 $A \oplus B$ 的语义：
$$\llbracket A \oplus B \rrbracket = \llbracket A \rrbracket + \llbracket B \rrbracket$$

### 3.2 操作语义

**定义 3.4 (仿射归约)**
仿射归约规则：
$$(\lambda x.e) v \rightarrow e[v/x]$$

**定理 3.1 (仿射归约保持类型)**
如果 $\Gamma \vdash e : \tau$ 且 $e \rightarrow e'$，则 $\Gamma \vdash e' : \tau$。

**证明：** 通过结构归纳法证明。对于每个归约规则：

1. **β归约**：$(\lambda x.e) v \rightarrow e[v/x]$
   - 仿射性在替换过程中得到保持
2. **投影归约**：$\pi_i(e_1, e_2) \rightarrow e_i$
   - 仿射性在投影过程中得到保持

### 3.3 仿射归约

**定义 3.5 (仿射替换)**
仿射替换 $e[v/x]$ 满足：

1. 变量 $x$ 在 $e$ 中最多出现一次
2. 替换后 $v$ 在 $e[v/x]$ 中最多出现一次

**引理 3.1 (仿射替换引理)**
如果 $\Gamma, x : \tau_1 \vdash e : \tau_2$ 且 $\Gamma \vdash v : \tau_1$，则 $\Gamma \vdash e[v/x] : \tau_2$。

## 4. 仿射类型系统

### 4.1 仿射积类型

**公理 4.1 (仿射积引入)**
$$\frac{\Gamma \vdash e_1 : \tau_1 \quad \Gamma \vdash e_2 : \tau_2}{\Gamma \vdash (e_1, e_2) : \tau_1 \& \tau_2} \quad \text{(AffineProduct)}$$

**公理 4.2 (仿射积消除)**
$$\frac{\Gamma \vdash e : \tau_1 \& \tau_2}{\Gamma \vdash \pi_1(e) : \tau_1} \quad \text{(AffineProj1)}$$

**公理 4.3 (仿射积消除2)**
$$\frac{\Gamma \vdash e : \tau_1 \& \tau_2}{\Gamma \vdash \pi_2(e) : \tau_2} \quad \text{(AffineProj2)}$$

**定理 4.1 (仿射积性质)**
仿射积类型满足：

1. 支持弱化
2. 不支持收缩
3. 支持投影操作

### 4.2 仿射和类型

**公理 4.4 (仿射和引入1)**
$$\frac{\Gamma \vdash e : \tau_1}{\Gamma \vdash \text{inl}(e) : \tau_1 \oplus \tau_2} \quad \text{(AffineSum1)}$$

**公理 4.5 (仿射和引入2)**
$$\frac{\Gamma \vdash e : \tau_2}{\Gamma \vdash \text{inr}(e) : \tau_1 \oplus \tau_2} \quad \text{(AffineSum2)}$$

**公理 4.6 (仿射和消除)**
$$\frac{\Gamma \vdash e : \tau_1 \oplus \tau_2 \quad \Gamma, x : \tau_1 \vdash e_1 : \tau \quad \Gamma, y : \tau_2 \vdash e_2 : \tau}{\Gamma \vdash \text{case } e \text{ of } \text{inl}(x) \Rightarrow e_1 \mid \text{inr}(y) \Rightarrow e_2 : \tau} \quad \text{(AffineSumElim)}$$

### 4.3 仿射函数类型

**定义 4.1 (仿射函数)**
仿射函数满足：

1. 每个参数最多使用一次
2. 支持忽略未使用的参数
3. 不支持重复使用参数

**定理 4.2 (仿射函数性质)**
仿射函数满足：

1. **弱化**：可以忽略未使用的参数
2. **投影**：可以从积类型中提取分量
3. **选择**：可以从和类型中选择分支

## 5. 与线性类型的关系

### 5.1 线性到仿射的嵌入

**定义 5.1 (线性到仿射嵌入)**
线性类型可以嵌入到仿射类型中：
$$\tau_1 \multimap \tau_2 \hookrightarrow \tau_1 \rightarrow \tau_2$$

**定理 5.1 (嵌入保持性)**
如果 $\Gamma \vdash_L e : \tau$，则 $\Gamma \vdash_A e : \tau$。

**证明：** 通过结构归纳法证明。线性类型系统的每个规则都对应仿射类型系统的一个规则。

### 5.2 仿射到线性的嵌入

**定义 5.2 (仿射到线性嵌入)**
仿射类型可以嵌入到线性类型中：
$$\tau_1 \rightarrow \tau_2 \hookrightarrow \tau_1 \multimap \tau_2$$

**定理 5.2 (嵌入限制)**
仿射类型嵌入到线性类型需要额外的线性性检查。

### 5.3 混合类型系统

**定义 5.3 (混合类型系统)**
混合类型系统结合线性和仿射类型：
$$\tau ::= \text{Base} \mid \tau_1 \multimap \tau_2 \mid \tau_1 \rightarrow \tau_2 \mid \tau_1 \otimes \tau_2 \mid \tau_1 \& \tau_2 \mid \tau_1 \oplus \tau_2$$

**定理 5.3 (混合类型系统性质)**
混合类型系统满足：

1. 线性类型：精确使用一次
2. 仿射类型：最多使用一次
3. 类型转换：线性类型可以转换为仿射类型

## 6. 实际应用

### 6.1 Rust借用系统

Rust 的借用系统基于仿射类型理论：

```rust
// 基于仿射类型理论的借用系统
fn borrow_string(s: &String) {
    println!("Borrowed: {}", s);
}

fn main() {
    let s = String::from("hello");
    borrow_string(&s);
    // 可以继续使用 s
    println!("Still available: {}", s);
    
    // 也可以忽略借用
    let _ = &s;  // 忽略借用
    println!("Still available: {}", s);
}
```

**定理 6.1 (Rust 借用安全)**
Rust 的借用系统保证内存安全。

**证明：** 通过仿射类型系统的性质：

1. 每个借用最多使用一次
2. 支持忽略未使用的借用
3. 借用检查防止数据竞争

### 6.2 函数式编程

**定义 6.1 (仿射函数)**

```haskell
class Affine a where
  use :: a -> b -> b
  ignore :: a -> ()
  duplicate :: a -> (a, a)  -- 仅对非线性类型可用

instance Affine Int where
  use x f = f x
  ignore _ = ()
  duplicate x = (x, x)

instance Affine (AffineRef a) where
  use ref f = f (ReadRef ref)
  ignore ref = IgnoreRef ref
  duplicate _ = error "Cannot duplicate affine reference"
```
