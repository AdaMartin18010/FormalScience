# 会话结束保存清单

> 每次会话结束时执行此检查清单，确保工作成果被完整保存

---

## 📋 结束步骤

### 1. 工作成果整理

- [ ] 确认所有代码已保存
- [ ] 确认所有文档已更新
- [ ] 检查是否有临时文件需要清理
- [ ] 验证关键文件的完整性

### 2. 更新 SESSION_LOG.md

- [ ] 在文件顶部添加新的会话记录
- [ ] 记录本次会话的时间、时长
- [ ] 列出完成的工作内容
- [ ] 记录遇到的问题及解决方案
- [ ] 写下下一步计划

**会话记录模板**：

```markdown
## 会话 #___ - [会话标题]

**时间**：YYYY-MM-DD HH:MM
**时长**：X 小时 X 分钟
**类型**：[开发/文档/测试/修复]

### 完成内容
- [x] 任务项 1
- [x] 任务项 2

### 遇到的问题
- 问题描述及解决方案

### 下一步计划
- 下一步工作安排
```

### 3. 更新 PROGRESS_TRACKER.md

- [ ] 更新各模块的完成百分比
- [ ] 检查里程碑状态
- [ ] 更新阻塞项列表
- [ ] 记录变更历史

### 4. 创建/更新 CHECKPOINT.md

- [ ] 创建新的断点记录
- [ ] 填写上下文摘要
- [ ] 列出已完成工作
- [ ] 记录待继续事项
- [ ] 标记相关文件位置

**断点记录模板**：

```markdown
## 🏁 最新断点

**断点编号**：CP-___
**创建时间**：YYYY-MM-DD HH:MM
**对应会话**：会话 #___
**状态**：[🟢 稳定 / 🟡 临时 / 🔴 风险]

### 📋 上下文摘要
[一句话描述当前状态]

### 🎯 已完成工作
- [x] 工作项 1
- [x] 工作项 2

### ⏳ 待继续事项
- [ ] 待办项 1
- [ ] 待办项 2

### 🔗 相关文件位置
- 文件路径 1
- 文件路径 2
```

### 5. 更新 TASK_QUEUE.md

- [ ] 检查是否有新任务需要添加
- [ ] 更新任务状态（进行中/已完成）
- [ ] 调整任务优先级
- [ ] 更新依赖关系

### 6. 更新 CURRENT_FOCUS.md

- [ ] 更新任务进度
- [ ] 记录新的工作笔记
- [ ] 更新注意事项
- [ ] 如有必要，切换到新任务
