# FormalScience 项目上下文提醒体系

## 🎯 体系概述

本上下文提醒体系旨在支持 FormalScience 项目重构工作的可持续性和连续性，确保任务可以安全中断并在后续会话中无缝恢复。

## 📁 文件结构说明

```
.context/
├── README.md                    # 本文件 - 体系使用指南
├── SESSION_LOG.md              # 会话记录 - 记录每次工作的详细情况
├── PROGRESS_TRACKER.md         # 进度追踪 - 各任务模块完成百分比
├── CHECKPOINT.md               # 断点记录 - 可恢复的工作状态快照
├── TASK_QUEUE.md               # 任务队列 - 待办任务按优先级排列
├── CURRENT_FOCUS.md            # 当前焦点 - 当前正在处理的任务详情
└── templates/                   # 模板文件
    ├── session_start.md        # 会话启动检查清单
    ├── session_end.md          # 会话结束保存清单
    └── task_template.md        # 任务记录标准格式
```

## 🚀 快速开始

### 每次会话开始前

1. **阅读 CHECKPOINT.md** - 了解上次工作的断点状态
2. **阅读 CURRENT_FOCUS.md** - 确认当前任务焦点
3. **使用 templates/session_start.md** - 完成启动检查清单

### 每次会话结束时

1. **更新 SESSION_LOG.md** - 记录本次会话的工作内容
2. **更新 PROGRESS_TRACKER.md** - 更新各模块完成百分比
3. **更新 CHECKPOINT.md** - 创建新的断点记录
4. **更新 TASK_QUEUE.md** - 调整任务优先级和状态
5. **使用 templates/session_end.md** - 完成结束保存清单

## 📝 文件使用规范

### 1. SESSION_LOG.md

- 按时间倒序记录每次会话
- 包含：时间、完成任务、遇到的问题、下一步计划

### 2. PROGRESS_TRACKER.md

- 维护各主要模块的完成百分比
- 标注关键里程碑
- 记录阻塞项

### 3. CHECKPOINT.md

- 创建可恢复的断点快照
- 包含：上下文摘要、文件位置、待确认事项
- 标记最后更新时间

### 4. TASK_QUEUE.md

- 按优先级排列待办任务
- 使用标签标记状态：🔴 紧急 / 🟡 重要 / 🟢 待安排
- 记录任务依赖关系

### 5. CURRENT_FOCUS.md

- 仅记录当前正在处理的单一任务
- 包含：任务描述、当前进度、相关文件、注意事项
- 会话结束时更新

## ⚠️ 重要原则

1. **及时更新** - 会话结束后立即更新相关文档
2. **保持简洁** - 记录关键信息，避免冗长
3. **版本一致** - 所有文档的时间戳保持一致
4. **断点安全** - 确保每个断点都是可恢复的状态
5. **优先级明确** - 任务队列始终保持清晰的优先级

## 🆘 中断恢复指南

如果工作中断，按以下步骤恢复：

1. 打开 `CHECKPOINT.md` - 获取上次工作状态
2. 打开 `CURRENT_FOCUS.md` - 了解当前任务
3. 检查 `PROGRESS_TRACKER.md` - 确认整体进度
4. 如有疑问，查看 `SESSION_LOG.md` 的历史记录
